JS1 — **var** is function-scoped and hoisted (declaration only). **let/const** are block-scoped and hoisted but in a Temporal Dead Zone (TDZ) until initialization. **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **Event Loop** manages execution. The **Call Stack** runs synchronous code. When async operations complete, callbacks go to the **Microtask Queue** (Promises) or **Macrotask Queue** (setTimeout). The loop runs the stack, then all microtasks, then one macrotask, repeating.

JS3 — A **closure** is a function retaining access to its lexical scope even when executed outside that scope. **Bug:** `var` in a `for` loop shares one variable reference. **Fix:** Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents eventual completion; **async/await** is syntactic sugar for chaining. Error handling uses `try/catch` with await or `.catch()`. **Promise.all** fails fast on first rejection; **Promise.allSettled** waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 — **Interface** defines contract (can be merged, implemented by classes); **Type** defines shape (unions, intersections, primitives). Use **Interface** for extensibility/classes, **Type** for unions/complex shapes. **Discriminated Unions** are unions with a literal property used to narrow types.

JS6 — **Prototypal inheritance** links objects via `__proto__`. **Object.create(proto)** creates an object with a specific prototype. **ES6 Classes** are syntactic sugar over prototype chains, adding `constructor`, `extends`, and `static`.

JS7 — **this** binding rules (priority): 1. **Explicit** (call/apply/bind), 2. **New** (constructor), 3. **Implicit** (obj.method), 4. **Default** (window/undefined). **bind()** returns a new function with bound context; it **cannot** be re-bound.

JS8 — **WeakRef** holds weak reference to object (GC can collect); **FinalizationRegistry** runs cleanup when object is collected. **WeakMap** keys must be objects (GC can collect keys if no strong refs exist), preventing memory leaks.

JS9 — V8 uses **Hidden Classes** (shapes) for property access optimization. **Inline Caching** caches property offsets. **Deoptimization** occurs when shapes change (e.g., adding properties dynamically), forcing fallback to slower dictionary mode.

JS10 — **Tree-shaking** removes unused exports. **ES Modules** have static `import/export` allowing static analysis. **CommonJS** uses dynamic `require`, preventing static analysis, so tree-shaking is ineffective.

JS11 — **undefined** (default value), **null** (intentional empty). `==` converts types (null == undefined is true); `===` checks type and value. **??** (nullish coalescing) returns right side if left is null/undefined. **?.** (optional chaining) safely accesses nested props.

JS12 — **map** transforms, **filter** selects, **reduce** accumulates. **reduce** with initial value prevents undefined first arg. **flatMap** maps then flattens one level, useful for normalizing nested arrays.

JS13 — **Template literals** use backticks for multi-line strings and `${}` interpolation. **Tagged templates** pass strings and values to a function. Example: `sql\`SELECT * FROM ${table}\`` allows sanitization before query execution.

JS14 — **Destructuring** extracts values: `const {a} = obj`, `const [x] = arr`. **Defaults:** `a=1`. **Renaming:** `{a: b}`. **Rest:** `...rest` collects remaining items into an array.

JS15 — **for...in** iterates enumerable keys/properties (including inherited). **for...of** iterates values of iterable objects. Avoid **for...in** on arrays as it includes non-index properties and is slower.

JS16 — **Generics** use `<T>` for type safety. **Constraints:** `T extends Foo`. **Conditional Types:** `T extends U ? X : Y`. **infer:** `infer R` extracts type from conditional (e.g., `ReturnType<T>`).

JS17 — **CommonJS** (`require`/`module.exports`) is synchronous, runtime resolution. **ES Modules** (`import`/`export`) are static, async. Node uses CJS by default; `type: "module"` in `package.json` enables ESM.

JS18 — **Structural typing** checks shape, not name. **Excess property checks** apply to object literals during assignment. **extends** checks assignability; **satisfies** checks shape without widening type. **Variance** (`in`/`out`) controls type parameter usage in generics.

JS19 — **Proxy** intercepts operations (get, set, etc.); **Reflect** provides default implementations. Example: Proxy validates property assignment before setting, enabling reactive data binding or logging.

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic ops (wait, notify). Workers communicate via `postMessage` or shared memory. **Ordering guarantees** ensure visibility across threads using memory barriers.

B1 — **$@** preserves arguments as separate words (respects quotes). **$*** joins arguments into one string. **"$@"** preserves individual arguments (safe for loops). **"$*"** joins them with first char of IFS.

B2 — **-e**: Exit immediately on error. **-u**: Treat unset variables as error. **-o pipefail**: Return error if any command in pipe fails.

B3 — **$(command)** is preferred over **`command`** (backticks). **$((expression))** is arithmetic expansion (shell built-in). **expr** is an external command, slower.

B4 — **Process substitution** `<(cmd)` creates a file descriptor. Example: `diff <(sort a.txt) <(sort b.txt)` compares sorted files without creating temp files.

B5 — Use `for f in *` or `find . -print0 | while IFS= read -r -d '' f`. **`for f in $(ls)`** breaks on spaces/newlines and globbing.

B6 — **Here document** (`<<EOF`) reads from stdin until EOF. **Here string** (`<<<`) passes a string. **`<<-EOF`** strips leading tabs from indentation.

B7 — `cmd > file 2>&1`: stdout to file, stderr to stdout (file). `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. `&> file`: both to file (bash specific). First two differ due to redirection order.

B8 — Use `mkdir` for locking (atomic). Function: `mkdir lockdir || exit 1`. **Trap** `cleanup` on EXIT/TERM to `rmdir lockdir`. `mkdir` avoids race conditions better than file creation.

B9 — **'...'**: Literal (no expansion). **"..."**: Expands vars/glob. **No quotes**: Expands vars/glob/split on IFS. Example: `echo $HOME` (no quotes) vs `echo "$HOME"` vs `echo '$HOME'`.

B10 — **File tests**: `-e` (exists), `-d` (dir), `-r` (readable). **[[ ]]** supports regex, logical operators, and is safer than `[ ]` (test).

B11 — **$?**: Exit code of last cmd. **$$**: Current PID. **$!**: PID of last background job. **$#**: Number of arguments.

B12 — Define: `func() {}` or `function func {}`. Return values: `return` (exit code) or `echo` (stdout). Functions run in current shell unless subshelled.

B13 — **PATH**: Colon-separated dirs searched for commands. **Builtin**: Shell internal (faster). **External**: Separate binary. Check with `type` or `which`.

B14 — **trap** sets handlers. `trap 'cmd' EXIT`. **EXIT** runs on shell exit. **ERR** on error. **INT/TERM** on signals. Order: EXIT handlers run last.

B15 — **xargs** builds command lines from stdin. **Command substitution** runs cmd immediately. **xargs -0** handles null-delimited input (from `find -print0`). **-P** runs jobs in parallel.

B16 — **Arrays**: `arr=(a b)`. **Associative**: `declare -A assoc`. Append: `arr+=("c")`. Length: `${#arr[@]}`. Syntax differs for keys/values in associative.

B17 — **(...)** creates subshell (new process, vars lost). **{ ...; }** group (current shell, vars persist). **Pipe into while** runs in subshell, losing variable changes outside loop.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (force, no catch). Graceful shutdown: Trap SIGTERM, wait for bg jobs, cleanup, exit.

B19 — Script: `split`, `xargs -P 4 -I {} process {}`, `cat results`. **Trade-offs**: Parallel I/O vs CPU. **Error propagation**: Check exit codes of parallel jobs, aggregate failures.

B20 — **exec** replaces shell process. **FD manipulation**: `exec 3>file` opens fd 3. **Coprocess**: `coproc` creates bidirectional pipe. Use exec for efficiency or coproc for persistent communication.

PS1 — **Write-Output** sends objects to pipeline. **Write-Host** writes to console (bypasses pipeline). Distinction matters because Host breaks pipeline flow.

PS2 — **Pipeline** passes objects. **ForEach-Object** (cmdlet) processes items one-by-one in pipeline. **foreach** (statement) loops over collection in memory. Use ForEach-Object for streaming.

PS3 — **[PSCustomObject]** creates typed object with properties. **Hashtable** is key/value pairs. Convert: `[PSCustomObject]@{Key="Value"}`. Use CustomObject for structured data, Hashtable for config.

PS4 — **Terminating**: Stops execution (throws). **Non-terminating**: Continues (warning). **-ErrorAction Stop** converts non-terminating to terminating. **try/catch** catches terminating errors.

PS5 — **Splatting** `@params` expands hashtable keys as parameters. Differs from passing hashtable as single param. Example: `Get-Process @params`.

PS6 — **Invoke-Command** runs one-off remote cmd. **Enter-PSSession** interactive session. Uses **WinRM** (HTTP/HTTPS) or **SSH** (v7+).

PS7 — Returns scalar if single item, array if multiple. **Pattern**: `$result = @()`. If function returns 0 items, `$x` is empty array, not null. Correct pattern: Always initialize array.

PS8 — **[scriptblock]** is code container. Pass via `-ArgumentList`. **Double-hop**: Kerberos delegation issue when accessing remote resource from remote session. Solve with CredSSP or explicit delegation.

PS9 — **@()** forces array. **[array]** casts. Indexing `$null` returns null. Indexing `@()` returns error if out of bounds.

PS10 — **-eq** (value), **-like** (wildcard), **-match** (regex). **-ceq** (case-sensitive). **-contains** (object in collection), **-in** (value in collection).

PS11 — **.psm1** (code), **.psd1** (manifest), **Binary** (DLL). **Import-Module** loads module into session. Manifest defines metadata/dependencies.

PS12 — **"..."** interpolates. **'...'** literal. **@"..."@** (here-string) for multi-line. Expressions: `"${var}"`.

PS13 — **Get-Member** lists members. **Properties** (data), **Methods** (func), **NoteProperties** (dynamic). **Select-Object** (subset props), **Where-Object** (filter objects).

PS14 — **Classes** (v5+) resemble C#. Limitations: No interfaces (v5), limited inheritance. Use classes for complex state, functions for simple logic.

PS15 — **[CmdletBinding()]** enables advanced features (ShouldProcess, parameters). **Begin/Process/End** blocks handle streaming (Begin init, Process items, End finalize).

PS16 — **Providers** abstract data sources. **Get-ChildItem** works on FileSystem, Registry, etc. Registry: `Get-ChildItem HKLM:\Software`.

PS17 — **[runspace]** (thread), **[runspacepool]** (pool). **ForEach-Object -Parallel** (v7+) uses pool. Manual runspace for fine control. Thread safety: Avoid shared mutable state.

PS18 — **PSScriptAnalyzer** enforces standards. **Rules** (checks), **Severity** (Warning/Error). Suppress via `#pragma`. Integrate via CI/CD (exit code).

PS19 — **$using:** scope passes local vars to remote. **-ArgumentList** passes arguments. **Serialization** (CLIXML) limits complex objects (e.g., custom classes).

PS20 — **DSC** defines desired state. **LCM** applies config. **Resources** manage state. Replaced/Supplemented by **Ansible**, **Chef**, or **PowerShell Desired State Configuration** modules in modern IaC.
