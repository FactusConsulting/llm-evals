DN1 — **Class vs Struct**: `class` is a reference type (heap-allocated, mutable, supports inheritance). `struct` is a value type (stack-allocated, immutable by default, no inheritance). Use `struct` for small, lightweight data objects (e.g., `Point`, `Color`) where copying is cheap and value semantics are required.

DN2 — **IEnumerable<T> vs IQueryable<T>**: `IEnumerable<T>` executes queries in-memory (client-side). `IQueryable<T>` builds expression trees translated to server-side queries (e.g., SQL). This matters for performance: `IQueryable` pushes filtering/sorting to the database, reducing data transfer and memory usage.

DN3 — **Async/Await & Task vs ValueTask**: `async/await` uses a state machine to suspend/resume execution without blocking threads. `Task` is a reference type (heap); `ValueTask` is a value type (stack). Use `ValueTask` for high-frequency async methods that often complete synchronously to reduce allocations. Do not use `ValueTask` if awaiting multiple times (struct copy issue).

DN4 — **Dependency Injection (DI) & Lifetimes**: DI injects dependencies rather than creating them. **Transient**: New instance every time. **Scoped**: New instance per request (web). **Singleton**: Single instance for app lifetime. **Bug**: Injecting a Scoped service into a Singleton captures the Singleton's lifetime, potentially causing stale data or disposing the Scoped service while the Singleton still holds a reference.

DN5 — **string vs ReadOnlySpan<char>**: `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a mutable view over memory (stack or heap) without allocation. Use `Span<T>` for high-performance parsing or string manipulation to avoid garbage pressure and copying.

DN6 — **Garbage Collector (GC)**: The GC manages memory on the managed heap. **Generations**: Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **LOH**: Large Object Heap (>85KB), collected less frequently. **IDisposable**: Required for unmanaged resources (files, DB connections) because the GC only manages managed memory; `Dispose` ensures timely cleanup.

DN7 — **ConfigureAwait(false)**: Default `await` captures the `SynchronizationContext` (e.g., UI thread). `ConfigureAwait(false)` avoids this, allowing continuation on any thread. Use in libraries to prevent deadlocks and improve perf. Do NOT use in UI/ASP.NET Core if you need to update UI or rely on context-specific data.

DN8 — **.NET 8/9 Features**: **Primary Constructors**: `public class Service(IDataService data) : IService { }`. **Collection Expressions**: `[1, 2, 3]`. **FrozenDictionary**: Immutable, high-performance lookup. **Example**:
```csharp
public class MyService(IDataService data) : IService {
    public void Run() => data.Load();
}
```

DN9 — **DbContext Concurrency Issue**: `DbContext` is not thread-safe. If registered as Singleton (or injected into a Singleton), concurrent requests share the same instance, causing race conditions and `ObjectDisposedException`. **Fix**: Register `DbContext` as **Scoped** (per HTTP request) in DI.

DN10 — **Record vs Class**: `record` is a reference type with value-based equality (positional) and immutability by default. `class` is mutable reference type. `record struct` is a value type with value equality. Use `record` for DTOs/Models, `class` for entities with identity/state, `record struct` for small value objects.

DN11 — **ref, out, in**: `ref`: Pass by reference, must be initialized before call. `out`: Pass by reference, must be assigned inside method. `in`: Pass by reference, read-only (no copy, no modify). Use `ref` for swapping, `out` for multiple returns, `in` for large readonly structs.

DN12 — **Nullable Reference Types (NRT)**: `string?` indicates a reference type can be null (compile-time warning). `int?` is a nullable value type (runtime null). NRTs differ by scope: NRTs are compile-time checks for references; nullable value types are runtime types (`Nullable<T>`).

DN13 — **LINQ**: Language Integrated Query. **Method Syntax**: `list.Where(...)`. **Query Syntax**: `from x in list where ...`. Both compile to same IL. **Deferred Execution**: Query logic runs only when enumerated (e.g., `ToList()`), not when defined.

DN14 — **Generic Constraints**: Limits type parameters. `where T : class` (reference), `where T : struct` (value), `where T : new()` (parameterless ctor), `where T : IComparable<T>`. Combine with `where T : class, IDisposable` to enforce multiple requirements.

DN15 — **Abstract Class vs Interface**: `abstract` allows code + contract + fields. `interface` is contract only (mostly). **C# 8+ Default Interface Methods**: Interfaces can now contain method implementations, blurring the line (interfaces can provide behavior).

DN16 — **Middleware Pipeline**: **Use**: Adds middleware to pipeline. **Map**: Branches pipeline for specific paths. **Run**: Terminal middleware (no next). **Order**: Critical (Auth before Controller). **Exception Handling**: `UseExceptionHandler` wraps pipeline to catch unhandled errors globally.

DN17 — **Channel<T>**: Async producer-consumer queue. **BlockingCollection<T>**: Sync wrapper. **Bounded Channels**: `Channel.CreateBounded` limits size; if full, producers wait (backpressure). Example: Limiting worker threads processing a stream.

DN18 — **Source Generators**: Compile-time code generation. **vs Reflection**: No runtime overhead. **vs IL Weaving**: No post-build step. **Incremental**: Regenerates only if source changes (faster). Preferred over `ISourceGenerator` for performance in large solutions.

DN19 — **TPL Dataflow**: Pipeline processing library. **ActionBlock**: Consumes items. **TransformBlock**: Processes items. **BufferBlock**: Stores items. **vs Channels**: Dataflow offers complex flow control (linking, completion, buffering); Channels is simpler, lower-level async streams.

DN20 — **Minimal APIs**: Lightweight endpoints (`app.MapGet`). **vs Controllers**: Less boilerplate, faster startup. **Trade-offs**: Controllers better for complex auth/versioning; Minimal APIs support OpenAPI via attributes. **Route Groups**: `app.MapGroup` organizes endpoints. **Endpoint Filters**: `IEndpointFilter` intercepts requests.

P1 — **List vs Tuple**: `List` is mutable, dynamic size. `Tuple` is immutable, fixed size. **Set vs Frozenset**: `Set` is mutable. `Frozenset` is immutable (hashable, usable as dict key).

P2 — **GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects **CPU-bound** workloads (threading ineffective). Workarounds: **Multiprocessing**, C-extensions (release GIL), or **Asyncio** for I/O.

P3 — **Deepcopy vs Copy**: `copy` creates new container but references same objects (shallow). `deepcopy` recursively copies objects. **Example**: `list = [[1], [2]]`. Shallow copy inner lists remain shared; modifying one affects the other.

P4 — **Decorators**: Functions wrapping functions. **Time Logger**:
```python
import time
def timeit(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        res = func(*args, **kwargs)
        print(f"{func.__name__}: {time.perf_counter() - start}")
        return res
    return wrapper
```

P5 — **Generators & Yield**: `yield` pauses function, returning a value. **vs List**: Lazy evaluation, lower memory. **yield from**: Delegates to another generator.

P6 — **Asyncio Event Loop**: Single-threaded scheduler. **run()**: Entry point. **gather()**: Run tasks concurrently. **create_task()**: Schedule task. **time.sleep()**: Blocks loop; use `await asyncio.sleep()` to yield control.

P7 — **MRO (Method Resolution Order)**: Python 3 uses **C3 Linearization**. **Diamond Inheritance**: `D(B, C)`. MRO: `D -> B -> C -> A`. `D().who()` calls `C.who()` because `C` appears before `A` in MRO. Prints "C".

P8 — **Metaclass**: Class of a class. **__init_subclass__**: Hook for subclass creation (easier, instance-level). **Metaclass**: Control class creation (harder, class-level). **Use Case**: Metaclass for Registry pattern; `__init_subclass__` for auto-registration or validation.

P9 — **Multiprocessing vs Threading**: **Multiprocessing**: Separate processes, no GIL, heavy overhead. **Threading**: Shared memory, GIL, light. **Concurrent.futures**: High-level API for both. **Serialization**: Multiprocessing requires pickling (objects must be serializable).

P10 — **__slots__**: Restricts attributes to a fixed set. **Prevents**: Dynamic attribute creation (`__dict__` removed). **Memory**: Reduces memory footprint significantly for many instances.

P11 — **Virtual Environment**: Isolated Python environment. **Why**: Dependency management. **venv**: Standard library (Python 3.3+). **virtualenv**: Third-party, older. **conda**: Package manager (binary packages, envs).

P12 — **Comprehensions**: Concise loops. `[x for x in list]`. **vs map/filter**: Comprehensions are more Pythonic/readable. Prefer comprehensions for simple transformations; `map/filter` for functional pipelines.

P13 — **== vs is**: `==` compares value. `is` compares identity (memory address). **Interning**: Small integers (-5 to 256) are cached. `a is b` returns True for small ints due to interning.

P14 — **args/kwargs**: `*args` = tuple of positional. `**kwargs` = dict of keyword. **Markers**: `/` = positional-only before. `*` = keyword-only after. Example: `def f(a, /, b, *, c):`.

P15 — **Context Managers**: Resource management (`with`). **__enter__/__exit__**: Manual implementation. **@contextmanager**: Generator-based. **Example**: Locking a resource or creating a temp directory.

P16 — **Type Hinting**: Static type annotations. **Union**: `int | str`. **Optional**: `str | None`. **Literal**: `Literal["a"]`. **TypeVar**: Generic constraints. **Protocol**: Structural typing. **mypy/pyright**: Analyze code without running to catch type errors.

P17 — **Data Model (Dunder)**: `__repr__` (dev), `__str__` (user). `__eq__` vs `__hash__`: If `__eq__` defined, `__hash__` becomes None (unhashable) unless explicitly defined. `__getattr__` (missing attr), `__getattribute__` (all attr access).

P18 — **Descriptors**: Control attribute access (`__get__`, `__set__`, `__delete__`). **property**: Uses descriptors for getter/setter. **classmethod/staticmethod**: Implemented via descriptors to bind/unbind methods.

P19 — **Import System**: **Finders**: Search `sys.path` for modules. **Loaders**: Load module code. **Relative vs Absolute**: Relative (`from . import x`) vs Absolute (`import package.x`). **__init__.py**: Marks package (optional in PEP 420 namespace packages).

P20 — **Dataclasses**: Boilerplate reduction (`@dataclass`). **vs attrs/Pydantic**: `attrs` more features, `Pydantic` validation. **__post_init__**: Logic after init. **field(default_factory)**: Mutable defaults. **Choose Pydantic**: For API models requiring validation, serialization, and type coercion.
