JS1 — **var** is function-scoped and hoisted (declaration only). **let** and **const** are block-scoped and hoisted but in a Temporal Dead Zone (TDZ) until initialization. **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages asynchronous execution. The **call stack** executes synchronous code. When the stack is empty, the loop checks the **microtask queue** (Promises, queueMicrotask) and runs all tasks before checking the **macrotask queue** (setTimeout, I/O).

JS3 — A **closure** is a function retaining access to its outer scope variables. Example: `function outer() { let x=1; return () => x; }`. **For loop bug**: `var` in loop shares one reference. **Fix**: Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents async completion. **async/await** is syntactic sugar for promises. Error handling: `try/catch` with `await`, `.catch()` on promises. **Promise.all** rejects on first failure; **Promise.allSettled** waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 — **Interface** defines shape, supports declaration merging, and is used for classes. **Type** is more flexible (unions, primitives). Use **Interface** for public API contracts, **Type** for unions/intersections. **Discriminated unions** are unions with a literal type property used to narrow types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — **Prototypal inheritance** links objects via `__proto__`. **Object.create(proto)** creates a new object with `proto` as its prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` and `constructor` to manage the prototype chain.

JS7 — **this** binding rules (priority): 1. `new` (new instance), 2. Explicit (`call`, `apply`, `bind`), 3. Object method (`obj.method()`), 4. Default (`window`/`undefined`). **bind()** returns a new function with bound `this`. It **cannot** be re-bound.

JS8 — **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** notifies when a weakly held object is collected. **WeakMap** keys must be objects (GC can collect keys if no strong refs exist), preventing memory leaks from cached data.

JS9 — **V8** uses **hidden classes** (shapes) for property access optimization and **inline caching** (IC) to cache property locations. **Deoptimization** occurs when assumptions break (e.g., polymorphic calls, dynamic property addition). Patterns causing deopt: adding properties to objects after creation, changing hidden classes.

JS10 — **Tree-shaking** removes unused exports. **ES Modules** are static (compile-time analysis), enabling effective tree-shaking. **CommonJS** (`require`) is dynamic (runtime), preventing static analysis and tree-shaking.

JS11 — **undefined** is unassigned; **null** is intentional empty. `== null` checks both. **`??`** (nullish coalescing) defaults only for null/undefined. **`?.`** (optional chaining) safely accesses nested properties without throwing errors.

JS12 — **map()** transforms, **filter()** selects, **reduce()** accumulates. **reduce()** with initial value prevents using the first array element as the accumulator. **flatMap()** maps then flattens one level, useful for transforming arrays of arrays.

JS13 — **Template literals** use backticks for multi-line strings and interpolation. **Tagged templates** call a function with string parts and values. Example: `sql`tag`SELECT * FROM users WHERE id = ${id}`; function escapes values to prevent SQL injection.

JS14 — **Destructuring** extracts values. **Object**: `{a}`. **Array**: `[b]`. **Defaults**: `=val`. **Renaming**: `{a: b}`. **Rest**: `...rest` collects remaining items. Works in function arguments and declarations.

JS15 — **for...in** iterates over enumerable property keys (including inherited). **for...of** iterates over values of iterable objects. Avoid **for...in** on arrays because it iterates indices/keys, not values, and includes non-enumerable or inherited properties.

JS16 — **Generics** use `<T>` for type safety. **Constraints**: `T extends Foo`. **Conditional types**: `T extends U ? X : Y`. **infer**: `infer R` extracts type from a type (e.g., `type UnwrapPromise<T> = T extends Promise<infer R> ? R : T`).

JS17 — **CommonJS** (`require`/`module.exports`) is synchronous and dynamic. **ES Modules** (`import`/`export`) are static and asynchronous. **Node.js** uses CJS by default; `"type": "module"` in `package.json` enables ESM.

JS18 — **Structural typing** checks shape, not name. **Excess property checks** apply to object literals assigned to interfaces. **extends** changes the variable type; **satisfies** checks type without changing it. **Variance** (`in`/`out`) controls type compatibility in generics (TS 4.7+).

JS19 — **Proxy** intercepts operations (get, set, apply). **Reflect** provides default implementations. Example: `new Proxy(obj, { set: (t, k, v) => { validate(k, v); t[k] = v; } })`. Used for validation, logging, or reactive frameworks.

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic operations (load, store, compareExchange). **Ordering**: Memory barriers ensure visibility. **Atomics.wait** blocks until **Atomics.notify** wakes threads, enabling synchronization without locks.

B1 — **$@** expands to separate quoted arguments (preserves word boundaries). **$*** expands to a single string (words joined by first char of IFS). **"$@"** preserves arguments as separate words; **"$*"** joins them into one string.

B2 — **set -e**: Exit on error. **-u**: Exit on undefined variable. **-o pipefail**: Exit if any command in a pipeline fails. Combined, they make scripts fail fast on errors.

B3 — **$(command)** is modern command substitution. **`command`** is legacy. **$((expression))** is arithmetic expansion (shell built-in). **expr** is an external command, slower and less feature-rich.

B4 — **Process substitution** `<(command)` creates a file descriptor (e.g., `/dev/fd/63`). Used to treat command output as a file. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — Use `for f in *` or `find . -print0 | xargs -0 -I {} sh -c '...'`. **`for f in $(ls)`** is dangerous because it splits on whitespace, breaking filenames with spaces or newlines.

B6 — **Here document** `<<EOF` reads stdin until EOF. **Here string** `<<<` passes a string as stdin. **`<<-EOF`** strips leading tabs from the script body, not the input.

B7 — `cmd > file 2>&1`: stdout to file, then stderr to stdout (file). **Both to file**. `cmd 2>&1 > file`: stderr to stdout (terminal), then stdout to file. **Stderr to terminal, stdout to file**. `&>` redirects both to file.

B8 — Use `mkdir` for locking (atomic).
```bash
lock_dir="/tmp/lock"
trap 'rm -rf "$lock_dir"' EXIT
mkdir "$lock_dir" || exit 1
# ... work ...
```
`mkdir` is atomic; if it exists, it fails. `trap` ensures cleanup on SIGTERM/EXIT.

B9 — **Single quotes**: Literal (no expansion). **Double quotes**: Expansion (variables, commands). **No quotes**: Word splitting and globbing. Example: `echo $HOME` (no quotes) vs `echo "$HOME"` (quoted) vs `echo '$HOME'` (literal).

B10 — **File tests**: `-e` (exists), `-d` (directory), `-r` (readable). **`[[ ]]`** supports regex and logical operators. **`[ ]`** is POSIX `test`, slower, no regex.

B11 — **$?**: Exit code of last command. **$$**: Current script PID. **$!**: PID of last background job. **$#**: Number of arguments passed.

B12 — **Define**: `func() { ... }` or `function func { ... }`. **Return**: `return` sets exit code; `echo` outputs to stdout. Functions inherit global scope unless `local` is used.

B13 — **PATH**: Colon-separated list of directories. Bash searches PATH for executables. **Builtin**: Internal (e.g., `cd`). **External**: Binary file. **Check**: `type command` or `which command`.

B14 — **trap**: `trap 'cmd' EXIT`. Handles signals (INT, TERM). **Execution order**: EXIT handlers run on exit. Multiple signals: Last registered handler runs (unless `set -E` or specific signal handling).

B15 — **xargs** builds command lines from stdin. **Substitution** runs command immediately. **xargs -0** handles null-delimited input (from `find -print0`). **xargs -P** runs jobs in parallel.

B16 — **Arrays**: `arr=(a b)`. **Associative**: `declare -A assoc`. **Append**: `arr+=("c")`. **Length**: `${#arr[@]}`. **Iterate**: `for i in "${arr[@]}"`. Associative keys are strings.

B17 — **Subshell** `(...)`: New process, variables lost. **Group** `{ ...; }`: Same shell, variables persist. **Pipe to while**: `cmd | while read; do var=1; done` loses `var` because loop runs in subshell.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (force kill, no catch). **Graceful shutdown**: `trap 'kill $(jobs -p)' TERM`. Stop background jobs before exiting.

B19 —
```bash
split -l 1000 log.txt part_
xargs -P 4 -I {} sh -c 'process {}' < <(ls part_*)
cat part_* > merged.txt
```
**Trade-offs**: Parallelism vs I/O contention. **Error propagation**: Check exit codes in worker function; if non-zero, log and exit parent.

B20 — **exec**: Replaces shell process. **FD manipulation**: `exec 3>file` opens FD 3. `exec 3>&-` closes it. **Coprocess**: `coproc CMD` creates bidirectional pipe. Use for persistent interactive sessions or daemon-like scripts.

PS1 — **Write-Output** sends objects to pipeline (usable by other commands). **Write-Host** writes directly to console (breaks pipeline). Use **Write-Output** for automation, **Write-Host** for UI only.

PS2 — **Pipeline** passes objects. **ForEach-Object** is a cmdlet (streaming, parallelizable). **foreach** is a statement (loops over collection in memory). Use **ForEach-Object** for large streams; **foreach** for small collections needing control flow.

PS3 — **[PSCustomObject]** creates a typed object with properties. **[hashtable]** is dynamic key-value. Convert: `[PSCustomObject]@{Key="Val"}`. Use **CustomObject** for structured data, **Hashtable** for dynamic config.

PS4 — **Terminating**: Stops execution (throw). **Non-terminating**: Continues (warning). **-ErrorAction Stop** converts non-terminating to terminating. **try/catch** catches terminating errors.

PS5 — **Splatting** `@params` passes a hashtable as arguments. **Difference**: Splatting expands keys to parameters; passing hashtable as single param treats it as one argument.

PS6 — **Invoke-Command -ComputerName**: One-off remote execution. **Enter-PSSession**: Interactive remote shell. **Protocol**: WinRM (default) or SSH (v7+). **WinRM** uses HTTP/HTTPS.

PS7 — Returns **scalar** if one item, **array** if multiple. `$x.GetType().Name` might be `String` instead of `Object[]`. **Correct pattern**: Always return `@()` (empty array) if no items, or force array `$result = @($result)`.

PS8 — **[scriptblock]** is code object. Pass via `-ArgumentList` or `-ScriptBlock`. **Double-hop**: CredSSP needed for second hop (local to remote to remote). **Solution**: Use `-ArgumentList` to pass data, or configure CredSSP/SSH.

PS9 — **@()** forces array. **[array]** casts. **Indexing $null**: Returns `$null`. **Indexing empty array**: Returns `$null`. **Force single**: `$x = @($singleItem)`.

PS10 — **-eq** (value), **-like** (glob), **-match** (regex). **-ceq** (case-sensitive). **-contains** (collection contains item). **-in** (item in collection). `-in` is more readable for "is item in list".

PS11 — **Module**: Reusable code. **.psm1**: Script module (code). **.psd1**: Manifest (metadata). **Binary**: Compiled DLL. **Import-Module**: Loads module into session.

PS12 — **"..."**: Interpolation (`$var`). **'...'**: Literal. **Here-string**: `@"..."@` for multi-line. **Expression**: `$var` inside quotes.

PS13 — **Get-Member**: Inspect object members. **Properties**: Data. **Methods**: Functions. **NoteProperties**: Added dynamically. **Select-Object**: Selects properties. **Where-Object**: Filters objects.

PS14 — **Classes** (v5+): Define types. **Limitations**: No private members, limited inheritance (no interfaces). **Use**: Complex data structures. **Function**: Simple logic.

PS15 — **CmdletBinding**: Enables advanced features (parameters, pipeline). **SupportsShouldProcess**: `-WhatIf`/`-Confirm`. **Begin/Process/End**: Setup, stream processing, cleanup.

PS16 — **Providers**: Virtual file systems. **Get-ChildItem** works on Registry (`HKLM:\`), Cert (`Cert:\`), Env. **Registry**: `Get-ChildItem HKLM:\Software`.

PS17 — **Runspaces**: Isolated execution contexts. **RunspacePool**: Reuses threads. **ForEach-Object -Parallel** (v7+): High-level parallelism. **Manual**: `RunspaceFactory` for fine-grained control.

PS18 — **PSScriptAnalyzer**: Linting tool. **Rules**: Checks coding standards. **Severity**: Warning/Error. **Suppress**: `#pragma warning disable`. **CI/CD**: Integrate as pre-commit or pipeline step.

PS19 — **$using:**: Passes local variable to remote. **-ArgumentList**: Passes data. **Serialization**: Objects serialized to CLIXML. **Limits**: Depth limits, complex objects may fail.

PS20 — **DSC**: Declarative config. **Resources**: Tasks (e.g., File, Service). **LCM**: Agent enforcing config. **WinPS vs PS7**: DSC in PS7 uses PowerShell modules. **Replaced by**: Ansible, Terraform, Chef.
