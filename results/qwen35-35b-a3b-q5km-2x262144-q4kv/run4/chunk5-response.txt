DN1 — **Class vs Struct**: `class` is a reference type (heap), `struct` is a value type (stack/inline). `struct` is immutable by default (mostly), `class` is mutable. Use `struct` for small, immutable data containers (e.g., `Point`, `Vector`) to avoid allocation overhead and ensure value semantics. Use `class` for objects with identity, inheritance, or large state.

DN2 — **IEnumerable<T> vs IQueryable<T>**: `IEnumerable<T>` executes in-memory (client-side). `IQueryable<T>` builds an expression tree that can be translated to a query language (e.g., SQL) and executed on the server. This matters for DB queries because `IQueryable` pushes filtering/sorting to the database, reducing network traffic and improving performance. `IEnumerable` pulls all data first, then filters.

DN3 — **async/await**: Compiles to a state machine. `Task` represents an asynchronous operation that may return a value or void. `ValueTask` is a struct wrapper for `Task` or a value, avoiding heap allocation if the operation completes synchronously. Use `ValueTask` in hot paths (e.g., high-frequency async methods) where allocation is costly, but avoid it if the caller expects a `Task` (e.g., `Task.WhenAll`).

DN4 — **Dependency Injection (DI)**: Pattern for decoupling dependencies. Lifetimes: **Transient** (new instance every time), **Scoped** (new instance per request), **Singleton** (single instance for app lifetime). **Bug**: Injecting a Scoped service into a Singleton captures the Scoped instance for the Singleton's lifetime, causing stale data or memory leaks (Captive Dependency).

DN5 — **string vs ReadOnlySpan<char>**: `string` is immutable, heap-allocated. `ReadOnlySpan<char>` is a stack-allocated view over memory (can be heap or stack) allowing zero-copy slicing. Use `Span<T>` for high-performance string processing, parsing, or avoiding allocations during substring operations.

DN6 — **Garbage Collector (GC)**: Manages memory by tracking roots and collecting unreachable objects. **Generations**: Gen 0 (short-lived), Gen 1 (buffer), Gen 2 (long-lived). **LOH**: Large Object Heap (>85KB) for large objects, collected less frequently. **IDisposable**: Needed for unmanaged resources (files, DB connections) that GC cannot clean up automatically. Implement `Dispose` to release these deterministically.

DN7 — **ConfigureAwait(false)**: Default `await` captures the current `SynchronizationContext` (e.g., UI thread) to resume there. `ConfigureAwait(false)` skips this, resuming on a thread pool thread. Use `false` in library code to avoid deadlocks and improve throughput. Do NOT use `false` in UI code where you need to update the UI thread after await.

DN8 — **.NET 8/9 Features**:
*   **Primary Constructors**: `public class Person(string Name) { }`.
*   **Collection Expressions**: `[1, 2, 3]`, `[..arr]`.
*   **FrozenDictionary**: Immutable, high-performance dictionary.
*   **DI Example**:
    ```csharp
    public class Service(string Name) : IService { }
    // Program.cs: builder.Services.AddSingleton<Service>();
    ```

DN9 — **DbContext Exception**: Cause: `DbContext` is Scoped by default. If injected into a Singleton or reused across requests, it holds stale connections or disposes prematurely under concurrency. Fix: Ensure `DbContext` is registered as Scoped (`AddDbContext`) and injected into Scoped services (Controllers/Endpoints), not Singletons.

DN10 — **record vs class**: `record` provides value-based equality (`Equals` compares properties) and immutability by default. `class` uses reference equality. `record struct` combines value type semantics with record equality. Use `record` for DTOs/Models, `class` for entities with identity, `record struct` for small value objects.

DN11 — **ref, out, in**: `ref` passes by reference (must be initialized, can modify). `out` passes by reference (must be assigned inside method, returns value). `in` passes by reference (read-only, no modification). Use `ref` for swapping, `out` for multiple returns, `in` for large structs to avoid copy.

DN12 — **Nullable Reference Types (NRT)**: `?` on reference types (e.g., `string?`) tells compiler it can be null. `int?` is a nullable value type (`Nullable<int>`). NRTs are compile-time warnings for reference types to prevent `NullReferenceException`. Value types use `?` to wrap in `Nullable<T>`.

DN13 — **LINQ**: Language Integrated Query. **Method Syntax**: `.Where()`, **Query Syntax**: `from x in ...`. **Deferred Execution**: Query isn't executed until enumerated (e.g., `ToList()`). Query syntax compiles to method syntax.

DN14 — **Generic Constraints**: `where T : class` (reference), `where T : struct` (value), `where T : new()` (parameterless ctor), `where T : IComparable<T>` (interface). Combine with `and` (e.g., `where T : class, IComparable<T>`). Use to enforce capabilities on generic types.

DN15 — **abstract class vs interface**: Abstract class can have implementation and state; Interface defines contract only (pre-C# 8). **Default Interface Methods** (C# 8+) allow interfaces to have implementation, blurring the line. Use abstract for shared code/state, interface for polymorphic contracts.

DN16 — **Middleware Pipeline**: Request flows through components. `app.Use()` adds middleware (can call next). `app.Map()` branches pipeline for specific paths. `app.Run()` is terminal (no next). Ordering matters: Auth before Routing, Exception Handling early. Exception middleware catches errors and returns response.

DN17 — **Channel<T>**: Async producer-consumer pattern. `BlockingCollection<T>` is sync. **Bounded Channel**: Set `boundedCapacity` to limit queue size, applying backpressure to producers when full. Example: `Channel.CreateBounded<int>(100)`.

DN18 — **Source Generators**: Compile-time code generation via Roslyn. Faster than reflection, no IL weaving (like Fody). **Incremental Generator**: Processes inputs incrementally, faster than `ISourceGenerator` for large projects. Preferred for reducing build times and enabling better IDE support.

DN19 — **TPL Dataflow**: `ActionBlock` (processes items, no output), `TransformBlock` (processes and outputs), `BufferBlock` (stores items). Compared to `Channels`: Dataflow offers built-in backpressure, batching, and cancellation tokens. Use Dataflow for complex pipelines; Channels for simpler async queues.

DN20 — **Minimal APIs**: Entry point in `Program.cs` without controllers. **Trade-offs**: Less boilerplate, harder to test (no controller base), middleware integration is direct. **Route Groups**: `MapGroup()` for shared prefixes/middleware. **Endpoint Filters**: Apply logic (e.g., validation) to specific endpoints.

P1 — **List vs Tuple**: `List` is mutable, dynamic size. `Tuple` is immutable, fixed size. `Set` is mutable collection of unique items. `FrozenSet` is immutable set. Use `Tuple` for small, fixed data groups; `List` for collections.

P2 — **GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects **CPU-bound** workloads. Workarounds: `multiprocessing` (processes bypass GIL), C extensions (release GIL), `asyncio` (I/O-bound).

P3 — **deepcopy vs copy**: `copy` creates shallow copy (new object, references nested objects). `deepcopy` creates new object and recursively copies nested objects. **Bug**: Modifying a nested object in a shallow copy affects the original.

P4 — **Decorators**: Functions that wrap other functions.
```python
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 — **Generators**: Functions using `yield` return generator objects (lazy). Differ from lists by not storing all items in memory. `yield from` delegates to another generator.

P6 — **asyncio**: Event loop manages async tasks. `asyncio.run()` starts loop. `asyncio.gather()` runs tasks concurrently. `asyncio.create_task()` schedules task. `time.sleep()` blocks the loop; use `await asyncio.sleep()`.

P7 — **MRO**: Method Resolution Order. Python 3 uses **C3 Linearization**. In diamond inheritance, it resolves to the first class in the linearized order. `D(B, C)` -> `D, B, C, A`. `D().who()` calls `C.who()` because `C` appears before `A` in MRO.

P8 — **Metaclass**: Class of a class. `__init_subclass__` is a hook called when a subclass is defined. Use **Metaclass** for enforcing class structure globally (e.g., ORM registration). Use `__init_subclass__` for simpler subclass hooks without metaclass complexity.

P9 — **Multiprocessing vs Threading**: **Multiprocessing** (processes, bypasses GIL, heavy, pickling required). **Threading** (threads, GIL, light, shared memory). **concurrent.futures** (abstraction for both). Use Multiprocessing for CPU-bound, Threading for I/O-bound. Pickling fails for lambdas/inner functions.

P10 — **__slots__**: Prevents creation of `__dict__` for instances. Saves memory (no dict per instance). Prevents adding dynamic attributes. Use for large numbers of instances with fixed attributes.

P11 — **Virtual Environment**: Isolated Python environment. Prevents dependency conflicts. **venv** (stdlib, Python 3.3+), **virtualenv** (older, more features), **conda** (package manager + env, binary packages).

P12 — **Comprehensions**: Concise list/dict/set creation. `map()`/`filter()` return iterators (lazy). Prefer comprehensions for readability. Use `map`/`filter` for functional style or when functions already exist.

P13 — **== vs is**: `==` checks value equality. `is` checks identity (memory address). **Interning**: Small integers (-5 to 256) are cached. `a is b` returns True for small ints due to interning, not equality logic.

P14 — **args/kwargs**: `*args` collects positional args into tuple. `**kwargs` collects keyword args into dict. `/` marks positional-only params. `*` marks keyword-only params (after `*`).

P15 — **Context Managers**: `with` statement ensures setup/teardown. Implement `__enter__`/`__exit__` or use `@contextmanager`. Example: Database transaction lock, temporary directory creation.

P16 — **Type Hinting**: Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar` (generics), `Protocol` (structural typing). `mypy`/`pyright` use these to catch errors before runtime.

P17 — **Data Model**: `__repr__` (dev/debug), `__str__` (user). `__eq__` defines equality. `__hash__` must be consistent with `__eq__`. If `__eq__` defined but not `__hash__`, object becomes unhashable (cannot be dict key).

P18 — **Descriptors**: Objects implementing `__get__`, `__set__`, `__delete__`. `property`, `classmethod`, `staticmethod` are descriptors. Used for computed attributes, validation, and method binding.

P19 — **Import System**: `sys.path` lists directories. **Finders** locate modules, **Loaders** load them. Relative imports (`from . import x`) vs Absolute (`from pkg import x`). `__init__.py` marks package (optional in 3.3+ namespace packages).

P20 — **Dataclasses**: Auto-generates `__init__`, `__repr__`, etc. `__post_init__` runs after init. `field()` configures fields. **Pydantic**: Adds validation (types, constraints). Choose Pydantic for API validation, Dataclasses for simple data containers.
