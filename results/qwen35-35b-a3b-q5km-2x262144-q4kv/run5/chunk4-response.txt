G1 — **Slice vs Array:** An array is a fixed-size value type `[N]T`. A slice is a dynamic view descriptor `{ptr, len, cap}` pointing to an underlying array. **Append to capacity:** If a slice reaches capacity, `append` allocates a new, larger backing array (usually doubling capacity), copies existing elements, and updates the slice header to point to the new array.

G2 — **`var` vs `:=`:** `var x int` declares a variable with a zero value and explicit type. `x := 0` is a short variable declaration that infers the type. **Cannot use `:=`:** You cannot use `:=` to redeclare a variable in the same scope, or at the package level (outside functions).

G3 — **Goroutine vs Thread:** A goroutine is a lightweight thread managed by the Go runtime, not the OS. **GMP Model:** **G** is the goroutine state, **M** is the OS thread, **P** is the logical processor (context for execution). The scheduler multiplexes many Gs onto fewer Ms using Ps to manage concurrency efficiently.

G4 — **Channels:** Channels are typed conduits for communication. **Buffered vs Unbuffered:** Unbuffered channels block sender until receiver is ready; buffered channels allow sending until full. **Send to full buffered:** The sender blocks until a receiver reads from the channel.

G5 — **Context:** Used for cancellation, deadlines, and request-scoped values. **Example:**
```go
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()
req, _ := http.NewRequestWithContext(ctx, "GET", url, nil)
client.Do(req) // Cancels if timeout exceeded
```

G6 — **Interface Satisfaction:** Go interfaces are satisfied implicitly (duck typing). **Difference:** Java/C# require explicit declaration (`implements`). **Empty Interface (`any`):** Holds any type. Powerful for flexibility (e.g., `interface{}`), dangerous because type assertions can panic if the type is wrong.

G7 — **Memory Model:** Go has no guaranteed memory ordering without synchronization. **Unsafe:** The loop `for !ready {}` creates a data race; the writer goroutine may cache `data` or reorder writes. **Fix:** Use `sync/atomic` (e.g., `atomic.Bool`) or `sync.Mutex` to establish a happens-before relationship.

G8 — **Mutex vs RWMutex:** `Mutex` is for exclusive access. `RWMutex` allows multiple readers or one writer. **sync.Map:** Optimized for read-heavy workloads with sparse keys. **Trade-off:** `sync.Map` has higher overhead for writes and dense keys compared to `map` + `Mutex`.

G9 — **Modules:** `go.mod` declares dependencies and Go version. `go.sum` stores checksums for verification. **`go mod tidy`:** Removes unused imports and adds missing dependencies. **MVS:** Selects the lowest version of a dependency that satisfies all constraints in the dependency graph.

G10 — **Defer:** Schedules a function call to run after the surrounding function returns. **Order:** LIFO (Last In, First Out). **Loop Gotcha:** Deferring a function using a loop variable captures the variable by reference, not value, leading to the final loop value being used in all deferred calls.

G11 — **Pointer:** A reference to a memory address. **Zero Value:** `nil`. **Struct Literal:** `&MyStruct{Field: value}` creates a pointer to a new struct instance.

G12 — **Init:** Runs automatically before `main()`. **Multiple:** Yes, a package can have multiple `init()` functions. **Order:** Executed in source file order, then within each file in declaration order.

G13 — **Make vs New:** `new(T)` allocates zeroed memory and returns a pointer `*T`. `make(T)` initializes slices, maps, or channels and returns the initialized type (not a pointer).

G14 — **Error Handling:** Errors are values implementing the `error` interface (`Error() string`). **Idiomatic:** `if err != nil { return err }`. **`errors.Is/As`:** `Is` checks error chain equality (wrapping). `As` checks type and assigns to a variable. `==` only checks direct equality.

G15 — **Composite Types:**
*   **Map:** Zero value is `nil`.
*   **Slice:** Zero value is `nil`.
*   **Channel:** Zero value is `nil`.
*   **Struct:** Zero value has all fields set to their zero values.

G16 — **Race Condition:** Multiple goroutines access shared data without synchronization. **`-race` flag:** Detects data races at runtime. **Fix:**
```go
var mu sync.Mutex
mu.Lock()
data = "hello"
mu.Unlock()
```

G17 — **Select:** Switch statement for channels. **Difference:** `switch` is deterministic; `select` chooses randomly among ready cases. **Timeout:** `select { case <-ch: ... case <-time.After(1s): ... }`.

G18 — **Escape Analysis:** Compiler determines if a variable lives on the stack or heap. **Heap:** If variable escapes function scope (e.g., returned). **Inspect:** `go build -gcflags="-m"`. **Performance:** Heap allocations trigger GC; stack is faster.

G19 — **Reflection:** `reflect.Type` is metadata about a type. `reflect.Value` is the actual value. **Cost:** Slow due to dynamic dispatch and type checking. **Justified:** For frameworks (serialization, DI) where code generation is too complex.

G20 — **Embed:** `//go:embed` compiles files into binary. **Usage:** `embed.FS` for directories/files. **Limitations:** Compile-time only, no runtime modification. **Interaction:** Implements `io/fs.FS` interface for standard file operations.

R1 — **Ownership:** Rust's core system. **Rules:** 1. Each value has one owner. 2. When owner drops, value is dropped. 3. Can move or borrow, but not both simultaneously.

R2 — **String vs &str:** `String` is owned, heap-allocated, mutable. `&str` is a borrowed slice of bytes (usually from a string literal). **Function Signatures:** Use `&str` for read-only input to avoid allocation.

R3 — **Borrowing:** **Rules:** 1. Immutable refs (`&T`) can be many. 2. Mutable ref (`&mut T`) is unique. 3. No mutable + immutable refs at same time. **Borrow Checker:** Ensures these rules are followed at compile time.

R4 — **Result/Option:** `Result<T, E>` for recoverable errors. `Option<T>` for nullable values. **`?` Operator:** Propagates errors/None up the call stack. **Example:**
```rust
fn read_file(path: &str) -> Result<String, io::Error> {
    std::fs::read_to_string(path)
}
```

R5 — **Box/Rc/Arc:** `Box<T>`: Single owner, heap allocation. `Rc<T>`: Reference counted, single-threaded. `Arc<T>`: Atomic reference counted, thread-safe. **Use:** `Box` for recursion/size. `Rc` for shared ownership in single thread. `Arc` for threads.

R6 — **Traits:** Define shared behavior. **Comparison:** Similar to Go interfaces but can have default implementations. **Trait Object (`dyn Trait`):** Dynamic dispatch (vtable). **Generic (`T: Trait`):** Static dispatch (monomorphization).

R7 — **Lifetimes:** Ensure references are valid. **Failure:** Compiler doesn't know if `x` or `y` outlives the return. **Fix:** `fn longest<'a>(x: &'a str, y: &'a str) -> &'a str { ... }`.

R8 — **Send/Sync:** `Send` means ownership can be transferred across threads. `Sync` means `&T` can be shared. **Rc:** Not `Send` (ref count not atomic). **Arc:** `Send` + `Sync` (atomic ref count).

R9 — **Async/Await:** Compiles to a state machine (`Future`). **Executor:** Polls the `Future` until ready. **Difference:** Go uses M:N threading (preemptive). Rust uses cooperative multitasking (explicit yield points).

R10 — **Unsafe:** Bypasses safety checks. **3 Things:** 1. Dereference raw pointers. 2. Call unsafe functions/FFI. 3. Modify `static mut`. **Justified:** FFI, performance-critical code, or interfacing with hardware.

R11 — **Pattern Matching:** `match` exhaustively checks all possibilities. `if let` checks one pattern. **Exhaustiveness:** Compiler enforces `match` covers all enum variants; `if let` does not.

R12 — **Vec vs Array:** `Vec` is dynamic heap memory. `[T; N]` is fixed stack memory. **Vec Capacity:** When full, allocates new larger buffer (usually double), copies elements, drops old buffer.

R13 — **Enums:** Algebraic Data Types (ADTs) holding data. **Difference:** Can hold variants with data (unlike C/Java). **Option:** `Some(T)` or `None`. **No Null:** Forces explicit handling of absence, preventing null pointer exceptions.

R14 — **Impl Block:** Defines methods. **`&self`:** Borrow. **`&mut self`:** Mutable borrow. **`self`:** Consume ownership. **Associated Functions:** `fn new()` do not take `self`, often constructors.

R15 — **Derive Macro:** Generates trait implementations. **Common:** `Debug`, `Clone`, `PartialEq`. **Cannot Derive:** If fields don't implement the trait (e.g., `Fn` trait cannot be derived).

R16 — **Cow:** Clone on Write. **Use:** Avoids allocation if data is only read. **Example:** `Cow::Borrowed(&str)` vs `Cow::Owned(String)`. If mutated, clones to owned; otherwise borrows.

R17 — **Module System:** `mod` declares module. `use` imports. `pub` exposes. **File System:** `mod.rs` or `module/mod.rs`. **2015 vs 2018:** 2018 allows `use` paths relative to current module, not just crate root.

R18 — **Pin:** Prevents memory movement. **Needed:** For self-referential structs and async futures (which may hold references to themselves). **Unpin:** Marker trait indicating a type *can* be moved safely.

R19 — **Macros:** `macro_rules!` (declarative) matches tokens. **Proc Macros:** Rust code manipulating AST (derive, attribute). **Hygiene:** Prevents macro variables from leaking into surrounding scope.

R20 — **Zero-Cost:** Abstractions have no runtime overhead. **Generics:** Monomorphization creates concrete types per usage (like C++ templates). **Iterators:** Inlined loops. **Traits:** Static dispatch avoids vtable lookups.
