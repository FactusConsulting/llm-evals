## .NET / C#

**DN1** — **Class** is a reference type (heap, GC managed, supports inheritance). **Struct** is a value type (stack/inline, no inheritance, copy-on-assign). Use **struct** for small, immutable data (e.g., coordinates, IDs) to avoid heap allocation and reference overhead.

**DN2** — `IEnumerable<T>` executes in-memory (deferred enumeration). `IQueryable<T>` builds an expression tree for server-side execution (e.g., SQL). This matters for DBs because `IQueryable` translates to optimized SQL queries, whereas `IEnumerable` pulls all data to memory first.

**DN3** — `async/await` uses a state machine to suspend/resume without blocking threads. `Task` is a reference type (heap allocation). `ValueTask` is a value type (stack allocation, zero allocation). Use `ValueTask` in hot paths where `Task` allocation is a bottleneck.

**DN4** — DI injects dependencies. **Transient**: new instance every request. **Scoped**: new instance per scope (e.g., HTTP request). **Singleton**: one instance for app lifetime. **Bug**: Injecting a Scoped service into a Singleton captures the Singleton's lifetime, causing stale data or `ObjectDisposedException` when the scope ends.

**DN5** — `string` is immutable and GC-managed. `ReadOnlySpan<char>` is a stack-based view into memory (no GC). Use `Span<T>` for high-performance parsing or string manipulation to avoid allocations and copying.

**DN6** — GC manages memory by reclaiming unreachable objects. **Gen 0**: short-lived, frequent collection. **Gen 1**: medium. **Gen 2**: long-lived, infrequent. **LOH**: objects >85KB (not compacted). `IDisposable` is needed for unmanaged resources (files, handles) because GC only manages managed memory.

**DN7** — `ConfigureAwait(false)` prevents capturing the current `SynchronizationContext`. Use it in libraries or non-UI code to avoid deadlocks and improve throughput. Do NOT use it in UI code (WPF/WinForms) where context is needed for UI updates.

**DN8** — **Primary Constructors**: `class C(int x) { }`. **Collection Expressions**: `[1, 2, 3]`. **FrozenDictionary**: Immutable, fast lookup. Example: `public class Service(int Id) : IService { }` allows DI injection of constructor args directly.

**DN9** — **Cause**: `DbContext` is not thread-safe. High concurrency + Singleton lifetime causes race conditions/disposal. **Fix**: Register `DbContext` as **Scoped** in DI. EF Core expects one context per logical unit of work (request).

**DN10** — `record` provides value equality and immutability (by default). `class` provides reference equality. `record struct` combines value semantics with record features. Choose `record` for DTOs/Models, `class` for mutable entities.

**DN11** — `ref`: pass by reference, modify allowed. `out`: pass by reference, must be assigned inside method. `in`: pass by reference, read-only. Use `ref`/`out` for large structs to avoid copy; use `in` for read-only large structs.

**DN12** — **NRT**: Compiler checks for nullability on reference types (`string?`). **Nullable Value Types**: `int?` is a runtime nullable struct (`Nullable<int>`). NRT is static analysis (compile-time), value types are runtime behavior.

**DN13** — **LINQ**: Language Integrated Query. **Method**: `list.Where(...)`. **Query**: `from x in list where ...`. **Deferred Execution**: Query logic runs only when enumerated (e.g., `ToList()`).

**DN14** — **Constraints**: `where T : class` (ref type), `struct` (val type), `new()` (paramless ctor), `IComparable<T>` (interface). Combine with commas: `where T : class, IComparable<T>`. Used to enforce type capabilities at compile time.

**DN15** — **Abstract Class**: Can have implementation + state. **Interface**: Contract only (C# 8+ allows default methods). **Change**: Default interface methods allow interfaces to have logic, blurring the line with abstract classes.

**DN16** — **Middleware**: Components in pipeline. `Use`: adds middleware. `Map`: branches pipeline. `Run`: terminal handler. **Order**: Critical (e.g., Auth before Controller). **Exception Handling**: Middleware catches exceptions and returns error responses.

**DN17** — `Channel<T>`: Async, bounded/unbounded queue. `BlockingCollection<T>`: Sync. **Bounded Channel**: Limits capacity to apply backpressure (producer waits if full), preventing memory exhaustion in high-throughput pipelines.

**DN18** — **Source Generators**: Compile-time code generation. **Reflection**: Runtime. **IL Weaving**: Post-compile. **Incremental**: Watches files, regenerates only changed parts (faster). Preferred over `ISourceGenerator` for performance in large solutions.

**DN19** — **TPL Dataflow**: Pipeline blocks. `ActionBlock` (process), `TransformBlock` (map), `BufferBlock` (queue). **vs Channels**: Dataflow offers complex flow control (linking, cancellation), Channels are simpler for basic producer-consumer.

**DN20** — **Minimal APIs**: Delegate-based endpoints. **vs Controllers**: Less boilerplate, harder to scale complex logic. **Trade-offs**: Less middleware integration, OpenAPI via Swashbuckle. **Route Groups**: Organize endpoints by feature. **Filters**: Apply logic (e.g., validation) to endpoints.

## Python

**P1** — **List**: Mutable, ordered. **Tuple**: Immutable, ordered. **Set**: Mutable, unique, unordered. **Frozenset**: Immutable, hashable. Use tuples for fixed data, sets for uniqueness.

**P2** — **GIL**: Locks bytecode execution to one thread. Affects **CPU-bound** workloads. **Workarounds**: Multiprocessing (processes), C-extensions (release GIL), Asyncio (I/O bound).

**P3** — **Deepcopy**: Recursive copy of objects. **Copy**: Shallow (copies container, references inner objects). **Bug**: Modifying a nested list in a shallow copy affects the original list.

**P4** — **Decorator**: Function wrapping another. Example:
```python
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        res = func(*args, **kwargs)
        print(f"{func.__name__}: {time.time()-start}")
        return res
    return wrapper
```

**P5** — **Generator**: Lazy iterator. **Yield**: Pauses function, returns value. **vs List**: No memory for all items. **Yield from**: Delegates to another generator.

**P6** — **Event Loop**: Single-threaded scheduler. `run()`: starts loop. `gather()`: waits for tasks. `create_task()`: schedules. `time.sleep()` blocks loop; use `asyncio.sleep()` for non-blocking wait.

**P7** — **MRO**: Method Resolution Order. **Algorithm**: C3 Linearization. **Diamond**: D(B, C) -> B(A), C(A). MRO: D, B, C, A. `D().who()` calls `C.who()` because C appears before A in MRO.

**P8** — **Metaclass**: Class of a class. `__init_subclass__`: Hook for subclass creation (simpler). **Metaclass**: Enforce class-level constraints (e.g., registry). Use `__init_subclass__` for plugin registration.

**P9** — **Multiprocessing**: Processes, bypass GIL, heavy (pickle). **Threading**: Threads, GIL, shared memory. **Concurrent.futures**: High-level API. Use MP for CPU, Thread for I/O. Pickling limits object types.

**P10** — `__slots__`: Fixed attributes, no `__dict__`. **Use**: Save memory, prevent dynamic attributes. **Impact**: Reduces memory per instance significantly.

**P11** — **Venv**: Isolated environment. **Why**: Dependency conflicts. **venv**: Stdlib (Python 3.3+). **virtualenv**: 3rd party (older). **conda**: Package manager + env (data science).

**P12** — **Comprehensions**: Concise loops. **map/filter**: Functional. **Prefer**: Comprehensions for readability. Use `map/filter` for lazy evaluation or complex logic.

**P13** — `==`: Value equality. `is`: Identity (memory address). **Interning**: Small ints cached. `a is b` True for small ints due to shared memory.

**P14** — `*args`: Tuple of positional. `**kwargs`: Dict of keywords. `/`: Positional-only. `*`: Keyword-only. Used for flexible API signatures.

**P15** — **Context Manager**: Setup/teardown. `__enter__/__exit__`: Class-based. `@contextmanager`: Generator-based. **Example**: `Lock`, `TempDir`.

**P16** — **Type Hints**: Static analysis. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. **mypy/pyright**: Check types before runtime to catch errors.

**P17** — **Dunder**: `__repr__` (dev), `__str__` (user). `__eq__` vs `__hash__`: If `__eq__` defined, `__hash__` becomes `None` (unhashable) unless defined. `__getattr__` (missing attr), `__getattribute__` (all attrs).

**P18** — **Descriptors**: Control attribute access (`__get__`, `__set__`, `__delete__`). **property**, `classmethod`, `staticmethod` use descriptors. Used for validation or computed properties.

**P19** — **Import**: `sys.path` search. **Relative**: `from . import`. **Absolute**: `from pkg import`. `__init__.py`: Marks package (Python 3.3+ namespace packages omit it). **Finder/Loader**: Find module, load code.

**P20** — **Dataclasses**: Auto `__init__`, `__repr__`. **vs attrs/Pydantic**: attrs (more features), Pydantic (validation). **Pydantic**: Choose for API models needing validation (e.g., FastAPI). `__post_init__`: Run after init. `frozen`: Immutable.
