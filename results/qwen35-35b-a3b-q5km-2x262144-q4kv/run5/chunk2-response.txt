K1 — A **Deployment** manages stateless applications, ensuring a desired number of pod replicas are running. It allows for rolling updates and rollbacks. A **StatefulSet** manages stateful applications, providing stable, unique network identifiers (hostnames), ordered deployment/termination, and stable persistent storage. Use Deployments for web servers; use StatefulSets for databases or clustered apps requiring stable IDs.

K2 — A **ClusterIP** Service creates a virtual IP within the cluster. The **kube-proxy** component (using iptables or IPVS) updates the node's routing rules to forward traffic destined for the Service IP to the Pod IPs listed in the Service's **Endpoints** object. This allows pods to communicate via the Service IP without knowing individual Pod IPs.

K3 — A **PersistentVolume (PV)** is a cluster-level storage resource (e.g., NFS, cloud disk). A **PersistentVolumeClaim (PVC)** is a user request for storage (e.g., "10Gi"). A **StorageClass** defines the provisioning policy (e.g., "fast SSD"). They relate as follows: A PVC requests storage; if a StorageClass is specified, Kubernetes dynamically provisions a PV matching the class and binds it to the PVC.

K4 — 1. **Check Logs:** `kubectl logs <pod>` to see application errors. 2. **Check Events:** `kubectl describe pod <pod>` for OOMKilled, CrashLoopBackOff reasons, or probe failures. 3. **Check Resources:** Verify if memory/CPU limits are too low causing OOM kills. 4. **Check Dependencies:** Ensure external dependencies (DB, API) are reachable from the pod network.

K5 — **CoreDNS** provides DNS resolution for services and pods within the cluster. If CoreDNS pods are down, pods cannot resolve internal service names (e.g., `my-service.default.svc`) or external domains, causing connection failures for any application relying on DNS.

K6 — The scheduler uses **Filtering** (Predicate) to eliminate nodes that don't meet requirements (e.g., insufficient resources, taints). It then uses **Scoring** to rank remaining nodes. Factors considered: 1. **Resource Requests** (CPU/Mem fit). 2. **Node Affinity/Anti-Affinity** (placement rules). 3. **Taints and Tolerations** (node restrictions).

K7 — **Yes, it will be scheduled.** Node has 4GB (4096Mi), 3.5GB (3584Mi) requested. Available = 512Mi. Pod request is 256Mi, which fits. **At runtime:** If the pod uses 600Mi, it exceeds the `limits.memory: 512Mi`. The OOM Killer will terminate the container, causing a CrashLoopBackOff.

K8 — 1. **External LB** forwards traffic to the Ingress Controller's NodePort/LoadBalancer IP. 2. **Ingress Controller** (NGINX) parses the Ingress resource rules (host/path). 3. **Service** routes traffic to the backend **Endpoints** (Pod IPs). 4. **kube-proxy** (iptables/IPVS) on the node ensures packets reach the correct Pod IP. 5. **Pod** receives the request.

K9 — A **NetworkPolicy** restricts traffic flow to/from pods. **Default behavior:** If no policy exists, all traffic is allowed (ingress and egress). **When applied:** Traffic is restricted *only* for pods selected by the policy. Other pods in the namespace remain unaffected unless a policy selects them.

K10 — **RollingUpdate** replaces pods gradually (maxSurge/maxUnavailable), ensuring zero downtime. **Recreate** kills all existing pods before creating new ones. Choose **Recreate** for stateful applications where simultaneous instances are not supported, or when configuration changes require a full restart to take effect.

K11 — A **Namespace** provides a logical partition for resources within a cluster (isolation). **Default namespaces:** `default`, `kube-system`, `kube-public`, `kube-node-lease`.

K12 — `kubectl describe` shows detailed state and events. Unlike `get`, it shows: 1. **Events** (warnings, errors). 2. **Conditions** (Ready status details). 3. **Mounts** (Volumes, ConfigMaps, Secrets attached).

K13 — A **ConfigMap** stores non-confidential configuration data as key-value pairs. Pods consume it via: 1. **Environment Variables** (injected into container env). 2. **Volume Mounts** (mounted as files in the container filesystem).

K14 — **ClusterIP:** Internal virtual IP (default). **NodePort:** Exposes service on a static port on every Node's IP. **LoadBalancer:** Provisions an external cloud Load Balancer to expose the service to the internet.

K15 — A **DaemonSet** ensures a copy of a pod runs on all (or specific) nodes. Use it for node-level agents like logging collectors (Fluentd), monitoring agents (Prometheus Node Exporter), or network plugins (Calico).

K16 — **Taints** are applied to nodes to repel pods (unless they have a **Toleration**). **Node Affinity** attracts pods to nodes. **Use Case:** Taint a node for GPU workloads (only pods with GPU tolerations run there). Use Affinity to co-locate pods on the same zone for low latency.

K17 — A **Kubernetes Operator** is a custom controller that manages the lifecycle of a specific application (e.g., database backup, scaling). It differs from a standard controller by encapsulating domain-specific knowledge (CRDs + Logic). It solves operational complexity for stateful, complex applications.

K18 — etcd uses **Raft consensus** for strong consistency. If quorum is lost (e.g., 2 of 3 nodes down), the cluster becomes read-only/unavailable. **Recovery:** Restore etcd from a snapshot on a new node, remove the dead nodes from the cluster membership, and rejoin the new node to restore quorum.

K19 — **RBAC** controls access. **Role/ClusterRole** define permissions (verbs/resources). **RoleBinding/ClusterRoleBinding** assign roles to users/groups.
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: pod-reader
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list", "watch"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: read-pods-binding
  namespace: default
subjects:
- kind: User
  name: developer
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io
```

K20 — **ndots** defines how many dots a name must have to be treated as absolute. Default is 5. If a name has fewer dots (e.g., `google.com`), the resolver tries search domains first (e.g., `google.com.default.svc.cluster.local`), causing multiple queries. **Fix:** Set `dnsConfig: options: [{name: ndots, value: "1"}]` to query the FQDN immediately.

D1 — **Rebase** rewrites history to create a linear commit path (changes are applied on top). **Merge** creates a new commit preserving history. **Avoid rebasing** on public/shared branches to prevent disrupting other developers' history.

D2 — **TCP** is connection-oriented, reliable (ordered delivery). **UDP** is connectionless, unreliable (faster). **TCP:** HTTP, SSH. **UDP:** DNS, VoIP.

D3 — A **race condition** occurs when the output depends on the sequence/timing of uncontrollable events. **Example:** Two threads increment a shared counter `x++` simultaneously. Without a lock, both read `x`, increment, and write back, resulting in `x` increasing by 1 instead of 2.

D4 — **Symmetric** uses one key for encryption/decryption. **Asymmetric** uses a public/private key pair. **TLS:** Asymmetric keys exchange a session key securely; Symmetric keys encrypt the actual data transfer for performance.

D5 — A **layer** is a read-only filesystem diff in a Docker image. **Caching:** Docker caches layers. If a layer changes, subsequent layers are rebuilt. **Best Practice:** Copy dependency files (e.g., `package.json`) before copying source code to cache dependencies and speed up builds.

D6 — 1. **Check Load Balancer:** Verify upstream health checks. 2. **Check App Logs:** Look for exceptions or timeouts. 3. **Check Resources:** Verify CPU/Memory limits (OOM). 4. **Check Dependencies:** Verify DB or external API connectivity.

D7 — **CAP Theorem:** A distributed system can only guarantee 2 of 3: Consistency, Availability, Partition Tolerance. **CP:** Zookeeper (guarantees consistency, may block during partition). **AP:** Cassandra (guarantees availability, may return stale data during partition).

D8 — **Horizontal Scaling:** Adding more nodes/machines. **Vertical Scaling:** Adding more resources (CPU/RAM) to existing nodes. **Patterns:** Stateless services, sharding, and load balancing enable horizontal scaling.

D9 — An **environment variable** is a dynamic value affecting process behavior. To persist across sessions on Linux, export it in `~/.bashrc` or `~/.profile` (e.g., `export MY_VAR=value`).

D10 — **JSON** is a strict key-value format. **YAML** is a human-readable data serialization format. **JSON Advantage:** Standard for APIs. **YAML Advantage:** Supports comments and indentation for config files.

D11 — **REST API** is an architectural style using HTTP methods. **GET:** Retrieve data. **POST:** Create resource. **PUT:** Update/Replace resource. **DELETE:** Remove resource.

D12 — **Version pinning** specifies exact dependency versions (e.g., `package==1.0.0`). **Importance:** Ensures reproducible builds across environments and prevents breaking changes from unexpected updates.

D13 — **Compiled:** Source code is converted to binary before execution (e.g., C, Go). **Interpreted:** Source code is executed line-by-line at runtime (e.g., Python, JavaScript).

D14 — **Reverse Proxy:** Receives client requests and forwards them to backend servers (e.g., Nginx, HAProxy). **Forward Proxy:** Acts on behalf of clients to access external resources. **Use Case:** Nginx (SSL termination), HAProxy (Load balancing).

D15 — A **deadlock** is a situation where processes wait indefinitely for each other. **Coffman Conditions:** 1. Mutual Exclusion. 2. Hold and Wait. 3. No Preemption. 4. Circular Wait.

D16 — **Eventual Consistency:** Updates propagate asynchronously; data becomes consistent over time. **Cassandra:** Uses **Read Repair** and **Tunable Consistency Levels** (e.g., ONE, QUORUM, ALL) to balance latency and consistency.

D17 — **mTLS** requires both client and server to present certificates. **Handshake:** Client Hello -> Server Hello -> Server Cert -> Client Cert -> Verify -> Key Exchange. **Use Case:** Service Mesh (Istio) for secure service-to-service communication.

D18 — A **hash table** maps keys to indices via a hash function. **Collision Resolution:** **Chaining** (linked lists at index) or **Open Addressing** (probe for next slot). **Complexity:** Avg O(1) for insert/lookup/delete. Worst O(n) if all keys collide.

D19 — **Circuit Breaker** prevents cascade failures. **States:** **Closed** (normal), **Open** (fail fast, no calls), **Half-Open** (test if service recovered). **Relation:** If circuit is Open, retries are skipped to prevent overloading the failing service.

D20 — **Optimistic:** Assumes no conflict; checks version on commit (e.g., `SELECT ... FOR UPDATE` is not used). **Pessimistic:** Locks data immediately (e.g., `SELECT ... FOR UPDATE`). **Choose Optimistic** for high concurrency/low conflict. **Choose Pessimistic** for critical data with high conflict risk.
