JS1 — **var** is function-scoped and hoisted (initialized as `undefined`). **let** and **const** are block-scoped and hoisted but in a Temporal Dead Zone (TDZ) until declaration. **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages asynchronous execution. The **call stack** executes synchronous code. When the stack is empty, the loop checks the **microtask queue** (Promises, `queueMicrotask`) first, then the **macrotask queue** (`setTimeout`, `setInterval`). Microtasks drain completely before the next macrotask.

JS3 — A **closure** is a function bundled with its lexical environment. Example: A factory function returning an inner function accessing outer variables. **For loop bug:** `var` in a `for` loop creates one shared variable; all callbacks reference the final value. **Fix:** Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents an async result. **async/await** is syntactic sugar for Promise chaining. Error handling: `try/catch` with `await`, or `.catch()` on Promises. **Promise.all** fails fast on first rejection. **Promise.allSettled** waits for all to settle (fulfilled or rejected), returning status objects.

JS5 — **interface** defines shape, supports declaration merging, and is used for classes. **type** defines aliases, supports primitives, unions, intersections. Use **interface** for public API contracts/classes. Use **type** for unions/complex shapes. **Discriminated unions** are unions with a literal type property used to narrow types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — **Prototypal inheritance** links objects via a `__proto__` chain. **Object.create(proto)** creates a new object with `proto` as its prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` to set up the prototype chain and `constructor` for initialization.

JS7 — **this** binding rules (priority): 1. **New** (constructor call), 2. **Explicit** (`call`, `apply`, `bind`), 3. **Implicit** (object method call), 4. **Default** (global/window or undefined in strict mode). **bind()** returns a new function with bound `this`. It **cannot** be re-bound.

JS8 — **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** notifies when an object is collected. Use for caches or cleanup. **WeakMap** keys must be objects because primitives are copied by value; objects are referenced, allowing GC to collect keys if no other references exist.

JS9 — **V8** optimizes via **hidden classes** (object shapes) and **inline caching** (caching property access locations). **Deoptimization** occurs when hidden classes change (e.g., adding properties dynamically). Patterns causing this: dynamic property addition, deleting properties, or polymorphic behavior in tight loops.

JS10 — **Tree-shaking** removes unused exports. **ES Modules** (static `import/export`) allow static analysis for tree-shaking. **CommonJS** (`require`) is dynamic, preventing static analysis, so bundlers cannot safely remove unused code.

JS11 — **null** is intentional absence of value; **undefined** is unassigned. `==` converts types (`null == undefined` is true); `===` checks type and value. **Nullish coalescing (`??`)** returns right operand if left is `null` or `undefined`. **Optional chaining (`?.`)** safely accesses nested properties without throwing errors.

JS12 — **map()** transforms elements. **filter()** selects elements. **reduce()** accumulates values. **Initial value** in `reduce()` prevents `undefined` on first iteration. **flatMap()** maps then flattens one level; useful for transforming arrays of arrays into a single array.

JS13 — **Template literals** use backticks for multi-line strings and interpolation (`${}`). **Tagged templates** call a function with the string parts and values. Example: `sql`tag`SELECT * FROM users WHERE id = ${id}`; the function can escape values to prevent SQL injection.

JS14 — **Destructuring** extracts values from objects/arrays. **Object:** `{ a, b }`. **Array:** `[a, b]`. **Defaults:** `{ a = 1 }`. **Renaming:** `{ a: b }`. **Rest operator:** `{ ...rest }` collects remaining properties/elements into an array/object.

JS15 — **for...in** iterates over enumerable property keys (including inherited). **for...of** iterates over iterable values. Avoid **for...in** on arrays because it includes non-index properties and order is not guaranteed.

JS16 — **Generics** use type parameters (e.g., `T`). **Constraints:** `T extends SomeType`. **Conditional types:** `T extends U ? X : Y`. **infer:** Extracts a type from a type (e.g., `infer ReturnType`). Example: `type UnwrapPromise<T> = T extends Promise<infer U> ? U : T`.

JS17 — **CommonJS** (`require`) is synchronous, dynamic, and object-based. **ES Modules** (`import`) are static, asynchronous, and tree-shakeable. **Node.js** supports both; `type: "module"` in `package.json` treats `.js` files as ESM by default.

JS18 — **Structural typing** checks shape compatibility, not name. **Excess property checks** apply to object literals assigned to types to prevent typos. **extends** checks assignability; **satisfies** checks type without widening the inferred type. **Variance:** `in` (contravariant), `out` (covariant) control type parameter usage in generics.

JS19 — **Proxy** intercepts operations (get, set, etc.). **Reflect** provides default implementations. Example: A Proxy can validate property assignments before setting them. `Reflect.set(target, prop, value)` is the default behavior.

JS20 — **SharedArrayBuffer** allows memory sharing between threads. **Atomics** provide atomic operations (add, wait, notify). Workers communicate via `postMessage` or shared memory. **Ordering guarantees:** Memory barriers ensure visibility. **Atomics.wait** blocks until **Atomics.notify** wakes it.

B1 — **$@** expands to individual quoted arguments (`"$1" "$2"`). **$*** expands to all arguments as a single string (`"$1 $2"`). **"$@"** preserves argument boundaries (safe for loops). **"$*"** joins them with the first character of IFS.

B2 — **set -e**: Exit immediately if a command exits non-zero. **-u**: Treat unset variables as errors. **-o pipefail**: Return exit status of the last command in a pipeline that fails.

B3 — **$(command)** is modern command substitution (nested). **`command`** is legacy. **$((expression))** is arithmetic expansion (shell built-in). **expr** is an external command for arithmetic (slower, less feature-rich).

B4 — **Process substitution** `<(command)` creates a file descriptor pointing to the command's output. Use when a command expects a filename but needs dynamic input. Example: `diff <(sort a.txt) <(sort b.txt)` compares sorted files without creating temp files.

B5 — Use globbing (`for f in *`) or `find`. **`for f in $(ls)`** is dangerous because it splits on whitespace/newlines, breaking filenames with spaces. Quote variables: `for f in "$@"`.

B6 — **Here document (`<<EOF`)** reads until EOF line. **Here string (`<<<`)** passes a string as input. **`<<-EOF`** strips leading tabs from the input lines.

B7 — `cmd > file 2>&1`: Stdout to file, Stderr to Stdout (file). `cmd 2>&1 > file`: Stderr to Stdout (terminal), Stdout to file. `cmd &> file`: Same as first (shorthand). First two differ because redirection order matters.

B8 — Use `mkdir` for locking (atomic). Function: `mkdir /tmp/lock` (fails if exists). Trap `EXIT` to `rmdir`. Trap `TERM` to cleanup. `mkdir` is preferred because it's atomic on POSIX filesystems, preventing race conditions.

B9 — **Single quotes**: Literal string (no expansion). **Double quotes**: Expand variables/commands. **No quotes**: Word splitting and globbing. Example: `echo $HOME` (no quotes) expands; `echo '$HOME'` (single) prints literal.

B10 — **`[ -e file ]`** (exists), **`[ -d dir ]`** (directory), **`[ -r file ]`** (readable). **`[[ ]]`** is Bash-specific, supports regex (`=~`), and handles spaces better. **`[ ]`** is POSIX.

B11 — **`$?`**: Exit status of last command. **`$$`**: PID of current shell. **`$!`**: PID of last background job. **`$#`**: Number of arguments passed to script.

B12 — Define: `func() { ... }` or `function func { ... }`. Return values: `return` sets exit code; `echo` prints to stdout (captured via `$()`).

B13 — **PATH**: Colon-separated list of directories. Bash searches PATH for external commands. **Builtin**: Shell internal (e.g., `cd`). **External**: Binary file. Check with `type` or `which`.

B14 — **trap** sets handlers. `trap 'cleanup' EXIT`. `trap 'cleanup' INT TERM`. Execution order: Handlers run on signal arrival. Multiple signals: Last one registered often wins, or signals are queued.

B15 — **xargs** builds command lines from stdin. **command substitution** runs command immediately. **`xargs -0`** with **`find -print0`** handles null-delimited filenames (spaces/newlines). **`xargs -P`** runs jobs in parallel.

B16 — **Arrays**: `arr=(1 2)`. **Associative**: `declare -A assoc`. Append: `arr+=(val)`. Length: `${#arr[@]}`. Iterate: `for i in "${arr[@]}"`. Assoc keys: `${!assoc[@]}`.

B17 — **Subshell (`(...)`)**: New process, variables don't persist. **Group (`{...}`)**: Same shell, variables persist. **Pipe into while**: Runs in subshell, so variable changes inside loop are lost after loop.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (terminal close), **SIGKILL** (force kill, no handler). Graceful shutdown: Trap signals, cleanup resources, `wait` for background jobs before exit.

B19 — Script: `split -l 1000 log.log part_`. `xargs -P 4 -I {} sh -c 'process {}'`. Merge results: `cat part_* > merged.log`. Trade-offs: Parallelism vs I/O contention. Error propagation: Check exit codes of parallel jobs, aggregate failures.

B20 — **exec** replaces the current shell process. **FD manipulation**: `exec 3>file` opens FD 3. `exec 3>&-` closes it. **Coprocess**: `coproc CMD` creates bidirectional pipe. Use for persistent communication with external processes.

PS1 — **Write-Output** sends objects to the pipeline. **Write-Host** writes directly to the console (bypasses pipeline). Distinction matters because `Write-Host` output cannot be captured or piped.

PS2 — **Pipeline** passes objects. **ForEach-Object** (cmdlet) processes pipeline items one-by-one. **foreach** (statement) iterates a collection in memory. Use `ForEach-Object` for streaming data; `foreach` for in-memory collections.

PS3 — **[PSCustomObject]** creates a typed object with properties. **[hashtable]** is a key/value collection. Convert: `@{Key='Value'} | ConvertTo-Object` or `[PSCustomObject]@{}`. Use CustomObject for structured data; Hashtable for dynamic config.

PS4 — **Terminating error** stops execution (throw). **Non-terminating** records error but continues. **-ErrorAction** overrides `$ErrorActionPreference`. **try/catch** handles terminating errors.

PS5 — **Splatting** (`@params`) passes a hashtable as individual parameters. **Hashtable param** passes the whole hashtable as one argument. Splatting allows dynamic parameter passing.

PS6 — **Invoke-Command** runs commands on remote machines (one-off). **Enter-PSSession** enters an interactive session. Uses **WinRM** (HTTP/HTTPS) or **SSH** (v7+).

PS7 — Pipeline type inference can be inconsistent. `$result += "one"` on `@()` creates `String[]`. If function returns 0 items, `$x` is empty array (`Object[]`). **Correct pattern**: Always return an array explicitly (`return @()`) to ensure consistent type.

PS8 — **[scriptblock]** is a code block object. Pass to remote via `-ArgumentList` or `-ScriptBlock`. **Double-hop**: Remote server cannot authenticate to a third server. **Solve**: Enable CredSSP or use `-ArgumentList` to pass credentials securely.

PS9 — **@()** creates an array. **[array]** casts to array. **Indexing `$null`**: Returns `$null`. **Indexing `@()`**: Returns empty array. Force single result: `$x = @($result)[0]`.

PS10 — **-eq** (equal), **-like** (wildcard), **-match** (regex), **-contains** (left operand contains right), **-in** (left operand in right). Case-sensitive variants: **-ceq**, **-clike**, **-cmatch**. **-contains** checks if array contains value; **-in** checks if value is in array.

PS11 — **Module**: Reusable code. **.psm1** (script), **.psd1** (manifest), **.dll** (binary). **Import-Module** loads module into session. Manifest defines metadata (version, author).

PS12 — **Interpolation**: `"Hello $name"`. **'...'**: Literal. **Here-string**: `@"..."@` for multi-line. Include expressions: `$var` inside double quotes.

PS13 — **Get-Member** lists properties/methods. **Properties** (data), **Methods** (actions), **NoteProperties** (added dynamically). **Select-Object** selects properties. **Where-Object** filters objects.

PS14 — **Classes** (v5) define structure. Limitations: No private members, limited inheritance (single), constructors must be `function`. Use classes for complex state; functions for simple logic.

PS15 — **[CmdletBinding()]** enables advanced features. **SupportsShouldProcess** enables `-WhatIf`/`-Confirm`. **Begin/Process/End**: Setup, stream processing, cleanup. **ValueFromPipeline** allows pipeline input.

PS16 — **Providers** expose data as file system. **Get-ChildItem** works on Registry (`HKLM:\`), Certs (`Cert:\`), Env. Access registry: `Get-ChildItem HKLM:\Software`.

PS17 — **Runspaces**: Isolated execution environments. **RunspacePool**: Reuse threads. **ForEach-Object -Parallel** (v7+) uses runspace pool. Manual management: `[PowerShell]::Create()`.

PS18 — **PSScriptAnalyzer**: Static code analysis. **Rules**: Enforce standards. **Severity**: Error, Warning, Info. **Suppress**: `#pragma disable`. **CI/CD**: Integrate via `Invoke-ScriptAnalyzer`.

PS19 — **$using:** scope passes local variables to remote blocks. **$args/-ArgumentList**: Passes arguments. **Serialization limits**: CLIXML depth limits complex objects. Use simple types or serialize to JSON for complex data.

PS20 — **DSC**: Configuration management. **Configurations**: Desired state. **Resources**: Actions. **LCM**: Applies config. **WinPS vs PS7**: PS7 uses DSC v3 (cross-platform). **Replaced by**: Ansible, Terraform, or native PS7 DSC modules in modern IaC.
