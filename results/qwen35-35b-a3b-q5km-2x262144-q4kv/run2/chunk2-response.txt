## Kubernetes

**K1** — A **Deployment** manages stateless applications, ensuring a desired number of identical pod replicas are running. A **StatefulSet** manages stateful applications, providing stable, unique network identifiers (hostnames), ordered deployment/termination, and stable persistent storage.

**K2** — A **ClusterIP** Service assigns a virtual IP within the cluster. **kube-proxy** (using iptables or IPVS) updates network rules on nodes to route traffic sent to the Service IP to the backend Pod IPs based on the Service selector.

**K3** — **PersistentVolume (PV)** is a cluster storage resource. **PersistentVolumeClaim (PVC)** is a user request for storage. **StorageClass** defines the provisioning mechanism (dynamic). A PVC binds to a PV; if no PV exists, the StorageClass triggers dynamic provisioning.

**K4** — 1. Check `kubectl logs` for application errors. 2. Check `kubectl describe pod` for Liveness Probe failures or OOMKilled events. 3. Verify resource limits vs. actual usage. 4. Check image pull errors or ConfigMap/Secret mounting issues.

**K5** — CoreDNS provides DNS service discovery for the cluster. If CoreDNS pods are down, internal service discovery (e.g., `svc.namespace`) fails. External DNS resolution may still work if the pod's `/etc/resolv.conf` points to external nameservers, but internal cluster communication breaks.

**K6** — The scheduler **filters** nodes (removing those that don't meet constraints) then **scores** remaining nodes. Factors: 1. Resource requests (CPU/Mem). 2. Node Affinity/Taints. 3. Pod Affinity/Topology Spread Constraints.

**K7** — Yes, it will be scheduled (4GB - 3.5GB = 0.5GB available; 256Mi < 0.5GB). If it uses 600Mi at runtime (exceeding 512Mi limit), the container will be **OOMKilled** by the kernel.

**K8** — 1. Request hits External LB. 2. LB forwards to Ingress Controller Pod. 3. Ingress Controller parses Ingress rules (host/path). 4. Traffic routed to Service ClusterIP. 5. kube-proxy/iptables directs to Pod IPs (Endpoints). 6. App Pod receives request.

**K9** — A **NetworkPolicy** restricts traffic flow between pods. Default behavior (no policy): All traffic allowed. When a policy is applied: Traffic is denied by default unless explicitly allowed by the policy rules.

**K10** — **RollingUpdate** replaces pods gradually (zero downtime). **Recreate** kills all existing pods before starting new ones. Choose **Recreate** for stateful applications where simultaneous instances cause conflicts (e.g., database migrations).

**K11** — A **Namespace** is a logical cluster partition for resource isolation. Default namespaces: `default`, `kube-system`, `kube-public`, `kube-node-lease`.

**K12** — Used for detailed debugging. Shows: 1. **Events** (scheduling, pull errors). 2. **Conditions** (Ready, PodScheduled). 3. **Mounts** (Volumes, ConfigMaps). `kubectl get` only shows Status, Age, Ready count.

**K13** — A **ConfigMap** stores non-sensitive configuration data. Pods consume it via: 1. **Environment Variables**. 2. **Volume Mounts** (files).

**K14** — **ClusterIP**: Internal cluster access only. **NodePort**: Exposes service on a static port on every Node's IP. **LoadBalancer**: Provisions an external cloud Load Balancer.

**K15** — A **DaemonSet** ensures a copy of a pod runs on all (or specific) nodes. Use for logging agents, monitoring daemons, or CNI plugins where node-level access is required, unlike Deployments which manage application replicas.

**K16** — **Taints** repel pods; **Tolerations** allow pods to schedule on tainted nodes. **Node Affinity** attracts pods to nodes. Use Case: Taint a GPU node (repel general pods); Affinity to co-locate app pods with cache pods.

**K17** — An **Operator** is a custom controller + Custom Resource Definition (CRD) that automates complex app management. It differs from a standard controller by managing the lifecycle of stateful applications (e.g., backups, upgrades) rather than just resource reconciliation.

**K18** — etcd uses **strong consistency (linearizable)**. If quorum is lost, the cluster becomes read-only. Recovery: Restore a snapshot on the remaining node, start new etcd nodes, and rejoin them to the cluster using the snapshot data.

**K19** — **Role** (namespace-scoped permissions), **ClusterRole** (cluster-scoped), **RoleBinding** (links Role to Subject in namespace), **ClusterRoleBinding** (links ClusterRole to Subject cluster-wide).
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata: namespace: dev
name: pod-reader
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list", "watch"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata: namespace: dev
name: read-pods
subjects:
- kind: User
  name: developer
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io
```

**K20** — `ndots` defines how many dots are needed to treat a name as FQDN. `ndots: 5` causes queries for `google.com` (1 dot) to append search domains, creating 5+ queries. Fix: Set `dnsConfig` in pod spec: `options: [{ name: "ndots", value: "1" }]`.

## Development

**D1** — **Rebase** rewrites history to create a linear commit chain. **Merge** preserves history with a merge commit. Avoid rebasing on shared/public branches to prevent history conflicts for others.

**D2** — **TCP** is connection-oriented and reliable. **UDP** is connectionless and faster. TCP: HTTP, SSH. UDP: DNS, VoIP.

**D3** — A **race condition** occurs when outcome depends on the sequence/timing of uncontrollable events. Example: Two threads incrementing a shared counter without a lock, resulting in lost updates.

**D4** — **Symmetric** uses one key for encryption/decryption. **Asymmetric** uses a public/private key pair. TLS uses Asymmetric for the handshake (key exchange) and Symmetric for the data transfer (performance).

**D5** — A **layer** is a filesystem diff in a container image. Layer caching improves build speed. Best practice: Order Dockerfile instructions so frequently changing files (e.g., `COPY .`) are at the bottom to preserve cached base layers.

**D6** — 1. Check upstream service health. 2. Verify database connectivity. 3. Check rate limiting/throttling logs. 4. Monitor resource exhaustion (CPU/Memory).

**D7** — **CAP Theorem**: A distributed system can only guarantee 2 of 3: Consistency, Availability, Partition Tolerance. **CP**: Zookeeper (guarantees consistency). **AP**: Cassandra (guarantees availability during partitions).

**D8** — **Horizontal Scaling** adds more nodes. **Vertical Scaling** adds more resources to a node. Patterns enabling horizontal scaling: Stateless services, Sharding, Load Balancing.

**D9** — An **environment variable** is a dynamic value affecting process behavior. To persist across sessions on Linux, export it in `~/.bashrc` or `~/.profile`.

**D10** — **JSON** is strict key-value pairs. **YAML** is human-readable with indentation. JSON advantage: Strict parsing. YAML advantage: Readability for config files.

**D11** — **REST** is an architectural style for APIs. **GET** (retrieve), **POST** (create), **PUT** (replace), **DELETE** (remove).

**D12** — **Version pinning** specifies exact dependency versions (e.g., `package==1.0.0`). Important for reproducible builds to prevent breakage from upstream updates.

**D13** — **Compiled** languages translate code to binary before execution. **Interpreted** languages execute code line-by-line. Compiled: C, Go. Interpreted: Python, JavaScript.

**D14** — A **reverse proxy** receives client requests and forwards them to backend servers. A **forward proxy** acts on behalf of clients to access the internet. Servers: Nginx (web server), HAProxy (load balancer).

**D15** — A **deadlock** is a situation where processes wait indefinitely for each other. Coffman Conditions: 1. Mutual Exclusion. 2. Hold and Wait. 3. No Preemption. 4. Circular Wait.

**D16** — **Eventual consistency** means updates propagate over time. Cassandra uses **Read Repair** and **Hinted Handoff**. Tunable levels: ONE, QUORUM, ALL.

**D17** — **mTLS** requires both client and server to present certificates. Handshake: Client Hello -> Server Hello -> Server Cert -> Client Cert -> Key Exchange. Used in: Service Meshes (Istio), Internal API security.

**D18** — A **hash table** maps keys to indices via a hash function. Collision resolution: **Chaining** (linked lists) or **Open Addressing** (probing). Time Complexity: Avg O(1), Worst O(n).

**D19** — A **circuit breaker** prevents cascade failures. States: **Closed** (normal), **Open** (fail fast), **Half-open** (test recovery). Relates to retry: Do not retry if circuit is open.

**D20** — **Optimistic** concurrency checks version on write (no lock). **Pessimistic** locks data on read. Choose Optimistic for high concurrency/low conflict. Choose Pessimistic for high conflict/critical data (e.g., `SELECT FOR UPDATE`).
