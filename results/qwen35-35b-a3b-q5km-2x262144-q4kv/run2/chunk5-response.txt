DN1 — **Class vs Struct**: `class` is a reference type (heap-allocated, mutable by default), `struct` is a value type (stack-allocated, immutable by convention). Use `struct` for small, immutable data (e.g., coordinates, IDs) to avoid heap allocation and copying overhead.

DN2 — **IEnumerable<T> vs IQueryable<T>**: `IEnumerable<T>` executes queries in-memory (client-side). `IQueryable<T>` builds expression trees translated to server-side queries (e.g., SQL). This matters for DB performance; `IQueryable` pushes filtering/sorting to the database, reducing data transfer.

DN3 — **Async/Await**: Compiles to a state machine. `Task` represents an async operation (heap-allocated). `ValueTask` avoids allocation if the result is already available (e.g., cached). Use `ValueTask` in hot paths where `Task` allocation is a bottleneck.

DN4 — **Dependency Injection**: Pattern for decoupling dependencies. **Transient**: New instance every request. **Scoped**: New instance per request (e.g., HTTP). **Singleton**: Single instance for app lifetime. **Bug**: Injecting a Scoped service into a Singleton causes the Singleton to hold a reference to the Scoped service, potentially leading to stale data or `ObjectDisposedException` when the scope ends.

DN5 — **string vs ReadOnlySpan<char>**: `string` is immutable, heap-allocated. `ReadOnlySpan<char>` is a stack-only ref struct allowing zero-copy, mutable views of memory. Use `Span<T>` for high-performance parsing or string manipulation to avoid allocations.

DN6 — **GC & Generations**: GC manages heap memory. **Gen 0**: Short-lived objects. **Gen 1**: Buffer. **Gen 2**: Long-lived. **LOH**: Large Object Heap (>85KB) for arrays/strings, collected less frequently. **IDisposable**: Required for unmanaged resources (files, DB connections) because GC only manages managed memory; `Dispose` ensures timely cleanup.

DN7 — **ConfigureAwait(false)**: Default behavior captures `SynchronizationContext` (e.g., UI thread) to resume on it. `ConfigureAwait(false)` avoids capturing context, resuming on thread pool. **Use**: In libraries or non-UI code. **Don't Use**: In UI/ASP.NET Core where context is needed (though ASP.NET Core doesn't use SynchronizationContext by default, legacy code might).

DN8 — **New Features**: **Primary Constructors**: `class Service(int Id) { }`. **Collection Expressions**: `[1, 2, 3]`. **FrozenDictionary**: Immutable, fast lookup. **Example**:
```csharp
public class Service(int Id) : IService { }
// DI: services.AddSingleton<Service>(new Service(1));
```

DN9 — **DbContext Exception**: Likely cause is `DbContext` registered as Singleton instead of Scoped. EF Core `DbContext` is not thread-safe. **Fix**: Register as `AddDbContext<DbContext>(ServiceLifetime.Scoped)`. High concurrency exposes race conditions if lifetime is wrong.

DN10 — **Record vs Class**: `record` provides value-based equality (positional) and `ToString()` by default. `class` uses reference equality. **Record Struct**: Value type with value equality. Choose `record` for DTOs/immutable data, `class` for mutable entities.

DN11 — **Ref/Out/In**: `ref`: Pass by reference, can modify. `out`: Pass by reference, must be assigned before return. `in`: Pass by reference, read-only (avoids copy of large structs).

DN12 — **Nullable Reference Types**: `string?` warns if null is assigned to non-nullable type (compile-time). `int?` is `Nullable<int>` (runtime value type). NRTs track reference nullability; value types use `Nullable<T>`.

DN13 — **LINQ**: Language Integrated Query. **Method**: `list.Where(...)`. **Query**: `from x in list where ...`. **Deferred Execution**: Query defined now, executed when enumerated (e.g., `ToList()`).

DN14 — **Generic Constraints**: Limits type parameters. `where T : class` (ref), `struct` (val), `new()` (paramless ctor), `IComparable<T>`. Combine with `and` (e.g., `where T : class, IComparable<T>`).

DN15 — **Abstract vs Interface**: `abstract` allows code + contract. `interface` is contract only. **C# 8+**: Default interface methods allow implementation in interfaces, blurring the line (interfaces can now have state via default methods).

DN16 — **Middleware Pipeline**: `app.Use()`: Adds middleware (calls `next`). `app.Map()`: Branches pipeline. `app.Run()`: Terminal handler. **Order**: Auth before Authz. **Exception Handling**: Middleware wraps pipeline to catch exceptions globally.

DN17 — **Channel<T>**: Async producer-consumer. `BlockingCollection<T>` is sync. **Bounded Channel**: Limits queue size. **Backpressure**: Producer blocks when channel is full, preventing memory exhaustion.

DN18 — **Source Generators**: Compile-time code generation. **Vs Reflection**: Faster, no runtime cost. **Vs IL Weaving**: No post-build assembly modification. **Incremental**: Watches files, regenerates only changed parts (faster build).

DN19 — **TPL Dataflow**: Pipeline blocks. `ActionBlock` (consume), `TransformBlock` (transform), `BufferBlock` (store). **Vs Channels**: Dataflow offers more flow control (linking, cancellation), Channels are simpler for basic pipelines.

DN20 — **Minimal APIs**: Delegate-based endpoints. **Vs Controllers**: Less boilerplate, no classes. **Trade-offs**: Less structure, harder to scale complex auth/areas. **Route Groups**: Organize endpoints (`app.MapGroup("/api")`). **Filters**: Apply logic (e.g., validation) to endpoints.

P1 — **List vs Tuple**: List is mutable, Tuple is immutable. **Set vs Frozenset**: Set is mutable, unique items; Frozenset is immutable, hashable.

P2 — **GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. **Affects**: CPU-bound threading. **Workarounds**: Multiprocessing, C-extensions, Asyncio.

P3 — **Deepcopy vs Copy**: `copy` creates shallow copy (nested objects shared). `deepcopy` creates recursive copy. **Example**: `list = [[1]]; copy = list.copy(); copy[0].append(2)` modifies original.

P4 — **Decorators**: Wrap functions. **Time Logger**:
```python
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        res = func(*args, **kwargs)
        print(f"{func.__name__}: {time.time()-start}s")
        return res
    return wrapper
```

P5 — **Generators**: Functions with `yield`. Return iterator, not list. Memory efficient. `yield from` delegates to another generator.

P6 — **Asyncio**: Event loop runs coroutines. `run()` (entry), `gather()` (wait all), `create_task()` (fire & forget). `time.sleep()` blocks loop; use `asyncio.sleep()`.

P7 — **MRO**: Method Resolution Order. Python 3 uses **C3 Linearization**. **Diamond**: `D(B, C)`. MRO: D, B, C, A. `D().who()` prints "C" because `C` overrides `who` and appears after `B` in MRO.

P8 — **Metaclass**: Class of a class. Controls class creation. `__init_subclass__`: Simpler hook for subclassing (e.g., auto-register). **Meta**: Complex control (e.g., ORM model registration).

P9 — **Multiprocessing/Threading/Futures**: **Multiprocessing**: CPU-bound, separate memory (pickling needed). **Threading**: I/O-bound, shared memory (GIL limits CPU). **Futures**: Abstraction for async execution.

P10 — **__slots__**: Prevents `__dict__` creation. Saves memory, prevents dynamic attributes. **Impact**: Reduces per-instance memory overhead significantly.

P11 — **Virtual Environment**: Isolates dependencies. **venv**: Stdlib (Python 3.3+). **virtualenv**: Older, external. **conda**: Env + package manager (binary packages).

P12 — **Comprehensions**: Concise loops. `map/filter` functional. **Prefer**: Comprehensions for readability, `map/filter` for lazy evaluation or functional pipelines.

P13 — **== vs is**: `==` compares value, `is` compares identity. **Interning**: Small ints (-5 to 256) cached. `a is b` True for small ints due to reuse.

P14 — **Args/Kwargs**: `*args` (tuple), `**kwargs` (dict). `/` (positional only), `*` (keyword only). Enforces signature constraints.

P15 — **Context Managers**: `with` statement. `__enter__/__exit__` (class) or `@contextmanager` (generator). **Example**: DB connection management.

P16 — **Type Hinting**: Static analysis. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. **Mypy/Pyright**: Check types before runtime to catch errors.

P17 — **Data Model**: Dunder methods. `__repr__` (dev), `__str__` (user). `__eq` sets `__hash__` to None unless defined. **Hash**: Required for dict/set keys.

P18 — **Descriptors**: Manage attribute access (`__get__`, `__set__`, `__delete__`). **property**, `classmethod`, `staticmethod` are descriptors.

P19 — **Import System**: **Finder**: Locates module. **Loader**: Loads module. `sys.path` search path. **Relative**: Based on current package. **Namespace**: No `__init__.py` (PEP 420).

P20 — **Dataclasses**: Auto-generates `__init__`, `__repr__`, etc. **Vs Pydantic**: Pydantic adds validation. **Use Pydantic**: For API models requiring validation. `__post_init__` runs after init.
