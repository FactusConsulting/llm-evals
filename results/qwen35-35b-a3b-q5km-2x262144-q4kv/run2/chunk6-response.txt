JS1 — **var** is function-scoped and hoisted (initialized as `undefined`). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages async execution. The **call stack** runs synchronous code. **Microtasks** (Promises, queueMicrotask) run after the call stack clears but before macrotasks. **Macrotasks** (setTimeout, I/O) run after the current macrotask completes. Microtasks have higher priority.

JS3 — A **closure** is a function retaining access to its outer scope. Example: Counter factory. **Bug**: `var` in `for` loops shares one variable reference. **Fix**: Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents async completion. **async/await** is syntactic sugar for promises. **Error handling**: `try/catch` with await, or `.catch()` on promises. **Promise.all** fails on first rejection; **Promise.allSettled** waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 — **Interface** defines shape, supports merging, and is erased at runtime. **Type** is a union/intersection of types, supports primitives. Use **interface** for public API contracts, **type** for unions/complex shapes. **Discriminated unions** use a literal property (tag) to narrow types in `switch`/`if`.

JS6 — **Prototypal inheritance** uses a prototype chain for property lookup. **Object.create(proto)** creates an object with `proto` as its prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` to set up the prototype chain and `constructor` for initialization.

JS7 — **this** depends on call site. **Binding rules**: 1. `new` (new instance), 2. Explicit (`call`/`apply`/`bind`), 3. Implicit (`obj.method()`), 4. Default (`window`/`undefined`). **bind()** returns a new function with bound context. **Cannot re-bind** a bound function.

JS8 — **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** runs cleanup when a weakly held object is collected. **WeakMap** keys must be objects (GC-safe). This allows memory management for caches without preventing garbage collection.

JS9 — **Hidden classes** optimize property access by grouping objects with same shape. **Inline caching** caches method lookup results. **Deoptimization** occurs when hidden classes change (e.g., adding properties dynamically). Patterns causing deopt: dynamic property addition, changing object shape.

JS10 — **Tree-shaking** removes unused exports. **ES Modules** allow static analysis (static `import`), enabling tree-shaking. **CommonJS** (`require`) is dynamic, preventing static analysis, so tree-shaking is ineffective.

JS11 — **null** is intentional absence; **undefined** is uninitialized. **==** converts types (null == undefined is true); **===** checks type and value. **??** (nullish coalescing) returns right side if left is null/undefined. **?.** (optional chaining) safely accesses nested properties.

JS12 — **map()** transforms elements. **filter()** selects elements. **reduce()** accumulates values. **Initial value** in reduce prevents using first element as accumulator. **flatMap()** maps then flattens one level; useful for transforming arrays of arrays.

JS13 — **Template literals** use backticks for multi-line strings and interpolation `${}`. **Tagged templates** call a function with string parts and values. Example: `html` tag to escape HTML or build SQL queries safely.

JS14 — **Destructuring** extracts values from objects/arrays. **Object**: `{a, b}`. **Array**: `[a, b]`. **Defaults**: `{a = 1}`. **Renaming**: `{a: b}`. **Rest operator** (`...`) gathers remaining elements into an array.

JS15 — **for...in** iterates over enumerable keys (objects). **for...of** iterates over values (iterables). Avoid **for...in** on arrays because it iterates keys, includes non-index properties, and is slower.

JS16 — **Generics** use type parameters (e.g., `T`). **Constraints** (`extends`) limit types. **Conditional types** (`T extends U ? X : Y`) select types based on condition. **infer** extracts types from conditional types. Example: `ReturnType<T>`.

JS17 — **CommonJS** (`require`) is synchronous, dynamic, Node.js default. **ES Modules** (`import`) are static, async, browser standard. Node.js supports both. **`"type": "module"`** in `package.json` forces Node to treat `.js` files as ES Modules.

JS18 — **Structural typing** checks shape, not name. **Excess property checks** apply to object literals assigned to types. **extends** creates subtype; **satisfies** checks type without widening. **Variance** (`in`/`out`) controls covariance/contravariance in generics (TS 4.7+).

JS19 — **Proxy** intercepts operations (get, set, etc.). **Reflect** provides default implementations. Example: Validation Proxy intercepts `set` to validate values before assignment.

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic ops. **Ordering guarantees** ensure visibility across threads. **Atomics.wait** blocks until notified; **Atomics.notify** wakes waiting threads. Requires cross-origin isolation headers.

B1 — **$@** expands to separate quoted arguments. **$*** expands to a single string. **"$@"** preserves individual arguments (safe for loops). **"$*"** joins them into one string (loses argument boundaries).

B2 — **`set -e`**: Exit on error. **`-u`**: Exit on undefined vars. **`-o pipefail`**: Pipeline fails if any command fails. Ensures strict error handling.

B3 — **$(command)** is modern command substitution. **`command`** is legacy. **$((expression))** is arithmetic expansion (shell built-in). **expr** is an external command (slower).

B4 — **Process substitution** `<(command)` creates a file descriptor. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — **Safe iteration**: `for f in *.txt`. **`for f in $(ls)`** is dangerous because `ls` output parsing breaks on spaces/newlines in filenames. Use globbing or `find`.

B6 — **Here document** `<<EOF` reads stdin until EOF. **Here string** `<<<` passes a string. **`<<-EOF`** strips leading tabs from the heredoc content.

B7 — **`cmd > file 2>&1`**: Stdout to file, stderr to stdout (file). **`cmd 2>&1 > file`**: Stderr to stdout (terminal), stdout to file. **`&> file`**: Both to file (bash specific). Order matters because redirections are processed left-to-right.

B8 — Use `mkdir` for locking (atomic on POSIX). Function:
```bash
lock() {
  mkdir "$1.lock" 2>/dev/null && trap 'rmdir "$1.lock"' EXIT
}
```
`mkdir` fails if lock exists. `trap` ensures cleanup on exit.

B9 — **'...'**: No expansion. **"..."**: Expansion ($, `, \). **No quotes**: Word splitting and globbing. Example: `echo $HOME` (expands), `echo '$HOME'` (literal), `echo $HOME` (unquoted).

B10 — **`[ -e file ]`** exists, **`-d`** directory, **`-r`** readable. **`[[ ]]`** supports regex, is safer, handles spaces better. `[` is a command, `[[` is a keyword.

B11 — **`$?`**: Exit code. **`$$`**: Current PID. **`$!`**: Last background PID. **`$#`**: Number of arguments.

B12 — **`foo() { ... }`** or **`function foo { ... }`**. Return values via `exit` code or `echo` (captured). `return` only works in functions.

B13 — **PATH** is colon-separated dirs. Bash searches PATH for external commands. **Builtin** (e.g., `cd`) is internal. **`type command`** shows which is used.

B14 — **`trap`** sets handlers. **`trap 'cmd' EXIT`** runs on exit. **`trap 'cmd' INT TERM`** on signals. Order: EXIT handlers run last.

B15 — **xargs** builds command lines. **`find -print0 | xargs -0`** handles spaces/newlines. **`-P`** runs jobs in parallel.

B16 — **Arrays**: `arr=(a b)`. **Associative**: `declare -A assoc`. **Iterate**: `for i in "${arr[@]}"`. **Length**: `${#arr[@]}`.

B17 — **Subshell** `(...)` creates new process (vars lost). **Group** `{...}` same shell. **Pipe** `| while` runs loop in subshell, so vars don't persist outside.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (force, no catch). **Graceful shutdown**: `trap 'cleanup' TERM` and wait for background jobs.

B19 — Script: `split`, `xargs -P 4 -I {} process {}`, `cat results`. **Trade-offs**: Parallel I/O vs CPU. **Error propagation**: Check exit codes in loop, aggregate errors.

B20 — **exec** replaces shell process. **FD manipulation**: `exec 3>file` opens fd 3. **Coprocess**: `coproc CMD` creates pipe. Use for persistent background communication.

PS1 — **Write-Output** sends objects to pipeline. **Write-Host** writes to screen (bypasses pipeline). Pipeline breaks if Write-Host is used (objects not passed downstream).

PS2 — **Pipeline** streams objects. **ForEach-Object** (cmdlet) processes each object. **foreach** (statement) iterates collection. Use **ForEach-Object** for pipeline, **foreach** for in-memory collections.

PS3 — **[PSCustomObject]** is a typed object. **[hashtable]** is untyped key/value. Convert: `[PSCustomObject]@{Key="Value"}`. Use **PSCustomObject** for structured data, **hashtable** for dynamic config.

PS4 — **Terminating** (throw) stops execution. **Non-terminating** (continue) logs error. **-ErrorAction** overrides preference. **try/catch** catches terminating errors.

PS5 — **Splatting** `@params` passes hashtable as named arguments. Differs from passing hashtable as single param (which treats it as one object).

PS6 — **Invoke-Command** runs one-off remote. **Enter-PSSession** interactive. Uses **WinRM** (default) or **SSH**.

PS7 — Returns **String[]** (array) even for one item. If 0 items, returns empty array `@()`. **Correct pattern**: Initialize `$result = @()` and return it to ensure array consistency.

PS8 — **[scriptblock]** is code container. Pass via `-ArgumentList`. **Double-hop** = auth chain failure. Solve with **CredSSP** or Kerberos delegation.

PS9 — **@()** forces array. **[array]** casts. Indexing `$null` throws error. Indexing empty array throws error.

PS10 — **-eq** (value), **-like** (glob), **-match** (regex). **-ceq** (case-sensitive). **-contains** (left), **-in** (right).

PS11 — **Module** = collection of commands. **.psm1** (code), **.psd1** (manifest). **Import-Module** loads it.

PS12 — **"..."** interpolates. **'...'** literal. **@"..."@** multi-line. Expressions: `"$(var)"`.

PS13 — **Get-Member** introspects. **Properties** (data), **Methods** (func). **Select-Object** (subset), **Where-Object** (filter).

PS14 — **Classes** (v5) are C#-like. Limitations: No private members, limited inheritance. Use for complex state management.

PS15 — **CmdletBinding** enables advanced features. **SupportsShouldProcess** enables -WhatIf/-Confirm. **Begin/Process/End** handle pipeline stages.

PS16 — **Providers** abstract data sources. **Get-ChildItem** works on Registry (`HKLM:\`). Access registry like filesystem.

PS17 — **[runspace]** = thread. **[runspacepool]** = pool. **ForEach-Object -Parallel** (v7+) uses pool. Manual runspace for fine control.

PS18 — **PSScriptAnalyzer** = linting. **Rules** enforce standards. **Severity** (Error/Warning). **Suppress** via comment. Integrate via CI.

PS19 — **$using:** passes local var to remote. **-ArgumentList** passes value. **Serialization** (CLIXML) limits complex objects.

PS20 — **DSC** = config management. **Resources** = tasks. **LCM** = manager. **Replaced** by Ansible/Terraform in modern IaC.
