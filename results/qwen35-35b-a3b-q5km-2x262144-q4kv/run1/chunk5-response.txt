DN1 — **Class** is a reference type (heap-allocated, mutable by default). **Struct** is a value type (stack-allocated or inline, immutable by default). Use a **struct** for small, immutable data types that represent a single value (e.g., `Point`, `Complex`) to avoid heap allocation and ensure value semantics.

DN2 — **IEnumerable<T>** represents in-memory collections (LINQ to Objects). **IQueryable<T>** represents a queryable data source (e.g., database) and translates expressions into SQL (LINQ to SQL/EF). This distinction matters because **IQueryable** allows filtering/sorting to happen on the server, reducing data transfer, whereas **IEnumerable** pulls all data into memory first.

DN3 — **async/await** compiles into a state machine that suspends execution without blocking threads. **Task** is a reference type representing an async operation. **ValueTask** is a value type (struct) representing an async operation that may complete synchronously. Use **ValueTask** in hot paths (e.g., high-frequency methods) to avoid heap allocation when the result is already available.

DN4 — **Dependency Injection (DI)** decouples object creation from usage. **Transient**: new instance every time requested. **Scoped**: one instance per scope (e.g., HTTP request). **Singleton**: one instance per application lifetime. **Bug**: Injecting a **Scoped** service into a **Singleton** causes the Singleton to capture the Scoped service from the first request, leading to stale data or scope leaks.

DN5 — **string** is immutable; every modification creates a new object. **ReadOnlySpan<char>** is a mutable view over memory (stack or heap) without copying. Use **Span<T>** for high-performance string processing (parsing, slicing) to eliminate temporary allocations and improve cache locality.

DN6 — The **.NET GC** manages memory automatically. **Generations**: Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **Large Object Heap (LOH)** stores objects >85KB (e.g., large strings/arrays) and is collected less frequently. **IDisposable** is needed for **unmanaged resources** (files, DB connections) because the GC only manages managed memory; `Dispose` ensures timely cleanup.

DN7 — **ConfigureAwait(false)** prevents capturing the current **SynchronizationContext** (e.g., UI thread) after an await. Use it in **library code** or background services to avoid deadlocks and improve performance. **Do NOT use it** in UI applications (WPF, WinForms, Blazor) where you need to update the UI thread after the await.

DN8 — **.NET 8 Features**:
1. **Primary Constructors**: Define constructor parameters directly in the class signature.
2. **Collection Expressions**: `[1, 2, 3]` syntax for arrays/lists.
3. **FrozenDictionary**: Immutable, high-performance dictionary.
**Example**:
```csharp
public class Service(IDb db) { public void Run() => db.Execute(); }
```

DN9 — **Cause**: `DbContext` is **not thread-safe**. High concurrency with a **Singleton** lifetime causes multiple requests to share one context, leading to `ObjectDisposedException` or race conditions. **Fix**: Register `DbContext` as **Scoped** (per HTTP request) in DI. EF Core's `DbContext` lifetime must match the request scope to ensure isolation.

DN10 — **Record**: Immutable by default, value-based equality (compares properties). **Class**: Mutable, reference-based equality. **Record Struct**: Value type with record semantics (value equality). Choose **Record** for DTOs/Models, **Class** for entities requiring mutation, **Record Struct** for small value objects needing value equality.

DN11 — **ref**: Pass by reference (read/write, must be initialized). **out**: Pass by reference (write only, must be assigned before return). **in**: Pass by reference (read-only, prevents copy of large structs). Use **ref** for swapping, **out** for multiple return values, **in** for large read-only parameters.

DN12 — **Nullable Reference Types (NRT)**: Compile-time warnings for reference types (`string?`). **Nullable Value Types**: Runtime wrappers for value types (`int?` is `Nullable<int>`). NRT helps prevent `NullReferenceException` at compile time; nullable value types allow `null` for value types at runtime.

DN13 — **LINQ**: Language Integrated Query. **Method Syntax**: `list.Where(...)` (fluent). **Query Syntax**: `from x in list where ...` (SQL-like). **Deferred Execution**: Query logic is not executed until enumeration (e.g., `ToList()` or `foreach`).

DN14 — **Generic Constraints**: Restrict type arguments at compile time. `where T : class` (ref type), `where T : struct` (val type), `where T : new()` (parameterless constructor), `where T : IComparable<T>` (comparable). Combine with commas: `where T : class, IComparable<T>`.

DN15 — **Abstract Class**: Can contain implementation and state. **Interface**: Contract only (no state). **Default Interface Methods** (C# 8+) allow interfaces to provide implementation, reducing breaking changes when adding methods to existing interfaces.

DN16 — **Middleware Pipeline**: HTTP request processing chain. **app.Use()**: Adds middleware to pipeline. **app.Map()**: Branches pipeline for specific paths. **app.Run()**: Terminal middleware (no next). **Ordering**: Critical (e.g., Auth before Endpoints). **Exception Handling**: Middleware wraps pipeline to catch and handle errors globally.

DN17 — **Channel<T>**: Async producer-consumer pattern. **BlockingCollection<T>**: Sync version. **Bounded Channel**: Limits capacity; if full, producer blocks (backpressure). Example: `Channel.CreateBounded<int>(100)` ensures memory safety under load.

DN18 — **Source Generators**: Generate code at **compile-time** (faster than reflection/IL weaving). **Incremental Generator**: Uses Roslyn's incremental compilation API for better caching/performance. Preferred over `ISourceGenerator` because it avoids regenerating unchanged code, improving build times.

DN19 — **TPL Dataflow**: Async pipeline library. **ActionBlock**: Consumes data. **TransformBlock**: Processes and outputs. **BufferBlock**: Stores data. **Channels**: Simpler, lower-level async queues. **Dataflow**: Better for complex flow control, backpressure, and parallelism.

DN20 — **Minimal APIs**: Lightweight endpoints without controllers. **Trade-offs**: Less boilerplate, but harder to scale complex auth/middleware compared to Controllers. **Route Groups**: Organize endpoints by feature. **Endpoint Filters**: Apply logic (e.g., validation) to specific endpoints.

P1 — **List**: Mutable, ordered collection. **Tuple**: Immutable, ordered collection. **Set**: Mutable, unordered unique items. **FrozenSet**: Immutable, unordered unique items. Use **Tuple** for fixed data, **Set** for uniqueness checks.

P2 — **GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects **CPU-bound** workloads. **Workarounds**: Multiprocessing (separate processes), C-extensions (release GIL), or `asyncio` for I/O-bound tasks.

P3 — **Deepcopy**: Recursive copy of object and all nested objects. **Shallow Copy**: Copy of container, references to nested objects. **Example**: `list1 = [[1], [2]]`, `list2 = copy(list1)`. Modifying `list2[0][0]` affects `list1` because inner lists are shared.

P4 — **Decorator**: Function that wraps another function.
```python
import time
def timeit(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 — **Generators**: Functions that yield values lazily. **Yield**: Pauses execution, returns value, resumes later. **Yield from**: Delegates to another generator. **Difference**: Generators save memory (no list creation) and support streaming.

P6 — **Event Loop**: Single-threaded scheduler for async tasks. **asyncio.run()**: Entry point. **asyncio.gather()**: Run multiple tasks concurrently. **asyncio.create_task()**: Schedule task immediately. **time.sleep()** blocks the loop; use **asyncio.sleep()** to yield control.

P7 — **MRO**: Method Resolution Order (C3 linearization). **Diamond Inheritance**: Resolved by order of bases. **Code**: `D(B, C)`. MRO: `D -> B -> C -> A`. `D().who()` calls `C.who()` because `C` appears before `A` in the MRO chain.

P8 — **Metaclass**: Class of a class (controls class creation). **__init_subclass__**: Hook for subclass initialization (easier than metaclass). **Use Case**: Metaclass for ORM model registration; `__init_subclass__` for configuring subclass behavior (e.g., auto-registering handlers).

P9 — **Multiprocessing**: Separate processes (bypass GIL, high CPU). **Threading**: Shared memory (GIL bound, high I/O). **concurrent.futures**: High-level API for both. **Serialization**: Multiprocessing requires **pickling** objects to send between processes.

P10 — **__slots__**: Declares fixed attributes. **Prevents**: Dynamic attribute creation (`__dict__` not created). **Memory Impact**: Reduces memory per instance significantly (no dict overhead). Use for large numbers of similar objects.

P11 — **Virtual Environment**: Isolated Python environment. **Why**: Prevent dependency conflicts. **venv**: Standard library (Python 3.3+). **virtualenv**: Third-party (older). **conda**: Package manager (binary packages, data science).

P12 — **Comprehensions**: Concise loops (`[x for x in list]`). **map/filter**: Functional transforms. **Prefer**: Comprehensions for readability in Python. Use `map/filter` for lazy evaluation or complex logic.

P13 — **==**: Value equality. **is**: Identity equality. **Interning**: Small integers (-5 to 256) are cached. `a is b` returns `True` for small ints because they point to the same cached object.

P14 — **\*args**: Tuple of positional args. **\*\*kwargs**: Dict of keyword args. **/**: Positional-only parameters (before `/`). **\***: Keyword-only parameters (after `*`). Example: `def f(a, /, b, *, c): ...`.

P15 — **Context Manager**: Manages resources (`with` statement). **__enter__/__exit__**: Manual implementation. **@contextmanager**: Decorator for generator-based context. **Example**: Locking a resource, managing DB connections.

P16 — **Type Hinting**: Static type annotations. **Union**: `int | str`. **Optional**: `int | None`. **Literal**: Specific values. **TypeVar**: Generic constraints. **Protocol**: Structural typing. **mypy/pyright**: Analyze code without running to catch type errors.

P17 — **Dunder Methods**: Magic methods (`__init__`, `__str__`). **__repr__**: Unambiguous (dev). **__str__**: Readable (user). **__eq__**: Equality. **__hash__**: Hashable. Defining `__eq__` without `__hash__` makes object **unhashable** (cannot be dict key).

P18 — **Descriptors**: Objects controlling attribute access (`__get__`, `__set__`, `__delete__`). **property**: Getter/Setter descriptor. **classmethod/staticmethod**: Class/instance descriptors. Used internally to manage attribute behavior.

P19 — **Import System**: **Finder**: Locates module. **Loader**: Loads/executes module. **Relative**: Based on current package. **Absolute**: Based on `sys.path`. **__init__.py**: Marks package (optional in PEP 420 namespace packages). **Namespace Packages**: Split across directories without `__init__.py`.

P20 — **Dataclasses**: Auto-generates `__init__`, `__repr__`, etc. **attrs**: More features (validation, converters). **Pydantic**: Validation + dataclasses. **__post_init__**: Logic after init. **Choose Pydantic** for API models requiring validation (e.g., JSON parsing).
