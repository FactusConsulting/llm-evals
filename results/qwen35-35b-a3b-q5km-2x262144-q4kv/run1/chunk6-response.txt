JS1 — Easy: `var` is function-scoped and hoisted (initialized as `undefined`). `let` and `const` are block-scoped and hoisted but not initialized (Temporal Dead Zone). `const` cannot be reassigned. Hoisting is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — Easy: The event loop manages asynchronous execution. The call stack executes synchronous code. When the stack is empty, the loop checks the microtask queue (Promises, `queueMicrotask`) first, then the macrotask queue (setTimeout, I/O). Microtasks drain completely before the next macrotask.

JS3 — Medium: A closure is a function retaining access to its outer lexical scope. Example: `function outer() { let x=1; return () => x; }`. Classic bug: `for (var i=0; i<3; i++) setTimeout(() => console.log(i), 0)` logs 3, 3, 3. Fix: use `let` (block scope) or an IIFE to capture the current value.

JS4 — Medium: `Promise` represents eventual completion. `async/await` syntactic sugar for promises. `try/catch` handles errors in both. `Promise.all` rejects on first failure; `Promise.allSettled` waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 — Medium: `interface` defines shape (can extend, merge) and is erased at runtime. `type` is a union/intersection alias (can define primitives, tuples). Use `interface` for public API contracts, `type` for unions or primitives. Discriminated unions use a literal property to distinguish types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — Medium: Prototypes link objects; `Object.create(proto)` creates an object with `proto` as its internal `[[Prototype]]`. ES6 classes are syntactic sugar over prototypes; `class` defines a constructor and methods on the prototype chain.

JS7 — Hard: `this` binding depends on call site. Priority: 1. `new` binding, 2. Explicit (`call`, `apply`, `bind`), 3. Implicit (`obj.method()`), 4. Default (`window`/`undefined`). `bind()` returns a new function with permanently bound `this`. It cannot be re-bound.

JS8 — Hard: `WeakRef` holds a weak reference to an object (GCable). `FinalizationRegistry` notifies when a weakly held object is collected. Use for caches or cleanup. `WeakMap` keys must be objects because primitives are copied, not referenced, preventing weak reference semantics.

JS9 — Hard: V8 uses hidden classes (shapes) for property access optimization. Inline caching (IC) caches the hidden class of the last object seen. Deoptimization occurs when hidden classes change (e.g., adding properties dynamically), forcing V8 to drop optimizations. Avoid dynamic property addition in tight loops.

JS10 — Medium: Tree-shaking removes unused exports. ES Modules (static `import`/`export`) allow static analysis. CommonJS (`require`) is dynamic, so bundlers cannot statically analyze dependencies, making tree-shaking ineffective.

JS11 — Easy: `undefined` is the default value for uninitialized variables; `null` is an assignment value representing "no value". `==` converts types (`null == undefined` is true); `===` checks type and value. `??` returns right operand if left is `null` or `undefined`. `?.` safely accesses nested properties.

JS12 — Easy: `map` transforms, `filter` selects, `reduce` accumulates. `reduce` takes a callback and optional initial value (otherwise first element is accumulator). `flatMap` maps then flattens one level; useful for transforming arrays of arrays into a single array.

JS13 — Easy: Template literals use backticks for multi-line strings and `${}` interpolation. Tagged templates pass the string array and values to a function. Example: `html` strings for escaping SQL/HTML to prevent injection.

JS14 — Easy: Destructuring extracts values. Object: `{a, b}`. Array: `[a, b]`. Defaults: `{a = 1}`. Renaming: `{a: b}`. Rest operator `...` collects remaining elements/properties into an array/object.

JS15 — Easy: `for...in` iterates over enumerable property keys (including inherited). `for...of` iterates over iterable values (array elements). Avoid `for...in` on arrays because it includes non-index properties and is slower.

JS16 — Medium: Generics use `<T>` for type parameters. Constraints: `T extends SomeType`. Conditional types: `T extends U ? X : Y`. `infer` extracts types from conditional types. Example: `type First<T> = T extends (infer U)[] ? U : never`.

JS17 — Medium: CommonJS uses `require`/`module.exports` (dynamic, synchronous). ES Modules use `import`/`export` (static, async). Node supports both; `"type": "module"` in `package.json` forces ESM interpretation for `.js` files.

JS18 — Hard: TypeScript is structural (shape-based). Excess property checks apply only to object literals in assignments. `extends` checks assignability; `satisfies` checks shape without widening types. Variance (in/out) controls type compatibility for generics (e.g., `Array<out T>`).

JS19 — Hard: `Proxy` intercepts operations (get, set, apply). `Reflect` provides default implementations. Example: `new Proxy(obj, { get: (t, k) => console.log(k), get: Reflect.get })`. Used for validation, logging, or reactive frameworks (Vue/React).

JS20 — Hard: `SharedArrayBuffer` allows shared memory between workers. `Atomics` provides atomic operations. `Atomics.wait` blocks until a value changes; `Atomics.notify` wakes waiting threads. Guarantees memory ordering (sequential consistency for atomics).

B1 — Easy: `$@` expands to individual quoted arguments (`"$1" "$2"`). `$*` expands to a single string (`"$1 $2"`). `"$@"` preserves argument boundaries; `"$*"` joins them. Use `"$@"` for safe iteration.

B2 — Easy: `set -e` exits on error. `set -u` errors on undefined vars. `set -o pipefail` returns error if any command in a pipeline fails. Together they make scripts fail fast and safely.

B3 — Medium: `$(command)` is preferred over backticks (nested easier). `$((expression))` is arithmetic expansion. `expr` is an external command; `$((...))` is built-in and faster.

B4 — Medium: Process substitution `<(cmd)` creates a file descriptor pointing to the command's output. Example: `diff <(sort a) <(sort b)` avoids creating temp files.

B5 — Medium: Use `for f in *; do ...` or `find . -print0 | while IFS= read -r -d '' f; do ...`. `for f in $(ls)` breaks on spaces/newlines and parses output incorrectly.

B6 — Medium: Here doc `<<EOF` reads until EOF line. Here string `<<<` passes a string. `<<-EOF` allows leading tabs to be stripped from the script body.

B7 — Hard: `cmd > file 2>&1`: Redirects stdout to file, then stderr to stdout (file). `cmd 2>&1 > file`: Redirects stderr to stdout (terminal), then stdout to file (stderr goes to terminal). `&> file` redirects both to file. First two differ due to redirection order.

B8 — Hard: Use `mkdir` for locking (atomic). Function: `mkdir /tmp/lockdir || exit 1; trap 'rmdir /tmp/lockdir' EXIT`. `mkdir` fails if dir exists, preventing race conditions. `touch` is not atomic.

B9 — Easy: Single quotes `'...'` disable expansion. Double quotes `"..."` allow expansion. No quotes allow word splitting/globbing. Example: `echo $HOME` (no quotes) expands; `echo '$HOME'` (single) prints literal.

B10 — Easy: `[ -f file ]`, `[ -d dir ]`, `[ -r file ]`. `[[ ]]` is a keyword (supports `&&`, `||`, regex), `[ ]` is a command (POSIX). `[[` is safer for bash scripts.

B11 — Easy: `$?` exit status. `$$` current PID. `$!` last background PID. `$#` argument count. Use `$?` for error checking, `$$` for temp files, `$!` for job control, `$#` for argument validation.

B12 — Easy: `func() { ... }` or `function func { ... }`. Return values via `return` (exit code) or `echo` (stdout). Functions share global scope unless `local` is used.

B13 — Easy: `PATH` is colon-separated dirs. Bash searches `PATH` for executables. Builtins are internal (e.g., `cd`); externals are files (e.g., `ls`). Use `type` or `which` to check.

B14 — Medium: `trap 'cmd' EXIT` runs on exit. `trap 'cmd' ERR` on error. `trap 'cmd' INT TERM` on signals. Signals are queued; handlers run sequentially. `trap -` unsets.

B15 — Medium: `xargs` builds command lines from stdin. `command substitution` runs immediately. `xargs -0` handles null-delimited input (`find -print0`). `xargs -P` runs jobs in parallel.

B16 — Medium: Arrays: `arr=(a b)`. Append: `arr+=("c")`. Length: `${#arr[@]}`. Associative: `declare -A arr`. Syntax: `arr=(["key"]="val")`. Iteration: `for k in "${!arr[@]}"`.

B17 — Hard: Subshell `(...)` runs in child process (vars lost). Group `{ ...; }` runs in current shell. Piping into `while` creates a subshell, so variable changes inside loop are lost after loop. Use process substitution `while ... done < <(cmd)` to fix.

B18 — Hard: SIGINT (Ctrl+C), SIGTERM (kill), SIGHUP (hangup), SIGKILL (force kill, uncatchable). Handle graceful shutdown by trapping SIGTERM, cleaning up, then exiting. Background jobs need `wait` or `disown`.

B19 — Hard: `split -l 1000 log.log part_`, `xargs -P 4 -I {} sh -c 'process {}'`, merge results. Trade-off: I/O vs CPU. Error propagation: check exit codes of parallel jobs, aggregate failures, exit non-zero if any fail.

B20 — Hard: `exec` replaces shell process. `exec 3>file` opens FD 3. `exec 3>&-` closes it. Coprocess `coproc CMD` creates bidirectional pipe. Use for persistent connections or long-running daemons within a script.

PS1 — Easy: `Write-Output` sends objects to pipeline. `Write-Host` writes to console (bypasses pipeline). Use `Write-Output` for automation; `Write-Host` only for interactive UI.

PS2 — Easy: Pipeline passes objects. `ForEach-Object` is a cmdlet (pipeline-aware, parallelizable). `foreach` is a statement (iterates collection in memory). Use `ForEach-Object` for streaming data.

PS3 — Medium: `[PSCustomObject]` creates a typed object with properties. `[hashtable]` is a key-value collection. Convert: `[PSCustomObject]@{Key="Val"}` or `.GetEnumerator()` for hashtable. Use CustomObject for structured data.

PS4 — Medium: Terminating errors stop execution (try/catch). Non-terminating continue (Warning). `-ErrorAction Stop` converts non-terminating to terminating. `$ErrorActionPreference` sets default.

PS5 — Medium: Splatting `@params` passes hashtable as arguments. `func @params` vs `func $params`. Splatting allows dynamic argument lists. Passing hashtable as single param treats it as one argument.

PS6 — Medium: `Invoke-Command -ComputerName` runs one-off. `Enter-PSSession` interactive. Uses WinRM (HTTP/HTTPS) or SSH (v7+). SSH requires `Enable-PSRemoting -Transport SSH`.

PS7 — Hard: Returns array even if single item. `$x.GetType().Name` is `Object[]`. If 0 items, returns `$null`. Correct pattern: `$result = @(); $result += ...; return $result`.

PS8 — Hard: `[scriptblock]` is code object. Pass via `-ArgumentList`. Double-hop: Client -> Server -> Target. Solve with CredSSP (insecure) or Kerberos delegation (secure).

PS9 — Easy: `@()` forces array. `[array]` casts. `$null[0]` throws error. `@($null)[0]` returns `$null`. Indexing `$null` fails; empty array returns empty.

PS10 — Easy: `-eq` (equal), `-like` (glob), `-match` (regex). `-ceq` case-sensitive. `-contains` checks if collection contains value. `-in` checks if value is in collection (newer, cleaner).

PS11 — Easy: Module is reusable code. `.psm1` script, `.psd1` manifest (metadata), binary (DLL). `Import-Module` loads it. Manifest defines exports, dependencies, version.

PS12 — Easy: `"Hello $var"` interpolates. `'...'` literal. `@"...@` here-string (multi-line). Escape `$` with `\$`.

PS13 — Easy: `Get-Member` lists properties/methods. Properties (data), Methods (actions), NoteProperties (added dynamically). `Select-Object` picks properties; `Where-Object` filters.

PS14 — Medium: Classes define structure. Limitations: No multiple inheritance, limited interfaces. Use classes for complex state management, functions for simple logic.

PS15 — Medium: `[CmdletBinding()]` enables advanced features (ShouldProcess, Verbose). `Begin` (init), `Process` (pipeline items), `End` (finalize). `ValueFromPipeline` binds input to parameter.

PS16 — Medium: Providers expose data as file system. `Get-ChildItem HKLM:\Software` lists registry keys. `Set-Location` navigates providers.

PS17 — Hard: `[runspace]` single thread. `[runspacepool]` thread pool. `ForEach-Object -Parallel` (v7+) uses pool. Manual runspace for complex control. Thread safety: avoid shared mutable state.

PS18 — Medium: `PSScriptAnalyzer` checks code quality. Rules (e.g., naming). Severity (Error, Warning). Suppress via `#pragma`. Integrate via CI (e.g., GitHub Actions).

PS19 — Hard: `$using:` passes variable value into remote session. `-ArgumentList` passes arguments. Serialization limits (CLIXML) affect complex objects (e.g., custom classes). Use JSON or custom serialization for complex data.

PS20 — Hard: DSC configures state. Resources (e.g., File, Service). LCM applies config. PowerShell 7+ uses `DSC` module. Replaced by Ansible, Terraform, or native OS tools (e.g., `systemd`).
