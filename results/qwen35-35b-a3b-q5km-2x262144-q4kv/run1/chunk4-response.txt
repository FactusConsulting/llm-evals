G1 ‚Äî Easy: An **array** is a fixed-size value type `[N]T`. A **slice** is a dynamic view over an array, defined as `[start:end]T` (struct containing pointer, length, capacity). When appending to a slice at capacity, the runtime allocates a new, larger backing array, copies existing elements, and returns a new slice pointing to it.

G2 ‚Äî `var x int` declares a variable with explicit type and zero value. `x := 0` is a short declaration that infers type and declares a new variable. You **cannot** use `:=` if the variable is already declared in the current scope (e.g., reassigning `x := 1` where `x` exists).

G3 ‚Äî A **goroutine** is a lightweight thread managed by the Go runtime, not the OS. It differs from OS threads by being cheaper to create and context-switch. The **GMP model** maps Goroutines (G) to OS Threads (M) via Processor slots (P). The scheduler multiplexes Gs onto Ms to maximize CPU utilization.

G4 ‚Äî **Channels** are typed conduits for communication. **Unbuffered** channels block sender and receiver until both are ready. **Buffered** channels allow sending without blocking until the buffer is full. Sending to a **full buffered channel** blocks the sender until a receiver reads from it.

G5 ‚Äî The **context** package manages cancellation, deadlines, and request-scoped values.
```go
ctx, cancel := context.WithTimeout(context.Background(), 1*time.Second)
defer cancel()
resp, err := http.GetContext(ctx, url)
```

G6 ‚Äî Go interfaces are **implicitly satisfied** (duck typing); no `implements` keyword needed. Unlike Java/C#, interfaces are defined by usage, not declaration. The **empty interface** (`interface{}` or `any`) holds any value. It is powerful for generic containers but dangerous because type assertions are required at runtime, risking panics if types mismatch.

G7 ‚Äî This is **unsafe** due to a **data race**: one goroutine writes `data` and `ready`, another reads `ready` and `data` without synchronization. The compiler/CPU may reorder operations. **Fix:** Use `sync.Mutex` to protect both, or `sync/atomic` for `ready` and a channel for `data`.

G8 ‚Äî `sync.Mutex` provides exclusive locking. `sync.RWMutex` allows multiple readers or one writer. `sync.Map` is optimized for read-heavy workloads or when keys are known at startup. **Trade-off:** `sync.Map` has higher overhead for writes and lookups compared to a standard `map` protected by a `Mutex`.

G9 ‚Äî **Go modules** manage dependencies. `go.mod` lists requirements and versions. `go.sum` stores checksums for integrity. `go mod tidy` removes unused dependencies and adds missing ones. **MVS** (Minimum Version Selection) selects the lowest version of a dependency that satisfies all constraints in the dependency tree.

G10 ‚Äî `defer` schedules a function call to run after the surrounding function returns. Multiple defers execute in **LIFO** (Last In, First Out) order. **Gotcha:** In a loop, `defer` captures the loop variable by reference (pre-Go 1.22), causing all deferred calls to see the final loop value.

G11 ‚Äî A **pointer** holds a memory address. Its zero value is `nil`. Create a pointer to a struct literal using the address-of operator: `&MyStruct{Field: "value"}`.

G12 ‚Äî `init()` runs automatically before `main()` executes. It runs once per package. A package can have multiple `init()` functions; they execute in **declaration order** within the file, and files are processed in alphabetical order.

G13 ‚Äî `make` allocates and initializes **built-in types** (map, slice, channel) and returns the typed value. `new` allocates zeroed memory for **any type** and returns a pointer to it.

G14 ‚Äî Errors are **values**. The `error` interface has `Error() string`. Idiom: `if err != nil { return err }`. `errors.Is` checks semantic equality (wrapping). `errors.As` performs type assertion to find a specific error type. `==` only works for concrete error types, not wrapped ones.

G15 ‚Äî **Composite types:**
*   **Map:** Zero value is `nil`.
*   **Slice:** Zero value is `nil`.
*   **Channel:** Zero value is `nil`.
*   **Struct:** Zero value has zero values for all fields.

G16 ‚Äî A **race condition** occurs when concurrent accesses to shared data lack synchronization. The `-race` flag adds instrumentation to detect data races at runtime. **Fix:** Use `sync.Mutex` to lock critical sections or channels to serialize access.

G17 ‚Äî `select` waits on multiple channel operations. Unlike `switch`, it chooses a ready case **randomly** if multiple are ready. **Timeout:** Add a `case <-time.After(duration)` to the select block.

G18 ‚Äî **Escape analysis** determines if a variable lives on the stack or heap. If a reference escapes the function scope, it moves to the heap. Inspect with `go build -gcflags="-m"`. It matters because heap allocations trigger GC, impacting latency and throughput.

G19 ‚Äî **Reflection** (`reflect`) allows runtime inspection of types and values. `reflect.Type` is metadata; `reflect.Value` is the data. **Cost:** High CPU overhead and no compile-time type safety. Justified for frameworks (serialization, DI) where code generation is too complex.

G20 ‚Äî `//go:embed` embeds files/directories at compile time into a `fs.FS`. Works with single files, directories (`...`), and glob patterns. **Limitations:** Static at compile time (cannot embed runtime-generated files). Interacts with `io/fs` by implementing the `FS` interface, allowing standard file operations on embedded content.

R1 ‚Äî **Ownership** ensures memory safety without GC. **Rules:** 1. Each value has one owner. 2. When owner goes out of scope, value is dropped. 3. You can borrow (reference) a value, but rules apply to borrowing.

R2 ‚Äî `String` is an owned, heap-allocated, mutable string. `&str` is a borrowed, immutable slice of bytes. Use `String` for ownership; use `&str` in function signatures for flexibility (accepts both `String` and string literals).

R3 ‚Äî **Borrowing** creates references. **Rules:** 1. Either immutable (`&T`) OR mutable (`&mut T`) references, not both. 2. References must always be valid. 3. Mutable references must be exclusive.

R4 ‚Äî `Result<T, E>` represents success (`Ok`) or failure (`Err`). `Option<T>` represents `Some(T)` or `None`. The `?` operator propagates errors.
```rust
fn read_file() -> Result<String, std::io::Error> {
    std::fs::read_to_string("file.txt")
}
```

R5 ‚Äî `Box<T>`: Heap allocation, single owner. `Rc<T>`: Reference counted, single thread. `Arc<T>`: Atomic reference counted, thread-safe for multi-threading. Use `Arc` when sharing data across threads.

R6 ‚Äî **Traits** define shared behavior. Like Go interfaces, but can have default implementations. `dyn Trait` uses dynamic dispatch (vtable, runtime cost). `T: Trait` uses static dispatch (monomorphization, zero cost).

R7 ‚Äî Fails because the compiler cannot verify the returned reference lives long enough. `x` and `y` might have different lifetimes. **Fix:** Add lifetime parameter `'a` linking inputs and output: `fn longest<'a>(x: &'a str, y: &'a str) -> &'a str`.

R8 ‚Äî `Send` means ownership can be transferred across threads. `Sync` means shared references can be sent. `Rc<T>` is not `Send` because its ref-count is not atomic. `Arc<T>` uses atomic ref-counting, making it `Sync` and `Send`.

R9 ‚Äî `async/await` creates a **Future** (state machine). The **executor** (e.g., Tokio) polls the Future to completion. Differs from Go: Go uses OS threads (M:N), Rust Futures are cooperative and run on fewer OS threads.

R10 ‚Äî `unsafe` bypasses safety checks. **Actions:** Raw pointer dereference, union access, calling unsafe functions. **Justified:** Performance-critical code, interfacing with C, or implementing safe abstractions over unsafe primitives.

R11 ‚Äî **Pattern matching** (`match`) exhaustively checks values. `if let` matches a single pattern. The compiler enforces **exhaustiveness**, requiring all possible enum variants to be handled (or `_` wildcard).

R12 ‚Äî `Vec<T>` is a growable heap array. `[T; N]` is a fixed-size stack array. When `Vec` capacity is exceeded, it allocates a new larger buffer, copies elements, and drops the old one.

R13 ‚Äî **Enums** can hold data (Algebraic Data Types). Unlike C/Java, they are type-safe and can be pattern matched. `Option<T>` replaces `null` with `Some(T)` or `None`, forcing explicit error handling.

R14 ‚Äî `impl` defines methods. `&self` borrows, `&mut self` mutably borrows, `self` consumes the value. **Associated functions** (e.g., `String::new()`) do not take `self` and act like static methods.

R15 ‚Äî `derive` generates boilerplate code. Common traits: `Debug`, `Clone`, `PartialEq`. Cannot derive if fields contain types that don't support the trait (e.g., `Clone` on `Fn` closures).

R16 ‚Äî `Cow<'a, T>` (Clone on Write) holds either a borrowed `&'a T` or an owned `Box<T>`. Useful when data is often read-only but occasionally modified. Avoids cloning if the data is never mutated.

R17 ‚Äî **Module system:** `mod` declares a module, `use` imports paths, `pub` exposes items. File system: `mod.rs` or `mod.rs` in a directory. **2018 vs 2015:** 2018 allows `use` paths relative to the crate root by default; 2015 required paths relative to the module.

R18 ‚Äî `Pin<T>` prevents a value from being moved in memory. Needed for **self-referential structs** (e.g., `Future` holding a reference to itself). `Unpin` is a marker trait indicating a type can be moved safely.

R19 ‚Äî **Macros** generate code. `macro_rules!` is declarative (pattern matching). Procedural macros are Rust functions generating code (derive, attribute). **Hygiene** ensures macro variables don't leak into the caller's scope.

R20 ‚Äî **Zero-cost abstractions** mean high-level features compile to efficient machine code. **Iterators** compile to loops. **Generics** use monomorphization (copying code per type). **Traits** use static dispatch. No runtime overhead compared to C.
