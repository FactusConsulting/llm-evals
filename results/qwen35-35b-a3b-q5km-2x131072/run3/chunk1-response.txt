**N1**
A Layer 2 switch forwards frames based on MAC addresses within a single broadcast domain (bridging). A Layer 3 switch forwards packets based on IP addresses (routing) and can perform inter-VLAN routing. Layer 3 switches combine switching speed with routing capabilities.

**N2**
**Network Address:** 10.0.0.0
**Broadcast Address:** 10.0.15.255
*Calculation:* /20 mask (255.255.240.0) creates blocks of 16 in the third octet. 10.0.5.37 falls in the 10.0.0.0–10.0.15.255 range.

**N3**
The handshake consists of: 1) Client sends SYN, 2) Server replies SYN-ACK, 3) Client sends ACK. If the final ACK is lost, the server times out and closes the connection. The client may retransmit the SYN, causing a duplicate connection attempt.

**N4**
**SNAT (Source NAT):** Modifies source IP (outbound). Example: Internal servers accessing the internet via a public gateway IP.
**DNAT (Destination NAT):** Modifies destination IP (inbound). Example: Port forwarding external traffic on port 80 to an internal web server.

**N5**
1. The certificate's Common Name (CN) or Subject Alternative Name (SAN) does not match the domain in the URL.
2. The server's clock is skewed, causing certificate validity checks to fail.
3. The certificate is self-signed or issued by an untrusted CA not in the client's trust store.

**N6**
**VLAN:** Logical L2 segmentation limited to physical topology (max 4096 IDs).
**VxLAN:** L2 overlay over L3 (UDP encapsulation), supports up to 16 million segments.
**Choice:** Choose VxLAN for multi-tenant clouds, data center fabrics, or when spanning L2 domains across L3 boundaries (WAN).

**N7**
**Cause:** The default route points to eth0. Return traffic from eth1 uses the default route (eth0) instead of the specific route for 192.168.1.0/24.
**Fix:** Use Policy-Based Routing (PBR).
1. `ip rule add from 192.168.1.0/24 table 100`
2. `ip route add default via 192.168.1.1 dev eth1 table 100`

**N8**
BGP selects the best path based on attributes. Cisco default order (top to bottom):
1. Weight (Highest)
2. Local Preference (Highest)
3. Locally Originated
4. AS Path Length (Shortest)
5. Origin Type (IGP < EGP < Incomplete)

**N9**
**Overhead:** VxLAN adds ~36 bytes (IP+UDP+VxLAN). IPsec adds ~50 bytes (ESP+Auth). Total ~86 bytes.
**Math:** 1500 (Physical MTU) - 86 (Overhead) = 1414 bytes.
**Max Inner Payload:** 1414 bytes (excluding inner Ethernet header).

**N10**
**ARP:** Resolves IPv4 addresses to MAC addresses.
**Gratuitous ARP:** An unsolicited ARP reply used to announce IP ownership, update neighbor caches, and detect IP address conflicts on the network.

**N11**
**Public:** Routable on the internet. **Private:** Reserved for internal use (RFC 1918).
**Ranges:**
1. 10.0.0.0 – 10.255.255.255 (/8)
2. 172.16.0.0 – 172.31.255.255 (/12)
3. 192.168.0.0 – 192.168.255.255 (/16)

**N12**
**DNS:** Resolves domain names to IP addresses.
**A Record:** Maps name to IPv4.
**AAAA Record:** Maps name to IPv6.
**CNAME Record:** Maps name to another domain name (alias).

**N13**
**DHCP:** Dynamically assigns IP configuration.
**DORA Process:**
1. **Discover:** Client broadcasts to find servers.
2. **Offer:** Server offers an IP.
3. **Request:** Client requests the offered IP.
4. **Acknowledge:** Server confirms lease.

**N14**
**Hub:** Layer 1 (Physical). Repeats signals to all ports.
**Switch:** Layer 2 (Data Link). Forwards frames based on MAC addresses.
**Router:** Layer 3 (Network). Routes packets based on IP addresses.

**N15**
**Half-Duplex:** One direction at a time (collision domain). **Full-Duplex:** Simultaneous send/receive (no collisions).
**Modern Ethernet:** Full-duplex (requires switch, not hub).

**N16**
**TTL:** Time To Live, decremented by each router to prevent loops.
**Traceroute:** Sends packets with TTL=1, 2, 3... Routers decrement TTL to 0 and return ICMP "Time Exceeded," revealing the hop.

**N17**
**NAT Gateway:** Translates private IPs to public IPs.
**Static NAT:** 1:1 mapping (fixed).
**Dynamic NAT:** Pool of public IPs mapped to private IPs.
**PAT (NAPT):** Many private IPs map to one public IP using unique ports.

**N18**
**ECMP:** Routes traffic across multiple paths with equal cost.
**Hashing:** Uses src/dst IP and ports to select path.
**Problem:** Asymmetric flows (outbound path A, return path B) cause stateful firewalls to drop return packets if state isn't synced.

**N19**
**Problem:** Bandwidth-Delay Product (BDP) is 10MB, but TCP window is limited to 64KB. The link is underutilized because the window cannot fill the pipe.
**Solution:** TCP Window Scaling (RFC 1323) allows the window size to be shifted (multiplied) beyond 65535 bytes, matching the BDP.

**N20**
**802.1Q:** Adds a 4-byte tag to Ethernet frames containing VLAN ID.
**Linux Config:** Create sub-interfaces: `ip link add link eth0 name eth0.10 type vlan id 10`.
**Frame Format:** Ethernet Header + 802.1Q Tag (TPID + TCI) + Payload + FCS.

**L1**
**Hard Link:** Points to the same inode (data). Deleting one name doesn't delete data.
**Symbolic Link:** Points to a file path. Deleting the target breaks the link.

**L2**
`find /var/log -type f -size +100M`

**L3**
**State D:** Uninterruptible sleep (usually waiting for I/O).
**Kill:** Cannot be killed with `kill -9`. The process must wait for the I/O to complete or the system to reboot.

**L4**
**cgroups v1:** Separate controllers (cpu, memory) with separate hierarchies.
**cgroups v2:** Unified hierarchy, single mount point.
**Systemd:** Prefers v2 for unified resource management and better cgroup2 features (e.g., memory pressure).

**L5**
**nftables:** Modern replacement, single table structure, faster.
**iptables:** Legacy, multiple tables (filter, nat, mangle).
**Kernel:** Modern kernels use `nftables` backend for `iptables` commands (iptables-nft) or support both via compatibility modules.

**L6**
**Cause:** A file was deleted but a process still has it open (FD held). Space is not freed until the process closes it.
**Fix:** `lsof +L1` to find the process, then restart it or truncate the file.

**L7**
1. **UEFI:** Initializes hardware, loads bootloader.
2. **Bootloader (GRUB):** Loads kernel and initramfs.
3. **Kernel:** Initializes hardware, mounts initramfs.
4. **Initramfs:** Loads drivers, mounts real root.
5. **Root:** Mounts real filesystem.
6. **Systemd:** Starts services, reaches target (e.g., multi-user).

**L8**
**Cause:** `available` accounts for reclaimable cache. If `available` is low, the system is out of memory. OOM killer fires to free memory.
**Tunables:** `vm.swappiness` (controls swap usage), `vm.vfs_cache_pressure` (controls inode/dentry reclaim).

**L9**
**ip netns:** Creates isolated network namespaces (separate routing tables, interfaces, sockets).
**Containers:** Use namespaces to isolate network stacks per container.

**L10**
**SIGTERM (15):** Graceful termination (catchable).
**SIGKILL (9):** Force kill (uncatchable).
**SIGQUIT (3):** Quit with core dump (catchable).
**Catchable:** SIGTERM, SIGQUIT.

**L11**
**stdin:** Input (0). **stdout:** Standard output (1). **stderr:** Standard error (2).
**Redirect:** `command 2> error.log` (stderr to file, stdout remains on terminal).

**L12**
**755:** Owner (7=rwx), Group (5=r-x), Others (5=r-x).
**Meaning:** Owner can read/write/execute; Group and Others can read/execute.

**L13**
**Root:** UID 0, unrestricted system access.
**Sudo:** Regular user granted temporary root privileges via `/etc/sudoers` policy.

**L14**
**Package Manager:** Software for installing/updating packages.
**Debian/Ubuntu:** `apt` (frontend), `dpkg` (backend).
**RHEL/Fedora:** `dnf` (frontend), `rpm` (backend).

**L15**
**fstab:** Static filesystem mount configuration at boot.
**Misconfigured:** System may fail to boot (drop to emergency mode) or fail to mount specific filesystems.

**L16**
**LVM:** Abstraction layer for storage.
**PV:** Physical Volume (disk/partition). **VG:** Volume Group (pool of PVs). **LV:** Logical Volume (virtual partition).
**Extend:** `lvextend -L +1G /dev/vg/lv` followed by `xfs_growfs` or `resize2fs`.

**L17**
**Scheduler:** Orders I/O requests to disk.
**mq-deadline:** Optimized for SSDs (latency). **bfq:** Optimized for fairness (interactive). **none:** No scheduling (NVMe).
**Change:** `echo deadline > /sys/block/sda/queue/scheduler`.

**L18**
**Load Average:** Number of processes in running or uninterruptible sleep (D state).
**I/O Wait:** High `wa` in `top` indicates disk bottleneck inflating load.
**Diagnose:** `iostat` (disk), `vmstat` (memory/IO), `pidstat -d` (per-process IO).

**L19**
**SELinux:** Enforces labels (contexts) and policies.
**Modes:** Enforcing (block), Permissive (log only), Disabled (off).
**Troubleshoot:** Check `audit.log` for denials. Use `semanage port -a -t http_port_t -p tcp 8080` to allow port.

**L20**
**Page Fault:** Access to unmapped memory. **Minor:** Page in memory (swap). **Major:** Page on disk (swap).
**THP:** Uses 2MB pages instead of 4KB. Improves TLB efficiency.
**Disable:** If application suffers from latency spikes or fragmentation (e.g., databases). `echo never > /sys/kernel/mm/transparent_hugepage/enabled`.
