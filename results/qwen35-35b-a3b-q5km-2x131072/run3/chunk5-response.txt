## .NET / C#

**DN1**
**Class** is a reference type (heap-allocated, stores reference). **Struct** is a value type (stack-allocated or inline, stores value). Use a **struct** for small, immutable data (e.g., Point, Color) where copying is cheap and value semantics are required. Avoid structs for large data or mutable state to prevent performance penalties from copying.

**DN2**
**IEnumerable<T>** represents an in-memory collection; LINQ queries execute immediately in memory. **IQueryable<T>** represents a queryable data source (e.g., database); it builds an expression tree that is translated to SQL (or provider-specific query) and executed on the server. This distinction matters because `IQueryable` allows server-side filtering/paging, reducing network traffic, whereas `IEnumerable` pulls all data to memory first.

**DN3**
**async/await** uses a state machine to suspend execution without blocking threads. **Task** is a reference type representing an async operation. **ValueTask** is a struct (value type) representing an async operation. Use **ValueTask** when the operation is likely to complete synchronously or to avoid the allocation overhead of a `Task` object in hot paths.

**DN4**
**Dependency Injection (DI)** decouples object creation from usage. **Transient**: New instance every time requested. **Scoped**: One instance per scope (e.g., HTTP request). **Singleton**: One instance per application lifetime. **Bug**: Injecting a **Scoped** service into a **Singleton** captures the Scoped service from the first request's scope. Subsequent requests use the stale/disposed instance, causing data inconsistency or ObjectDisposedException.

**DN5**
**string** is immutable and heap-allocated. **ReadOnlySpan<char>** is a stack-only view into memory (can be on stack, heap, or unmanaged). Use **Span<T>** for high-performance string processing (parsing, slicing) to avoid allocations and copying. It enables zero-copy operations but cannot be stored in fields or returned from methods (must be stack-local).

**DN6**
The **GC** automatically reclaims memory. **Gen 0**: Short-lived objects. **Gen 1**: Intermediate. **Gen 2**: Long-lived. **Large Object Heap (LOH)**: Objects >85KB, collected less frequently. **IDisposable** is needed for **unmanaged resources** (file handles, DB connections) that the GC does not track. Relying solely on GC for these causes resource leaks.

**DN7**
**ConfigureAwait(false)** prevents capturing the current **SynchronizationContext** (e.g., UI thread) after an await. Use it in **library code** to avoid deadlocks and improve performance. **Do NOT use it** in UI applications (WPF, WinForms) where you must update the UI thread after await, or in ASP.NET Core where context is less critical but `false` is generally preferred for server-side logic.

**DN8**
**Primary Constructors**: Define constructor parameters directly in the class definition (e.g., `public class Person(string Name)`). **Collection Expressions**: `[1, 2, 3]` syntax for arrays/lists. **FrozenDictionary**: Immutable, high-performance dictionary.
*Example:*
```csharp
public class UserService(IRepository repo) : IUserService {
    public void DoWork() => repo.Save();
}
```

**DN9**
**Cause**: DbContext is registered as **Singleton** or captured in a Singleton, but it is designed to be **Scoped**. Under high concurrency, the Singleton instance holds a reference to a Scoped context that gets disposed after the first request. **Fix**: Register DbContext as `AddDbContext<DbContext>(options => ..., ServiceLifetime.Scoped)`. Ensure it is injected into Scoped services, not Singletons.

**DN10**
**Record**: Reference type with value-based equality (structurally compared) and immutability by default. **Class**: Reference type with reference equality. **Record Struct**: Value type with record features. Choose **Record** for immutable data models (DTOs, Entities). Choose **Class** for mutable state or polymorphic behavior. Choose **Record Struct** for small, value-type data requiring structural equality.

**DN11**
**ref**: Pass by reference (read/write). **out**: Pass by reference (must be assigned before return). **in**: Pass by reference (read-only). Use **ref** to modify arguments. Use **out** for multiple return values. Use **in** for large structs to avoid copying while preventing modification.

**DN12**
**Nullable Reference Types (NRT)**: Compile-time warnings for reference types (e.g., `string?`). **Nullable Value Types**: Runtime nullability for value types (e.g., `int?`). NRT helps catch null reference exceptions at compile time; `int?` allows the value to be null at runtime.

**DN13**
**LINQ** (Language Integrated Query). **Method Syntax**: Fluent API (`list.Where(...)`). **Query Syntax**: SQL-like (`from x in list where ...`). Both compile to the same IL. **Deferred Execution**: Query logic is not executed until the result is enumerated (e.g., `foreach`, `.ToList()`).

**DN14**
**Generic Constraints** restrict type arguments. `where T : class` (reference type), `where T : struct` (value type), `where T : new()` (parameterless constructor), `where T : IComparable<T>` (implements interface). Combine with `&` (e.g., `where T : class, IComparable<T>`). Use when logic requires specific capabilities from the type.

**DN15**
**Abstract Class**: Can contain implementation and state. **Interface**: Contract only (no state). **Default Interface Methods** (C# 8+) allow interfaces to have method implementations, blurring the line. Use **Abstract Class** for shared code/state. Use **Interface** for polymorphism/contracts.

**DN16**
**Middleware Pipeline**: Requests pass through a chain of components. **app.Use()**: Wraps next middleware. **app.Map()**: Branches pipeline for specific paths. **app.Run()**: Terminal middleware (no next). **Order**: Authentication before Routing. **Exception Handling**: Middleware can catch exceptions and return responses (e.g., `UseExceptionHandler`).

**DN17**
**Channel<T>**: Async-safe producer-consumer queue. **BlockingCollection<T>**: Sync-safe. **Bounded Channel**: Limits capacity to apply **backpressure** (producer waits if full). Example: `Channel.CreateBounded<int>(100)` ensures memory doesn't grow unbounded under load.

**DN18**
**Source Generators**: Compile-time code generation. **Reflection**: Runtime inspection (slow). **IL Weaving**: Post-build modification (e.g., Fody). **Incremental Generator**: Only regenerates code when source files change (faster than `ISourceGenerator`). Preferred for performance in large solutions.

**DN19**
**TPL Dataflow**: Async pipeline library. **ActionBlock**: Consumes data. **TransformBlock**: Consumes and produces. **BufferBlock**: Stores data. **Channels**: Simpler, lower-level async queues. **Dataflow**: Better for complex flow control, backpressure, and dependency management between blocks.

**DN20**
**Minimal APIs**: Lightweight, single-file endpoints without Controllers. **Trade-offs**: Less boilerplate, but harder to scale for complex auth/middleware. **Route Groups**: Organize endpoints by feature. **Endpoint Filters**: Apply logic (e.g., validation) to specific endpoints. OpenAPI support is now robust in .NET 8+.

## Python

**P1**
**List**: Mutable, ordered sequence. **Tuple**: Immutable, ordered sequence. **Set**: Mutable, unordered unique elements. **FrozenSet**: Immutable set. Use **Tuple** for fixed data, **Set** for membership testing.

**P2**
**GIL (Global Interpreter Lock)**: Ensures only one thread executes Python bytecode at a time. Affects **CPU-bound** workloads. **Workarounds**: Use **multiprocessing** (separate processes), **C-extensions** (release GIL), or **asyncio** for I/O-bound tasks.

**P3**
**copy**: Shallow copy (new container, same objects). **deepcopy**: Recursive copy (new container, new objects). **Example**: `list1 = [[1], [2]]`, `list2 = copy(list1)`. Modifying `list2[0][0]` changes `list1[0][0]` because inner lists are shared.

**P4**
**Decorator**: Function that wraps another function.
```python
import time
def log_time(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.perf_counter() - start}s")
        return result
    return wrapper
```

**P5**
**Generators**: Functions that yield values lazily. **yield**: Pauses execution, returns value. **Differ from list**: No memory for all items, iterates once. **yield from**: Delegates to another generator.

**P6**
**Event Loop**: Single-threaded scheduler for async tasks. **asyncio.run()**: Entry point. **asyncio.gather()**: Run tasks concurrently. **asyncio.create_task()**: Schedule task. **time.sleep()** blocks the loop; use **asyncio.sleep()** to yield control.

**P7**
**MRO (Method Resolution Order)**: C3 linearization algorithm. **Diamond Inheritance**: Resolved by order of bases. **Code**: `D(B, C)`. MRO: `D -> B -> C -> A`. `D().who()` calls `C.who()` (returns "C") because C appears before A in the MRO.

**P8**
**Metaclass**: Class of a class (controls class creation). **__init_subclass__**: Hook for subclass initialization (easier, no metaclass needed). **Use Case**: Metaclass for ORM registration; `__init_subclass__` for plugin discovery.

**P9**
**Multiprocessing**: Separate processes (bypass GIL, CPU-bound). **Threading**: Shared memory (GIL, I/O-bound). **concurrent.futures**: High-level API for both. **Serialization**: Multiprocessing requires **pickling** arguments/results, which fails for some objects (e.g., lambdas).

**P10**
**__slots__**: Declares fixed attributes. **Prevents**: Dynamic attribute creation (`__dict__` not created). **Memory**: Reduces memory footprint significantly for many instances.

**P11**
**Virtual Environment**: Isolated Python environment. **venv**: Standard library (Python 3.3+). **virtualenv**: Third-party (older). **conda**: Package manager (binary packages, data science). Use venv to avoid dependency conflicts.

**P12**
**Comprehensions**: Concise list/dict/set creation. **map/filter**: Functional iteration. **Prefer**: Comprehensions for readability. Use `map/filter` for lazy evaluation or complex logic.

**P13**
**==**: Value equality. **is**: Identity equality. **Interning**: Small integers (-5 to 256) are cached. `a is b` returns True for small ints because they point to the same object.

**P14**
***args**: Tuple of positional args. **kwargs**: Dict of keyword args. **/**: Positional-only (before). **\***: Keyword-only (after). Use `/` for API compatibility, `*` for clarity.

**P15**
**Context Manager**: Manages resources (`with` statement). **__enter__/__exit__**: Manual implementation. **@contextmanager**: Decorator for generator-based. **Example**: Database connection manager.

**P16**
**Type Hinting**: Static type annotations. **Union**: `int | str`. **Optional**: `int | None`. **Literal**: Specific values. **TypeVar**: Generic constraints. **Protocol**: Structural typing. **mypy/pyright**: Check types before runtime.

**P17**
**Dunder Methods**: Special methods (`__repr__`, `__str__`). **repr**: Debug/Dev. **str**: User. **eq/hash**: If `__eq__` defined, `__hash__` becomes `None` (unhashable) unless defined. **getattr/getattribute**: `getattr` calls hook; `getattribute` is called for every access.

**P18**
**Descriptors**: Objects controlling attribute access (`__get__`, `__set__`, `__delete__`). **property**, **classmethod**, **staticmethod** are descriptors. Used for validation, computed properties, and method binding.

**P19**
**Import System**: **sys.path**: Search paths. **PYTHONPATH**: Env var override. **__init__.py**: Marks package (Python 3.3+ namespace packages allow omitting). **Finder**: Locates module. **Loader**: Loads module. **Relative**: `from . import x`. **Absolute**: `from package import x`.

**P20**
**Dataclasses**: Auto-generates `__init__`, `__repr__`, etc. **attrs**: More features (validation). **Pydantic**: Validation + serialization. **__post_init__**: Logic after init. **Frozen**: Immutable. **Choose Pydantic** for API models requiring validation (e.g., FastAPI).
