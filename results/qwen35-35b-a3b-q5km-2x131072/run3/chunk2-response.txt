K1 — A **Deployment** manages stateless applications, creating and replacing Pods without guaranteed order or identity. A **StatefulSet** manages stateful applications, providing stable network IDs, ordered deployment/termination, and persistent storage guarantees.

K2 — A **ClusterIP** Service assigns a virtual IP within the cluster. **kube-proxy** (using iptables or IPVS) updates node networking rules to forward traffic destined for the Service IP to the backend Pod IPs.

K3 — A **PersistentVolume (PV)** is a cluster storage resource. A **PersistentVolumeClaim (PVC)** is a user request for storage. A **StorageClass** defines the provisioning class for dynamic PV creation. A PVC binds to a PV (or triggers dynamic provisioning via StorageClass).

K4 — 1. Check `kubectl logs` for application errors. 2. Check `kubectl describe pod` for events (e.g., OOMKilled, probe failures). 3. Verify resource limits (CPU/Memory) are not exceeded. 4. Inspect Liveness/Readiness probe configuration for incorrect thresholds.

K5 — **CoreDNS** provides DNS resolution for Kubernetes services. If CoreDNS pods are down, internal service discovery (e.g., `service.namespace.svc`) fails, though external DNS resolution may still work via fallback.

K6 — The scheduler **filters** nodes (feasibility) then **scores** them (ranking). Factors: 1. Resource availability (CPU/Memory). 2. Node Affinity/Taints. 3. Pod Affinity/Anti-affinity.

K7 — Yes, it will be scheduled. Node free RAM = 4GB - 3.5GB = 0.5GB (512MB). Pod request (256Mi) < 512MB. At runtime, if usage hits 600Mi > limit (512Mi), the container will be **OOMKilled**.

K8 — 1. Request hits External Load Balancer. 2. LB forwards to Ingress Controller (NGINX). 3. NGINX matches Ingress rules to Service. 4. Service forwards to **Endpoints** (Pod IPs). 5. **kube-proxy** (iptables/IPVS) routes traffic to the specific Pod.

K9 — A **NetworkPolicy** restricts traffic flow. Default behavior: All traffic is allowed. When a policy is applied, traffic is denied by default unless explicitly allowed by the policy rules.

K10 — **RollingUpdate** replaces pods gradually (zero downtime). **Recreate** kills all existing pods before starting new ones. Choose **Recreate** for stateful apps requiring a full restart or atomic database migrations.

K11 — A **Namespace** is a logical cluster partition for resource isolation. Default namespaces: `default`, `kube-system`, `kube-public`, `kube-node-lease`.

K12 — `kubectl describe` shows detailed status. Information not in `get`: 1. **Events** (warnings/errors). 2. **Conditions** (Ready status details). 3. **Mounts/Env vars** (configuration details).

K13 — A **ConfigMap** stores non-sensitive configuration data. Pods consume it via: 1. **Environment Variables**. 2. **Volume Mounts** (files).

K14 — **ClusterIP**: Internal cluster access only. **NodePort**: Exposes service on a static port on every Node IP. **LoadBalancer**: Provisions an external cloud Load Balancer.

K15 — A **DaemonSet** ensures a copy of a Pod runs on all (or specific) Nodes. Use for logging agents, monitoring daemons, or storage daemons.

K16 — **Taints** repel pods from nodes; **Tolerations** allow pods to schedule on tainted nodes. **Node Affinity** attracts pods to nodes. Use case: Taint for dedicated GPU nodes; Affinity for co-locating app and cache.

K17 — An **Operator** is a custom controller managing a CRD. It differs from standard controllers by automating complex stateful app lifecycle (backup, scaling, upgrades). It solves operational complexity of stateful systems.

K18 — etcd uses **strong consistency** (Raft). If quorum is lost, the cluster becomes read-only/unavailable. Recovery: Restore from backup on the remaining node, then rejoin lost nodes or add new ones.

K19 — **Role** (NS rules), **ClusterRole** (Cluster rules), **RoleBinding** (NS binding), **ClusterRoleBinding** (Cluster binding).
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: dev
  name: pod-reader
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list", "watch"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  namespace: dev
  name: read-pods-binding
subjects:
- kind: User
  name: developer
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io
```

K20 — `ndots` (default 5) causes the resolver to try appending up to 5 suffixes to short hostnames, causing excessive queries. Fix: Set `dnsConfig: options: [{name: ndots, value: "1"}]` in the pod spec.

D1 — **Rebase** rewrites history linearly; **Merge** preserves history with a merge commit. Avoid rebasing on shared/public branches to prevent history rewriting conflicts.

D2 — **TCP**: Reliable, connection-oriented. **UDP**: Unreliable, connectionless. TCP: HTTP, SSH. UDP: DNS, VoIP.

D3 — A **race condition** occurs when output depends on the timing of events. Example: Two threads incrementing a shared counter without a lock, resulting in lost updates.

D4 — **Symmetric**: Single key for encryption/decryption. **Asymmetric**: Public/Private key pair. TLS uses Asymmetric for handshake/key exchange, then Symmetric for bulk data encryption.

D5 — A **layer** is a filesystem diff in a container image. Caching: Place dependency installation before application code in the Dockerfile to reuse cached layers.

D6 — 1. Check upstream service health. 2. Verify database connectivity. 3. Check rate limiting/throttling logs. 4. Monitor resource exhaustion (CPU/Memory).

D7 — **CAP**: Consistency, Availability, Partition tolerance. **CP**: Zookeeper (prioritizes consistency). **AP**: DynamoDB (prioritizes availability).

D8 — **Horizontal**: Add more nodes. **Vertical**: Add more resources to existing nodes. Patterns: Stateless design, Sharding, Load Balancing.

D9 — An **environment variable** is runtime configuration. Persist across sessions: Add `export VAR=value` to `~/.bashrc` or `~/.profile`.

D10 — **JSON**: Strict syntax. **YAML**: Human-readable indentation. JSON: Machine parseable. YAML: Less verbose.

D11 — **REST**: Architectural style for APIs. **GET** (read), **POST** (create), **PUT** (replace), **DELETE** (remove).

D12 — **Version pinning** locks dependencies to exact versions. Important for reproducible builds and security (prevents unexpected breaking changes).

D13 — **Compiled**: Source to binary (C, Go). **Interpreted**: Source to execution (Python, JS).

D14 — **Reverse Proxy**: Client -> Proxy -> Server (e.g., Nginx for SSL termination). **Forward Proxy**: Client -> Proxy -> Internet (e.g., corporate firewall).

D15 — A **deadlock** is a process waiting indefinitely for another. Conditions: 1. Mutual exclusion. 2. Hold and wait. 3. No preemption. 4. Circular wait.

D16 — **Eventual consistency**: Updates propagate over time. Cassandra uses **Read Repair** and tunable levels (ONE, QUORUM, ALL) to balance consistency vs. latency.

D17 — **mTLS**: Mutual TLS authentication. Handshake: Client sends cert, Server verifies; Server sends cert, Client verifies. Used in: Service Mesh (Istio), Internal API security.

D18 — **Hash Table**: Hash function maps key to index. Collision: **Chaining** (linked list), **Open Addressing** (probe). Complexity: Avg O(1), Worst O(n).

D19 — **Circuit Breaker**: Prevents cascade failures. States: **Closed** (normal), **Open** (fail fast), **Half-open** (test). Relates to retry: When open, retries are skipped to prevent overload.

D20 — **Optimistic**: Check version on commit (e.g., `version` column). **Pessimistic**: Lock on read (e.g., `SELECT FOR UPDATE`). Choose Optimistic for low conflict; Pessimistic for high conflict.
