JS1 — `var` is function-scoped and hoisted (initialized as `undefined`). `let` and `const` are block-scoped and hoisted but not initialized (Temporal Dead Zone). `const` cannot be reassigned. Hoisting means declarations are moved to the top of their scope during compilation.

JS2 — The event loop manages execution. The call stack runs synchronous code. When async operations complete, callbacks go to the microtask queue (Promises) or macrotask queue (setTimeout). The event loop processes the call stack, then all microtasks, then one macrotask, repeating.

JS3 — A closure is a function retaining access to its outer scope variables. Example: `function outer() { let x = 1; return () => x; }`. The classic bug: `for (var i=0; i<3; i++) setTimeout(() => console.log(i), 0)` logs 3, 3, 3. Fix: use `let` (block scope) or an IIFE to capture `i`.

JS4 — `Promise` represents async completion. `async/await` syntactic sugar for promises. Error handling: `try/catch` with `await`, or `.catch()` on promises. `Promise.all` rejects on first failure; `Promise.allSettled` waits for all to settle, returning status (`fulfilled`/`rejected`) for each.

JS5 — `interface` defines shape for objects/classes (can extend/implement). `type` defines aliases (can be primitives, unions, intersections). Use `interface` for public API contracts, `type` for unions/complex logic. Discriminated unions use a literal property to distinguish types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — Prototypal inheritance: objects link to a prototype via `[[Prototype]]`. `Object.create(proto)` creates an object with `proto` as its prototype. ES6 classes are syntactic sugar over prototypes; `class` defines a constructor and prototype methods.

JS7 — `this` binding rules (priority): 1. `new` (new instance), 2. Explicit (`call`, `apply`, `bind`), 3. Implicit (object method), 4. Default (global/window or undefined in strict mode). `bind()` returns a new function with bound `this`. It cannot be re-bound.

JS8 — `WeakRef` holds weak reference to object (GC can collect it). `FinalizationRegistry` notifies when object is collected. Use for caches or cleanup. `WeakMap` keys must be objects because primitives are immutable and GC relies on object identity.

JS9 — V8 optimizes via hidden classes (objects with same shape share class) and inline caching (cache property lookup). Deoptimization occurs when assumptions break (e.g., adding property to object, using `arguments` object, dynamic property access).

JS10 — Tree-shaking removes unused exports. Works with ES Modules (static `import`/`export`) because structure is known at build time. CommonJS (`require`) is dynamic, so bundlers cannot statically analyze dependencies effectively.

JS11 — `undefined` is default value; `null` is intentional absence. `==` converts types (`null == undefined` is true). `===` checks type and value. `??` returns right operand if left is `null`/`undefined`. `?.` safely accesses nested properties.

JS12 — `map` transforms, `filter` selects, `reduce` accumulates. `reduce` needs initial value to avoid using first element as initial. `flatMap` maps then flattens one level; useful for transforming arrays of arrays into single arrays.

JS13 — Template literals use backticks for multi-line strings and `${}` interpolation. Tagged templates pass the string array and values to a function. Example: `html` tag to escape values and prevent XSS.

JS14 — Destructuring extracts values. Object: `{a, b}`. Array: `[a, b]`. Defaults: `{a = 1}`. Renaming: `{a: b}`. Rest operator `...` collects remaining items into an array or object.

JS15 — `for...in` iterates over enumerable property keys (including inherited). `for...of` iterates over iterable values. Avoid `for...in` on arrays because it includes non-numeric keys and order isn't guaranteed.

JS16 — Generics use `<T>` for type flexibility. Constraints: `T extends SomeType`. Conditional types: `T extends U ? X : Y`. `infer` extracts types from conditional types. Example: `type First<T> = T extends (infer U)[] ? U : never`.

JS17 — CommonJS uses `require`/`module.exports` (synchronous, dynamic). ES Modules use `import`/`export` (static, async). Node.js supports both; `"type": "module"` in `package.json` forces `.js` files to be treated as ES Modules.

JS18 — Structural typing: types compatible if shapes match. Excess property checks apply to object literals in assignments. `extends` creates inheritance/constraints. `satisfies` checks shape without widening type. Variance: `in` (contravariant), `out` (covariant) for generic parameters.

JS19 — `Proxy` intercepts operations (get, set, apply). `Reflect` provides default implementations. Example: Proxy for validation or reactive data binding (Vue/React use similar concepts). Intercept `set` to trigger updates.

JS20 — `SharedArrayBuffer` allows shared memory between workers. `Atomics` provides atomic operations. Ordering guarantees via memory barriers. `Atomics.wait` blocks until `notify` is called. Used for high-performance inter-thread communication.

B1 — `$@` preserves arguments as separate words (respects quotes). `$*` joins arguments into one string. `"$@"` expands to separate quoted arguments. `"$*"` expands to one string. Use `"$@"` for safe argument passing.

B2 — `set -e`: exit on error. `set -u`: error on undefined vars. `set -o pipefail`: exit if any command in pipe fails. Ensures scripts fail fast and predictably.

B3 — `$(command)` is preferred over backticks (nested easier). `$((expression))` is arithmetic expansion. `expr` is external command (slower, legacy). Prefer `$(( ))` for math.

B4 — Process substitution `<(command)` creates a file descriptor. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files. Useful for comparing command outputs directly.

B5 — Use `while IFS= read -r -d '' file; do ... done < <(find . -print0)`. `for f in $(ls)` breaks on spaces/newlines. `find -print0` + `read -d ''` handles special chars safely.

B6 — Here doc `<<EOF` reads until EOF. Here string `<<<` passes string as stdin. `<<-EOF` allows leading tabs to be stripped. Use here docs for multi-line input, here strings for single-line.

B7 — `cmd > file 2>&1`: stdout to file, stderr to stdout (file). Both in file. `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. Only stdout in file. `&>` is shorthand for both to file. Order matters.

B8 — Use `mkdir /tmp/lock` (atomic). `trap 'rm -rf /tmp/lock' EXIT`. Check if dir exists before creating. `mkdir` is preferred over file creation because it's atomic on POSIX, preventing race conditions.

B9 — Single quotes: literal (no expansion). Double quotes: expansion (vars, commands). No quotes: word splitting/globbing. Example: `echo '$HOME'` (literal), `echo "$HOME"` (expanded), `echo $HOME` (expanded + glob).

B10 — `[ -f file ]` (file), `[ -d dir ]` (directory), `[ -r file ]` (readable). `[[ ]]` is bash builtin (supports `&&`, `||`, regex). `[ ]` is external `test` command. Prefer `[[ ]]` in bash scripts.

B11 — `$?`: exit code of last command. `$$`: current PID. `$!`: PID of last background job. `#`: number of arguments. Use `$?` for error checking, `$$` for temp files.

B12 — `function name() {}` or `name() {}`. Return values via `echo` (stdout) or `return` (exit code). `return` only works for exit status. Capture output via command substitution.

B13 — `PATH` is colon-separated dirs. Bash searches `PATH` for executables. Builtin is internal (faster). External is binary. Use `type` or `which` to check. `type -a` shows all.

B14 — `trap 'cmd' EXIT` runs on exit. `ERR` on error. `INT`/`TERM` on signals. Execution order: specific signal handlers run, then `EXIT` trap. Multiple signals: last one wins unless handled.

B15 — `xargs` builds command lines from stdin. `command substitution` runs command immediately. `xargs -0` + `find -print0` handles newlines/spaces. `xargs -P` runs jobs in parallel.

B16 — Arrays: `arr=(1 2)`. Associative: `declare -A arr`. Append: `arr+=(val)`. Length: `${#arr[@]}`. Iteration: `for i in "${arr[@]}"`. Associative keys are strings, not integers.

B17 — Subshell `(...)` runs in new process (vars lost). Group `{ ...; }` runs in current shell (vars persist). Piping into `while` creates subshell, so loop vars don't persist outside. Use process substitution to avoid.

B18 — SIGINT (Ctrl+C), SIGTERM (kill), SIGHUP (hangup), SIGKILL (force kill, no handler). Handle graceful shutdown by trapping SIGTERM, cleaning up, then exiting. Background jobs need `wait` or `disown`.

B19 — `split` file, `xargs -P` parallel jobs, merge results. Trade-off: speed vs complexity. Error propagation: check exit codes in parallel jobs, aggregate errors, exit non-zero if any fail. Use `set -o pipefail`.

B20 — `exec` replaces shell process. `exec 3>file` opens fd 3. `exec 3>&-` closes fd. Coprocess (`coproc`) creates bidirectional pipe. Use for persistent connections or long-running processes.

PS1 — `Write-Output` sends objects to pipeline. `Write-Host` writes to console (bypasses pipeline). Use `Write-Output` for scripting/piping, `Write-Host` for UI only.

PS2 — Pipeline passes objects. `ForEach-Object` is cmdlet (pipeline aware). `foreach` is statement (iterates collection in memory). Use `ForEach-Object` for streaming, `foreach` for pre-loaded collections.

PS3 — `[PSCustomObject]` creates typed object with properties. `[hashtable]` is key-value pairs. Convert: `[PSCustomObject]@{Key='Val'}`. Use CustomObject for structured data, hashtable for config.

PS4 — Terminating errors stop script. Non-terminating continue. `-ErrorAction Stop` converts non-terminating to terminating. `$ErrorActionPreference` sets default. `try/catch` handles terminating errors.

PS5 — Splatting `@params` passes hashtable as arguments. `func @params` vs `func (Get-Content params)`. Splatting expands arguments. Passing hashtable as single param treats it as one argument.

PS6 — `Invoke-Command -ComputerName` runs remotely. `Enter-PSSession` interactive. Uses WinRM (HTTP/HTTPS) or SSH (v7+). WinRM is default on Windows.

PS7 — `+=` on array creates new array. `Get-Items` returns array. `$x.GetType().Name` is `Object[]`. If 0 items, returns `$null`. Correct pattern: `$result = @(); $result += ...; return $result`.

PS8 — `[scriptblock]` is code object. Pass via `-ArgumentList`. Double-hop: remote session to another remote. Solve with CredSSP (insecure) or `Invoke-Command -ArgumentList` to pass creds securely.

PS9 — `@()` forces array. `[array]` casts. Indexing `$null` throws error. Indexing `@()` returns empty. Use `@()` to ensure array type.

PS10 — `-eq` (equal), `-like` (glob), `-match` (regex). `-ceq` (case-sensitive). `-contains` checks if array contains value. `-in` checks if value is in array (newer, cleaner).

PS11 — Module: reusable code. `.psm1` script, `.psd1` manifest (metadata), binary (DLL). `Import-Module` loads module. Manifest defines exports, aliases, functions.

PS12 — `"..."` interpolates. `'...'` literal. `@"..."@` here-string (multi-line). Expressions: `"Value: $var"`. Escape `$` with backtick.

PS13 — `Get-Member` lists properties/methods. Properties (data), Methods (actions), NoteProperties (added dynamically). `Select-Object` selects props. `Where-Object` filters objects.

PS14 — Classes (v5+) define structure. Limitations: no interfaces, limited inheritance. Use classes for complex state, functions for simple logic. Classes compile to C#.

PS15 — `[CmdletBinding()]` enables advanced features. `SupportsShouldProcess` for `-WhatIf`. `Begin/Process/End` blocks for pipeline stages. `ValueFromPipeline` binds pipeline input.

PS16 — Providers expose data as file system. `Get-ChildItem` works on Registry, Certs, Env. Access registry: `Get-ChildItem HKLM:\Software`. Treats keys as folders, values as files.

PS17 — Runspaces: isolated execution. `[runspacepool]` manages threads. `ForEach-Object -Parallel` (v7+) uses runspace pool. Manual runspace for complex control. Thread safety: avoid shared mutable state.

PS18 — PSScriptAnalyzer: linting tool. Rules enforce standards. Severity: Error, Warning, Info. Suppress with `#pragma`. Integrate in CI to block bad code.

PS19 — `$using:` passes local vars to remote. `-ArgumentList` passes arguments. Serialization: CLIXML. Limits: depth 1000, complex objects may fail. Use simple types for remote.

PS20 — DSC: declarative config. Resources define state. LCM applies config. WinPS DSC uses XML. PS7 uses YAML. Replaced by Ansible, Terraform, or native PS7 DSC (experimental).
