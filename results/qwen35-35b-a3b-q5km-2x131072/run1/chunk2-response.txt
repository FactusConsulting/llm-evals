K1: A **Deployment** manages stateless applications with interchangeable pods (random names, no guaranteed order). A **StatefulSet** manages stateful applications requiring stable network IDs, ordered deployment/termination, and persistent storage per pod.

K2: A **ClusterIP** Service creates a virtual IP. **kube-proxy** (using iptables or IPVS) updates rules on nodes to route traffic destined for the Service IP to the backend Pod IPs listed in the Service's **Endpoints** object.

K3: A **PersistentVolume (PV)** is a cluster storage resource. A **PersistentVolumeClaim (PVC)** is a user request for storage. A **StorageClass** defines the provisioner for dynamic PV creation. A PVC binds to an existing PV (static) or triggers a PV creation via a StorageClass (dynamic).

K4: 1. Check **application logs** (`kubectl logs`) for errors. 2. Check **pod events** (`kubectl describe`) for OOMKilled or probe failures. 3. Verify **resource limits** (CPU/Memory) are not exceeded. 4. Check **dependencies** (e.g., database connectivity) or configuration mounts.

K5: **CoreDNS** provides DNS resolution for services and pods within the cluster. If CoreDNS pods are down, internal service discovery (e.g., `service-name.namespace`) fails, though external DNS may still work if fallback is configured.

K6: The scheduler uses **Filtering** (remove ineligible nodes) and **Scoring** (rank eligible nodes). Factors considered: 1. **Resource Requests** (CPU/Mem fit), 2. **Node Affinity/Taints**, 3. **Pod Affinity/Anti-Affinity**.

K7: Node has 4GB, 3.5GB requested = 0.5GB (512Mi) free. Pod requests 256Mi. **Yes, it will be scheduled** (256Mi < 512Mi). If it uses 600Mi at runtime, it exceeds the 512Mi limit and will be **OOMKilled**.

K8: 1. Request hits **External LB**. 2. LB forwards to **Ingress Controller** (NGINX). 3. Controller routes based on rules to a **Service**. 4. Service resolves to **Endpoints** (Pod IPs). 5. **kube-proxy/iptables** on the node routes traffic to the specific **Pod**.

K9: A **NetworkPolicy** defines pod communication rules (firewall). Default behavior (no policy): **All traffic allowed**. When applied: Default **deny** for ingress/egress unless explicitly allowed by the policy selectors.

K10: **RollingUpdate** replaces pods gradually (zero downtime). **Recreate** kills all existing pods before starting new ones. Choose **Recreate** for stateful apps requiring full shutdown or when configuration changes necessitate a complete restart.

K11: A **Namespace** provides logical isolation for resources. Default namespaces: `default`, `kube-system`, `kube-public`, `kube-node-lease`.

K12: `kubectl describe pod` is used for **debugging**. It shows: 1. **Events** (scheduling, pull errors), 2. **Conditions** (Ready status), 3. **Mounts/Env Vars** (detailed config), which `kubectl get` omits.

K13: A **ConfigMap** stores non-sensitive configuration data. Pods consume it via: 1. **Environment Variables**, 2. **Volume Mounts** (files).

K14: **ClusterIP**: Internal cluster access only. **NodePort**: Exposes service on each node's IP at a static port. **LoadBalancer**: Provisions an external cloud load balancer.

K15: A **DaemonSet** ensures a copy of a pod runs on **all (or selected) nodes**. Use for logging agents, monitoring daemons, or storage services that must run on every node.

K16: **Taints** repel pods from nodes; **Tolerations** allow pods to schedule on tainted nodes. **Node Affinity** attracts pods to nodes. Use Case: Taint GPU nodes (repel general pods); Affinity to co-locate app and cache pods.

K17: A **Kubernetes Operator** is a custom controller that manages CRDs to automate complex application lifecycle (e.g., DB backups). It differs from standard controllers by encoding operational knowledge and managing stateful resources deeply.

K18: etcd uses **strong consistency** (linearizable). If quorum is lost, the cluster becomes **read-only or unavailable**. Recovery: Restore from a **snapshot** on new node(s) to re-establish a quorum (e.g., add nodes to replace lost ones).

K19: **Role** (NS-scoped permissions), **ClusterRole** (cluster-scoped), **RoleBinding** (links Role to subject in NS), **ClusterRoleBinding** (links ClusterRole to subject cluster-wide).
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata: namespace: my-ns
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list", "watch"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata: namespace: my-ns
subjects:
- kind: User
  name: developer
roleRef:
  kind: Role
  name: pod-reader
```

K20: `ndots` defines how many dots a domain must have to be treated as FQDN. `ndots:5` causes queries for `google.com` to try `google.com`, `google.com.ns1`, etc., causing delays. Fix: Set `dnsConfig` in pod spec: `options: [{name: "ndots", value: "1"}]`.

D1: **Rebase** rewrites history to create a linear commit chain. **Merge** preserves history with a merge commit. Avoid rebasing on **shared branches** to prevent history conflicts for collaborators.

D2: **TCP** is connection-oriented and reliable (HTTP, SSH). **UDP** is connectionless and fast (DNS, VoIP).

D3: A **race condition** occurs when multiple threads access shared data concurrently, leading to unpredictable results. Example: Two threads increment a counter without a lock, resulting in a lost update.

D4: **Symmetric** uses one key for encryption/decryption. **Asymmetric** uses a public/private key pair. TLS uses **Asymmetric** for the handshake (key exchange) and **Symmetric** for the actual data transfer (performance).

D5: A **container image layer** is an immutable filesystem snapshot. **Layer caching** improves build speed; place frequently changing instructions (e.g., `COPY .`) at the bottom of the Dockerfile to cache stable dependencies.

D6: 1. Check **application logs** for errors. 2. Verify **database/dependency health**. 3. Check **resource usage** (CPU/Memory throttling). 4. Review **load balancer health checks**.

D7: **CAP Theorem**: Consistency, Availability, Partition Tolerance. **CP** (Consistency + Partition Tolerance): etcd, HBase. **AP** (Availability + Partition Tolerance): Cassandra, DynamoDB.

D8: **Horizontal Scaling** adds more nodes; **Vertical Scaling** adds resources to existing nodes. Patterns enabling horizontal scaling: **Stateless services**, **Sharding**, **Message Queues**.

D9: An **environment variable** is runtime configuration. To persist across sessions on Linux, export it in `~/.bashrc` or `/etc/environment`.

D10: **JSON** is strict and machine-readable. **YAML** is human-readable and supports comments. Advantage JSON: Web standard. Advantage YAML: Readability for config files.

D11: **REST API** uses HTTP methods. **GET** (retrieve), **POST** (create), **PUT** (replace), **DELETE** (remove).

D12: **Version pinning** locks dependencies to specific versions (e.g., `==1.0.0`). It ensures **reproducible builds** by preventing unexpected breakage from upstream updates.

D13: **Compiled** languages (C++, Go) translate to binary before execution. **Interpreted** languages (Python, JavaScript) execute line-by-line at runtime.

D14: A **reverse proxy** sits in front of servers (Nginx, HAProxy) to handle client requests. A **forward proxy** sits in front of clients. Use Case: Nginx for SSL termination; HAProxy for load balancing.

D15: A **deadlock** is a state where processes wait for each other indefinitely. Coffman Conditions: 1. Mutual Exclusion, 2. Hold and Wait, 3. No Preemption, 4. Circular Wait.

D16: **Eventual Consistency** ensures data converges over time. Cassandra uses **Read Repair** and **Hinted Handoff**. Tunable levels: **ONE, QUORUM, ALL**.

D17: **mTLS** requires both client and server to present certificates. Handshake: Client Hello -> Server Hello + Cert -> Client Verify -> Server Verify. Commonly used in **Service Meshes** (e.g., Istio) for internal service security.

D18: A **hash table** maps keys to buckets via a hash function. Collision resolution: **Chaining** (linked lists) or **Open Addressing** (probing). Time Complexity: Average **O(1)**, Worst **O(n)** (all collisions).

D19: A **circuit breaker** prevents cascade failures. States: **Closed** (normal), **Open** (fail fast), **Half-open** (test). It relates to retries/backoff by stopping retries when the circuit opens and allowing limited probes when half-open.

D20: **Optimistic** concurrency checks version on commit (e.g., `UPDATE ... WHERE version = old`). **Pessimistic** locks data on read (e.g., `SELECT ... FOR UPDATE`). Choose Optimistic for low contention; Pessimistic for high contention.
