DN1 — **Class vs Struct**: `class` is a reference type (heap-allocated, mutable by default), while `struct` is a value type (stack-allocated or inline, immutable by default). Use a `struct` for small, immutable data types that represent a single value (e.g., `Point`, `ComplexNumber`) to avoid heap allocation and copying overhead.

DN2 — **IEnumerable<T> vs IQueryable<T>**: `IEnumerable<T>` executes queries in-memory (client-side), while `IQueryable<T>` builds an expression tree that is translated to a query language (e.g., SQL) and executed on the server. This distinction matters for database queries because `IQueryable` allows filtering, projection, and sorting to happen on the database server, reducing network traffic and improving performance.

DN3 — **Async/Await & Task vs ValueTask**: `async/await` uses a state machine to suspend and resume execution without blocking threads. `Task` is a reference type; `ValueTask` is a value type (struct). Use `ValueTask` for high-performance scenarios where the operation completes synchronously most of the time or is a single-use operation to avoid heap allocation.

DN4 — **Dependency Injection (DI) & Lifetimes**: DI manages object creation and dependencies. **Transient**: New instance every request. **Scoped**: One instance per request (web). **Singleton**: One instance for the app lifetime. **Bug**: Injecting a Scoped service into a Singleton captures the Singleton's lifetime, potentially causing the Scoped service to be reused across requests (stale data) or disposed prematurely.

DN5 — **string vs ReadOnlySpan<char>**: `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a stack-only, zero-copy view over memory. Use `Span<T>` (and `ReadOnlySpan`) for high-performance string processing to avoid allocations during substring operations or parsing.

DN6 — **Garbage Collector (GC)**: The GC automatically reclaims memory. **Generations**: Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **LOH**: Large Object Heap (>85KB) for large objects, collected less frequently. **IDisposable**: Needed for unmanaged resources (files, DB connections) because the GC only manages managed memory; `Dispose` ensures timely cleanup.

DN7 — **ConfigureAwait(false)**: Default behavior captures the `SynchronizationContext` (e.g., UI thread) to resume on the same context. `ConfigureAwait(false)` skips this. Use `false` in library code to avoid deadlocks and improve performance. Do NOT use it in UI code where you must update the UI thread after the await.

DN8 — **.NET 8/9 Features**: **Primary Constructors**: `public class Service(int Id) : IService { }`. **Collection Expressions**: `[1, 2, 3]`. **FrozenDictionary**: Immutable, high-performance dictionary. **Example**: `public class UserService(IRepository repo) : IUserService { }` (Constructor injection without explicit fields).

DN9 — **DbContext Exception**: Likely cause: `DbContext` registered as `Singleton` instead of `Scoped`. EF Core `DbContext` is not thread-safe. Under high concurrency, multiple requests share the same context instance, causing race conditions and `ObjectDisposedException`. Fix: Register as `services.AddScoped<DbContext>()`.

DN10 — **Record vs Class**: `record` provides value-based equality and immutable properties by default. `class` provides reference equality. `record struct` combines value type semantics with record equality. Choose `record` for DTOs/Models, `class` for entities requiring identity/reference semantics.

DN11 — **ref, out, in**: `ref`: Pass by reference (read/write, must be initialized). `out`: Pass by reference (write only, must be assigned in method). `in`: Pass by reference (read-only, prevents copy of large structs).

DN12 — **Nullable Reference Types (NRT)**: `?` annotation (e.g., `string?`) warns about potential nulls for reference types at compile-time. `int?` (nullable value type) allows `null` for value types at runtime. NRTs improve safety without runtime overhead for references.

DN13 — **LINQ**: Language Integrated Query. **Method Syntax**: `list.Where(...)`. **Query Syntax**: `from x in list where ...`. Both compile to the same IL. **Deferred Execution**: Query is not executed until enumerated (e.g., `ToList()`), allowing composition.

DN14 — **Generic Constraints**: `where T : class` (reference type), `where T : struct` (value type), `where T : new()` (parameterless constructor), `where T : IComparable<T>`. Combine constraints (e.g., `where T : class, IDisposable`) to enforce multiple compile-time requirements.

DN15 — **Abstract Class vs Interface**: Abstract class can have implementation and state; Interface defines a contract. **C# 8+ Default Interface Methods**: Interfaces can now have method implementations, blurring the line and allowing evolution without breaking implementations.

DN16 — **Middleware Pipeline**: Request processing chain. `app.Use()` adds middleware (calls `next`). `app.Map()` branches the pipeline. `app.Run()` ends the pipeline. **Order**: Auth before MVC. **Exception Handling**: Middleware can catch exceptions and return custom responses.

DN17 — **Channel<T>**: Async producer-consumer pattern. **BlockingCollection<T>**: Synchronous. **Bounded Channels**: Limit capacity; producers block when full, providing backpressure. Example: `Channel.CreateBounded<int>(100)`.

DN18 — **Source Generators**: Compile-time code generation (faster than reflection). **Incremental Generators**: Only regenerate when source changes (preferred over `ISourceGenerator` for performance). **IL Weaving**: Modifies IL at build time (e.g., Fody); Generators emit C# code.

DN19 — **TPL Dataflow**: Pipeline processing library. **ActionBlock**: Process items (void return). **TransformBlock**: Map items. **BufferBlock**: Queue items. **Vs Channels**: Dataflow offers complex flow control (fan-out, fan-in, cancellation); Channels are simpler for basic producer-consumer.

DN20 — **Minimal APIs**: Lightweight, no controllers. **Trade-offs**: Less boilerplate, but harder to scale complex auth/middleware compared to Controllers. **Route Groups**: Organize endpoints (`MapGroup`). **Endpoint Filters**: Apply logic (e.g., validation) to specific endpoints.

P1 — **List vs Tuple**: `list` is mutable, ordered collection. `tuple` is immutable, ordered collection. `set` is mutable, unordered unique items. `frozenset` is immutable, hashable set.

P2 — **GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects **CPU-bound** workloads. **Workarounds**: Multiprocessing (separate processes), C-extensions (release GIL), or async I/O.

P3 — **Deepcopy vs Copy**: `copy` creates a new container but references same objects (shallow). `deepcopy` recursively copies objects. **Example**: `list1 = [[1]]`, `list2 = copy(list1)`. Modifying `list2[0][0]` affects `list1`.

P4 — **Decorators**: Functions wrapping other functions. **Time Logger**:
```python
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.perf_counter() - start}s")
        return result
    return wrapper
```

P5 — **Generators & Yield**: Generators produce values lazily. `yield` pauses execution and returns a value. **Vs List**: No memory for all items. `yield from` delegates to another generator.

P6 — **Asyncio Event Loop**: Single-threaded scheduler. `asyncio.run()` starts loop. `asyncio.gather()` waits for multiple tasks. `asyncio.create_task()` schedules task. `time.sleep()` blocks the loop; use `await asyncio.sleep()`.

P7 — **MRO (Method Resolution Order)**: Python 3 uses **C3 Linearization**. For `class D(B, C)`, MRO is `D -> B -> C -> A`. `D().who()` calls `C.who()` because `C` appears before `A` in the linearization of `D`. Prints "C".

P8 — **Metaclass**: Class of a class. **Metaclass**: Control class creation (e.g., enforcing attributes). `__init_subclass__`: Easier hook for subclass initialization (e.g., auto-registering subclasses). Use Metaclass for complex class-level logic; `__init_subclass__` for simpler subclass hooks.

P9 — **Multiprocessing vs Threading**: **Multiprocessing**: Separate processes (bypass GIL, good for CPU-bound). **Threading**: Shared memory (GIL limits CPU-bound, good for I/O). **concurrent.futures**: High-level API for both. **Serialization**: Multiprocessing requires pickling arguments (objects must be picklable).

P10 — **__slots__**: Prevents creation of `__dict__` for instances. **Use**: Save memory, prevent dynamic attributes. **Impact**: Reduces memory per instance significantly but prevents adding new attributes at runtime.

P11 — **Virtual Environment**: Isolated Python environment. **Why**: Dependency management. **Differences**: `venv` (stdlib, Python 3.3+), `virtualenv` (3rd party, supports older), `conda` (binary packages, cross-platform).

P12 — **Comprehensions**: Concise loop syntax (`[x for x in list]`). **Vs map/filter**: Comprehensions are more Pythonic/readable. Prefer comprehensions for simple transformations; `map/filter` for functional pipelines.

P13 — **== vs is**: `==` checks value equality. `is` checks identity (memory address). **Interning**: Small integers (-5 to 256) are cached. `a is b` returns True for small ints due to interning.

P14 — **args/kwargs**: `*args` (tuple of positional), `**kwargs` (dict of keyword). **Markers**: `/` (positional-only before), `*` (keyword-only after). Example: `def f(a, /, b, *, c):`.

P15 — **Context Managers**: Manage resources (`with` statement). **__enter__/__exit__**: Class-based. **@contextmanager**: Generator-based. **Example**: Database connection manager ensuring commit/rollback.

P16 — **Type Hinting**: Static type annotations. **Union**: `A | B`. **Optional**: `A | None`. **Literal**: Specific values. **TypeVar**: Generic constraints. **Protocol**: Structural subtyping. **mypy/pyright**: Analyze code without running to catch type errors.

P17 — **Data Model (Dunder)**: `__repr__` (dev), `__str__` (user). `__eq__` (equality), `__hash__` (hashability). **Rule**: If `__eq__` is defined, `__hash__` becomes `None` (unhashable) unless explicitly defined. `__getattr__` (missing attr), `__getattribute__` (all attr access).

P18 — **Descriptors**: Objects controlling attribute access (`__get__`, `__set__`, `__delete__`). **property**, `classmethod`, `staticmethod` are descriptors. Used for validation, computed properties, and method binding.

P19 — **Import System**: **sys.path**: Search paths. **Relative**: `from . import mod`. **Absolute**: `from pkg import mod`. **__init__.py**: Marks package (PEP 420 namespace packages omit this). **Finder**: Locates module. **Loader**: Loads module.

P20 — **Dataclasses**: Reduces boilerplate (`__init__`, `__repr__`). **Vs attrs/Pydantic**: attrs (more flexible), Pydantic (validation). **__post_init__**: Logic after init. **Frozen**: Immutable. **Choose Pydantic**: When runtime validation (e.g., JSON schema) is required.
