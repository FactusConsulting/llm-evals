JS1 — `var` is function-scoped and hoisted with initialization to `undefined`. `let` and `const` are block-scoped and hoisted but not initialized (Temporal Dead Zone). Hoisting is the mechanism where declarations are moved to the top of their scope during compilation. `const` cannot be reassigned, but its properties can be mutated.

JS2 — The event loop manages asynchronous execution. The call stack executes synchronous code. When the stack is empty, the loop checks the microtask queue (Promises, `queueMicrotask`) first, then the macrotask queue (setTimeout, I/O). Microtasks drain completely before the next macrotask.

JS3 — A closure is a function bundled with its lexical environment. Example: `function outer() { let x=1; return () => x; }`. The classic bug: `for (var i=0; i<3; i++) setTimeout(() => console.log(i), 0)` logs `3, 3, 3` because `i` is shared. Fix: use `let` (block scope) or an IIFE to capture `i`.

JS4 — `Promise` represents an async operation's eventual completion. `async/await` syntactic sugar for promises. Error handling: `try/catch` for `await`, `.catch()` for promises. `Promise.all` rejects on first failure; `Promise.allSettled` waits for all to settle, returning status (`fulfilled`/`rejected`) for each.

JS5 — `interface` defines shape only, supports `extends` and `implements`, and can be merged. `type` defines aliases, supports primitives, unions, intersections, and tuples. Use `interface` for public API contracts; `type` for complex unions or primitives. Discriminated unions use a literal property to narrow types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — Prototypal inheritance links objects via `[[Prototype]]`. `Object.create(proto)` creates an object with `proto` as its prototype. ES6 `class` is syntactic sugar over prototypes; `extends` sets the prototype chain, `constructor` initializes `this`.

JS7 — `this` binding depends on call site. Priority: 1. `new` (binds to instance), 2. Explicit (`call`, `apply`, `bind`), 3. Receiver object (`obj.method()`), 4. Default (`window`/`undefined` in strict mode). `bind()` returns a new function with bound `this`. It cannot be re-bound.

JS8 — `WeakRef` holds a weak reference to an object (GC can collect it). `FinalizationRegistry` notifies when a weakly held object is collected. Use for caches or cleanup. `WeakMap` keys must be objects because primitives are copied, not referenced, preventing garbage collection tracking.

JS9 — V8 optimizes via hidden classes (shapes) and inline caching (IC). Objects with same structure share hidden classes. IC caches property access offsets. Deoptimization occurs when structure changes (e.g., adding properties dynamically). Avoid dynamic property addition in tight loops.

JS10 — Tree-shaking removes unused exports. ES Modules (static `import`/`export`) allow static analysis. CommonJS (`require`) is dynamic, so bundlers cannot statically determine dependencies, making tree-shaking ineffective without transpilation to ESM.

JS11 — `undefined` is a primitive type/value (unassigned). `null` is an assignment value (empty object pointer). `==` converts types (`null == undefined` is true). `===` checks type and value. `??` returns right operand if left is `null` or `undefined`. `?.` safely accesses nested properties.

JS12 — `map` transforms, `filter` selects, `reduce` accumulates. `reduce` takes `(acc, curr)`. Initial value prevents `acc` being the first element. `flatMap` maps then flattens one level. Useful for transforming nested arrays into flat lists.

JS13 — Template literals use backticks, support interpolation `${}` and multiline. Tagged templates pass the string array and values to a function. Example: `html` tag to escape HTML: `function html(strings, ...values) { return strings.join('') + values.join(''); }`.

JS14 — Destructuring extracts values. Object: `{a, b}`. Array: `[a, b]`. Defaults: `{a = 1}`. Renaming: `{a: b}`. Rest operator `...` collects remaining items. In destructuring, `...rest` gathers remaining elements into an array.

JS15 — `for...in` iterates over enumerable property keys (including inherited). `for...of` iterates over iterable values (array elements). Avoid `for...in` on arrays because it includes non-numeric keys and order isn't guaranteed.

JS16 — Generics use `<T>` for type flexibility. Constraints: `T extends SomeType`. Conditional types: `T extends U ? X : Y`. `infer` infers types in conditional types. Example: `type UnwrapPromise<T> = T extends Promise<infer U> ? U : T`.

JS17 — CommonJS uses `require`/`module.exports` (dynamic, synchronous). ES Modules use `import`/`export` (static, async). Node.js supports both; `type: "module"` in `package.json` forces ESM. ESM supports top-level await; CJS does not.

JS18 — TypeScript uses structural typing (shape matches). Excess property checks apply to object literals assigned to variables. `extends` creates a subtype relationship. `satisfies` checks shape without widening type. Variance: `in` (contravariant), `out` (covariant) for generics in 4.7+.

JS19 — `Proxy` intercepts operations (get, set, apply). `Reflect` provides default implementations. Example: Validation Proxy. `new Proxy(obj, { set(target, key, val) { if (val < 0) throw Error; return true; } })`. `Reflect` methods ensure correct `this` binding.

JS20 — `SharedArrayBuffer` allows memory sharing between workers. `Atomics` provides atomic operations. Workers communicate via `postMessage` or shared memory. Ordering guarantees: `Atomics.wait` blocks until `notify` or timeout. Prevents race conditions on shared integers.

B1 — `$@` expands to separate quoted arguments (preserves word boundaries). `$*` expands to a single string joined by IFS. `"$@"` preserves each argument as a separate word. `"$*"` joins them into one string. Use `"$@"` for passing arguments to commands.

B2 — `set -e` exits on error. `set -u` errors on undefined variables. `set -o pipefail` returns error if any command in pipe fails. Together, they enforce strict error handling in scripts.

B3 — `$(command)` is preferred over backticks `` `command` `` (modern, nested). `$((expression))` evaluates arithmetic. `expr` is external and slower. `$(( ))` is built-in and safer for arithmetic.

B4 — Process substitution `<(command)` creates a file descriptor `/dev/fd/N`. Used when a command expects a filename but needs stdin. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — Use `while IFS= read -r -d '' file; do ... done < <(find . -print0)`. `for f in $(ls)` fails on spaces/newlines and parses output incorrectly. `find -print0` with `read -d ''` handles special characters safely.

B6 — Here document `<<EOF` reads until EOF line. Here string `<<<` passes a string as stdin. `<<-EOF` allows leading tabs to be stripped from the delimiter line.

B7 — `cmd > file 2>&1`: stdout to file, stderr to stdout (file). `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. `&> file`: redirects both to file (bash extension). First two differ due to redirection order.

B8 — Use `mkdir` for locking (atomic). Function: `lock_dir=$(mktemp -d); trap 'rm -rf "$lock_dir"' EXIT; mkdir "$lock_dir" || exit 1`. `mkdir` is atomic on POSIX filesystems. `trap` ensures cleanup on SIGTERM/EXIT.

B9 — Single quotes `'...'`: literal, no expansion. Double quotes `"..."`: variable/command expansion. No quotes: word splitting/globbing. Example: `echo '$HOME'` prints `$HOME`. `echo "$HOME"` prints path. `echo *` prints files.

B10 — `[ -f file ]` checks file. `[ -d dir ]` checks directory. `[ -r file ]` checks readable. `[[ ]]` supports pattern matching (`==`) and logical operators (`&&`, `||`) without quoting. `[ ]` is POSIX, `[[ ]]` is Bash.

B11 — `$?`: exit status of last command. `$$`: PID of current shell. `$!`: PID of last background job. `#`: number of positional parameters. Use `$?` for error checking, `$$` for temp file naming.

B12 — `function name() { ... }` or `name() { ... }`. Return values via `return` (exit code) or `echo` (stdout). Functions share global scope unless `local` is used. `return` exits function; `echo` outputs data.

B13 — `PATH` is colon-separated list of directories. Bash searches `PATH` for executables. Builtin (e.g., `cd`) is internal. External (e.g., `ls`) is binary. `type -a cmd` shows all locations.

B14 — `trap 'cmd' SIGNAL`. `EXIT` runs on script end. `ERR` on error. `INT`/`TERM` on signals. Execution order: `EXIT` runs last. Multiple signals: `trap` handles one at a time; `trap -` unsets.

B15 — Command substitution `$(cmd)` captures output. `xargs` builds command lines from stdin. `xargs -0` handles null-delimited input (`find -print0`). `xargs -P N` runs N processes in parallel.

B16 — Arrays: `arr=(a b c)`. Append: `arr+=("d")`. Length: `${#arr[@]}`. Associative: `declare -A map`. Keys: `map=(["key"]="val")`. Iterate: `for k in "${!map[@]}"`.

B17 — Subshell `(...)` runs in new process (vars lost). Group `{ ...; }` runs in current shell (vars persist). Piping `while` creates subshell, so `var` changes inside loop are lost. Use process substitution `< <(cmd)` to avoid subshell.

B18 — SIGINT (Ctrl+C), SIGTERM (kill), SIGHUP (hangup), SIGKILL (force kill). `trap` handles SIGTERM. Background jobs: `wait` to sync. Use `trap` to kill child processes on script exit.

B19 — `split -l 1000 log.log part_`, `find . -name 'part_*' -print0 | xargs -0 -P 4 -I {} bash process.sh {}`. Merge: `cat part_* > result.log`. Trade-off: I/O vs CPU. Error propagation: check exit codes in loop, `set -o pipefail`.

B20 — `exec` replaces shell process. `exec 3>file` opens FD 3. `exec 3>&-` closes it. Coprocess: `coproc CMD`. Used for persistent communication channels. `exec` is faster for long-running scripts.

PS1 — `Write-Output` sends objects to pipeline. `Write-Host` writes to console (bypasses pipeline). Distinction matters: `Write-Host` breaks pipeline chaining and makes scripting harder. Use `Write-Output` for data flow.

PS2 — Pipeline passes objects. `ForEach-Object` is a cmdlet (pipeline-aware, processes one at a time). `foreach` is a statement (iterates collection in memory). Use `ForEach-Object` for streaming; `foreach` for in-memory loops.

PS3 — `[PSCustomObject]` creates typed objects with properties. `[hashtable]` is key-value pairs. Convert: `[PSCustomObject]@{Key='Val'}`. Use CustomObject for structured data/pipeline; Hashtable for config/maps.

PS4 — Terminating errors stop script (throw). Non-terminating continue (warning). `-ErrorAction Stop` converts non-terminating to terminating. `try/catch` handles terminating errors. `$ErrorActionPreference` sets default behavior.

PS5 — Splatting `@params` passes hashtable as arguments. `func @params` vs `func $params`. Splatting expands keys to parameters. Passing hashtable as single param treats it as one argument.

PS6 — `Invoke-Command -ComputerName` runs one-off. `Enter-PSSession` interactive. Uses WS-Management (WinRM) or SSH (v7+). WinRM is default on Windows; SSH requires config.

PS7 — Returns array even for single item. `$x.GetType().Name` is `Object[]`. If 0 items, returns `$null`. Correct pattern: `$result = @(); $result += ...`. Or force array: `$x = @(Get-Items)`.

PS8 — `[scriptblock]` is code object. Pass via `-ArgumentList` or `Invoke-Command -ScriptBlock`. Double-hop: CredSSP allows auth forwarding. `Invoke-Command -ArgumentList` serializes variables.

PS9 — `@()` forces array. `[array]` casts. Indexing `$null` returns `$null`. Indexing empty array returns `$null`. `@()` ensures consistent array type for iteration.

PS10 — `-eq` (equal), `-like` (glob), `-match` (regex). `-ceq` (case-sensitive). `-contains` checks if collection contains value. `-in` checks if value is in collection (newer, cleaner).

PS11 — Module: reusable code. `.psm1` script, `.psd1` manifest (metadata), binary (DLL). `Import-Module` loads it. Manifest defines exports, dependencies, and version.

PS12 — `"..."` interpolates `$var`. `'...'` literal. `@"..."@` here-string (multiline). Expressions: `"Value: $var"`. Escape `$` with backtick `` ` ``.

PS13 — `Get-Member` lists properties/methods. Properties: data. Methods: functions. NoteProperties: added dynamically. `Select-Object` selects props. `Where-Object` filters objects.

PS14 — Classes define structure. Limitations: no interfaces, limited inheritance (single). Constructors: `function New-Object`. Use classes for complex state; functions for simple logic.

PS15 — `[CmdletBinding()]` enables advanced features. `SupportsShouldProcess` adds `-WhatIf/-Confirm`. `Begin/Process/End` blocks handle pipeline stages. `ValueFromPipeline` binds input.

PS16 — Providers expose data as file system. `Get-ChildItem` works on Registry (`HKLM:\`). `cd HKLM:\Software` navigates. Allows uniform cmdlet usage across data sources.

PS17 — Runspaces: isolated execution contexts. `RunspacePool` manages threads. `ForEach-Object -Parallel` (v7+) uses pool. Manual runspace: `[PowerShell]::Create()`. Thread safety: avoid shared mutable state.

PS18 — PSScriptAnalyzer: static analysis. Rules: naming, style. Severity: Error, Warning, Info. Suppress: `#pragma`. CI/CD: run in pipeline, fail build on errors.

PS19 — `$using:` passes local var to remote. `-ArgumentList` serializes. Limits: CLIXML depth (complex objects fail). Use simple types or serialize to JSON for complex data.

PS20 — DSC: declarative config. Resources define state. LCM applies config. WinPS: XML-based. PS7+: YAML/JSON. Replaced by Ansible/Terraform in modern IaC. DSC still used for Windows compliance.
