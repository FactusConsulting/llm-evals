DN1 — **Class vs Struct**: `class` is a reference type (heap, mutable, inheritance), `struct` is a value type (stack/inline, copy-on-assign, no inheritance). Use `struct` for small, immutable data (e.g., coordinates, IDs) where copying is cheap and value semantics are required.

DN2 — **IEnumerable<T> vs IQueryable<T>**: `IEnumerable<T>` executes queries in-memory (client-side). `IQueryable<T>` builds expression trees translated to server-side queries (e.g., SQL). Distinction matters for DB queries to avoid pulling entire tables into memory before filtering.

DN3 — **async/await**: Compiles to state machines. `Task` represents an async operation (heap allocated). `ValueTask` avoids allocation if the operation completes synchronously. Use `ValueTask` in hot paths where synchronous completion is common to reduce GC pressure.

DN4 — **Dependency Injection (DI)**: Pattern for loose coupling. Lifetimes: **Transient** (new instance each time), **Scoped** (new per request), **Singleton** (single instance). Bug: Injecting a Scoped service into a Singleton captures the Singleton's lifetime, causing the Scoped service to live forever or throw `ObjectDisposedException` if disposed prematurely.

DN5 — **string vs ReadOnlySpan<char>**: `string` is immutable, heap-allocated. `ReadOnlySpan<char>` is a stack-only view into memory (no allocation). Use `Span<T>` for high-performance string manipulation, parsing, or avoiding allocations in loops.

DN6 — **Garbage Collector (GC)**: Manages memory by reclaiming unreachable objects. **Generations**: Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **LOH**: Objects >85KB go here (not compacted by default). `IDisposable`: Required for unmanaged resources (files, DB connections) because GC is non-deterministic; `Dispose` ensures timely release.

DN7 — **ConfigureAwait(false)**: Prevents capturing the current `SynchronizationContext`. Use in library code to avoid deadlocks and improve performance. Do NOT use in UI apps (WPF/WinForms) where context is needed to marshal back to the UI thread. ASP.NET Core has no context by default, so it's less critical but still good practice.

DN8 — **.NET 8/9 Features**: **Primary Constructors**: `class Service(IDataService data) { }`. **Collection Expressions**: `[1, 2, 3]`. **FrozenDictionary**: Immutable, high-performance dictionary. Example: `public class MyService(IDataService data) : IMyService { }`.

DN9 — **DbContext Exception**: Likely cause: `DbContext` injected as Singleton instead of Scoped. EF Core `DbContext` is not thread-safe. Fix: Register `DbContext` as Scoped (`AddDbContext`). High concurrency on a Singleton context causes race conditions and disposal issues.

DN10 — **record vs class**: `record` provides value-based equality and immutability by default. `record struct` is a value type with record semantics. Use `class` for mutable state/identity, `record` for data transfer objects (DTOs), `record struct` for small value objects needing equality.

DN11 — **ref, out, in**: `ref` passes by reference (must be initialized). `out` passes by reference (must be assigned inside method). `in` passes by reference (read-only). Use `ref` for swapping, `out` for multiple returns, `in` for large structs to avoid copy.

DN12 — **Nullable Reference Types (NRT)**: `?` annotation warns about potential null dereferences at compile time (reference types). Differs from `int?` (nullable value types) which allows `null` at runtime for value types. NRTs are opt-in (`#nullable enable`).

DN13 — **LINQ**: Language Integrated Query. **Method syntax** (`Where`) vs **Query syntax** (`from`). **Deferred Execution**: Query logic executes only when enumerated (e.g., `ToList()`). Prevents unnecessary processing.

DN14 — **Generic Constraints**: `where T : class` (reference), `struct` (value), `new()` (parameterless constructor), `IComparable<T>` (comparable). Combine constraints (e.g., `where T : class, IComparable<T>`) to enforce specific capabilities on generic types.

DN15 — **abstract class vs interface**: Abstract class can have implementation and state; interface defines contract only. **Default interface methods** (C# 8+) allow interfaces to have implementation, blurring the line but still favoring composition over inheritance.

DN16 — **Middleware Pipeline**: `app.Use()` (calls next, can wrap), `app.Map()` (branches pipeline), `app.Run()` (terminal, no next). Ordering matters (Auth before Routing). Exception handling middleware (`UseExceptionHandler`) catches unhandled exceptions globally.

DN17 — **Channel<T>**: Async producer-consumer. `BlockingCollection<T>` is sync. **Bounded channels** limit capacity; if full, producers block (backpressure). Example: `Channel.CreateBounded<int>(100)` to prevent memory overflow during high load.

DN18 — **Source Generators**: Compile-time code generation. Differs from Reflection (runtime) and IL Weaving (post-build). **Incremental Generator**: Processes inputs incrementally (faster, supports caching) vs `ISourceGenerator` (runs on every build). Preferred for performance.

DN19 — **TPL Dataflow**: `ActionBlock` (processes items, no output), `TransformBlock` (processes and outputs), `BufferBlock` (stores items). Compared to `Channels`: Dataflow offers more complex flow control (fan-out, fan-in, cancellation), Channels are simpler for basic pipelines.

DN20 — **Minimal APIs**: Direct route handlers without controllers. Trade-offs: Less boilerplate, harder to scale complex auth/middleware, OpenAPI support requires extra setup. **Route Groups**: `MapGroup` for shared prefixes/middleware. **Endpoint Filters**: Interceptors for logic (e.g., validation) applied to specific endpoints.

P1 — **List vs Tuple**: List is mutable, dynamic size. Tuple is immutable, fixed size. **Set vs Frozenset**: Set is mutable, unordered unique items. Frozenset is immutable, hashable (can be dict key).

P2 — **GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects CPU-bound workloads. Workarounds: Multiprocessing, C-extensions (release GIL), or async I/O.

P3 — **deepcopy vs copy**: `copy` creates shallow copy (nested objects shared). `deepcopy` recursively copies all objects. Example: `list1 = [[1]]; list2 = copy(list1); list2[0].append(2)` modifies `list1[0]`.

P4 — **Decorators**: Functions wrapping others.
```python
import time, functools
def log_time(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time()-start}s")
        return result
    return wrapper
```

P5 — **Generators**: Functions using `yield` to produce items lazily. Memory efficient vs `return` list. `yield from` delegates to another generator/sub-generator.

P6 — **asyncio**: Event loop manages coroutines. `asyncio.run()` starts loop. `asyncio.gather()` waits for multiple tasks. `asyncio.create_task()` schedules task. `time.sleep()` blocks the loop; use `await asyncio.sleep()`.

P7 — **MRO**: Method Resolution Order. Python 3 uses **C3 Linearization**. Diamond inheritance: `D(B, C)`. MRO: D -> B -> C -> A. `D().who()` prints "C" because C is visited before A in the linearization.

P8 — **Metaclass**: Class of a class. `__init_subclass__` is a simpler hook for subclass initialization (no metaclass needed). Use Metaclass for complex class creation logic (e.g., ORM registration). Use `__init_subclass__` for simpler subclass hooks (e.g., auto-registration).

P9 — **Multiprocessing vs Threading**: Multiprocessing (CPU-bound, separate memory, pickling required). Threading (I/O-bound, shared memory, GIL limits CPU). `concurrent.futures` provides high-level API for both. Pickling limits multiprocessing to serializable objects.

P10 — **__slots__**: Prevents creation of `__dict__` for instances. Saves memory (no dict overhead). Prevents adding dynamic attributes. Use for classes with many instances and fixed attributes.

P11 — **Virtual Environment**: Isolated Python environment. `venv` (stdlib, modern), `virtualenv` (older, more features), `conda` (package manager + env, binary packages). Prevents dependency conflicts.

P12 — **Comprehensions**: Concise list/dict/set creation. `map()`/`filter()` return iterators. Prefer comprehensions for readability and Pythonic style. Use `map`/`filter` for functional pipelines or lazy evaluation without list creation.

P13 — **== vs is**: `==` compares values. `is` compares identity (memory address). **Interning**: Small integers (-5 to 256) are cached. `a is b` returns True for small ints due to interning, not value equality.

P14 — **args/kwargs**: `*args` (tuple of positional), `**kwargs` (dict of keyword). `/` (positional-only before), `*` (keyword-only after). Used for flexible function signatures.

P15 — **Context Managers**: `with` statement ensures cleanup. `__enter__`/`__exit__` define protocol. `@contextmanager` uses generator. Example: Database connection manager ensuring rollback on error.

P16 — **Type Hinting**: Static typing annotations. `Union` (A|B), `Optional` (A|None), `Literal` (specific value), `TypeVar` (generic type), `Protocol` (structural typing). `mypy`/`pyright` use these for static analysis without runtime cost.

P17 — **Data Model**: `__repr__` (dev/debug), `__str__` (user). `__eq` defines equality, `__hash` defines hashability. Defining `__eq__` without `__hash__` makes object unhashable (cannot be dict key/set member). `__getattr__` (fallback), `__getattribute__` (always called).

P18 — **Descriptors**: Objects implementing `__get__`, `__set__`, `__delete__`. `property` is a descriptor. `classmethod` wraps function to pass class. `staticmethod` wraps function without passing class. Used for validation, lazy loading, etc.

P19 — **Import System**: `sys.path` lists directories. `PYTHONPATH` overrides. `__init__.py` marks package (optional in namespace packages). **Finder**: Locates module. **Loader**: Loads module. Relative imports (`from . import x`) vs Absolute (`from pkg import x`).

P20 — **dataclasses**: Reduces boilerplate (`__init__`, `__repr__`). `attrs` (more features, validation). `Pydantic` (validation, serialization). `__post_init__` runs after init. Frozen dataclasses are immutable. Choose Pydantic for validation-heavy models (APIs).
