JS1 ‚Äî `var` is function-scoped and hoisted (initialized as undefined). `let` and `const` are block-scoped and hoisted but not initialized (Temporal Dead Zone). Hoisting is moving declarations to the top of their scope during compilation.

JS2 ‚Äî The event loop manages execution. Call stack runs sync code. Microtask queue (Promises) runs before macrotask queue (setTimeout). Loop: Run stack -> Flush microtasks -> Render -> Run macrotasks.

JS3 ‚Äî A closure is a function retaining access to its lexical scope. Example: Counter factory. Bug: `var` in loop captures reference, not value. Fix: Use `let` (block scope) or IIFE to capture value.

JS4 ‚Äî Promise represents async result. `async/await` syntactic sugar. Error handling: `try/catch` for await, `.catch()` for promises. `Promise.all` fails on first rejection; `allSettled` waits for all, returning status.

JS5 ‚Äî `interface` defines shape (supports merging, types only). `type` defines union/intersection/primitives. Use `interface` for public API contracts, `type` for unions. Discriminated unions: Union with literal type field to narrow types.

JS6 ‚Äî Objects inherit from prototypes via `[[Prototype]]`. `Object.create(proto)` creates object with specific proto. ES6 classes are syntactic sugar over prototype chains (`class` defines constructor and prototype methods).

JS7 ‚Äî `this` binding rules: 1. New (constructor), 2. Explicit (bind/call/apply), 3. Implicit (object method), 4. Default (window/undefined). `bind()` returns new function. Cannot re-bind `bind()`ed functions.

JS8 ‚Äî `WeakRef` holds weak reference (GC collects target). `FinalizationRegistry` runs cleanup when target GC'd. `WeakMap` keys must be objects (GC can collect keys). Prevents memory leaks for caches.

JS9 ‚Äî V8 uses hidden classes (shapes) for property access. Inline caching caches lookup results. Deoptimization occurs when shape changes (e.g., adding properties dynamically). Avoid dynamic property addition in hot paths.

JS10 ‚Äî Tree-shaking removes unused exports. ES Modules (static `import`) allow static analysis. CommonJS (`require`) is dynamic, preventing static analysis, so tree-shaking is ineffective.

JS11 ‚Äî `null` is intentional absence; `undefined` is unassigned. `== null` matches both. `??` (nullish coalescing) returns right if left is null/undefined. `?.` (optional chaining) prevents errors on null/undefined.

JS12 ‚Äî `map` transforms, `filter` selects, `reduce` accumulates. `reduce` with initial value prevents undefined first arg. `flatMap` maps then flattens one level; useful for transforming arrays of arrays.

JS13 ‚Äî Template literals use backticks for multi-line strings and interpolation. Tagged templates call a function with string array + args. Example: `sql`SELECT * FROM ${table}`` for SQL query building.

JS14 ‚Äî Destructuring extracts values. Object: `{a}`. Array: `[a]`. Defaults: `{a=1}`. Renaming: `{a: b}`. Rest: `...rest` collects remaining. Works in function args and declarations.

JS15 ‚Äî `for...in` iterates enumerable keys (including prototype). `for...of` iterates values (iterables). Avoid `for...in` on arrays due to prototype pollution and non-guaranteed order.

JS16 ‚Äî Generics use type params `<T>`. Constraints: `T extends Foo`. Conditional: `T extends U ? X : Y`. `infer` extracts type from conditional. Example: `type Unwrap<T> = T extends Promise<infer U> ? U : T`.

JS17 ‚Äî CommonJS: `require`/`module.exports` (dynamic, sync). ES Modules: `import`/`export` (static, async). Node `type: "module"` in `package.json` forces ESM interpretation for `.js` files.

JS18 ‚Äî Structural typing: compatibility based on shape. Excess property checks apply to object literals assigned to types. `extends` checks assignability; `satisfies` checks shape without widening type. Variance: `in` (contravariant), `out` (covariant).

JS19 ‚Äî `Proxy` intercepts operations (get, set, etc.). `Reflect` provides default implementations. Example: Proxy for validation on property assignment. `Reflect` methods ensure correct `this` binding.

JS20 ‚Äî `SharedArrayBuffer` allows shared memory between workers. `Atomics` provides atomic ops. Ordering guarantees via memory barriers. `Atomics.wait` blocks; `Atomics.notify` wakes threads. Enables lock-free algorithms.

B1 ‚Äî `$@` expands to individual quoted arguments. `$*` expands to single string. `"$@"` preserves argument boundaries. `"$*"` joins args with first char of IFS.

B2 ‚Äî `-e`: Exit on error. `-u`: Treat unset vars as error. `-o pipefail`: Pipe fails if any command fails. Ensures strict error handling.

B3 ‚Äî `$(...)` is modern command substitution (nested). `` `...` `` is legacy. `$((...))` is arithmetic expansion. `expr` is external command for math (slower).

B4 ‚Äî Process substitution `<(cmd)` creates `/dev/fd/N`. Example: `diff <(sort a) <(sort b)` compares sorted files without temp files.

B5 ‚Äî Use `for f in *` or `find . -print0 | xargs -0`. `for f in $(ls)` breaks on spaces/newlines and parses output incorrectly.

B6 ‚Äî Here doc `<<EOF` feeds stdin. Here string `<<<` feeds single line. `<<-EOF` strips leading tabs from indentation.

B7 ‚Äî `> file 2>&1`: Stdout to file, stderr to stdout (file). `2>&1 > file`: Stderr to stdout (terminal), stdout to file. `&>`: Shorthand for `> file 2>&1`.

B8 ‚Äî Use `mkdir` for locking (atomic). Trap `SIGTERM` to cleanup. Example: `mkdir /var/lock/mylock` then `trap 'rmdir /var/lock/mylock' EXIT`.

B9 ‚Äî Single quotes: literal. Double quotes: expand vars/globs. No quotes: globbing/word split. Example: `echo '$var'` vs `echo "$var"`.

B10 ‚Äî `-f` (file), `-d` (dir), `-r` (readable). `[` is POSIX `test`. `[[` is Bash keyword (supports `&&`, `||`, regex).

B11 ‚Äî `$?`: Exit code. `$$`: Current PID. `$!`: Last background PID. `#`: Arg count.

B12 ‚Äî `func() {}` or `function func {}`. Return via `return` (exit code) or `echo` (stdout).

B13 ‚Äî PATH: Colon-separated dirs. Bash searches PATH. `type` shows builtin/alias/external. `which` shows external.

B14 ‚Äî `trap 'cmd' EXIT`. Handlers run on exit. Multiple signals: Last one wins unless handled. `EXIT` runs on shell exit.

B15 ‚Äî `xargs` builds command lines. `find -print0 | xargs -0` handles spaces/newlines. `-P` runs jobs in parallel.

B16 ‚Äî `declare -a` (indexed), `declare -A` (associative). Append: `arr+=()`. Length: `${#arr[@]}`. Associative keys are strings.

B17 ‚Äî Subshell `(...)` forks. Group `{...}` runs in current shell. Pipe `while` runs in subshell, losing vars. Use process substitution or redirection.

B18 ‚Äî SIGINT (Ctrl+C), SIGTERM (kill), SIGHUP (hangup), SIGKILL (force). Trap handlers for cleanup. `wait` for background jobs.

B19 ‚Äî `split` file, `xargs -P` parallel jobs, merge results. Trade-off: I/O vs CPU. Error: Check exit codes, aggregate failures.

B20 ‚Äî `exec` replaces shell process. FD manipulation: `exec 3>file`, `exec 3>&-`. Coprocess `coproc` creates bidirectional pipe. Use for persistent connections.

PS1 ‚Äî `Write-Output` sends to pipeline. `Write-Host` writes to screen (bypasses pipeline). Pipeline matters for chaining commands.

PS2 ‚Äî Pipeline passes objects. `ForEach-Object` (cmdlet) processes stream. `foreach` (loop) iterates collection. Use cmdlet for streaming, loop for logic.

PS3 ‚Äî `[PSCustomObject]` is typed object. `[hashtable]` is key/value. Convert: `[PSCustomObject]@{Key="Val"}`. Use CustomObject for structured data.

PS4 ‚Äî Terminating (throws), Non-terminating (continues). `-ErrorAction Stop` makes non-terminating terminate. `try/catch` catches terminating errors.

PS5 ‚Äî Splatting `@params` passes hashtable as separate args. Passing hashtable as single param passes it as one object. Splatting improves readability.

PS6 ‚Äî Remoting uses WinRM (HTTP) or SSH. `Invoke-Command` runs one-off. `Enter-PSSession` interactive. Requires configuration.

PS7 ‚Äî Returns array even if 1 item. Returns `$null` if 0 items. Pattern: `$result = @(); $result += ...`. Check `$result.Count`.

PS8 ‚Äî `[scriptblock]` is code object. Pass via `-ArgumentList`. Double-hop: Auth relay issue. Solve with CredSSP or delegation.

PS9 ‚Äî `@()` creates array. `[array]` casts. Index `$null` returns `$null`. Index empty array returns `$null`.

PS10 ‚Äî `-eq` (equal), `-like` (glob), `-match` (regex). `-c` variants case-sensitive. `-contains` (object in array), `-in` (value in array).

PS11 ‚Äî Module: `.psm1` (code), `.psd1` (manifest), Binary (DLL). `Import-Module` loads. Manifest defines exports/dependencies.

PS12 ‚Äî Interpolation: `"Hello $var"`. Single quotes: literal. Here-string: `@"..."@` for multi-line.

PS13 ‚Äî `Get-Member` lists properties/methods. Properties (data), Methods (funcs), NoteProperties (dynamic). `Select-Object` (subset), `Where-Object` (filter).

PS14 ‚Äî Classes (v5) support inheritance. Limitations: No interfaces, limited constructors. Use for stateful objects, functions for logic.

PS15 ‚Äî `CmdletBinding` enables advanced features. `SupportsShouldProcess` for `-WhatIf`. `Begin/Process/End` for pipeline stages.

PS16 ‚Äî Providers expose data as file system. `Get-ChildItem HKLM:\` accesses registry. `Get-ChildItem` works across providers.

PS17 ‚Äî Runspaces: Threads. `RunspacePool` manages pool. `ForEach-Object -Parallel` (v7) simplifies. Manual runspace for complex control.

PS18 ‚Äî PSScriptAnalyzer: Linter. Rules/Severity. Suppress via `#Requires -Modules`. Integrate in CI for quality gates.

PS19 ‚Äî `$using:` passes local var to remote. `-ArgumentList` serializes. Limits: CLIXML depth. Complex objects may fail serialization.

PS20 ‚Äî DSC: Configs, Resources, LCM. WinPS uses DSC. PS7 uses DSC Core. Replaced by Ansible/Choco in some IaC contexts.
