JS1 — **var** is function-scoped and hoisted (initialized as `undefined`). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages execution. The **call stack** runs synchronous code. When empty, it processes the **microtask queue** (Promises, `queueMicrotask`) before the **macrotask queue** (`setTimeout`, `setInterval`). Microtasks drain completely before the next macrotask.

JS3 — A **closure** is a function retaining access to its lexical scope. Example: `function outer() { let x = 1; return () => x; }`. **Bug:** `var` in a `for` loop captures the same variable reference. **Fix:** Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents async completion. **async/await** is syntactic sugar for Promises. Error handling uses `try/catch` for async/await and `.catch()` for Promises. **Promise.all** fails on first rejection; **Promise.allSettled** waits for all to settle (fulfilled or rejected).

JS5 — **Interface** defines shape (supports merging, only types). **Type** defines value (supports primitives, unions, intersections). Use **Interface** for public API contracts, **Type** for unions or primitives. **Discriminated unions** use a literal property to distinguish types (e.g., `{type: 'a', data: A} | {type: 'b', data: B}`).

JS6 — **Prototypal inheritance** links objects via `[[Prototype]]`. **Object.create(proto)** creates an object with `proto` as its prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` and `constructor` to manage the prototype chain.

JS7 — **this** binding rules (priority): 1. `new` (new instance), 2. Explicit (`call`, `apply`, `bind`), 3. Implicit (`obj.method`), 4. Default (`window`/`undefined`). **bind()** returns a new function with bound `this`. It **cannot** be re-bound.

JS8 — **WeakRef** holds weak reference to object (GC can collect). **FinalizationRegistry** runs cleanup when object is collected. **WeakMap** keys must be objects because primitives are copied, not referenced, preventing GC tracking.

JS9 — **V8** optimizes via **hidden classes** (object shape) and **inline caching** (caching property lookup). **Deoptimization** occurs when shape changes (e.g., adding property to existing object, deleting property). Avoid dynamic property access on objects in tight loops.

JS10 — **Tree-shaking** removes unused exports. Works with **ES Modules** (static `import`/`export` allows static analysis). **CommonJS** (`require`) is dynamic, so bundlers cannot statically analyze dependencies effectively.

JS11 — **undefined** (variable declared but not assigned), **null** (intentional empty value). `==` converts types (`null == undefined` is true). `===` checks type and value. **`??`** (nullish coalescing) returns right side only if left is `null`/`undefined`. **`?.`** (optional chaining) prevents errors on null/undefined access.

JS12 — **map()** transforms elements. **filter()** selects elements. **reduce()** accumulates value. **Initial value** prevents using first element as accumulator. **flatMap()** maps then flattens one level (useful for array-of-arrays transformation).

JS13 — **Template literals** use backticks for multi-line strings and `${}` interpolation. **Tagged templates** pass the string array and values to a function. Example: `html` strings for escaping or SQL query builders.

JS14 — **Destructuring** extracts values from objects/arrays. **Defaults** (`{a = 1}`), **Renaming** (`{a: b}`). **Rest operator** (`...rest`) collects remaining elements into an array.

JS15 — **for...in** iterates over enumerable **keys** (properties). **for...of** iterates over **values** of iterables. Avoid **for...in** on arrays because it iterates over inherited properties and is slower.

JS16 — **Generics** (`<T>`) allow reusable types. **Constraints** (`T extends Foo`) limit types. **Conditional types** (`T extends U ? X : Y`) select types based on condition. **infer** extracts type from conditional (e.g., `infer R` in `Promise<R>`).

JS17 — **CommonJS** (`require`) is dynamic, Node default. **ES Modules** (`import`) are static, browser standard. Node supports both. **`"type": "module"`** in `package.json` forces Node to treat `.js` files as ES Modules.

JS18 — **Structural typing** checks shape, not name. **Excess property checks** apply to object literals assigned to types (e.g., `{a: 1, b: 2}` vs `{a: 1}`). **`satisfies`** checks type without widening. **Variance** (`in`/`out`) controls generic parameter usage (contravariant/covariant).

JS19 — **Proxy** intercepts operations (get, set, apply). **Reflect** provides default implementations. Example: **Validation Proxy** intercepts `set` to validate values before assignment.

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic ops. **Ordering guarantees** ensure visibility across threads. **Atomics.wait** blocks until notified; **Atomics.notify** wakes waiting threads.

B1 — **$@** preserves arguments as separate words. **$*** joins them into one string. **"$@"** preserves arguments as separate words (safe). **"$*"** joins them into one string (unsafe for spaces).

B2 — **`set -e`**: Exit on error. **`-u`**: Exit on undefined variable. **`-o pipefail`**: Exit if any command in pipe fails. **`-x`**: Print commands (debug).

B3 — **`$(command)`** is preferred over **`command`** (backticks). **`$((expression))`** is arithmetic expansion (shell built-in). **`expr`** is an external command (slower, legacy).

B4 — **Process substitution** `<(command)` treats command output as a file. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — Use **`for f in *.txt`** (globbing). **`for f in $(ls)`** is dangerous because it splits on whitespace/newlines, breaking filenames with spaces.

B6 — **Here document** (`<<EOF`) reads from stdin until EOF. **Here string** (`<<<`) passes a string. **`<<-EOF`** strips leading tabs from the script body.

B7 — **`cmd > file 2>&1`**: stdout to file, stderr to stdout (file). **`cmd 2>&1 > file`**: stderr to stdout (terminal), stdout to file. **`&> file`**: Both to file (bash specific). First two differ due to redirection order.

B8 — Use **`mkdir`** for locking (atomic). Function: `mkdir "$LOCK_DIR" 2>/dev/null || exit 1`. **Trap**: `trap 'rm -rf "$LOCK_DIR"' EXIT`. Handles SIGTERM via trap cleanup.

B9 — **'...'**: Literal (no expansion). **"..."**: Expansion (variables, commands). **No quotes**: Word splitting and globbing. Example: `echo $HOME` vs `echo "$HOME"`.

B10 — **`[ -e file ]`** (exists), **`-d`** (dir), **`-r`** (readable). **`[[ ]]`** supports regex and logical operators; **`[ ]`** is POSIX test.

B11 — **`$?`**: Exit code. **`$$`**: Script PID. **`$!`**: Last background PID. **`$#`**: Number of arguments.

B12 — **`foo() {}`** or **`function foo {}`**. Return values via **`return`** (exit code) or **`echo`** (stdout).

B13 — **PATH** is colon-separated directories. Bash searches PATH. **Builtin** (shell internal), **External** (binary). Use **`type`** or **`which`** to check.

B14 — **`trap`** sets handlers. **EXIT** runs on script exit. **ERR** on error. **INT/TERM** on signals. Order: Handlers run in order defined; EXIT runs last.

B15 — **xargs** batches arguments. **Subshell** loops line-by-line. **`xargs -0`** handles null-delimited input (from `find -print0`). **`-P`** runs jobs in parallel.

B16 — **Arrays**: `arr=(1 2)`. **Associative**: `declare -A map`. Append: `arr+=(val)`. Length: `${#arr[@]}`.

B17 — **Subshell** `(...)` creates new process (vars lost). **Group** `{ ...; }` same shell. **Pipe** `while` runs in subshell, so variable changes are lost.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (cannot catch). Handle shutdown with **`trap`** to kill background jobs before exit.

B19 — Script: `split -l 1000 log`, `xargs -P 4 -I {} process {}`, `cat part* > merged`. Trade-offs: Parallelism vs I/O contention. Error propagation: Check exit codes of parallel jobs.

B20 — **`exec`** replaces shell process. **FD manipulation**: `exec 3>file` opens FD 3. **Coprocess** (`coproc`) creates bidirectional pipe. Use for persistent communication.

PS1 — **Write-Output** sends objects to pipeline. **Write-Host** writes to console (bypasses pipeline). Distinction matters for piping data to other commands.

PS2 — **Pipeline** passes objects. **`ForEach-Object`** is a cmdlet (pipeline-aware). **`foreach`** is a statement (iterates collection). Use `ForEach-Object` for pipeline streaming.

PS3 — **`[PSCustomObject]`** creates typed objects (properties). **`hashtable`** is key/value pairs. Convert: `[PSCustomObject]@{Key='Value'}`. Use CustomObject for structured data.

PS4 — **Terminating** (throws exception, stops). **Non-terminating** (warning, continues). **`-ErrorAction`** overrides preference. **`try/catch`** handles terminating errors.

PS5 — **Splatting** `@params` passes hashtable as arguments. Differs from passing hashtable as single parameter (which treats it as one argument).

PS6 — **`Invoke-Command`** runs one-off command remotely. **`Enter-PSSession`** interactive session. Uses **WinRM** (default) or **SSH** (v7+).

PS7 — Returns scalar if single item. `$x.GetType().Name` is **String**. If 0 items, returns `$null`. Correct pattern: `return @($result)` to force array.

PS8 — **`[scriptblock]`** is code container. Pass via `-ArgumentList`. **Double-hop** (cred delegation) requires **CredSSP** or **Invoke-Command -ArgumentList** to pass creds securely.

PS9 — **`@()`** forces array. **`[array]`** casts. Indexing `$null` throws error. Indexing `@()` returns empty.

PS10 — **`-eq`** (value), **`-match`** (regex). **`-ceq`** (case-sensitive). **`-contains`** (object in collection), **`-in`** (value in collection).

PS11 — **Module** is reusable code. **.psm1** (code), **.psd1** (manifest), **Binary** (DLL). **`Import-Module`** loads it.

PS12 — **`"..."`** interpolates. **`'...'`** literal. **`@"..."@`** (here-string) for multi-line. Expressions: `$var`.

PS13 — **`Get-Member`** introspects objects. **Properties** (data), **Methods** (functions), **NoteProperties** (dynamic). **`Select-Object`** (filter props), **`Where-Object`** (filter rows).

PS14 — **Classes** (v5+) define structure. Limitations: No interfaces, limited inheritance. Use **Classes** for complex state, **Functions** for logic.

PS15 — **`[CmdletBinding()]`** enables advanced features. **`SupportsShouldProcess`** (Confirm). **`Begin/Process/End`** blocks (setup, stream, cleanup).

PS16 — **Providers** expose data as file system. **`Get-ChildItem`** works on Registry (`HKLM:\`). Access registry like files.

PS17 — **Runspaces** (threads). **RunspacePool** (pool). **`ForEach-Object -Parallel`** (v7+) simplifies parallelism. Manual runspace for complex control.

PS18 — **PSScriptAnalyzer** enforces standards. **Rules** (checks), **Severity** (Warning/Error). Suppress via comments. Integrate via CI/CD (e.g., GitHub Actions).

PS19 — **`$using:`** scope passes local vars to remote. **`-ArgumentList`** serializes. **CLIXML** limits depth. Complex objects may fail serialization.

PS20 — **DSC** manages state. **Resources** (tasks), **LCM** (manager). **PS7+** removed DSC. Replaced by **Bicep**, **Terraform**, **Ansible** for modern IaC.
