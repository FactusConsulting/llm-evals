N1 — A Layer 2 switch forwards frames based on MAC addresses within a broadcast domain (VLAN). A Layer 3 switch forwards packets based on IP addresses and can perform inter-VLAN routing. Layer 2 operates at the Data Link layer; Layer 3 operates at the Network layer.

N2 — **Network Address:** 10.0.0.0
**Broadcast Address:** 10.0.15.255
*Calculation:* /20 mask (255.255.240.0) creates blocks of 16 in the third octet. 5 falls in the 0-15 range.

N3 — **Handshake:** 1. Client sends SYN. 2. Server sends SYN-ACK. 3. Client sends ACK.
**Lost ACK:** The server does not receive the final ACK, so it retransmits the SYN-ACK. The client ignores the duplicate SYN-ACK (as it already sent the ACK) and the connection eventually times out on the server side.

N4 — **SNAT (Source NAT):** Modifies the source IP (e.g., outbound traffic masquerading). **DNAT (Destination NAT):** Modifies the destination IP (e.g., port forwarding inbound traffic).
**Example:** SNAT: Internal server accessing internet via public IP. DNAT: Public user accessing internal web server via public IP.

N5 — 1. The hostname in the `curl` command does not match the Common Name (CN) or Subject Alternative Name (SAN) in the certificate. 2. The server is presenting a certificate for a different domain (e.g., due to load balancer misconfiguration). 3. DNS resolution is pointing to a different server than expected.

N6 — **VLAN:** Logical L2 segmentation using 802.1Q tags (limit 4094). **VxLAN:** L2 overlay over L3 using UDP encapsulation (limit ~16M segments).
**Choose VxLAN:** When spanning L2 domains across L3 boundaries (e.g., multi-datacenter cloud, large scale tenant isolation) where VLAN limits or physical topology constraints exist.

N7 — **Cause:** The default route points to eth0. Return traffic from the server uses the default gateway (eth0) instead of the interface the request arrived on (eth1).
**Fix:** Use Policy-Based Routing (PBR).
1. `ip rule add from 192.168.1.0/24 table 100`
2. `ip route add default via 192.168.1.1 table 100`
3. Ensure `ip route add default via 10.0.0.1` remains for other traffic.

N8 — **Path Selection Order:** 1. Weight (Cisco proprietary), 2. Local Preference, 3. AS Path Length, 4. Origin Type (IGP < EGP < Incomplete), 5. MED (Multi-Exit Discriminator), 6. eBGP vs iBGP, 7. IGP Cost to Next Hop.

N9 — **Math:**
Physical MTU: 1500 bytes.
VxLAN Overhead: 50 bytes (20 IP + 8 UDP + 8 VxLAN + 14 Eth).
IPsec ESP Overhead: ~50 bytes (20 IP + 12 ESP + 10 Trailer + 8 Auth).
Total Overhead: 100 bytes.
**Max Inner Payload (IP Packet):** 1500 - 100 = 1400 bytes.
*(Note: TCP payload would be 1400 - 20 IP header = 1380 bytes).*

N10 — **ARP:** Resolves IP addresses to MAC addresses on a local network.
**Gratuitous ARP:** An unsolicited ARP reply used to announce a new IP assignment, update ARP caches on other hosts, and detect IP address conflicts.

N11 — **Public IP:** Routable on the internet. **Private IP:** Reserved for internal networks, not routable on the internet.
**RFC 1918 Ranges:**
1. 10.0.0.0/8
2. 172.16.0.0/12
3. 192.168.0.0/16

N12 — **DNS:** Domain Name System, resolves hostnames to IPs.
**A Record:** Maps hostname to IPv4 address.
**CNAME Record:** Maps hostname to another hostname (alias).
**AAAA Record:** Maps hostname to IPv6 address.

N13 — **DHCP:** Dynamic Host Configuration Protocol, assigns IP addresses automatically.
**DORA Process:**
1. **D**iscover (Client broadcasts request)
2. **O**ffer (Server offers IP)
3. **R**equest (Client requests offered IP)
4. **A**cknowledge (Server confirms lease)

N14 — **Hub:** L1 device, broadcasts to all ports. **Switch:** L2 device, forwards based on MAC. **Router:** L3 device, forwards based on IP.
**Layers:** Hub (Physical), Switch (Data Link), Router (Network).

N15 — **Half-Duplex:** Transmit or receive at one time (shared medium). **Full-Duplex:** Transmit and receive simultaneously (dedicated paths).
**Modern Ethernet:** Full-duplex.

N16 — **TTL:** Time To Live, decrements at each hop to prevent infinite loops.
**Traceroute:** Sends packets with incrementing TTL (1, 2, 3...). Routers drop packet when TTL=0 and send ICMP Time Exceeded back, revealing the hop.

N17 — **NAT Gateway:** Translates private IPs to public IPs.
**Static NAT:** 1:1 mapping. **Dynamic NAT:** Pool of public IPs mapped to private. **PAT:** Many private IPs map to one public IP using different ports.

N18 — **ECMP:** Routes traffic across multiple paths with equal cost.
**Hashing:** Typically src/dst IP and ports.
**Problem:** Asymmetric flows (request path A, reply path B) break stateful firewalls expecting return traffic on the same interface/state table.

N19 — **Problem:** Throughput limited by Window Size / RTT. 64KB window is too small for 10MB BDP, causing low utilization.
**Solution:** TCP Window Scaling (RFC 1323) allows the window size to be shifted left (multiplied), supporting windows up to 1GB.

N20 — **802.1Q:** Adds a 4-byte tag to Ethernet frames containing VLAN ID.
**Linux Config:** Create sub-interfaces (e.g., `eth0.10`, `eth0.20`).
**Frame Format:** Ethernet Header + 802.1Q Tag (TPID + TCI) + Payload + FCS.

L1 — **Hard Link:** Points to the same inode as the original file. **Symbolic Link:** Points to the path of the file. Hard links cannot cross filesystems; symlinks can.

L2 — `find /var/log -type f -size +100M`

L3 — **State D:** Uninterruptible sleep (usually waiting on I/O).
**Kill:** Cannot be killed with `kill -9` (SIGKILL is ignored). Must wait for I/O to complete or reboot.

L4 — **v1:** Separate controllers (cpu, memory, etc.) in a hierarchy. **v2:** Unified hierarchy, all controllers in one tree.
**Systemd:** Prefers v2 (newer, unified, better cgroup2 support).

L5 — **nftables:** Unified framework, faster, simpler syntax. **iptables:** Legacy, multiple tables.
**Kernel:** iptables rules are translated to nftables rules in modern kernels (nftables backend).

L6 — A file was deleted but a process still has it open (holding the inode). The space is not freed until the process closes the file.
**Check:** `lsof | grep deleted`.

L7 — **UEFI:** Initializes hardware, loads bootloader. **Bootloader (GRUB):** Loads kernel. **Kernel:** Initializes hardware, mounts initramfs. **Initramfs:** Temporary root, loads drivers. **Root:** Mounts real root filesystem. **Systemd:** Initializes services to target.

L8 — **Reason:** `available` accounts for reclaimable cache. If `available` is low, the system is under memory pressure. OOM fires when `available` drops below `min_free_kbytes` or a process requests more than available.
**Tunables:** `vm.swappiness` (swap usage), `vm.vfs_cache_pressure` (inode/dentry reclaim), `vm.min_free_kbytes`.

L9 — **ip netns:** Creates isolated network namespaces (separate network stack).
**Containers:** Use namespaces to isolate network interfaces, routing tables, and ports for each container.

L10 — **SIGTERM:** Graceful termination (catchable). **SIGKILL:** Force kill (uncatchable). **SIGQUIT:** Quit + dump core (catchable).
**Catchable:** SIGTERM, SIGQUIT.

L11 — **stdin:** Input (0). **stdout:** Output (1). **stderr:** Error (2).
**Redirect:** `command 2>stderr.log` (stderr to file, stdout to terminal).

L12 — **755:** Owner (7=rwx), Group (5=r-x), Other (5=r-x).

L13 — **Root:** UID 0, unrestricted access. **Sudo:** User with temporary root privileges via policy (`/etc/sudoers`).

L14 — **Package Manager:** Software for installing/updating packages.
**Debian/Ubuntu:** `apt` / `dpkg`.
**RHEL/Fedora:** `dnf` / `yum`.

L15 — **fstab:** Filesystem table, defines mount points at boot.
**Misconfigured:** System may fail to boot or drop to emergency/single-user mode.

L16 — **LVM:** Abstraction layer over physical disks. **PV:** Physical Volume (disk). **VG:** Volume Group (pool). **LV:** Logical Volume (partition).
**Extend:** `lvextend -L +1G /dev/vg/lv` then `resize2fs /dev/vg/lv`.

L17 — **Scheduler:** Manages disk I/O queue. **mq-deadline:** Latency focused. **bfq:** Fairness focused. **none:** No scheduling (NVMe).
**Change:** `echo deadline > /sys/block/sda/queue/scheduler`.

L18 — **Load Average:** Number of processes in Running or Uninterruptible Sleep (D state).
**I/O Wait:** Processes waiting on disk inflate load average without using CPU.
**Diagnose:** `iostat` (disk), `vmstat` (swap/IO), `pidstat` (per-process).

L19 — **SELinux:** Mandatory Access Control using labels. **Enforcing:** Blocks access. **Permissive:** Logs violations. **Disabled:** Off.
**Troubleshoot:** Check `/var/log/audit/audit.log` for denials. Use `audit2allow` to generate policy.

L20 — **Page Fault:** CPU requests page not in RAM. **Minor:** Page in RAM (swap). **Major:** Page on disk (swap).
**THP:** Uses 2MB pages instead of 4KB. Improves TLB performance but can cause latency spikes.
**Disable:** For latency-sensitive apps (e.g., HFT, databases) to prevent fragmentation/latency.
