K1 — Easy: **Deployment** manages stateless applications, allowing for rolling updates and rollbacks with replica sets. **StatefulSet** manages stateful applications, providing stable network identities (e.g., `pod-0`, `pod-1`), ordered deployment/termination, and stable persistent storage.

K2 — Easy: A **ClusterIP** Service exposes a virtual IP within the cluster. **kube-proxy** (using iptables or IPVS) maintains network rules on each node to route traffic from the Service IP to the backend Pod IPs based on the Service selector.

K3 — Medium: **PersistentVolume (PV)** is a cluster-level storage resource. **PersistentVolumeClaim (PVC)** is a user request for storage. **StorageClass** defines the provisioner for dynamic PV creation. **Relation:** A PVC binds to a PV (static) or triggers a PV creation via a StorageClass (dynamic).

K4 — Medium: 1. Check **logs** (`kubectl logs`) for application errors. 2. Check **events** (`kubectl describe`) for scheduling or OOMKilled reasons. 3. Verify **resource limits** (CPU/Memory) are not exceeded. 4. Check **probe failures** (liveness/readiness) in the pod status.

K5 — Medium: **CoreDNS** provides DNS resolution for services and pods within the cluster. If CoreDNS pods are down, internal service discovery (e.g., `service.namespace.svc`) fails, though external DNS resolution may still work if fallback is configured.

K6 — Hard: The scheduler uses **Filtering** (remove ineligible nodes) and **Scoring** (rank eligible nodes). **Factors:** 1. Resource requests (CPU/Mem). 2. Node Affinity/Taints. 3. Pod Affinity/Anti-Affinity. 4. Topology Spread Constraints.

K7 — Hard: **Yes**, it will be scheduled (256Mi request < 500Mi free). At runtime, if it uses 600Mi (exceeding 512Mi limit), the container will be **OOMKilled** and restart.

K8 — Hard: 1. **External LB** forwards traffic to Ingress Controller IP. 2. **Ingress Controller** parses rules and routes to Service. 3. **Service** resolves to **Endpoints** (Pod IPs). 4. **kube-proxy** (iptables/IPVS) directs traffic to the specific Pod.

K9 — Medium: A **NetworkPolicy** restricts traffic flow between pods. **Default:** If no policy exists, all traffic is allowed. **Applied:** If a policy selects pods, it defaults to deny all ingress/egress unless explicitly allowed by the policy rules.

K10 — Medium: **RollingUpdate** replaces pods gradually (zero downtime). **Recreate** kills all existing pods before starting new ones. Choose **Recreate** for stateful applications requiring exclusive access or when configuration changes require a full restart.

K11 — Easy: A **Namespace** provides logical isolation within a cluster. **Default namespaces:** `default`, `kube-system`, `kube-public`, `kube-node-lease`.

K12 — Easy: `kubectl describe` shows detailed status. **3 pieces of info:** 1. **Events** (warnings/errors). 2. **Conditions** (Ready status details). 3. **Mounts/Env Vars** (configuration details).

K13 — Easy: A **ConfigMap** stores non-sensitive configuration data. **Methods:** 1. As **Environment Variables** in the pod spec. 2. As **Volume Mounts** (files) inside the container.

K14 — Easy: **ClusterIP** (internal cluster access), **NodePort** (exposes port on every node's IP), **LoadBalancer** (provisions external cloud load balancer).

K15 — Easy: A **DaemonSet** ensures a copy of a pod runs on all (or specific) nodes. Use for **logging agents**, **monitoring daemons**, or **CNI plugins**.

K16 — Medium: **Taints** repel pods; **Tolerations** allow pods to tolerate taints. **Node Affinity** pulls pods to nodes. **Use Case:** Taint a GPU node (repel general pods); Affinity a pod to co-locate with a specific service.

K17 — Medium: An **Operator** is a custom controller managing a CRD. **Difference:** Standard controllers manage standard resources; Operators manage complex application lifecycles (e.g., backups, upgrades). **Solves:** Operational complexity of stateful apps.

K18 — Hard: **etcd** uses strong consistency (linearizable). **Quorum lost:** Cluster becomes read-only/unavailable. **Recovery:** Restore from a snapshot backup on new nodes and rejoin the cluster to restore quorum.

K19 — Hard: **RBAC** uses Roles (permissions) and Bindings (subjects). **Manifest:**
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata: namespace: my-ns
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list", "watch"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata: namespace: my-ns
subjects:
- kind: User
  name: developer
roleRef:
  kind: Role
  name: pod-reader
```

K20 — Hard: **ndots** (default 5) causes the resolver to try appending search domains to every query. If the domain isn't found, it retries with each suffix, causing delays. **Fix:** Set `dnsConfig.options: [{name: "ndots", value: "1"}]` in the pod spec.

D1 — Easy: **Rebase** rewrites history to create a linear commit chain. **Merge** preserves history with a merge commit. **Avoid Rebase** on shared/public branches to prevent history rewriting for collaborators.

D2 — Easy: **TCP** is connection-oriented and reliable (HTTP, SSH). **UDP** is connectionless and fast (DNS, VoIP).

D3 — Medium: A **race condition** occurs when output depends on the sequence of execution. **Example:** Two threads incrementing a shared counter without a lock, resulting in lost updates.

D4 — Medium: **Symmetric** uses one key; **Asymmetric** uses a key pair. **TLS:** Asymmetric encryption exchanges the session key securely; Symmetric encryption handles the bulk data transfer.

D5 — Medium: A **layer** is a filesystem diff in a Docker image. **Caching:** Place instructions with changing dependencies (e.g., `COPY package.json`) before static code to maximize layer reuse during builds.

D6 — Medium: 1. Check **application logs** for errors. 2. Check **upstream service health**. 3. Verify **database connectivity**. 4. Check **resource limits** (CPU/Memory throttling).

D7 — Hard: **CAP** states a distributed system can only guarantee 2 of 3 (Consistency, Availability, Partition tolerance). **CP:** Zookeeper (sacrifices availability during partition). **AP:** Cassandra (sacrifices consistency during partition).

D8 — Medium: **Horizontal** adds nodes; **Vertical** adds resources. **Patterns:** Stateless design, Sharding, Load Balancing.

D9 — Easy: An **environment variable** is runtime configuration. **Persist:** Add `export VAR=value` to `~/.bashrc` or `~/.profile`.

D10 — Easy: **JSON** is strict and code-friendly (advantage: standard for APIs). **YAML** is human-friendly (advantage: supports comments and indentation).

D11 — Easy: **REST** is an architectural style. **GET** (retrieve), **POST** (create), **PUT** (replace), **DELETE** (remove).

D12 — Easy: **Version pinning** locks dependencies to exact versions. **Importance:** Ensures reproducible builds and prevents unexpected breaking changes.

D13 — Easy: **Compiled** converts code to binary before execution (C, Go). **Interpreted** executes code line-by-line (Python, JavaScript).

D14 — Medium: **Reverse Proxy** sits in front of servers (client -> proxy -> server). **Forward Proxy** sits in front of clients (client -> proxy -> internet). **Servers:** Nginx (web serving), HAProxy (load balancing).

D15 — Medium: A **deadlock** is a permanent blocking of processes. **Conditions:** 1. Mutual Exclusion. 2. Hold and Wait. 3. No Preemption. 4. Circular Wait.

D16 — Hard: **Eventual Consistency** ensures updates propagate over time. **Cassandra:** Uses **Read Repair** and **Tunable Consistency** (e.g., ONE, QUORUM, ALL) to balance latency and consistency.

D17 — Hard: **mTLS** authenticates both client and server via certificates. **Handshake:** Client Hello -> Server Hello + Cert -> Client Verify -> Client Cert -> Server Verify -> Finished. **Used in:** Service Meshes (Istio) for secure service-to-service communication.

D18 — Hard: A **hash table** maps keys to indices via a hash function. **Collision:** Chaining (linked lists) or Open Addressing (probing). **Complexity:** Avg O(1), Worst O(n) (all collisions).

D19 — Hard: **Circuit Breaker** stops calls to failing services. **States:** Closed (normal), Open (fail fast), Half-open (test recovery). **Relation:** Prevents retries from overwhelming a failing service when the circuit is Open.

D20 — Hard: **Optimistic** checks conflicts at commit (row versioning). **Pessimistic** locks data at read (`SELECT FOR UPDATE`). **Choose Optimistic** for low conflict/high concurrency. **Choose Pessimistic** for high conflict/critical data.
