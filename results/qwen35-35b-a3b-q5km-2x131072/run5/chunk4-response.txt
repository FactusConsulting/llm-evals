G1 — **Difference:** Arrays are value types with fixed size `[N]T`. Slices are dynamic views over arrays `[T]` with length and capacity. **Append:** If capacity is reached, Go allocates a new underlying array (typically doubling capacity), copies existing elements, and updates the slice header.

G2 — **Difference:** `var x int` declares a variable with zero value (`0`). `x := 0` declares and initializes in one step (short declaration). **Cannot use `:=`:** When redeclaring a variable in the same scope (must use `=`), or outside a function (must use `var`).

G3 — **Goroutine:** Lightweight thread managed by the Go runtime. **Difference:** OS threads are kernel-managed (heavy); goroutines are user-space managed (light). **GMP Model:** **G** (goroutine), **M** (OS thread), **P** (logical processor). The scheduler maps Gs to Ms using Ps to manage concurrency on available CPUs.

G4 — **Channels:** Typed conduits for communication between goroutines. **Buffered:** Stores a fixed number of values; send blocks only if full. **Unbuffered:** Blocks sender until receiver is ready. **Full Buffer:** Sending to a full buffered channel blocks until a receiver reads a value.

G5 — **Usage:** Propagates cancellation, deadlines, and request-scoped values. **Example:**
```go
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()
resp, err := http.GetContext(ctx, url)
```

G6 — **Model:** Implicit satisfaction (no `implements` keyword). **Difference:** Java/C# require explicit declaration. **Empty Interface:** `interface{}` (or `any`) accepts any type. **Danger:** Requires type assertions to use, risking runtime panics if type is incorrect.

G7 — **Unsafe:** `ready` and `data` are accessed by multiple goroutines without synchronization. **Fix:** Use `sync.Once`, `atomic.Bool` for `ready`, or a channel to signal completion. **Model:** Go memory model requires synchronization (locks, channels, atomics) to guarantee visibility across goroutines.

G8 — **Mutex vs RWMutex:** `Mutex` allows one writer or one reader. `RWMutex` allows multiple readers or one writer. **sync.Map:** Optimized for sparse keys or read-heavy workloads. **Trade-off:** `sync.Map` has higher overhead for writes and contention compared to a `map` + `Mutex`.

G9 — **Modules:** Dependency management system. **go.mod:** Declares module path and dependencies. **go.sum:** Cryptographic checksums of dependencies. **tidy:** Removes unused imports/dependencies. **MVS:** Selects the lowest compatible version of each dependency to satisfy all requirements.

G10 — **Defer:** Delays function execution until surrounding function returns. **Order:** LIFO (Last In, First Out). **Gotcha:** Loop variable capture (e.g., `i` in a loop) is captured by reference; use a copy or pass `i` as an argument to avoid all callbacks seeing the final value.

G11 — **Pointer:** Memory address of a value. **Zero Value:** `nil`. **Struct Literal:** `&MyStruct{Field: value}`.

G12 — **init():** Runs before `main()`. **Multiple:** Yes, run in declaration order within a package. **Order:** Package `init()` functions run before `main()`, ordered by import dependency.

G13 — **make:** Initializes built-in types (`map`, `slice`, `chan`). Returns typed value. **new:** Allocates zeroed memory for any type. Returns pointer.

G14 — **Handling:** Errors are values. **Interface:** `type error interface { Error() string }`. **Pattern:** `if err != nil { return err }`. **Is/As:** `errors.Is` checks semantic equality (wrapping). `errors.As` checks type assertion (unwrapping). `==` checks pointer equality.

G15 — **Types:** `map`, `slice`, `channel`, `struct`. **Zero Values:** `nil` for map/slice/channel; zero values for struct fields.

G16 — **Race:** Unsynchronized concurrent access to shared data. **Flag:** `-race` adds instrumentation to detect races. **Fix:** Use `sync.Mutex` to protect access or channels for communication.

G17 — **Select:** Multiplexes channel operations. **Switch:** Selects on values. **Multiple Ready:** Randomly chooses one. **Timeout:** `case <-time.After(d):` inside select.

G18 — **Analysis:** Compiler determines if variable escapes stack to heap. **Decision:** Based on lifetime (e.g., returned pointer). **Inspect:** `go build -gcflags="-m"`. **Matter:** Heap allocations trigger GC; stack is faster.

G19 — **Reflection:** Runtime introspection. **Type:** Metadata about type. **Value:** Actual data. **Cost:** Slow, bypasses compile-time checks. **Justified:** Serialization, generic frameworks where code generation is not feasible.

G20 — **Directive:** `//go:embed`. **Work:** Embeds files/dirs at compile time into `embed.FS`. **Limitations:** Static at compile time, no symlinks. **FS:** Implements `io/fs.FS` interface for standard file operations.

R1 — **Ownership:** System ensuring memory safety. **Rules:** 1. Each value has one owner. 2. When owner drops, value drops. 3. Can transfer ownership (move).

R2 — **String:** Owned, heap-allocated, mutable. **&str:** Borrowed slice of bytes, immutable. **Usage:** Use `&str` for function arguments (zero-copy), `String` for owned data.

R3 — **Borrowing:** Accessing data without ownership. **Rules:** 1. Either one mutable OR any number of immutable refs. 2. No dangling references. 3. References must be valid.

R4 — **Result:** `Ok(T)` or `Err(E)`. **Option:** `Some(T)` or `None`. **? Operator:** Propagates error if `Err`/`None`. **Example:** `fs::read_to_string(path).map_err(|e| e.to_string())`.

R5 — **Box:** Single ownership, heap. **Rc:** Reference counting, single thread. **Arc:** Atomic reference counting, multi-thread. **Use:** `Box` for recursion/heap size, `Rc` for shared single-thread, `Arc` for shared multi-thread.

R6 — **Traits:** Define shared behavior. **Comparison:** Similar to Go interfaces. **dyn Trait:** Dynamic dispatch (vtable), runtime cost. **T: Trait:** Static dispatch (monomorphization), zero cost.

R7 — **Lifetimes:** Scope of references. **Failure:** Compiler cannot guarantee return ref outlives inputs without annotation. **Fix:** `fn longest<'a>(x: &'a str, y: &'a str) -> &'a str`. (Note: Modern Rust elides this, but explicit is safer).

R8 — **Send:** Ownership transferable across threads. **Sync:** Shared ref safe across threads. **Rc:** Not `Send` (ref count not atomic). **Arc:** `Send` + `Sync` (atomic ref count).

R9 — **Async:** State machine. **Future:** Pollable task. **Executor:** Runs futures (e.g., Tokio). **Diff:** Go goroutines are stackful (blocking); Rust futures are stackless (non-blocking).

R10 — **Unsafe:** Bypasses safety checks. **Actions:** Raw pointers, FFI, union access. **Justified:** FFI, performance-critical low-level ops, unsafe libraries.

R11 — **Match:** Exhaustive pattern matching. **If Let:** Partial match. **Exhaustiveness:** Compiler enforces all cases covered in `match`, preventing runtime errors.

R12 — **Vec:** Dynamic heap array. **Array:** Fixed stack array. **Capacity:** Vec doubles capacity when full, reallocating and copying elements.

R13 — **Enums:** Algebraic Data Types (ADT) with variants. **Diff:** Can hold data. **Option:** `Some(T)` or `None`. **Null:** Replaced by `Option` to prevent null pointer exceptions.

R14 — **Impl:** Defines methods. **&self:** Borrow. **&mut self:** Borrow mut. **self:** Consume. **Associated:** Functions without `self` (e.g., `String::new()`).

R15 — **Derive:** Auto-generates trait implementations. **Traits:** `Debug`, `Clone`, `PartialEq`. **Limit:** Cannot derive if fields don't support trait (e.g., `Fn` on `Clone`).

R16 — **Cow:** Clone on Write. **Use:** Borrow data initially, clone only if mutation occurs. **Benefit:** Avoids allocation if no mutation happens.

R17 — **Modules:** `mod` declares module, `use` imports, `pub` exposes. **File:** `mod.rs` or `mod name.rs`. **2015/2018:** 2018 unified paths (`crate::` vs `self::`).

R18 — **Pin:** Prevents moving memory. **Need:** Self-referential structs, `Future` polling. **Unpin:** Marker trait indicating type can be moved safely.

R19 — **Macros:** Code generation. **Declarative:** `macro_rules!` (pattern matching). **Procedural:** Rust code generating Rust code (derive, attribute). **Hygiene:** Identifiers scoped lexically to prevent collisions.

R20 — **Zero-Cost:** Abstractions compile to efficient machine code. **Iterators:** No allocation. **Generics:** Monomorphization (copy-paste code per type). **Traits:** Static dispatch (inline).
