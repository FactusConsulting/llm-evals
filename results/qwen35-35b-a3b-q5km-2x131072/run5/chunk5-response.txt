DN1 — **Class vs Struct**: `class` is a reference type (heap), supports inheritance, and is mutable by default. `struct` is a value type (stack/inline), does not support inheritance (except from `System.ValueType`), and is immutable by convention. Use `struct` for small, immutable data objects (e.g., coordinates, IDs) to avoid heap allocation and copying overhead.

DN2 — **IEnumerable<T> vs IQueryable<T>**: `IEnumerable<T>` executes queries in-memory (client-side). `IQueryable<T>` builds expression trees that are translated to server-side queries (e.g., SQL). This distinction matters for databases because `IQueryable` pushes filtering/sorting to the DB (efficient), whereas `IEnumerable` loads all data into memory first (inefficient).

DN3 — **Async/Await & Task/ValueTask**: `async/await` uses a state machine to suspend/resume execution without blocking threads. `Task` is a reference type (object allocation). `ValueTask` is a struct (zero allocation) wrapping a `Task` or a value. Use `ValueTask` in hot paths where the operation might complete synchronously to avoid allocation overhead.

DN4 — **Dependency Injection (DI)**: DI decouples object creation from usage. Lifetimes: **Transient** (new instance every request), **Scoped** (new instance per request/context), **Singleton** (one instance for app lifetime). **Bug**: Injecting a Scoped service into a Singleton captures the Scoped service at the Singleton's creation time (often null or stale), leading to data isolation issues or `ObjectDisposedException`.

DN5 — **String vs ReadOnlySpan<char>**: `string` is immutable and stored on the GC heap. `ReadOnlySpan<char>` is a stack-allocated view over memory (no GC pressure). Use `Span<T>` for high-performance string/buffer manipulation to avoid allocations during slicing or parsing.

DN6 — **Garbage Collector (GC)**: The GC manages heap memory. **Generations**: Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **LOH**: Large Object Heap (objects > 85KB) is not compacted frequently. **IDisposable**: Required for unmanaged resources (files, DB connections) because the GC only manages managed memory; `Dispose` ensures timely cleanup.

DN7 — **ConfigureAwait(false)**: Default behavior captures the `SynchronizationContext` (e.g., UI thread) to resume on it. `ConfigureAwait(false)` skips this, resuming on a thread pool thread. Use `false` in library code to avoid deadlocks and improve throughput. Do NOT use `false` in UI/ASP.NET Core code where context is needed for state (e.g., UI updates).

DN8 — **.NET 8/9 Features**: **Primary Constructors**: `public class Service(int Id) : IService { }`. **Collection Expressions**: `var arr = [1, 2, 3]`. **FrozenDictionary**: Immutable, high-performance dictionary. **Example**:
```csharp
public class UserService(IRepository repo) : IUserService { }
```

DN9 — **DbContext Exception**: Likely cause: **DbContext registered as Singleton**. EF Core DbContext is not thread-safe. Under high concurrency, multiple requests share the same context, causing race conditions and disposal errors. **Fix**: Register DbContext as **Scoped** (`AddDbContext`). Ensure Scoped services are not injected into Singletons.

DN10 — **Record vs Class**: `record` provides value-based equality and immutable properties by default. `class` provides reference equality. `record struct` combines value type semantics with record features. Choose `record` for DTOs/Models where equality matters; `class` for entities with identity or mutable state.

DN11 — **ref, out, in**: `ref` passes by reference (must be initialized). `out` passes by reference (must be assigned inside method). `in` passes by reference (read-only). Use `ref` for modification, `out` for multiple return values, `in` for large read-only structs to avoid copying.

DN12 — **Nullable Reference Types (NRT)**: `#nullable enable` adds compile-time warnings for potential null dereferences on reference types. `int?` is a nullable *value* type (struct). NRT distinguishes between `string` (can be null) and `string!` (never null) at compile time, unlike runtime null checks.

DN13 — **LINQ**: Language Integrated Query. **Method Syntax**: `list.Where(...)`. **Query Syntax**: `from x in list where ...`. Both compile to the same IL. **Deferred Execution**: Queries are not executed until enumerated (e.g., `ToList()`), allowing composition.

DN14 — **Generic Constraints**: `where T : class` (reference type), `where T : struct` (value type), `where T : new()` (parameterless constructor), `where T : IComparable<T>` (implements interface). Combine constraints to enforce specific capabilities (e.g., `where T : class, IDisposable`).

DN15 — **Abstract Class vs Interface**: Abstract class can contain implementation and state; Interface defines a contract (C# 8+ allows default methods). Use Abstract for shared code + contract; Interface for polymorphism across unrelated types. Default interface methods allow adding functionality without breaking existing implementations.

DN16 — **Middleware Pipeline**: Middleware components form a chain. `app.Use()` adds middleware to the chain. `app.Map()` branches the pipeline for specific paths. `app.Run()` adds a terminal delegate. **Order matters**: Middleware executes top-down, response bottom-up. Exception handling middleware (e.g., `UseExceptionHandler`) catches errors and returns responses.

DN17 — **Channel<T>**: Async, thread-safe queue. `BlockingCollection<T>` is blocking and older. **Bounded Channel**: Limits capacity. If full, producers wait (backpressure). Example: Producer writes to bounded channel; Consumer reads. Prevents memory overflow under load.

DN18 — **Source Generators**: Compile-time code generation. Faster/safer than reflection (no runtime cost) and IL weaving (no post-build steps). **Incremental Generator**: Only regenerates code when source files change, improving build times. Preferred over `ISourceGenerator` for performance and incremental compilation support.

DN19 — **TPL Dataflow**: Pipeline processing library. `ActionBlock` (process, no output), `TransformBlock` (process, output), `BufferBlock` (queue). **Vs Channels**: Dataflow offers more complex flow control (cancellation, completion, batching). Channels are simpler for basic producer-consumer.

DN20 — **Minimal APIs**: Lambda-based endpoints without controllers. **Trade-offs**: Less boilerplate, faster dev, but harder to scale/maintain for complex apps. **Middleware**: Same as MVC. **OpenAPI**: Supported via Swashbuckle. **Route Groups**: Organize endpoints (`MapGroup`). **Filters**: Apply logic (e.g., validation) to endpoints.

P1 — **List vs Tuple**: `List` is mutable, ordered collection. `Tuple` is immutable, ordered collection. `Set` is mutable, unordered, unique items. `Frozenset` is immutable set. Use `Tuple` for fixed data; `Set` for uniqueness.

P2 — **GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects **CPU-bound** workloads. **Workarounds**: Multiprocessing (separate processes), C-extensions (release GIL), or async I/O for I/O-bound tasks.

P3 — **Deepcopy vs Copy**: `copy` creates a shallow copy (nested objects shared). `deepcopy` creates a recursive copy. **Example**: `list = [[1]]`, `copy = list.copy()`. Modifying `copy[0][0]` changes `list[0][0]` because the inner list is shared.

P4 — **Decorators**: Functions that wrap other functions. **Time Logger**:
```python
import time
def timeit(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 — **Generators & Yield**: Generators produce values lazily. `yield` pauses execution and returns a value. **Vs List**: Generators save memory (no list storage). `yield from` delegates to another generator.

P6 — **Asyncio Event Loop**: `asyncio.run()` starts the loop. `asyncio.gather()` runs coroutines concurrently. `asyncio.create_task()` schedules a coroutine. `time.sleep()` blocks the loop; use `await asyncio.sleep()` to yield control.

P7 — **MRO**: Method Resolution Order. Python 3 uses **C3 Linearization**. **Diamond Inheritance**: `D(B, C)`. MRO: D -> B -> C -> A. `D().who()` prints "C" because `C` appears before `A` in the linearization of `D`.

P8 — **Metaclass**: Class of a class. Controls class creation. `__init_subclass__` is a simpler hook for subclass initialization (no metaclass needed). **Use Case**: Metaclass for ORM registration; `__init_subclass__` for plugin discovery.

P9 — **Multiprocessing vs Threading**: **Multiprocessing**: CPU-bound, separate memory (no GIL), requires pickling. **Threading**: I/O-bound, shared memory (GIL limits CPU). **Concurrent.Futures**: High-level abstraction for both. Pickling is required for passing args to processes.

P10 — **__slots__**: Restricts instance attributes to a fixed set. **Use**: Memory optimization for many instances. **Prevents**: Dynamic attribute creation (`__dict__` is omitted). **Impact**: Reduces memory footprint significantly.

P11 — **Virtual Environment**: Isolated Python environment. **Why**: Dependency management. **Differences**: `venv` (stdlib, Python 3.3+), `virtualenv` (older, external), `conda` (package manager, binary packages).

P12 — **Comprehensions**: Concise list/dict/set creation. `map()`/`filter()` return iterators. **Preference**: Comprehensions are more Pythonic and readable for simple transformations. Use `map/filter` for functional pipelines or lazy evaluation.

P13 — **== vs is**: `==` checks value equality. `is` checks identity (memory address). **Interning**: Small integers (-5 to 256) are cached. `a is b` is True for small ints due to interning, but False for large ints even if values match.

P14 — **args/kwargs**: `*args` (tuple of positional), `**kwargs` (dict of keyword). `/` forces positional-only before it. `*` forces keyword-only after it. Used for flexible function signatures.

P15 — **Context Managers**: Manage resources (`with` statement). `__enter__`/`__exit__` define behavior. `@contextmanager` decorator simplifies creation. **Example**: Database connection manager ensuring commit/rollback.

P16 — **Type Hinting**: Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. **mypy/pyright**: Analyze code without running to catch type errors. `Protocol` enables structural typing (duck typing).

P17 — **Data Model (Dunder)**: `__repr__` (dev/debug), `__str__` (user). `__eq__` defines equality. `__hash__` defines hashability. **Defining `__eq__` without `__hash__`**: Makes the object unhashable (cannot be used in sets/dicts).

P18 — **Descriptors**: Objects controlling attribute access (`__get__`, `__set__`, `__delete__`). **property**, `classmethod`, `staticmethod` are descriptors. Used for validation, computed properties, and access control.

P19 — **Import System**: `sys.path` determines search paths. **Relative**: `from . import mod`. **Absolute**: `from pkg import mod`. `__init__.py` marks packages (optional in 3.3+ for namespace). **Finder**: Locates modules. **Loader**: Loads/compiles them.

P20 — **Dataclasses**: Auto-generates `__init__`, `__repr__`, etc. **Vs attrs/Pydantic**: attrs is more flexible; Pydantic adds validation. **__post_init__**: Runs after init. **Frozen**: Immutable. **Choose Pydantic** for API models requiring validation (e.g., FastAPI).
