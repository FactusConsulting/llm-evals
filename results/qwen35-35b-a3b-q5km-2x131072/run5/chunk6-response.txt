JS1 — **var** is function-scoped and hoisted (initialized as `undefined`). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages async execution. The **call stack** runs synchronous code. **Microtasks** (Promises) run after the call stack clears but before the next macrotask. **Macrotasks** (setTimeout, I/O) run in the next event loop iteration.

JS3 — A **closure** is a function retaining access to its lexical scope. Example: Counter factory. **Bug**: `var` in `for` loops shares one variable reference. **Fix**: Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents async completion; **async/await** is syntactic sugar. Error handling: `.catch()` or `try/catch`. **Promise.all** fails on first rejection; **Promise.allSettled** waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 — **Interface** defines shape (can extend, implement); **type** defines value (unions, intersections). Use **interface** for public API contracts, **type** for unions/complex logic. **Discriminated unions** are unions with a literal type tag to narrow types safely.

JS6 — **Prototypal inheritance** links objects via `__proto__`. **Object.create(proto)** creates an object with `proto` as its prototype. **ES6 classes** are syntactic sugar over prototype chains (`class` extends `prototype`).

JS7 — **this** depends on call site. **Rules**: 1. Explicit (`call/apply/bind`), 2. Implicit (object method), 3. New (constructor), 4. Default (global/window). **bind()** returns a new function with bound `this`; it cannot be re-bound.

JS8 — **WeakRef** holds weak reference to object (GC can collect). **FinalizationRegistry** runs cleanup when object is collected. **WeakMap** keys must be objects (GC tracked); primitives cannot be keys because they aren't GC'd.

JS9 — **Hidden classes** optimize property access by grouping objects with same shape. **Inline caching** caches lookup results. **Deoptimization** occurs on shape changes, `eval`, or dynamic keys, forcing engine to revert to slower interpretation.

JS10 — **Tree-shaking** removes unused exports. **ES Modules** allow static analysis (static `import`), enabling tree-shaking. **CommonJS** (`require`) is dynamic, preventing static analysis, so tree-shaking is ineffective.

JS11 — **null** is intentional absence; **undefined** is uninitialized. **==** coerces types (`null == undefined` is true); **===** checks strict equality. **??** returns right side if left is `null`/`undefined`. **?.** short-circuits on `null`/`undefined`.

JS12 — **map()** transforms, **filter()** selects, **reduce()** accumulates. **reduce()** with initial value prevents type errors on empty arrays. **flatMap()** maps then flattens one level; useful for normalizing nested arrays.

JS13 — **Template literals** use backticks for multi-line strings and interpolation. **Tagged templates** call a function with the string parts and interpolated values. Example: `html` tag to escape HTML content before rendering.

JS14 — **Destructuring** extracts values from objects/arrays. **Defaults** provide fallbacks. **Renaming** uses `: newName`. **Rest operator** (`...`) collects remaining elements into an array.

JS15 — **for...in** iterates over enumerable keys (including prototype). **for...of** iterates over values of iterable objects. Avoid **for...in** on arrays because it includes non-enumerable/prototype properties and is slower.

JS16 — **Generics** allow reusable types (`T`). **Constraints** (`extends`) limit types. **Conditional types** (`T extends U ? X : Y`) select types based on condition. **infer** extracts types from conditional types (e.g., `ReturnType<T>`).

JS17 — **CommonJS** (`require`) is synchronous, dynamic, Node-specific. **ES Modules** (`import`) are static, async, standard. Node uses **type: "module"** in `package.json` to treat `.js` files as ESM by default.

JS18 — **Structural typing** means types compatible by shape, not name. **Excess property checks** apply to object literals to prevent typos. **extends** is for inheritance; **satisfies** checks shape without widening type. **Variance** is implicit in TS (structural), unlike C# explicit `in/out`.

JS19 — **Proxy** intercepts operations (get, set, etc.). **Reflect** provides default implementations. Example: Proxy for validation or logging. Intercept `set` to validate before assignment.

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic ops. **Ordering guarantees** ensure visibility across threads. **Atomics.wait** blocks until **Atomics.notify** wakes it; requires COOP/COEP headers in browsers.

B1 — **$@** expands to separate quoted arguments. **$*** expands to one string. **"$@"** preserves argument boundaries (safe for loops). **"$*"** joins args with first char of IFS (unsafe for spaces).

B2 — **-e** exits on error. **-u** errors on undefined vars. **-o pipefail** exits if any command in pipe fails. Together they enforce strict error handling.

B3 — **$(command)** is preferred over backticks (nested easier). **$((expression))** is shell arithmetic. **expr** is an external command (slower, POSIX).

B4 — **Process substitution** `<(cmd)` creates a file descriptor. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — Use `for f in *` (globbing). **`for f in $(ls)`** breaks on spaces/newlines. Safe iteration: `while IFS= read -r -d '' f; do ...; done < <(find . -print0)`.

B6 — **Here doc** (`<<EOF`) feeds stdin. **Here string** (`<<<`) feeds a single string. **`<<-EOF`** strips leading tabs from indentation.

B7 — `cmd > file 2>&1`: stdout to file, stderr to stdout (file). `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. `&> file`: both to file. First two differ due to redirection order.

B8 — Use `mkdir` for locking (atomic). `trap 'rmdir "$lock"' EXIT`. Handle SIGTERM by trapping and cleaning lock. `mkdir` preferred over file creation because it's atomic on POSIX.

B9 — **Single quotes** (`'`) literal. **Double quotes** (`"`) expand vars/glob. **No quotes** split on IFS/glob. Example: `echo $HOME` (no quotes) vs `echo "$HOME"` (quotes).

B10 — **`[ -e file ]`** exists, **`-d`** directory, **`-r`** readable. **`[[ ]]`** supports regex, `&&`/`||`, and is safer than `[ ]`.

B11 — **$?** exit code. **$$** script PID. **$!** last background PID. **$#** argument count. Use `$$` for temp files, `$?` for error checks.

B12 — **`foo() {}`** or **`function foo {}`**. Return via `exit code` or `echo`. Functions share global scope unless `local` used.

B13 — **PATH** is colon-separated dirs. Bash searches PATH for external commands. **Builtin** is internal (faster). Use `type` or `which` to check.

B14 — **trap** sets handlers. **EXIT** on script end, **ERR** on error, **INT/TERM** on signals. Order: Signal handler runs, then script continues or exits.

B15 — **xargs** builds command args. **`-0`** with **`-print0`** handles spaces/newlines. **`-P`** runs jobs in parallel (limit concurrency).

B16 — **Arrays**: `arr=(1 2)`. **Associative**: `declare -A map`. Append: `arr+=("val")`. Length: `${#arr[@]}`. Associative keys are strings.

B17 — **Subshell** `(...)` forks process (vars lost). **Group** `{...}` runs in current shell. **Pipe** to `while` creates subshell, losing variable changes. Use process substitution to fix.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (unblockable). Handle graceful shutdown by trapping SIGTERM, killing child jobs, then exiting.

B19 — **split** file, **xargs -P** process chunks, **merge** results. Trade-offs: I/O vs CPU. Error propagation: Check exit codes in each job, aggregate failures.

B20 — **exec** replaces shell process. **FD manipulation** (`exec 3>file`) keeps streams open. **Coprocess** (`coproc`) creates background pipe for bidirectional communication.

PS1 — **Write-Output** sends to pipeline (objects). **Write-Host** writes to screen (strings). Distinction matters because `Write-Host` breaks pipelines and cannot be captured.

PS2 — **Pipeline** passes objects. **ForEach-Object** (cmdlet) processes items in pipeline. **foreach** (statement) iterates a collection. Use cmdlet for pipeline, statement for loops.

PS3 — **[PSCustomObject]** is a typed object. **[hashtable]** is key/value pairs. Convert: `[PSCustomObject]@{Key="Val"}`. Use CustomObject for structured data, Hashtable for dynamic config.

PS4 — **Terminating** (throw) stops execution. **Non-terminating** (continue) logs error. **-ErrorAction** overrides preference. **try/catch** handles terminating errors.

PS5 — **Splatting** `@params` passes hashtable as arguments. Differs from passing hashtable as single param. Example: `Get-Process @params`.

PS6 — **Remoting** uses WinRM (HTTP) or SSH. **Invoke-Command** runs one-off. **Enter-PSSession** interactive. Requires configuration (TrustedHosts).

PS7 — Returns array even for single item. If 0 items, returns empty array. **Pattern**: Always initialize `$result = @()` and ensure return is array.

PS8 — **[scriptblock]** is code object. Pass via `-ArgumentList`. **Double-hop** = Kerberos delegation issue. Solve with CredSSP or `-ArgumentList` to pass creds.

PS9 — **@()** creates array. **[array]** casts. Indexing `$null` throws error. Indexing empty array returns `$null`.

PS10 — **-eq** (equal), **-like** (glob), **-match** (regex). **-ceq** (case-sensitive). **-contains** (member), **-in** (array membership).

PS11 — **Module** (.psm1 code, .psd1 manifest). **Binary** (.dll). **Import-Module** loads into session. Manifest defines dependencies/commands.

PS12 — **Interpolation** `"$var"`. **'...'** literal. **Here-string** `@"..."@` for multi-line. Escape `$` with backtick.

PS13 — **Get-Member** lists properties/methods. **Properties** (data), **Methods** (actions), **NoteProperties** (added). **Select-Object** (subset), **Where-Object** (filter).

PS14 — **Classes** (v5) support state. Limitations: No interfaces, limited inheritance. Use for stateful logic, functions for stateless.

PS15 — **CmdletBinding** enables advanced features. **SupportsShouldProcess** (Confirm). **Begin/Process/End** blocks (setup, stream, cleanup).

PS16 — **Providers** abstract data stores (Registry, File). **Get-ChildItem** works on `HKLM:` like `C:\`. Access registry via `Registry::` drive.

PS17 — **Runspaces** (threads). **RunspacePool** manages threads. **ForEach-Object -Parallel** (v7+) simplifies parallelism. Manual runspace for complex control.

PS18 — **PSScriptAnalyzer** enforces standards. **Rules** (severity). **Suppress** via comments. Integrate in CI to block builds on errors.

PS19 — **$using:** passes local vars to remote. **Serialization** (CLIXML) limits complex objects. **ArgumentList** avoids serialization limits for simple types.

PS20 — **DSC** configures state. **LCM** applies config. **WinPS** uses XML; **PS7** uses JSON. Replaced by **Ansible**, **Terraform**, or **Pulumi** in modern IaC.
