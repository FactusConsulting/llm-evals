DN1 — **Class vs Struct:** `class` is a reference type (heap-allocated, nullable, supports inheritance). `struct` is a value type (stack-allocated mostly, copy-on-assign, no inheritance). Use `struct` for small, immutable data types (e.g., `Point`, `Vector`) where value semantics are desired and performance (avoiding heap allocation) matters.

DN2 — **IEnumerable<T> vs IQueryable<T>:** `IEnumerable<T>` executes queries in-memory (client-side). `IQueryable<T>` builds expression trees for server-side execution (e.g., SQL). This matters for DB queries because `IQueryable` pushes filtering/pagination to the database, reducing network traffic and improving performance.

DN3 — **Async/Await & Task/ValueTask:** `async/await` uses a state machine to suspend/resume execution without blocking threads. `Task` is a reference type (heap allocation). `ValueTask` is a struct (stack allocation). Use `ValueTask` when the operation is often already completed (e.g., cached data) to avoid allocation overhead.

DN4 — **Dependency Injection (DI):** DI decouples object creation from usage. Lifetimes: **Transient** (new instance every request), **Scoped** (new instance per request), **Singleton** (one instance for app lifetime). **Bug:** Injecting a Scoped service into a Singleton captures the Scoped instance at Singleton creation time. When the request scope ends, the Scoped service is disposed, but the Singleton still holds the reference, causing `ObjectDisposedException` later.

DN5 — **String vs ReadOnlySpan<char>:** `string` is immutable, heap-allocated. `ReadOnlySpan<char>` is a stack-allocated view into memory (no allocation). Use `Span<T>` for high-performance string parsing, slicing, or formatting to avoid creating intermediate string objects.

DN6 — **Garbage Collector (GC):** GC manages memory by reclaiming unreachable objects. **Generations:** Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **LOH:** Large Object Heap (objects > 85KB) for large arrays/strings. **IDisposable:** Needed for unmanaged resources (files, DB connections) because GC only manages managed memory; `Dispose` ensures deterministic cleanup.

DN7 — **ConfigureAwait(false):** Default behavior captures `SynchronizationContext` (e.g., UI thread). `ConfigureAwait(false)` skips this. Use `false` in library code to prevent deadlocks and improve throughput. Do NOT use `false` in UI code where you need to update the UI thread after await.

DN8 — **.NET 8/9 Features:** **Primary Constructors:** `public class Service(int id) { }`. **Collection Expressions:** `[1, 2, 3]`. **FrozenDictionary:** Immutable, fast lookup. **Example:**
```csharp
public class Service(int id) : IService { public int Id => id; }
// DI registration: services.AddSingleton(new Service(1));
```

DN9 — **DbContext Exception:** Likely cause: `DbContext` is Scoped, but injected into a Singleton or BackgroundService. The Scoped context is disposed when the first request ends, but the Singleton retains the reference. **Fix:** Ensure `DbContext` is registered as Scoped and injected only into Scoped/Transient services. Never inject Scoped into Singleton.

DN10 — **Record vs Class:** `record` provides value-based equality and is immutable by default. `class` provides reference equality and is mutable. `record struct` combines value semantics with record features. Use `record` for DTOs/Models, `class` for entities with identity or complex behavior.

DN11 — **ref, out, in:** `ref` passes by reference (modify original). `out` passes by reference (must assign inside method). `in` passes by reference (read-only). Use `ref` for in-place modification, `out` for multiple return values, `in` for large read-only structs.

DN12 — **Nullable Reference Types (NRT):** `?` annotation warns about potential null dereferences at compile-time for reference types. `int?` is a nullable *value* type (runtime nullability). NRT helps catch bugs early; `int?` is a struct wrapping `int` with a `HasValue` flag.

DN13 — **LINQ:** Language Integrated Query. **Method vs Query:** Same IL, query syntax is syntactic sugar. **Deferred Execution:** Query logic runs only when enumerated (e.g., `ToList()`), not when defined.

DN14 — **Generic Constraints:** `where T : class` (reference type), `where T : struct` (value type), `where T : new()` (paramless constructor), `where T : IComparable<T>` (implements interface). Combine with commas: `where T : class, IComparable<T>`.

DN15 — **Abstract Class vs Interface:** Abstract class allows partial implementation and state. Interface defines a contract (no state). **Default Interface Methods (C# 8+):** Allow adding methods to interfaces without breaking existing implementations, blurring the line slightly.

DN16 — **Middleware Pipeline:** Requests pass through a chain. `app.Use()` adds middleware (calls `next()`). `app.Map()` branches the pipeline. `app.Run()` terminates the pipeline. **Order:** Auth before Controllers. **Exception Handling:** Middleware wrapping the pipeline catches exceptions and returns error responses.

DN17 — **Channel<T>:** Async producer-consumer pattern. `BlockingCollection<T>` is synchronous. **Bounded Channel:** Set `maxItems` to enforce backpressure (producers wait if buffer is full). Example: `Channel.CreateBounded<int>(100)`.

DN18 — **Source Generators:** Compile-time code generation (faster than reflection). **Incremental Generators:** Process files incrementally (faster rebuilds). Preferred over `ISourceGenerator` for performance and better integration with Roslyn APIs.

DN19 — **TPL Dataflow:** Pipeline processing blocks. `ActionBlock` (consume), `TransformBlock` (transform), `BufferBlock` (store). **Vs Channels:** Dataflow offers higher-level backpressure and linking; Channels are lower-level, simpler, and more flexible for custom flow control.

DN20 — **Minimal APIs:** Lightweight endpoints without controllers. **Trade-offs:** Less boilerplate, but harder to organize complex auth/middleware. **Route Groups:** Organize endpoints by feature. **Filters:** Intercept requests/responses (e.g., validation). OpenAPI support via Swashbuckle.

P1 — **List vs Tuple:** `List` is mutable, ordered collection. `Tuple` is immutable, ordered collection. **Set vs Frozenset:** `Set` is mutable, unordered unique items. `Frozenset` is immutable (hashable).

P2 — **GIL:** Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects CPU-bound workloads. **Workarounds:** Multiprocessing (separate processes), C-extensions (release GIL), or `asyncio` (I/O-bound).

P3 — **Deepcopy vs Copy:** `copy` creates a new container but references original items. `deepcopy` recursively copies items. **Example:** `list1 = [[1]]`, `list2 = copy(list1)`. Modifying `list2[0][0]` changes `list1[0][0]`.

P4 — **Decorators:** Functions wrapping other functions. **Time Decorator:**
```python
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 — **Generators & Yield:** Generators are lazy iterators. `yield` pauses execution and returns a value. **Vs List:** Generators save memory (no list creation). `yield from` delegates to another generator.

P6 — **Asyncio:** Event loop manages async tasks. `asyncio.run()` starts the loop. `asyncio.gather()` runs tasks concurrently. `asyncio.create_task()` schedules a task. **time.sleep()** blocks the loop; use `asyncio.sleep()` instead.

P7 — **MRO:** Method Resolution Order. Python 3 uses **C3 Linearization**. **Diamond:** `D(B, C)`. MRO: `D -> B -> C -> A`. `D().who()` calls `C.who()` because `C` appears before `A` in the linearization and overrides `who`.

P8 — **Metaclass:** Class of a class. `__init_subclass__` is easier for subclass hooks (no metaclass needed). **Use Case:** Metaclass for ORM registration; `__init_subclass__` for config validation on subclass creation.

P9 — **Multiprocessing vs Threading:** Multiprocessing bypasses GIL (CPU-bound), separate memory. Threading shares memory (GIL-bound, I/O-bound). `concurrent.futures` provides high-level API. **Serialization:** Multiprocessing requires pickling arguments, which can fail for complex objects.

P10 — **__slots__:** Fixed attribute list. Prevents dynamic attribute creation. **Impact:** Reduces memory usage significantly for many instances, but prevents `__dict__`.

P11 — **Virtual Environment:** Isolated Python environment. **Why:** Dependency management. **Difference:** `venv` (stdlib), `virtualenv` (3rd party, older), `conda` (package manager + env, data science).

P12 — **Comprehensions:** Concise loops. `map()`/`filter()` are functional. **Prefer:** Comprehensions for readability and Pythonic style. `map()`/`filter()` for lazy evaluation or functional pipelines.

P13 — **== vs is:** `==` compares values. `is` compares identity (memory address). **Interning:** Small integers (-5 to 256) are cached. `a is b` returns True for small ints because they point to the same object.

P14 — **args/kwargs:** `*args` = tuple of positional args. `**kwargs` = dict of keyword args. `/` = positional-only. `*` = keyword-only. Example: `def f(a, /, b, *, c):`.

P15 — **Context Managers:** Manage resources (`with` statement). **Implementation:** `__enter__`/`__exit__` or `@contextmanager`. **Example:** Locking a resource or managing a database transaction.

P16 — **Type Hinting:** Static typing annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. **Tools:** `mypy`/`pyright` check types before runtime to catch errors early.

P17 — **Data Model:** `__repr__` (dev/debug), `__str__` (user). `__eq__` vs `__hash__`: If `__eq__` is defined, `__hash__` becomes `None` (unhashable) unless explicitly defined. `__getattr__` (missing attr), `__getattribute__` (all attr access).

P18 — **Descriptors:** Control attribute access (`__get__`, `__set__`, `__delete__`). **property:** Uses descriptors for getter/setter. **classmethod/staticmethod:** Implemented via descriptors to bind/unbind methods.

P19 — **Import System:** **Finders** locate modules, **Loaders** load them. **Relative vs Absolute:** Relative (`from . import x`) vs Absolute (`from pkg import x`). `sys.path` determines search paths. `__init__.py` marks package. **Namespace Packages:** No `__init__.py`, allow splitting packages across directories.

P20 — **Dataclasses:** Reduce boilerplate for `__init__`, `__repr__`, etc. **Vs attrs/Pydantic:** `attrs` more flexible, `Pydantic` adds validation. `__post_init__` runs after init. `frozen` = immutable. **Choose Pydantic** for API validation (JSON schema, types).
