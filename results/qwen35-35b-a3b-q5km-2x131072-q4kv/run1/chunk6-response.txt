JS1 — Easy: `var` is function-scoped and hoisted (initialized as `undefined`). `let` and `const` are block-scoped and hoisted but not initialized (Temporal Dead Zone). `const` cannot be reassigned. Hoisting is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — Easy: The event loop manages execution. The call stack executes synchronous code. When async operations occur, callbacks are placed in queues. Microtasks (Promises, `queueMicrotask`) run after the call stack clears but before macrotasks. Macrotasks (setTimeout, I/O) run after microtasks. The loop repeats: stack empty → microtasks → render → macrotasks.

JS3 — Medium: A closure is a function bundled with its lexical environment. Example: `function outer() { let x=1; return () => x; }`. Classic bug: `for (var i=0; i<3; i++) setTimeout(() => console.log(i), 0)` logs 3, 3, 3 because `i` is shared. Fix: use `let` (block scope) or an IIFE to capture `i`.

JS4 — Medium: `Promise` represents eventual completion. `async/await` syntactic sugar for promises. `try/catch` handles errors in async functions. `Promise.all` rejects on first failure; `Promise.allSettled` waits for all to settle (fulfilled/rejected) and returns status objects for each.

JS5 — Medium: `interface` defines shape and can be extended/merged; `type` is a union/intersection alias. Use `interface` for public API contracts, `type` for unions/complex logic. Discriminated unions are unions with a literal type field (e.g., `{type: 'a', a: string}` | `{type: 'b', b: number}`) allowing type narrowing based on the discriminator.

JS6 — Medium: JS uses prototypal inheritance; objects link to a prototype. `Object.create(proto)` creates an object with `proto` as its prototype. ES6 `class` is syntactic sugar over prototypes; `extends` sets the prototype chain, `constructor` initializes instance properties.

JS7 — Hard: `this` refers to the execution context. Binding rules (priority): 1. `new` (new instance), 2. Explicit (`call`, `apply`, `bind`), 3. Implicit (`obj.method()`), 4. Default (`window`/`undefined`). `bind()` returns a new function with `this` permanently bound. You cannot re-bind a bound function.

JS8 — Hard: `WeakRef` holds a weak reference to an object (GC can collect it). `FinalizationRegistry` notifies when a weakly held object is collected. Use for caches or cleanup. `WeakMap` keys must be objects because primitives are copied, not referenced, preventing weak tracking. This allows GC to reclaim keys without breaking the map.

JS9 — Hard: V8 uses hidden classes (shapes) to optimize property access. Inline caching (IC) caches the hidden class of the last object seen at a property access site. Deoptimization occurs when hidden classes change (e.g., adding properties dynamically). Patterns causing deopt: dynamic property addition, `delete`, `with` statement, changing prototype.

JS10 — Medium: Tree-shaking removes unused exports. ES Modules (static `import`/`export`) allow static analysis for tree-shaking. CommonJS (`require`) is dynamic, so bundlers cannot statically analyze dependencies easily, making tree-shaking ineffective without transpilation to ESM.

JS11 — Easy: `undefined` is a primitive type (unassigned); `null` is an intentional empty value. `==` converts types (`null == undefined` is true); `===` checks type and value (`null === undefined` is false). `??` (nullish coalescing) returns right operand if left is `null` or `undefined`. `?.` (optional chaining) safely accesses nested properties.

JS12 — Easy: `map` transforms elements, `filter` selects elements, `reduce` accumulates values. `reduce` takes a callback and optional initial value (otherwise first element is accumulator). `flatMap` maps then flattens one level; useful for transforming arrays of arrays into a single array.

JS13 — Easy: Template literals use backticks `` ` `` for multi-line strings and `${}` interpolation. Tagged templates pass the string array and substitution values to a function. Example: `html` strings for escaping: `function html(strings, ...values) { return strings.join('') + values.join(''); }`.

JS14 — Easy: Destructuring extracts values from objects/arrays. `const {a, b} = obj`. Default values: `const {a=1} = obj`. Renaming: `const {a: x} = obj`. Rest operator `...` collects remaining items: `const [a, ...rest] = arr`.

JS15 — Easy: `for...in` iterates over enumerable property keys (including inherited). `for...of` iterates over iterable values (array elements, strings). Avoid `for...in` on arrays because it includes prototype properties and order isn't guaranteed.

JS16 — Medium: Generics (`<T>`) allow reusable types. Constraints (`T extends Foo`) limit types. Conditional types (`T extends U ? X : Y`) select types based on condition. `infer` infers types within conditional types. Example: `type First<T> = T extends (infer U)[] ? U : never;`.

JS17 — Medium: CommonJS uses `require`/`module.exports` (dynamic, synchronous). ES Modules use `import`/`export` (static, async). Node.js supports both; `type: "module"` in `package.json` forces ESM interpretation for `.js` files.

JS18 — Hard: Structural typing means types compatible by shape are assignable. Excess property checks prevent extra properties in object literals assigned to types. `extends` creates a subtype relationship; `satisfies` checks shape without widening the type. Variance (`in`, `out`) controls generic parameter covariance/contravariance (TS 4.7+).

JS19 — Hard: `Proxy` intercepts operations (get, set, apply). `Reflect` provides default implementations. Example: Validation proxy intercepts `set` to validate data before assignment. `Reflect` is used inside proxies to call default behavior.

JS20 — Hard: `SharedArrayBuffer` allows memory sharing between workers. `Atomics` provides atomic operations (load, store, compareExchange). Ordering guarantees ensure visibility across threads. `Atomics.wait` blocks until `Atomics.notify` wakes it, enabling synchronization without locks.

B1 — Easy: `$@` preserves arguments as separate words (respects quoting). `$*` joins arguments into a single string. `"$@"` expands to separate quoted arguments. `"$*"` expands to a single string with spaces.

B2 — Easy: `set -e` exits on error. `set -u` errors on undefined variables. `set -o pipefail` returns error if any command in a pipeline fails (not just the last).

B3 — Medium: `$(command)` is preferred over backticks `` `command` `` (nested easier). `$((expression))` is arithmetic expansion. `expr` is an external command for arithmetic (slower, legacy).

B4 — Medium: Process substitution `<(command)` creates a file descriptor `/dev/fd/N`. Used to pass command output as a file argument. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — Medium: Use `while IFS= read -r -d '' file; do ... done < <(find . -print0)`. `for f in $(ls)` breaks on spaces/newlines and parses glob expansion. Safe iteration avoids word splitting.

B6 — Medium: Here document `<<EOF` reads from stdin until EOF. Here string `<<<` passes a string to stdin. `<<-EOF` allows leading tabs to be stripped from the delimiter line.

B7 — Hard: `cmd > file 2>&1` redirects stdout to file, then stderr to stdout (file). `cmd 2>&1 > file` redirects stderr to original stdout (terminal), then stdout to file. `&>` redirects both to file. First two differ because redirections happen left-to-right.

B8 — Hard:
```bash
lock() {
  mkdir "$1.lock" 2>/dev/null && trap 'rmdir "$1.lock"' EXIT INT TERM
}
```
`mkdir` is atomic on POSIX, preventing race conditions. `trap` ensures cleanup on exit or signal.

B9 — Easy: Single quotes `'...'` disable expansion. Double quotes `"..."` allow expansion. No quotes allow globbing. Example: `echo '$HOME'` prints `$HOME`. `echo "$HOME"` prints path. `echo *` prints files.

B10 — Easy: `[ -f file ]` checks file. `[ -d dir ]` checks directory. `[ -r file ]` checks readable. `[[ ]]` supports regex (`=~`) and logical operators (`&&`, `||`) without quoting issues. `[ ]` is POSIX compliant.

B11 — Easy: `$?` exit status of last command. `$$` current PID. `$!` PID of last background job. `$#` number of arguments passed to script.

B12 — Easy: `function name() { ... }` or `name() { ... }`. Return values via `return` (exit code) or `echo` (stdout). Functions share global scope unless `local` is used.

B13 — Easy: `PATH` is colon-separated directories. Bash searches `PATH` for executables. Builtin is internal (e.g., `cd`), external is binary (e.g., `/bin/ls`). Use `type` or `which` to check.

B14 — Medium: `trap 'cmd' SIGNAL`. `EXIT` runs on script end. `ERR` on error. `INT`/`TERM` on signals. Multiple signals: `trap` handles the last registered handler for that signal unless `set -E` is used.

B15 — Medium: `xargs` builds command lines from stdin. `command substitution` runs command immediately. `xargs -0` handles null-delimited input (`find -print0`). `xargs -P` runs jobs in parallel.

B16 — Medium: Arrays: `arr=(1 2)`. Associative: `declare -A map`. Append: `arr+=(3)`. Length: `${#arr[@]}`. Syntax differs: `arr[0]` vs `map["key"]`.

B17 — Hard: Subshell `(...)` runs in new process (vars lost). Group `{ ...; }` runs in current shell. Piping into `while` creates subshell, so variable changes inside loop are lost. Use process substitution to avoid subshell.

B18 — Hard: `SIGINT` (Ctrl+C), `SIGTERM` (kill), `SIGHUP` (terminal close), `SIGKILL` (force kill, no handler). Graceful shutdown: `trap` SIGTERM to cleanup background jobs (`wait`) before exit.

B19 — Hard:
```bash
split -l 1000 log.log part_
for f in part_*; do
  xargs -P 4 -I {} bash -c 'process {}' < <(cat "$f")
done
cat part_* > merged.log
```
Trade-off: Parallelism vs I/O contention. Error propagation: Check exit codes of parallel jobs; if any fail, exit script.

B20 — Hard: `exec` replaces current shell process. `exec 3>file` opens FD 3. `exec 3>&-` closes it. Coprocess (`coproc`) creates bidirectional pipe. Use for persistent communication (e.g., interactive tools).

PS1 — Easy: `Write-Output` sends objects to pipeline. `Write-Host` writes to console (bypasses pipeline). Distinction matters because `Write-Host` output cannot be captured or piped.

PS2 — Easy: Pipeline passes objects. `ForEach-Object` is a cmdlet (pipeline-aware, processes each object). `foreach` is a statement (iterates collection in memory). Use `ForEach-Object` for streaming data.

PS3 — Medium: `[PSCustomObject]` creates a typed object with properties. `[hashtable]` is key-value pairs. Convert: `[PSCustomObject]@{Key='Value'}`. Use CustomObject for structured data, hashtable for dynamic config.

PS4 — Medium: Terminating errors stop execution (throw exception). Non-terminating continue (warning). `-ErrorAction Stop` converts non-terminating to terminating. `try/catch` handles terminating errors. `$ErrorActionPreference` sets default behavior.

PS5 — Medium: Splatting `@params` passes hashtable as arguments. `Cmdlet -Param1 $val` passes single param. Splatting allows dynamic argument lists.

PS6 — Medium: `Invoke-Command -ComputerName` runs one-off commands remotely. `Enter-PSSession` starts interactive session. Uses WinRM (HTTP/HTTPS) or SSH (v7+).

PS7 — Hard: `+=` on scalar creates array. `Get-Items` returns array even if one item. `$x.GetType().Name` is `String[]`. If 0 items, returns `$null`. Correct pattern: `$result = @(); $result += ...; return $result`.

PS8 — Hard: `[scriptblock]` is code object. Functions are named scriptblocks. Pass via `-ArgumentList`. Double-hop: CredSSP allows auth delegation. `Invoke-Command -ArgumentList` serializes variables.

PS9 — Easy: `@()` forces array. `[array]` casts. `$null[0]` throws error. `@()[0]` returns `$null`.

PS10 — Easy: `-eq` (equal), `-like` (glob), `-match` (regex), `-contains` (array contains), `-in` (value in array). Case-sensitive variants (`-ceq`) use culture-sensitive comparison. `-contains` checks array for value; `-in` checks if value is in array.

PS11 — Easy: Module: reusable code. `.psm1` script, `.psd1` manifest (metadata), binary (DLL). `Import-Module` loads module into session.

PS12 — Easy: `"..."` interpolates `$var`. `'...'` literal. `@"..."@` multi-line. Expressions: `"${var}"`.

PS13 — Easy: `Get-Member` lists properties/methods. Properties (data), Methods (actions), NoteProperties (added dynamically). `Select-Object` filters properties. `Where-Object` filters objects.

PS14 — Medium: Classes (v5+) support inheritance, constructors. Limitations: No interfaces, limited polymorphism. Use classes for complex state, functions for simple logic.

PS15 — Medium: `[CmdletBinding()]` enables advanced features. `SupportsShouldProcess` adds `-WhatIf/-Confirm`. `Begin/Process/End` blocks handle setup, streaming, and cleanup.

PS16 — Medium: Providers expose data as file system. `Get-ChildItem` works on Registry (`HKLM:\`). `cd HKLM:\` navigates registry like folders.

PS17 — Hard: Runspaces manage threads. `[runspacepool]` manages pool. `ForEach-Object -Parallel` (v7+) uses pool. Manual runspace for complex control. Thread safety: avoid shared mutable state.

PS18 — Medium: PSScriptAnalyzer checks code quality. Rules have severity (Error, Warning, Info). Suppress with `#pragma`. Integrate via CI (e.g., GitHub Actions).

PS19 — Hard: `$using:` passes local variable to remote/parallel. `-ArgumentList` passes serialized data. Limits: CLIXML depth (complex objects may fail). Use primitive types or serialize to JSON for complex data.

PS20 — Hard: DSC configures state. Resources define actions. LCM applies config. WinPS DSC uses XML; PS7 uses JSON. Modern IaC: Ansible, Terraform, or PS7 DSC Core (deprecated in favor of other tools).
