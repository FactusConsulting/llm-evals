JS1 — **var** is function-scoped and hoisted (declaration only). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages execution. The **call stack** runs synchronous code. **Microtasks** (Promises) run after the current stack clears but before the next macrotask. **Macrotasks** (setTimeout, I/O) run in the next event loop tick. Microtasks drain completely before macrotasks execute.

JS3 — A **closure** is a function retaining access to its lexical scope. Example: `function outer() { let x=1; return () => x; }`. **For loop bug:** `var` creates one shared scope; all callbacks reference the final loop value. **Fix:** Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents async completion. **async/await** is syntactic sugar for promises. Error handling uses `try/catch` with await or `.catch()`. **Promise.all** rejects on first error. **Promise.allSettled** waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 — **Interface** defines structure for objects/classes (declaration merging). **Type** defines data shapes (primitives, unions, intersections). Use **Interface** for public API contracts, **Type** for unions/complex shapes. **Discriminated Unions** use a literal property (tag) to narrow types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — **Prototypal inheritance** links objects via a prototype chain. **Object.create(proto)** creates an object with `proto` as its prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` and `constructor` to manage the chain.

JS7 — **this** refers to the execution context. **4 Binding Rules:** 1. **New** (constructor), 2. **Explicit** (call/apply/bind), 3. **Implicit** (obj.method), 4. **Default** (window/global). **bind()** returns a new function with bound context. **Re-binding is impossible**; the context is fixed.

JS8 — **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** runs cleanup when the object is collected. **WeakMap** keys must be objects (GC can track them). This allows storing metadata without preventing garbage collection of keys.

JS9 — **V8** optimizes via **hidden classes** (shape optimization) and **inline caching** (caching property access). **Deoptimization** occurs when hidden classes change (e.g., adding properties dynamically). Patterns causing this: dynamic property addition, `delete`, or `with` statements.

JS10 — **Tree-shaking** removes unused code. **ES Modules** are static (compile-time analysis), enabling tree-shaking. **CommonJS** is dynamic (`require`), preventing static analysis. Tree-shaking fails in CommonJS because dependencies are resolved at runtime.

JS11 — **null** is intentional absence; **undefined** is unassigned. **==** coerces types (null == undefined is true). **===** checks strict equality. **Nullish coalescing (??)** returns right side if left is null/undefined. **Optional chaining (?.)** safely accesses nested properties.

JS12 — **map()** transforms elements. **filter()** selects elements. **reduce()** accumulates values. **Initial value** in reduce prevents using the first array element as the accumulator. **flatMap()** maps then flattens one level; useful for normalizing nested arrays.

JS13 — **Template literals** use backticks for multi-line strings and interpolation (`${}`). **Tagged templates** call a function with the string parts and values. Example: `html`<div>${name}</div>` calls `html(['<div>','</div>'], name)`.

JS14 — **Destructuring** extracts values. **Object:** `{ a } = obj`. **Array:** `[a] = arr`. **Defaults:** `{ a = 1 }`. **Renaming:** `{ a: b }`. **Rest operator:** `{ ...rest }` collects remaining properties/values.

JS15 — **for...in** iterates over enumerable keys (including inherited). **for...of** iterates over values of iterables. Avoid **for...in** on arrays because it includes prototype properties and order is not guaranteed.

JS16 — **Generics** use type parameters (`<T>`). **Constraints:** `T extends U` limits types. **Conditional types:** `T extends U ? X : Y` selects types. **infer:** extracts types from conditional types. Example: `type First<T> = T extends (infer U)[] ? U : never`.

JS17 — **CommonJS** uses `require`/`module.exports` (dynamic). **ES Modules** use `import`/`export` (static). **Node.js** supports both (CJS default, ESM via `.mjs` or `"type": "module"`). `"type": "module"` in `package.json` treats `.js` files as ESM.

JS18 — **Structural typing** checks shape, not name. **Excess property checks** apply to object literals assigned to types (e.g., `{a:1, b:2} as {a:1}`). **extends** is inheritance; **satisfies** checks type without widening. **Variance** is inferred (covariant/contravariant) based on usage, not explicit keywords.

JS19 — **Proxy** intercepts operations (get, set, etc.). **Reflect** provides default implementations. Example: `new Proxy(obj, { set: (t, k, v) => { validate(k, v); t[k] = v; } })`. Used for validation, logging, or reactive data binding.

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic ops (wait, notify). Workers communicate via `postMessage` or shared memory. **Ordering guarantees** come from memory barriers. **Atomics.wait** blocks until notified; **notify** wakes waiting threads.

B1 — **$@** expands to separate arguments. **$*** expands to a single string (IFS dependent). **"$@"** preserves arguments as separate words (handles spaces). **"$*"** joins them into one string.

B2 — **set -e** exits on error. **-u** errors on undefined variables. **-o pipefail** exits if any command in a pipeline fails. Together they ensure robust script execution.

B3 — **$(command)** is modern command substitution. **`command`** is legacy. **$((expression))** is arithmetic expansion (built-in). **expr** is an external command for arithmetic (slower).

B4 — **Process substitution** `<(cmd)` creates a file descriptor. Used to treat command output as a file. Example: `diff <(sort a) <(sort b)` compares sorted outputs without temp files.

B5 — Use globbing (`for f in *`) or `find`. **`for f in $(ls)`** is dangerous because it splits on whitespace/newlines, breaking filenames with spaces.

B6 — **Here document (`<<EOF`)** feeds multi-line input. **Here string (`<<<`)** feeds a single line. **`<<-EOF`** strips leading tabs from the input lines.

B7 — **`cmd > file 2>&1`**: Redirects stdout to file, then stderr to stdout (file). **`cmd 2>&1 > file`**: Redirects stderr to stdout (terminal), then stdout to file (stderr remains on terminal). **`&> file`** redirects both to file.

B8 — Use `mkdir` for locking (atomic). Function: `mkdir /tmp/lock` (fails if exists). Trap `EXIT` to `rmdir /tmp/lock`. Handle SIGTERM by trapping and cleaning up lock before exit.

B9 — **Single quotes** (`'...'`) are literal. **Double quotes** (`"..."`) allow expansion. **No quotes** allow globbing/splitting. Example: `echo $HOME` (empty), `echo "$HOME"` (path), `echo '$HOME'` (literal string).

B10 — **`[ -e file ]`** exists, **`-d`** directory, **`-r`** readable. **`[[ ]]`** supports regex and logical operators, safer than `[ ]` (test).

B11 — **$?** (exit status), **$$** (current PID), **$!** (last background PID), **$#** (argument count). Use case: `$?` for error checking, `$$` for temp files.

B12 — **`function foo {}`** or **`foo() {}`**. Return values via `exit code` (0-255) or `echo` (stdout). Functions share global scope unless `local` is used.

B13 — **PATH** is colon-separated directories. Bash searches PATH for external commands. **Builtin** is internal (e.g., `cd`), **External** is binary. Use `type cmd` to check.

B14 — **trap** sets signal handlers. **EXIT** runs on shell exit. **ERR** on error. **INT/TERM** on signals. Execution order: Signal handler runs, then script continues or exits.

B15 — **xargs** builds command lines from stdin. **`find -print0 | xargs -0`** handles spaces/newlines. **`-P`** runs jobs in parallel (e.g., `xargs -P 4`).

B16 — **Indexed:** `arr=(1 2)`. **Associative:** `declare -A map`. Append: `arr+=(val)`. Length: `${#arr[@]}`. Associative keys are strings, values are strings.

B17 — **Subshell `(...)`** forks a new process (vars lost). **Group `{...}`** runs in current shell. **Pipe into while** creates subshell for loop, so vars don't persist outside.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (force, no trap). Handle graceful shutdown by trapping SIGTERM, cleaning up, then exiting.

B19 — Script: `split -l 1000 log.log part_`. `xargs -P 4 -I {} sh -c 'process {}'`. Merge results: `cat part_* > merged.log`. Trade-offs: Parallelism vs I/O contention. Error propagation: Check exit codes of parallel jobs.

B20 — **exec** replaces the shell process. **File descriptors:** `exec 3>file` opens FD 3. `exec 3>&-` closes it. **Coprocess** (`coproc`) creates a pipe to a command, allowing bidirectional communication.

PS1 — **Write-Output** sends objects to the pipeline. **Write-Host** writes to the UI (console). Pipeline breaks if Write-Host is used because it doesn't output objects.

PS2 — **Pipeline** passes objects. **ForEach-Object** is a cmdlet (processes items one-by-one). **foreach** is a statement (loop). Use **ForEach-Object** for pipeline streaming, **foreach** for logic loops.

PS3 — **[PSCustomObject]** creates a typed object with properties. **[hashtable]** is key/value pairs. Convert: `@{a=1} | ConvertTo-CustomObject`. Use **PSCustomObject** for structured data, **Hashtable** for dynamic config.

PS4 — **Terminating** errors stop execution (throw). **Non-terminating** continue (catch). **-ErrorAction** overrides preference. **try/catch** handles terminating errors.

PS5 — **Splatting** `@params` passes a hashtable as arguments. **Difference:** Splatting expands keys to params; passing hashtable as one param treats it as a single argument.

PS6 — **Invoke-Command** runs one-off commands remotely. **Enter-PSSession** interactive. Uses **WinRM** (HTTP/HTTPS) or **SSH**.

PS7 — Returns **Object[]** if items added. If 0 items, returns **$null** (not empty array). **Correct pattern:** Always return `@()` to ensure array type consistency.

PS8 — **[scriptblock]** is code object. Pass via `-ArgumentList` or `-ScriptBlock`. **Double-hop** (jumping from server to server) fails Kerberos auth. Solve with **CredSSP** or **Invoke-Command -ArgumentList** to pass creds.

PS9 — **@()** creates array. **[array]** casts to array. Indexing **$null** throws error. Indexing **empty array** returns empty. Force single result: `$x = @($single)`.

PS10 — **-eq** (equal), **-like** (wildcard), **-match** (regex). **-c** variants are case-sensitive. **-contains** checks if collection contains item. **-in** checks if item is in collection.

PS11 — **Module** is reusable code. **.psm1** (script), **.psd1** (manifest), **Binary** (compiled). **Import-Module** loads it into session.

PS12 — **Interpolation** `"$var"` expands variables. **Single quotes** `'...'` are literal. **Here-string** `@"..."@` allows multi-line interpolation.

PS13 — **Get-Member** lists properties/methods. **Properties** (data), **Methods** (functions), **NoteProperties** (dynamic). **Select-Object** filters properties. **Where-Object** filters objects.

PS14 — **Classes** (v5) define types. Limitations: No interfaces in v5, limited inheritance. Use **Class** for stateful objects, **Function** for stateless logic.

PS15 — **CmdletBinding** enables advanced features (parameters, ShouldProcess). **Begin/Process/End** blocks handle setup, streaming, and cleanup.

PS16 — **Providers** expose data as file system. **Get-ChildItem** works on Registry, Certs, Env. Access registry: `Get-ChildItem HKLM:\Software`.

PS17 — **[runspace]** is thread. **[runspacepool]** manages threads. **ForEach-Object -Parallel** (v7+) simplifies parallelism. Manual runspace for complex control.

PS18 — **PSScriptAnalyzer** enforces standards. **Rules** check code. **Severity** (Error, Warning). **Suppress** via comments. Integrate via CI/CD (e.g., GitHub Actions).

PS19 — **$using:** passes local vars to scriptblock. **ArgumentList** serializes objects. **Limits:** CLIXML depth limits complex objects. Use **$using:** for simple vars, **ArgumentList** for objects.

PS20 — **DSC** manages state. **Configurations** define state. **LCM** applies config. **PS7+** uses DSC Core. Replaced by **Ansible**, **Terraform**, or **Containerization** for modern IaC.
