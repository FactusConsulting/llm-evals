## Kubernetes

**K1** — A **Deployment** manages stateless applications, ensuring a desired number of identical pods are running. It uses a ReplicaSet and allows rolling updates. A **StatefulSet** manages stateful applications, providing stable, unique network identities (hostnames) and ordered deployment/termination (e.g., databases).

**K2** — A **ClusterIP** Service creates a virtual IP within the cluster. **kube-proxy** (using iptables or IPVS) programs the node's network rules to redirect traffic sent to the Service IP to the backend Pod IPs based on the Service's selector.

**K3** — **PersistentVolume (PV)** is a cluster-level storage resource. **PersistentVolumeClaim (PVC)** is a user request for storage. **StorageClass** defines the provisioner and parameters for dynamic provisioning. A PVC binds to a PV (static) or triggers a PV creation via StorageClass (dynamic).

**K4** — 1. Check `kubectl logs <pod>` for application errors. 2. Check `kubectl describe pod <pod>` for events (OOMKilled, CrashLoopBackOff). 3. Verify liveness/readiness probe configurations. 4. Inspect resource limits (CPU/Memory) and node capacity.

**K5** — CoreDNS provides DNS resolution for services and pods within the cluster. If CoreDNS pods are down, internal service discovery (e.g., `my-service.default.svc`) fails, breaking inter-service communication, though external DNS may still work if fallback exists.

**K6** — The scheduler uses **Filtering** (predicates) to eliminate nodes that don't meet requirements (e.g., taints, resource availability) and **Scoring** (priorities) to rank remaining nodes. Factors: 1. Resource requests (CPU/Mem), 2. Node Affinity/Taints, 3. Pod Affinity/Anti-Affinity.

**K7** — **Scheduling:** Yes. Node has 4GB, 3.5GB requested (0.5GB free). Pod requests 256Mi (< 512Mi). **Runtime:** If it uses 600Mi > 512Mi limit, the container will be **OOMKilled** and restart.

**K8** — 1. Request hits External LB. 2. LB forwards to Ingress Controller (NGINX). 3. NGINX matches Host/Path rules to a Kubernetes Service. 4. Service selects Pods via Endpoints. 5. kube-proxy routes traffic to Pod IP. 6. Pod handles request.

**K9** — A **NetworkPolicy** restricts traffic flow to/from pods. Default behavior (no policy): All traffic is allowed. When applied, it restricts traffic only to pods matching the policy's `podSelector` and defined ingress/egress rules.

**K10** — **RollingUpdate** replaces pods gradually (minReadySeconds). **Recreate** kills all existing pods before starting new ones. Choose **Recreate** for stateful apps where downtime is acceptable or to ensure a clean state before migration.

**K11** — A **Namespace** provides logical isolation for resources. Default namespaces: `default`, `kube-system`, `kube-public`, `kube-node-lease`.

**K12** — `kubectl describe` shows detailed status and events. 1. **Events** (e.g., scheduling failures). 2. **Conditions** (e.g., PodReady). 3. **Environment Variables** and **Mounts**. `kubectl get` only shows status, age, and ready count.

**K13** — A **ConfigMap** stores non-confidential configuration data. Pods consume it via: 1. **Environment Variables** (env: valueFrom configMapKeyRef). 2. **Volume Mounts** (volumes: configMap).

**K14** — **ClusterIP:** Internal cluster access only. **NodePort:** Exposes service on a static port on each node's IP. **LoadBalancer:** Provisions an external cloud load balancer to expose the service.

**K15** — A **DaemonSet** ensures a copy of a pod runs on all (or specific) nodes. Use for logging agents (Fluentd), monitoring (Prometheus Node Exporter), or network plugins (CNI).

**K16** — **Taints** repel pods; **Tolerations** allow pods to schedule on tainted nodes. **Node Affinity** pulls pods to nodes. Use Case: Taint a node for GPU workloads (only pods with GPU toleration schedule there); Affinity to co-locate app and cache pods.

**K17** — An **Operator** is a custom controller that manages complex stateful applications using CRDs. It differs from a standard controller by encoding operational knowledge (lifecycle, backup, scaling) into the application logic, solving manual management of stateful apps.

**K18** — etcd uses **Raft consensus** (strong consistency). If quorum (majority) is lost, the cluster becomes read-only/unavailable. Recovery: Restore a snapshot on a new node, join it to the cluster, and rejoin remaining nodes if possible.

**K19** — **Role/ClusterRole** define permissions. **RoleBinding/ClusterRoleBinding** grant them to users/groups.
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata: namespace: default
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata: namespace: default
subjects:
- kind: User
  name: developer
roleRef:
  kind: Role
  name: pod-reader
```

**K20** — `ndots` (default 5) causes the resolver to try appending search domains up to 5 times before treating the name as FQDN, causing multiple queries. Fix: Set `dnsConfig: ndots: 1` in the pod spec to reduce search attempts.

## Development

**D1** — **Rebase** rewrites history to create a linear commit path. **Merge** preserves history with a merge commit. Avoid rebasing on shared/public branches to prevent history conflicts for collaborators.

**D2** — **TCP** is connection-oriented, reliable (ACKs). **UDP** is connectionless, unreliable (fire-and-forget). TCP: HTTP, SSH. UDP: DNS, VoIP.

**D3** — A **race condition** occurs when output depends on the sequence/timing of uncontrollable events. Example: Two threads incrementing a shared counter `x++` without a lock, resulting in lost updates.

**D4** — **Symmetric:** Same key for encrypt/decrypt (fast). **Asymmetric:** Public/Private key pair (secure key exchange). TLS uses Asymmetric for the handshake (key exchange) and Symmetric for the session data (performance).

**D5** — A **layer** is a read-only filesystem diff in a Docker image. **Caching:** Place instructions that change frequently (e.g., `COPY .`) after dependencies (`RUN apt-get`) to maximize layer reuse during builds.

**D6** — 1. Check application logs for exceptions. 2. Verify upstream dependencies (DB, APIs). 3. Check resource limits (OOM/Kill). 4. Review load balancer health checks and rate limiting rules.

**D7** — **CAP:** Consistency, Availability, Partition Tolerance. **CP:** Zookeeper (guarantees consistency during partition). **AP:** Cassandra (guarantees availability during partition, eventual consistency).

**D8** — **Horizontal:** Add more nodes (scale out). **Vertical:** Add more resources to a node (scale up). Patterns enabling H-scale: Stateless services, sharding, load balancing.

**D9** — An **environment variable** is a dynamic value set at runtime. To persist across sessions on Linux, export it in `~/.bashrc` or `~/.profile`.

**D10** — **JSON** is strict key-value pairs. **YAML** is human-readable, indentation-based. JSON advantage: Strict parsing, standard in web APIs. YAML advantage: Comments, less verbose, supports anchors.

**D11** — **REST** is an architectural style using HTTP. **GET:** Retrieve. **POST:** Create. **PUT:** Replace resource. **DELETE:** Remove resource.

**D12** — **Version pinning** specifies exact dependency versions (e.g., `package==1.0.0`). It ensures reproducible builds by preventing unexpected breaking changes from upstream updates.

**D13** — **Compiled:** Source to binary before execution (C, Go). **Interpreted:** Source executed line-by-line at runtime (Python, JavaScript).

**D14** — **Reverse Proxy:** Receives client requests, forwards to backend server (e.g., Nginx serving static files). **Forward Proxy:** Client requests go through proxy to internet (e.g., corporate firewall). Use cases: Nginx (web server), HAProxy (load balancing).

**D15** — A **deadlock** is a permanent blocking of threads waiting for each other. **Coffman Conditions:** 1. Mutual Exclusion, 2. Hold and Wait, 3. No Preemption, 4. Circular Wait.

**D16** — **Eventual Consistency:** Updates propagate asynchronously; system converges over time. Cassandra uses **Read Repair** and **Hinted Handoff**. Consistency levels: **ONE** (fastest), **QUORUM** (balanced), **ALL** (slowest, strongest).

**D17** — **mTLS:** Both client and server present certificates. Handshake: Client Hello -> Server Hello -> Server Cert -> Client Cert -> Verify. Used in **Service Meshes** (Istio) and secure database connections.

**D18** — A **hash table** maps keys to indices via a hash function. **Collision:** Chaining (linked list at index) or Open Addressing (probe next slot). Complexity: Avg O(1), Worst O(n) (all collisions).

**D19** — **Circuit Breaker:** Prevents cascade failures. **Closed:** Normal operation. **Open:** Fail fast (reject requests). **Half-Open:** Test if service recovered. Relates to retries: Circuit opens to stop retry storms.

**D20** — **Optimistic:** Check conflicts at commit (e.g., version number). **Pessimistic:** Lock data on read (e.g., `SELECT FOR UPDATE`). Choose Optimistic for high concurrency/low conflict; Pessimistic for critical data/high conflict.
