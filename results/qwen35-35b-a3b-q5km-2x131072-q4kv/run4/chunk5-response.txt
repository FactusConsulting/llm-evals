DN1 ‚Äî **Class vs Struct:** `class` is a reference type (heap), supports inheritance, and is mutable by default. `struct` is a value type (stack/inline), does not support inheritance (except from `System.ValueType`), and is immutable by convention. **Use struct** for small, immutable data objects (e.g., `Point`, `Vector`) where copying overhead is low and value semantics are desired.

DN2 ‚Äî **IEnumerable<T> vs IQueryable<T>:** `IEnumerable<T>` executes queries in-memory (LINQ to Objects). `IQueryable<T>` builds expression trees that are translated to a query language (e.g., SQL) and executed on the data source (LINQ to SQL/EF). **Matters for DB:** `IQueryable` pushes filtering/aggregation to the database, reducing network traffic. `IEnumerable` pulls all data to memory first, then filters, causing performance issues.

DN3 ‚Äî **async/await:** Compiles to a state machine. `Task` represents an asynchronous operation returning an object. `ValueTask` is a struct wrapper that avoids heap allocation if the operation completes synchronously. **Use ValueTask** in high-frequency hot paths (e.g., inside a loop) where `Task` allocation overhead is significant.

DN4 ‚Äî **Dependency Injection (DI):** Pattern for decoupling dependencies. **Lifetimes:** `Transient` (new instance every request), `Scoped` (new instance per request scope), `Singleton` (one instance for app lifetime). **Bug:** Injecting a `Scoped` service into a `Singleton` causes the Singleton to hold a reference to the Scoped service's first instance, leading to stale data or `ObjectDisposedException` when the scope ends.

DN5 ‚Äî **string vs ReadOnlySpan<char>:** `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a stack-allocated reference to memory (can point to heap or stack) allowing zero-copy manipulation. **Use Span<T>** for high-performance parsing or string processing where avoiding allocations and copying is critical.

DN6 ‚Äî **GC:** Manages memory via generations. **Gen 0:** Short-lived objects. **Gen 1:** Intermediate. **Gen 2:** Long-lived. **LOH:** Large Object Heap (>85KB) for objects like large strings/arrays, collected less frequently. **IDisposable:** Required for unmanaged resources (files, DB connections) because GC only manages managed memory; `Dispose` ensures timely cleanup.

DN7 ‚Äî **ConfigureAwait(false):** Prevents capturing the current `SynchronizationContext`. **Use** in library code or server-side (ASP.NET) to avoid deadlocks and improve throughput. **Do NOT use** in UI applications (WPF/WinForms) where you need to marshal back to the UI thread to update controls.

DN8 ‚Äî **.NET 8/9 Features:** **Primary Constructors:** `class C(int x) { }`. **Collection Expressions:** `[1, 2, 3]`. **FrozenDictionary:** Immutable, high-performance dictionary. **Example:**
```csharp
public class Service(int x) : IService { public int Value => x; }
// DI: services.AddSingleton<Service>(new Service(10));
```

DN9 ‚Äî **DbContext Exception:** Likely cause is injecting `DbContext` as `Singleton` or reusing a disposed context across threads. **Fix:** Register `DbContext` as `Scoped` in DI container. **Interaction:** EF Core `DbContext` is not thread-safe; it must be scoped to a single request to ensure isolation and proper disposal.

DN10 ‚Äî **Record vs Class:** `record` provides value-based equality and immutability by default (via primary constructor). `class` provides reference equality. **Record Struct:** Value type with record semantics. **Choose Record** for DTOs/immutable data models. **Choose Class** for mutable entities or when inheritance is required.

DN11 ‚Äî **ref, out, in:** `ref` passes by reference (must be initialized). `out` passes by reference (must be assigned in method). `in` passes by reference (read-only). **Use** `ref`/`out` for large structs to avoid copy or multiple return values. `in` for large read-only data.

DN12 ‚Äî **Nullable Reference Types:** `string?` indicates a reference type *may* be null (compiler warning). `int?` (nullable value type) indicates a value type *can* be null. **Difference:** Ref types default to non-nullable unless annotated; value types default to nullable if `?` is added.

DN13 ‚Äî **LINQ:** Language Integrated Query. **Method vs Query:** Syntactic sugar; compiles to same IL. **Deferred Execution:** Query logic isn't executed until enumeration (e.g., `ToList()`, `foreach`). Prevents unnecessary work if result isn't used.

DN14 ‚Äî **Generic Constraints:** Restrict type arguments. `where T : class` (ref type), `where T : struct` (val type), `where T : new()` (paramless ctor), `where T : IComparable<T>` (interface). **Combine:** `where T : class, IComparable<T>` to enforce multiple requirements.

DN15 ‚Äî **Abstract vs Interface:** `abstract` allows code + contract, single inheritance. `interface` defines contract only (multiple inheritance). **C# 8+ Default Methods:** Interfaces can have method implementations, blurring the line by allowing shared logic without abstract classes.

DN16 ‚Äî **Middleware Pipeline:** Request processing chain. `app.Use()` adds middleware. `app.Map()` branches pipeline by path. `app.Run()` is terminal (no next). **Order:** First added executes first. **Exception Handling:** Middleware wrapping the pipeline catches exceptions; specific handlers can be added to stop propagation.

DN17 ‚Äî **Channel<T> vs BlockingCollection<T>:** `Channel<T>` is async-first, supports backpressure. `BlockingCollection<T>` is sync-first. **Bounded Channel:** Set max capacity. If full, producers wait (backpressure), preventing memory exhaustion under high load.

DN18 ‚Äî **Source Generators:** Compile-time code generation. **Vs Reflection:** No runtime overhead. **Vs IL Weaving:** Happens before compilation, no post-build step. **Incremental:** Processes only changed files (faster) vs `ISourceGenerator` (full re-run). Preferred for performance in large solutions.

DN19 ‚Äî **TPL Dataflow:** Pipeline processing library. `ActionBlock` (process, no output), `TransformBlock` (map input to output), `BufferBlock` (queue). **Vs Channels:** Dataflow offers higher-level flow control (linking, cancellation, completion) and built-in backpressure, whereas Channels are lower-level primitives.

DN20 ‚Äî **Minimal APIs:** Delegate-based endpoints without controllers. **Trade-offs:** Less boilerplate, harder to scale complex auth/middleware, OpenAPI support requires extensions (Swashbuckle). **Route Groups:** Organize endpoints by path prefix. **Endpoint Filters:** Apply logic (e.g., validation) to specific endpoints.

P1 ‚Äî **List vs Tuple:** `List` is mutable, ordered collection. `Tuple` is immutable, ordered collection. **Set vs Frozenset:** `Set` is mutable, unique items. `Frozenset` is immutable, hashable (can be dict key).

P2 ‚Äî **GIL:** Global Interpreter Lock allows only one thread to execute Python bytecode at a time. **Affects:** CPU-bound threading. **Workarounds:** Multiprocessing (processes bypass GIL), C-extensions (release GIL), or `asyncio` (I/O bound).

P3 ‚Äî **Deepcopy vs Copy:** `copy` creates shallow copy (nested objects shared). `deepcopy` creates recursive copy. **Example:** `list2 = copy(list1)` where `list1` contains dicts; modifying `list2`'s dict modifies `list1`'s dict.

P4 ‚Äî **Decorators:** Functions wrapping other functions. **Time Logger:**
```python
import time
def timeit(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 ‚Äî **Generators:** Functions using `yield` to produce iterators lazily. **Vs List:** No memory allocation for all items. **yield from:** Delegates to another generator (sub-generator).

P6 ‚Äî **Asyncio:** Event loop manages async tasks. `asyncio.run()` (entry), `asyncio.gather()` (wait multiple), `asyncio.create_task()` (schedule). **time.sleep()** blocks the loop; use `asyncio.sleep()` to yield control.

P7 ‚Äî **MRO:** Method Resolution Order. Python 3 uses **C3 Linearization**. **Diamond Inheritance:** `D(B, C)`. MRO: `D -> B -> C -> A`. `D().who()` prints "C" because `C` overrides `A` and appears after `B` in MRO.

P8 ‚Äî **Metaclass:** Class of a class. `__init_subclass__` is a subclass hook (easier, no metaclass needed). **Use Metaclass:** Enforce class structure (e.g., Django models). **Use `__init_subclass__`:** Configure subclass behavior (e.g., registering plugins).

P9 ‚Äî **Multiprocessing vs Threading:** Multiprocessing (processes, bypass GIL, heavy, pickling required). Threading (threads, GIL, I/O bound). **Concurrent.Futures:** High-level API for both. **Pickling:** Multiprocessing requires args to be serializable.

P10 ‚Äî **__slots__:** Fixed attribute list. **Prevents:** `__dict__` creation. **Impact:** Saves memory, prevents dynamic attribute assignment. Use for large numbers of objects with fixed attributes.

P11 ‚Äî **Virtual Environment:** Isolated Python environment. **Why:** Dependency management. **venv:** Standard library (Python 3.3+). **virtualenv:** Third-party, older. **conda:** Package manager + env (data science).

P12 ‚Äî **Comprehensions:** Concise loops. `map()`/`filter()` return iterators (functional). **Prefer Comprehensions:** For readability and list/set/dict creation. Use `map`/`filter` for complex logic or existing functions.

P13 ‚Äî **== vs is:** `==` checks value equality. `is` checks identity (memory address). **Interning:** Small integers (-5 to 256) are cached. `a is b` returns True for small ints due to interning.

P14 ‚Äî **args/kwargs:** `*args` (tuple), `**kwargs` (dict). **Markers:** `/` (positional only), `*` (keyword only). Example: `def f(a, /, b, *, c):`.

P15 ‚Äî **Context Managers:** `with` statement for setup/teardown. **__enter__/__exit__** (class-based) vs `@contextmanager` (decorator). **Example:** Locking a resource or managing a temporary directory.

P16 ‚Äî **Type Hinting:** Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. **mypy/pyright:** Analyze code without running to catch type errors early.

P17 ‚Äî **Data Model:** Dunder methods. `__repr__` (dev/debug), `__str__` (user). `__eq__` vs `__hash__`: If `__eq__` is defined, `__hash__` becomes None (unhashable) unless explicitly defined.

P18 ‚Äî **Descriptors:** Objects controlling attribute access (`__get__`, `__set__`, `__delete__`). **property:** Uses descriptors for getters/setters. **classmethod/staticmethod:** Implemented as descriptors to modify binding behavior.

P19 ‚Äî **Import System:** **Finders** locate modules, **Loaders** load them. **Relative:** Based on current package. **Absolute:** Based on sys.path. `__init__.py` marks package. **Namespace Packages:** No `__init__.py`, allow splitting package across directories.

P20 ‚Äî **Dataclasses:** Auto-generates `__init__`, `__repr__`, etc. **Vs attrs:** attrs more flexible. **Vs Pydantic:** Pydantic adds validation. **Choose Pydantic** for API models requiring validation (e.g., FastAPI). `__post_init__` runs after init.
