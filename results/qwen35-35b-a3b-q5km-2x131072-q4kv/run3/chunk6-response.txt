## JavaScript / TypeScript (20 questions)

**JS1 — Easy:** `var` is function-scoped and hoisted (initialized as `undefined`). `let` and `const` are block-scoped and hoisted but not initialized (Temporal Dead Zone). `const` cannot be reassigned. Hoisting is the behavior where declarations are moved to the top of their scope during compilation.

**JS2 — Easy:** The event loop manages execution. The **call stack** executes synchronous code. When async operations complete, callbacks go to the **microtask queue** (Promises, `queueMicrotask`) or **macrotask queue** (setTimeout, I/O). The event loop processes the call stack, then all microtasks, then one macrotask, repeating.

**JS3 — Medium:** A closure is a function remembering its lexical scope even when executed outside it. Example: `function outer() { let x = 1; return () => x; }`. Classic bug: `for (var i=0; i<3; i++) setTimeout(() => console.log(i), 0)` logs `3, 3, 3` because `var` is function-scoped. Fix: use `let` (block-scoped) or an IIFE.

**JS4 — Medium:** `Promise` represents eventual completion. `async/await` syntactic sugar for promises. Errors use `try/catch` with `await`. `Promise.all` rejects immediately on first failure. `Promise.allSettled` waits for all to settle, returning status (`fulfilled`/`rejected`) for each.

**JS5 — Medium:** `interface` defines shape only (can be extended/merged), `type` defines value (can be primitives, unions, tuples). Use `interface` for public API contracts, `type` for complex unions or primitives. Discriminated unions use a literal type property to narrow types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

**JS6 — Medium:** Prototypal inheritance links objects via `[[Prototype]]`. `Object.create(proto)` creates an object with `proto` as its prototype. ES6 `class` is syntactic sugar over prototypes; `extends` sets the prototype chain, `constructor` initializes instance properties.

**JS7 — Hard:** `this` binding depends on call site. Priority: 1. `new` (new instance), 2. Explicit (`call`, `apply`, `bind`), 3. Receiver object (`obj.method()`), 4. Default (`window`/`undefined`). `bind()` returns a new function with fixed `this`. It cannot be re-bound.

**JS8 — Hard:** `WeakRef` holds weak reference to object (GC can collect target). `FinalizationRegistry` notifies when target is collected. `WeakMap` keys must be objects (GC can track them). Used for caching metadata without preventing GC or cleanup logic.

**JS9 — Hard:** V8 uses **hidden classes** (shapes) for property access optimization. **Inline Caching (IC)** caches property lookup results. **Deoptimization** occurs when assumptions break (e.g., adding properties dynamically, polymorphic calls). Patterns causing deopt: dynamic property access, `arguments` object, `with` statement.

**JS10 — Medium:** Tree-shaking removes unused exports. Works with **ES Modules** (static `import`/`export` allows static analysis). **CommonJS** (`require`) is dynamic, so bundlers cannot statically analyze dependencies effectively, preventing tree-shaking.

**JS11 — Easy:** `undefined` is default uninitialized value; `null` is intentional absence. `==` converts types (`null == undefined` is true); `===` checks type and value. `??` (nullish coalescing) returns right side if left is `null`/`undefined`. `?.` (optional chaining) safely accesses nested properties.

**JS12 — Easy:** `map` transforms, `filter` selects, `reduce` accumulates. `reduce` takes `(acc, curr) => acc`. Initial value prevents `acc` being first element. `flatMap` maps then flattens one level; useful for transforming arrays of arrays into single arrays.

**JS13 — Easy:** Template literals use backticks `` ` `` for multi-line strings and `${}` interpolation. Tagged templates pass the string array and values to a function. Example: `html` strings for escaping: `function html(strings, ...values) { return strings.join('') + values.join(''); }`.

**JS14 — Easy:** Destructuring extracts values. Object: `{a, b}`. Array: `[a, b]`. Defaults: `{a = 1}`. Rename: `{a: b}`. Rest operator `...` collects remaining items: `const [first, ...rest] = arr`.

**JS15 — Easy:** `for...in` iterates over enumerable **property keys** (inherited too). `for...of` iterates over **iterable values** (array elements). Avoid `for...in` on arrays because it includes non-index properties and order isn't guaranteed.

**JS16 — Medium:** Generics (`<T>`) allow reusable types. Constraint: `function f<T extends string>(t: T)`. Conditional: `type IsString<T> = T extends string ? true : false`. `infer`: `type UnwrapPromise<T> = T extends Promise<infer U> ? U : T`.

**JS17 — Medium:** **CommonJS** (`require`) is synchronous, runtime resolution. **ES Modules** (`import`) are static, async, top-level. Node.js supports both; `"type": "module"` in `package.json` forces `.js` files to be treated as ESM.

**JS18 — Hard:** TypeScript is **structural**: types compatible if shapes match. **Excess property checks** apply only to object literals in assignments. `extends` creates inheritance/subtyping; `satisfies` checks shape without widening type. Variance: `in` (contravariant), `out` (covariant) for generics (TS 4.7+).

**JS19 — Hard:** `Proxy` intercepts operations (get, set, apply). `Reflect` provides default implementations. Example: Validation Proxy. `new Proxy(obj, { set(target, key, value) { if (key === 'age' && value < 0) throw Error; return true; } })`.

**JS20 — Hard:** `SharedArrayBuffer` allows shared memory between workers. `Atomics` provides atomic operations (load, store, wait, notify). `Atomics.wait` blocks until `notify` is called. Guarantees memory ordering (sequential consistency). Used for high-performance inter-thread communication.

## Bash (20 questions)

**B1 — Easy:** `$@` expands to individual quoted arguments (`"$1" "$2"`). `$*` expands to a single string (`"$1 $2"`). `"$@"` preserves argument boundaries; `"$*"` joins them with IFS.

**B2 — Easy:** `set -e` exits on error. `set -u` errors on undefined variables. `set -o pipefail` returns exit code of last failed command in a pipeline. Ensures scripts fail fast on any issue.

**B3 — Medium:** `$(command)` is preferred over backticks `` `command` `` (nested easier). `$((expression))` is arithmetic expansion (fast, built-in). `expr` is an external command (slower, legacy).

**B4 — Medium:** Process substitution `<(cmd)` creates a file descriptor `/dev/fd/N`. Example: `diff <(sort a) <(sort b)`. Useful when commands expect filenames but produce streams.

**B5 — Medium:** Use `for f in *; do ...` or `while IFS= read -r -d '' f; do ...`. `for f in $(ls)` breaks on spaces/newlines and parses output of `ls` (unsafe).

**B6 — Medium:** Here doc `<<EOF` reads until EOF line. Here string `<<<"str"` passes string as stdin. `<<-EOF` allows leading tabs to be stripped from indentation.

**B7 — Hard:** `cmd > file 2>&1`: stdout to file, stderr to stdout (file). `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. `&> file`: both to file (bash extension). First two differ because redirections are processed left-to-right.

**B8 — Hard:**
```bash
lock() {
  mkdir "$1.lock" 2>/dev/null && trap 'rmdir "$1.lock"' EXIT INT TERM
}
```
`mkdir` is atomic (fails if exists), acting as a lock. `trap` ensures cleanup on exit or signal.

**B9 — Easy:** Single quotes `'...'`: literal (no expansion). Double quotes `"..."`: variable/command expansion. No quotes: word splitting/globbing. Example: `echo $HOME` (empty), `echo "$HOME"` (path), `echo '$HOME'` (literal string).

**B10 — Easy:** `[ -f file ]` (file), `[ -d dir ]` (directory), `[ -r file ]` (readable). `[[ ]]` is a keyword (supports `&&`, `||`, regex), `[ ]` is a command (POSIX). `[[` is safer for bash.

**B11 — Easy:** `$?` last exit code. `$$` current PID. `$!` background job PID. `$#` number of arguments. Use `$?` for error checking, `$$` for temp files.

**B12 — Easy:** `func() { ... }` or `function func { ... }`. Return values via `return` (exit code) or `echo` (stdout). Functions share global scope unless `local` used.

**B13 — Easy:** `PATH` is colon-separated dirs. Bash searches `PATH` for executables. `type` or `command -v` checks. Builtin (shell internal) vs External (binary).

**B14 — Medium:** `trap 'cmd' SIGTERM`. Cleanup handlers run on EXIT, ERR, INT, TERM. Order: EXIT handlers run last. Multiple signals: queued, handled sequentially.

**B15 — Medium:** `xargs` builds command lines from stdin. `command substitution` runs command immediately. `xargs -0` + `find -print0` handles newlines/spaces. `xargs -P` runs jobs in parallel.

**B16 — Medium:** Arrays: `arr=(a b)`. Associative: `declare -A map`. Append: `arr+=("c")`. Length: `${#arr[@]}`. Syntax differs (`declare -A` required for associative).

**B17 — Hard:** Subshell `(...)` runs in child (vars lost). Group `{ ...; }` runs in current shell. Piping `| while` creates subshell for loop, losing variable changes. Use process substitution or `while ... done < <(cmd)`.

**B18 — Hard:** SIGINT (Ctrl+C), SIGTERM (kill), SIGHUP (hangup), SIGKILL (force kill, no handler). Graceful shutdown: `trap` SIGTERM to cleanup background jobs (`wait`) before exit.

**B19 — Hard:** Split log: `split -l 1000 log.log part_`. Process: `xargs -P 4 -I {} bash process.sh {}`. Merge: `cat part_* > merged.log`. Trade-off: I/O vs CPU. Error propagation: check exit codes of `xargs` jobs, aggregate non-zero.

**B20 — Hard:** `exec` replaces shell process (no new PID). File descriptors: `exec 3>file` opens fd 3. `exec 3>&-` closes it. Coprocess: `coproc CMD` creates pipe. Use for persistent background processes needing I/O.

## PowerShell (20 questions)

**PS1 — Easy:** `Write-Output` sends objects to pipeline. `Write-Host` writes to console (bypasses pipeline). Distinction matters for piping; `Write-Host` breaks pipelines.

**PS2 — Easy:** Pipeline passes objects. `ForEach-Object` is a cmdlet (pipeline-aware, processes items one-by-one). `foreach` is a statement (iterates collection in memory). Use `ForEach-Object` for streaming, `foreach` for logic loops.

**PS3 — Medium:** `[PSCustomObject]` creates typed objects with properties (pipeline friendly). `[hashtable]` is key-value pairs (dynamic). Convert: `[PSCustomObject]@{Key='Value'}`. Use CustomObject for structured data, hashtable for config.

**PS4 — Medium:** Terminating errors stop execution (`throw`). Non-terminating continue (`Write-Error`). `-ErrorAction Stop` converts non-terminating to terminating. `$ErrorActionPreference` sets default. `try/catch` handles terminating errors.

**PS5 — Medium:** Splatting `@params` passes hashtable as parameters. `Cmd -Param1 $val` vs `Cmd @params`. Splatting improves readability and dynamic parameter passing.

**PS6 — Medium:** `Invoke-Command -ComputerName` runs one-off. `Enter-PSSession` interactive. Uses WinRM (HTTP/HTTPS) or SSH (v7+). SSH requires `Enable-PSRemoting -Transport SSH`.

**PS7 — Hard:** Returns string `"one"`, not array. `$x.GetType().Name` is `String`. Returns 0 items: `$x` is `$null`. Correct pattern: `$result = @(); $result += "one"; return $result`.

**PS8 — Hard:** `[scriptblock]` is code object. Pass via `-ArgumentList` or `-ScriptBlock`. Double-hop: CredSSP allows auth delegation. Solve with `Invoke-Command -ArgumentList` to pass creds securely or use SSH.

**PS9 — Easy:** `@()` forces array. `[array]` casts. Indexing `$null` returns `$null`. Indexing `@()` returns `$null` (no error). `@()` ensures consistent array behavior.

**PS10 — Easy:** `-eq` (equal), `-like` (glob), `-match` (regex), `-contains` (array contains), `-in` (value in array). Case-sensitive variants: `-ceq`, `-clike`. `-contains` checks array for value; `-in` checks if value is in collection.

**PS11 — Easy:** Module: reusable code. `.psm1` script, `.psd1` manifest (metadata), binary (DLL). `Import-Module` loads into session. Manifest defines exports, dependencies, version.

**PS12 — Easy:** `"..."` interpolates `$var`. `'...'` literal. `@"..."@` here-string (multi-line). Expressions: `"Value: $var"`.

**PS13 — Easy:** `Get-Member` lists properties/methods. Properties (data), Methods (actions), NoteProperties (dynamic). `Select-Object` selects props, `Where-Object` filters.

**PS14 — Medium:** Classes (v5+) define types. Limitations: no private fields (use `private` keyword), limited inheritance vs C#. Use classes for complex state, functions for simple logic.

**PS15 — Medium:** `[CmdletBinding()]` enables advanced features. `SupportsShouldProcess` adds `-WhatIf/-Confirm`. `Begin` (init), `Process` (pipeline items), `End` (finalize).

**PS16 — Medium:** Providers expose data as file system. `Get-ChildItem` works on Registry (`HKLM:\`). `cd HKLM:\Software` navigates registry like folders.

**PS17 — Hard:** Runspaces: isolated execution contexts. `RunspacePool` manages threads. `ForEach-Object -Parallel` (v7+) uses runspace pool. Manual: `[RunspaceFactory]::CreateRunspacePool()`. Thread safety: avoid shared mutable state.

**PS18 — Medium:** PSScriptAnalyzer: static analysis. Rules enforce style (naming, length). Severity: Error, Warning, Info. Suppress: `# SuppressMessage`. Integrate via CI (e.g., GitHub Actions).

**PS19 — Hard:** `$using:` scope passes local vars to remote/parallel. `-ArgumentList` passes arguments. Serialization limits: CLIXML depth (complex objects may fail). Use simple types or serialize to JSON.

**PS20 — Hard:** DSC: declarative config. Configs define state. LCM applies config. WinPS DSC vs PS7 DSC (cross-platform). Replaced by modern IaC (Terraform, Ansible) for cloud-native, but DSC remains for Windows config.
