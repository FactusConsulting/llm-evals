## .NET / C#

**DN1 — Easy: What is the difference between class and struct in C#? When should you use a struct?**
*   **Class:** Reference type, stored on the heap, supports inheritance, default value is `null`.
*   **Struct:** Value type, stored on stack (or inline in heap object), no inheritance (implements interfaces only), default value is zero-initialized.
*   **Usage:** Use `struct` for small, immutable data types that represent a single value (e.g., `Point`, `Vector2`, `Money`). Avoid for large data or when inheritance is needed.

**DN2 — Easy: Explain the difference between IEnumerable<T> and IQueryable<T>. Why does this distinction matter for database queries?**
*   **IEnumerable<T>:** Executes in-memory. Filters/transforms are applied after data is retrieved.
*   **IQueryable<T>:** Executes on the server (e.g., SQL). Builds an expression tree that is translated into SQL.
*   **Distinction:** Using `IQueryable` pushes filtering to the database (efficient). Using `IEnumerable` pulls all data to memory first then filters (inefficient for large DBs).

**DN3 — Medium: How does async/await work in C#? What is a Task vs a ValueTask? When should you use ValueTask?**
*   **Async/Await:** Compiles to a state machine. `await` yields control until the operation completes without blocking the thread.
*   **Task:** Reference type, heap allocation. Represents an async operation.
*   **ValueTask:** Value type, zero allocation. Represents an async operation that might complete synchronously.
*   **Usage:** Use `ValueTask` in hot paths (e.g., high-frequency async methods) where the operation often completes synchronously to avoid GC pressure.

**DN4 — Medium: Explain dependency injection in .NET. What are the three service lifetimes (Transient, Scoped, Singleton), and what is a common bug when injecting a Scoped service into a Singleton?**
*   **DI:** Pattern for decoupling dependencies from implementation.
*   **Lifetimes:**
    *   **Transient:** New instance every time requested.
    *   **Scoped:** New instance per request (e.g., HTTP request).
    *   **Singleton:** Single instance for the application lifetime.
*   **Bug:** Injecting a Scoped service into a Singleton captures the first Scoped instance forever. This causes stale data or `ObjectDisposedException` if the Scoped service (like DbContext) is disposed at the end of the request.

**DN5 — Medium: What is the difference between string and ReadOnlySpan<char> for string processing? When would you use Span<T> and why?**
*   **String:** Immutable, heap-allocated, copies on substring.
*   **ReadOnlySpan<char>:** Stack-allocated (mostly), no allocation, views memory without copying.
*   **Usage:** Use `Span<T>` for high-performance string parsing, slicing, or processing where avoiding GC allocations is critical (e.g., parsing large logs).

**DN6 — Medium: Explain how the .NET garbage collector works. What are generations (Gen 0, 1, 2)? What is the Large Object Heap? What is IDisposable and why do you need it if you have a GC?**
*   **GC:** Automatic memory management. Collects unreachable objects.
*   **Generations:** Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). Objects survive collections move up.
*   **LOH:** Heap for objects > 85KB. Collected less frequently.
*   **IDisposable:** Required for unmanaged resources (files, DB connections, handles). GC only manages managed memory; it doesn't know how to close OS handles.

**DN7 — Hard: What is the difference between ConfigureAwait(false) and the default behavior? When should you use it, and when should you NOT? How does this relate to SynchronizationContext?**
*   **Default:** Captures `SynchronizationContext` (e.g., UI thread) to resume on the same context.
*   **ConfigureAwait(false):** Does not capture context. Resumes on thread pool thread.
*   **Usage:** Use `false` in library code or server-side logic to avoid deadlocks and improve throughput.
*   **Do NOT use:** In UI applications or ASP.NET Core request handlers where you need to update UI or access request-scoped context after await.

**DN8 — Hard: Explain the new .NET 8/9 features: primary constructors, collection expressions, and FrozenDictionary. Write a minimal example using primary constructors with DI.**
*   **Primary Constructors:** Define constructor parameters directly in class declaration (C# 12).
*   **Collection Expressions:** `[1, 2, 3]` syntax for arrays/lists.
*   **FrozenDictionary:** Immutable dictionary with high performance.
*   **Example:**
    ```csharp
    public class Service(int Id) : IService { public int Id => Id; }
    // DI Registration: services.AddSingleton<Service>(new Service(1));
    ```

**DN9 — Hard: You have a .NET API that works fine under low load but throws ObjectDisposedException on the DbContext under high concurrency. Diagnose the likely cause and explain the fix. How does EF Core's DbContext lifetime interact with DI scoping?**
*   **Cause:** `DbContext` is likely registered as `Singleton` or shared across threads. It is not thread-safe.
*   **Fix:** Register `DbContext` as `Scoped` (per HTTP request).
*   **Interaction:** EF Core expects one `DbContext` instance per logical unit of work (request). Sharing it violates thread safety and disposal lifecycle.

**DN10 — Medium: What is the difference between record and class in C#? What about record struct? When would you choose each?**
*   **Record:** Reference type, value equality (by content), immutable by default (init-only properties), positional.
*   **Class:** Reference type, reference equality, mutable.
*   **Record Struct:** Value type, value equality.
*   **Choice:** Use `record` for DTOs, configuration, or immutable data models. Use `class` for mutable entities or complex inheritance hierarchies. Use `record struct` for small, immutable value objects.

**DN11 — Easy: What is the difference between ref, out, and in parameter modifiers in C#? When would you use each?**
*   **ref:** Pass by reference, must be initialized before call, can be modified.
*   **out:** Pass by reference, must be assigned inside method, can be modified.
*   **in:** Pass by reference, read-only (no modification allowed).
*   **Usage:** `ref` for modifying existing values. `out` for returning multiple values. `in` for passing large structs without copying.

**DN12 — Easy: Explain nullable reference types in C# (the ? annotation and #nullable enable). How do they differ from nullable value types (int?)?**
*   **NRT:** Compile-time warnings for reference types (e.g., `string?`) to prevent NullReferenceExceptions.
*   **Nullable Value Types:** Runtime nullability for value types (e.g., `int?` is `Nullable<int>`).
*   **Difference:** NRT is static analysis (compiler); Nullable Value Types are actual runtime types that can hold `null`.

**DN13 — Easy: What is LINQ? Explain the difference between method syntax (list.Where(...)) and query syntax (from x in list where ...). What is deferred execution?**
*   **LINQ:** Language Integrated Query for data.
*   **Syntax:** Method syntax is more flexible (supports complex operators); Query syntax is more readable (SQL-like). Both compile to the same IL.
*   **Deferred Execution:** Query is not executed until enumerated (e.g., `ToList()`, `foreach`).

**DN14 — Easy: What are generic constraints in C#? Explain where T : class, where T : struct, where T : new(), and where T : IComparable<T>. When would you combine multiple constraints?**
*   **Constraints:** Limit what types `T` can be.
*   **Examples:** `class` (reference), `struct` (value), `new()` (parameterless constructor), `IComparable<T>` (interface).
*   **Combination:** `where T : class, IDisposable` ensures T is a reference type that can be disposed.

**DN15 — Easy: What is the difference between abstract class and interface in C#? How did default interface methods (C# 8+) change this distinction?**
*   **Abstract Class:** Can have state (fields), constructors, and implementation. Single inheritance.
*   **Interface:** Contract only (no state). Multiple inheritance.
*   **Default Methods:** Interfaces can now have method implementations. Allows adding functionality without breaking existing implementations, blurring the line with abstract classes.

**DN16 — Medium: Explain the middleware pipeline in ASP.NET Core. What is the difference between app.Use(), app.Map(), and app.Run()? How does ordering matter, and how does exception handling middleware work?**
*   **Pipeline:** Request passes through middleware components.
*   **Use:** Adds middleware to pipeline.
*   **Map:** Branches pipeline for specific paths.
*   **Run:** Terminal middleware (no next).
*   **Order:** Critical (e.g., Exception Handling first, Auth before Routing).
*   **Exception Handling:** Catches exceptions thrown downstream and returns a response.

**DN17 — Medium: What is Channel<T> in .NET? How does it compare to BlockingCollection<T>? Give an example of using bounded channels for producer-consumer backpressure.**
*   **Channel<T>:** Async-first producer-consumer queue.
*   **BlockingCollection:** Sync wrapper around concurrent collections.
*   **Bounded Channel:** Limits queue size. If full, producer waits (backpressure).
*   **Example:** `new Channel<int>(new BoundedChannelOptions(100) { FullMode = ChannelFullMode.Wait })`.

**DN18 — Hard: Explain Source Generators in .NET. How do they differ from reflection and IL weaving? What is an incremental generator, and why is it preferred over ISourceGenerator?**
*   **Source Generators:** Generate C# code at compile time.
*   **Difference:** Reflection is runtime (slow); IL weaving modifies binaries post-compile. Generators are compile-time (fast, type-safe).
*   **Incremental:** Uses `IncrementalGenerator` API. Only regenerates code when inputs change (performance). Preferred over `ISourceGenerator` for better caching and incremental compilation support.

**DN19 — Hard: What is the System.Threading.Tasks.Dataflow library (TPL Dataflow)? Explain ActionBlock, TransformBlock, and BufferBlock. How does it compare to System.Threading.Channels for pipeline processing?**
*   **TPL Dataflow:** Building blocks for async dataflow pipelines.
*   **Blocks:** `ActionBlock` (process), `TransformBlock` (map), `BufferBlock` (queue).
*   **Comparison:** Channels are simpler async queues. Dataflow offers complex linking, cancellation, completion propagation, and backpressure out of the box.

**DN20 — Hard: What are Minimal APIs in .NET? How do they differ from controller-based APIs? What are the trade-offs in terms of testability, middleware, and OpenAPI support? How do route groups and endpoint filters work?**
*   **Minimal APIs:** Delegate-based endpoints (no controllers).
*   **Difference:** Less boilerplate, direct dependency injection.
*   **Trade-offs:** Easier for small apps, harder to scale/test for large apps. OpenAPI support requires `Swashbuckle`.
*   **Route Groups:** `MapGroup` organizes endpoints.
*   **Filters:** `AddEndpointFilter` allows intercepting requests/responses globally.

## Python

**P1 — Easy: What is the difference between a list and a tuple? What about a set vs a frozenset?**
*   **List:** Mutable, ordered, allows duplicates.
*   **Tuple:** Immutable, ordered, allows duplicates.
*   **Set:** Mutable, unordered, unique elements.
*   **Frozenset:** Immutable, unique elements.

**P2 — Easy: Explain the GIL (Global Interpreter Lock). What types of workloads does it affect, and what are the workarounds?**
*   **GIL:** Lock allowing only one thread to execute Python bytecode at a time.
*   **Affects:** CPU-bound multi-threaded workloads.
*   **Workarounds:** Multiprocessing (separate processes), C-extensions (release GIL), Asyncio (I/O bound).

**P3 — Medium: What is the difference between deepcopy and copy? Give an example where shallow copy causes unexpected behavior.**
*   **Copy:** Shallow copy (new object, references nested objects).
*   **Deepcopy:** Recursive copy (new object, new nested objects).
*   **Example:** `list1 = [[1], [2]]`, `list2 = copy(list1)`. Modifying `list2[0][0]` changes `list1[0][0]` because inner lists are shared.

**P4 — Medium: Explain Python decorators. Write a decorator that logs the execution time of any function.**
*   **Decorators:** Functions that modify other functions/classes.
*   **Example:**
    ```python
    import time
    def timer(func):
        def wrapper(*args, **kwargs):
            start = time.time()
            result = func(*args, **kwargs)
            print(f"{func.__name__} took {time.time() - start}s")
            return result
        return wrapper
    ```

**P5 — Medium: What are generators and yield? How do they differ from returning a list? What is yield from?**
*   **Generators:** Functions that pause execution (`yield`) and resume later. Lazy evaluation.
*   **Difference:** No memory overhead for full list storage.
*   **Yield From:** Delegates to another generator (sub-generator).

**P6 — Medium: Explain Python's asyncio event loop. What is the difference between asyncio.run(), asyncio.gather(), and asyncio.create_task()? Why can't you call time.sleep() inside an async function?**
*   **Event Loop:** Manages execution of async tasks.
*   **Run:** Starts the loop (entry point).
*   **Gather:** Waits for multiple tasks to complete.
*   **Create Task:** Schedules a coroutine.
*   **Sleep:** `time.sleep()` blocks the entire thread/loop. Use `await asyncio.sleep()`.

**P7 — Hard: Explain Python's MRO (Method Resolution Order). What algorithm does Python 3 use? What happens with diamond inheritance, and why does the following print what it does?**
*   **MRO:** Order in which base classes are searched.
*   **Algorithm:** C3 Linearization.
*   **Code:** `D(B, C)`. MRO: `D -> B -> C -> A`. `D` calls `B` (no `who`), then `C` (has `who`).
*   **Output:** "C".

**P8 — Hard: What is a metaclass? When would you use __init_subclass__ vs a metaclass? Give a practical use case for each.**
*   **Metaclass:** Class of a class (controls class creation).
*   **__init_subclass__:** Hook called when subclass is defined (simpler).
*   **Metaclass Use:** ORM models (Django) to register fields automatically.
*   **__init_subclass__ Use:** Plugin registration systems where you want to register subclasses without complex metaclass logic.

**P9 — Hard: Explain the difference between multiprocessing, threading, and concurrent.futures in Python. When would you use each? What are the serialization implications of multiprocessing (pickling)?**
*   **Threading:** Shared memory, GIL limited (I/O bound).
*   **Multiprocessing:** Separate memory, no GIL (CPU bound).
*   **Concurrent.Futures:** High-level abstraction for both.
*   **Pickling:** Multiprocessing requires objects to be serializable (pickleable) to pass between processes.

**P10 — Medium: What is __slots__? When should you use it, and what does it prevent? What is the memory impact?**
*   **Slots:** Declares fixed set of attributes.
*   **Prevents:** Creation of `__dict__` for each instance.
*   **Impact:** Significant memory reduction and faster attribute access. Prevents adding new attributes dynamically.

**P11 — Easy: What is a virtual environment, and why should you use one? What is the difference between venv, virtualenv, and conda?**
*   **Virtual Env:** Isolated Python environment with its own packages.
*   **Why:** Dependency conflicts, clean project setup.
*   **Differences:** `venv` (stdlib, modern), `virtualenv` (older, more features), `conda` (package manager + env, handles non-Python deps).

**P12 — Easy: Explain list comprehensions, dict comprehensions, and set comprehensions. How do they relate to map() and filter()? When would you prefer one over the other?**
*   **Comprehensions:** Concise syntax for creating collections.
*   **Relation:** `map`/`filter` are functional; comprehensions are Pythonic.
*   **Preference:** Prefer comprehensions for readability and simplicity. Use `map`/`filter` for lazy evaluation or complex logic.

**P13 — Easy: What is the difference between == and is in Python? What is object interning, and why does a is b sometimes return True for small integers?**
*   **==:** Value equality (`__eq__`).
*   **is:** Identity (same memory address).
*   **Interning:** Small integers (-5 to 256) are cached in memory.
*   **Result:** `a = 1; b = 1; a is b` is `True` because they point to the same cached object.

**P14 — Easy: Explain Python's *args and **kwargs. How do positional-only (/) and keyword-only (*) parameter markers work in function signatures?**
*   **Args:** `*args` collects positional args into a tuple. `**kwargs` collects keyword args into a dict.
*   **Markers:** `/` forces args before it to be positional-only. `*` forces args after it to be keyword-only.

**P15 — Easy: What are context managers (with statement)? How do you write one using __enter__/__exit__ vs @contextmanager? Give an example beyond file handling.**
*   **Context Managers:** Manage resources (setup/teardown).
*   **Implementation:** Class with `__enter__`/`__exit__` or `@contextmanager` decorator.
*   **Example:** Database connection manager that commits on success and rolls back on error.

**P16 — Medium: What is type hinting in Python? Explain typing.Union, Optional, Literal, TypeVar, and Protocol. How does mypy or pyright use these for static analysis?**
*   **Type Hinting:** Annotations for static type checking.
*   **Types:** `Union` (A|B), `Optional` (Union[X, None]), `Literal` (specific value), `TypeVar` (generic), `Protocol` (structural typing).
*   **Analysis:** Tools like `mypy` check code against hints without running it, catching type errors early.

**P17 — Medium: Explain Python's data model (dunder methods). What are __repr__ vs __str__, __eq__ vs __hash__, and __getattr__ vs __getattribute__? What happens if you define __eq__ but not __hash__?**
*   **Dunder:** Special methods for object behavior.
*   **Repr/Str:** `__repr__` (dev/debug), `__str__` (user).
*   **Eq/Hash:** `__eq__` defines equality. `__hash__` defines hashability.
*   **Defining Eq:** If `__eq__` is defined without `__hash__`, the object becomes unhashable (cannot be used in sets/dicts).
*   **Getattr:** `__getattr__` (fallback), `__getattribute__` (always called).

**P18 — Hard: What are descriptors in Python? Explain the descriptor protocol (__get__, __set__, __delete__). How do property, classmethod, and staticmethod use descriptors under the hood?**
*   **Descriptors:** Objects that control attribute access.
*   **Protocol:** `__get__`, `__set__`, `__delete__`.
*   **Usage:** `property` uses descriptors to manage getters/setters. `classmethod`/`staticmethod` use descriptors to bind/unbind methods.

**P19 — Hard: Explain Python's import system in detail. What is the difference between relative and absolute imports? How does sys.path, PYTHONPATH, __init__.py, and namespace packages work? What is a finder and a loader?**
*   **System:** Finds and loads modules.
*   **Relative:** Based on current package (`from . import x`). Absolute: Based on `sys.path`.
*   **Path:** `sys.path` (runtime), `PYTHONPATH` (env var).
*   **Init:** `__init__.py` marks directory as package (PEP 420 allows namespace packages without it).
*   **Finder/Loader:** Finder locates module; Loader loads code.

**P20 — Hard: What is dataclasses and how does it compare to attrs and Pydantic? Explain __post_init__, field factories, frozen dataclasses, and how dataclasses interacts with inheritance. When would you choose Pydantic over dataclasses?**
*   **Dataclasses:** Auto-generates `__init__`, `__repr__`, etc.
*   **Comparison:** `attrs` (more features), `Pydantic` (validation).
*   **Features:** `__post_init__` (run after init), `field()` (config), `frozen=True` (immutable).
*   **Inheritance:** Supports standard inheritance.
*   **Pydantic:** Choose when runtime validation (types, constraints) is required (e.g., API request models).
