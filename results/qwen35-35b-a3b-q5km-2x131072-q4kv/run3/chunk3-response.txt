T1 — **Plan** calculates the execution plan (what will happen) without making changes. **Apply** executes the plan to create, update, or destroy resources. Plan is safe for review; Apply is the action.

T2 — **State** is the database mapping real-world resources to configuration. **Remote state** is crucial for collaboration (sharing state), locking (preventing concurrent edits), and security (storing secrets outside local files).

T3 — **count** uses integer indices; **for_each** uses map/set keys. **for_each** is preferred when resources depend on dynamic attributes (e.g., names) because changing the list order doesn't shift indices and cause unintended resource recreation.

T4 — OpenTofu detects this as **drift**. Running `tofu plan` will show the resource as "modified" (or "tainted"). OpenTofu will propose changes to reconcile the state with the configuration.

T5 —
```hcl
resource "aws_instance" "web" {
  for_each = toset(["web1", "web2", "web3"])
  ami      = "ami-12345"
  instance_type = "t2.micro"
  tags = {
    Name = each.key
  }
}
```

T6 — **terraform_remote_state** reads state from a backend (legacy). **Module outputs** expose data directly from the module's state. **Recommended:** Use Module Outputs. It is more secure (no direct state access), type-safe, and decouples modules from backend implementation details.

T7 — **Why:** The `null_resource` provisioner finishes before the RKE2 cluster is fully ready to serve the kubeconfig, or the file isn't flushed. **Fix:** Add a `local-exec` provisioner that polls the Kubernetes API until ready before the `null_resource` marks success, or use `depends_on` on the consumer resource and ensure the provisioner waits for the API.

T8 — **Import CLI** is manual, one-time, and not version-controlled. **Import Block** (1.5+) is declarative, version-controlled, and allows importing resources directly in code during `apply`. It improves maintainability and CI/CD integration.

T9 — **Variable** is input data passed into the module (e.g., `tfvars`). **Output** is data exposed from the module to the caller. Variables are inputs; outputs are results.

T10 — A **provider** is a plugin that interacts with APIs (AWS, Azure, etc.). Without `tofu init`, providers are not downloaded, and `tofu plan` fails with "Provider not installed" errors.

T11 — The **lock file** records exact provider versions used. It should be committed to ensure reproducible builds and prevent provider version mismatches across environments.

T12 — **Destroy** removes all managed resources. Target a single resource with `-target=aws_instance.example`.

T13 — A **data source** reads existing infrastructure data without managing it. **Example:** `aws_ami` to find the latest Ubuntu image instead of hardcoding an ID.

T14 — **Variables** are inputs from outside the module. **Locals** are computed values used internally within the module. Use locals for complex expressions or repeated values that don't need external configuration.

T15 — **Lifecycle** controls resource behavior.
*   `create_before_destroy`: Create new resource before destroying old (zero downtime).
*   `prevent_destroy`: Prevent accidental deletion (safety).
*   `ignore_changes`: Ignore drift in specific attributes (e.g., tags managed externally).

T16 — A **moved block** tells OpenTofu that a resource has changed address (module path) but is the same resource. It updates the state mapping without destroying and recreating the resource during refactoring.

T17 — The **dependency graph** determines execution order. **Implicit** dependencies come from resource references. **Explicit** (`depends_on`) forces order. `depends_on` reduces parallelism because it forces serialization, slowing down apply times.

T18 — Strategies:
1.  **`plan -refresh=false`**: Skips state refresh (faster, but may miss drift).
2.  **Remote Backend**: Reduces local I/O and enables locking (faster, requires setup).
3.  **State File Splitting**: Use modules to reduce graph size (faster, increases complexity).
4.  **Parallelism**: Increase `TF_parallelism` (faster, risks race conditions if not careful).

T19 — **Locking** prevents concurrent writes. If a lock isn't released (crash), subsequent runs hang. **Break lock:** `tofu force-unlock <LOCK_ID>`. **Risk:** Data corruption if two processes write simultaneously.

T20 — Process:
1.  Update `backend` block in code to S3/DynamoDB.
2.  Run `tofu init -migrate-state`.
3.  OpenTofu copies local state to S3.
4.  Verify with `tofu state list` and check S3 bucket.

A1 — **Playbook**: YAML file defining automation steps. **Role**: Reusable structure (tasks, vars, templates). **Task**: Single action (e.g., install package).

A2 — **ansible_facts**: System info gathered by setup module. Access OS family via `{{ ansible_os_family }}`.

A3 — **command**: Runs without shell (no pipes). **shell**: Runs with shell (supports pipes). **raw**: No setup (no Python module), useful for bootstrapping SSH.

A4 — **Idempotency**: Running a task multiple times yields the same result. **Idempotent:** `apt: name=nginx`. **Not:** `shell: rm -rf /tmp/*` (deletes every time).

A5 —
```yaml
- name: Install htop
  package:
    name: htop
  when: ansible_os_family == "Debian"

- name: Install htop on RedHat
  package:
    name: htop
  when: ansible_os_family == "RedHat"
  # Add epel-release task if needed
```

A6 — **Precedence (High to Low):** Extra vars (`-e`) > Command line > Play vars > Host vars > Group vars > Role vars > Role defaults > Inventory vars. `set_fact` overrides most at runtime.

A7 — Use **serial** in the play: `serial: 3`. Group nodes in inventory (e.g., `rke2_nodes`). Use `wait_for` in a task to ensure the first 3 nodes are ready before the next batch starts.

A8 — **ansible-vault** encrypts sensitive data. Encrypt a single variable using `ansible-vault encrypt_string "secret" --name "my_secret"`.

A9 — **Inventory**: List of hosts. **Static**: Defined in a file (INI/YAML). **Dynamic**: Generated by a script/API (e.g., AWS EC2).

A10 — `ansible webservers -m shell -a "df -h"`

A11 — **Handler**: Task that runs only if notified (e.g., restart service). Differs from regular tasks by conditional execution based on `notify`. Triggered via `notify: handler_name`.

A12 — **ansible.cfg**: Configuration file. Settings: `host_key_checking`, `forks` (parallelism), `remote_user`.

A13 — **register**: Captures task output into a variable. Use in subsequent tasks via `{{ registered_var.stdout }}` or `when: registered_var.rc == 0`.

A14 — **include_tasks**: Dynamic (evaluated at runtime, supports `when`). **import_tasks**: Static (evaluated at parse time, faster, no `when` on import). Use `include` for conditionals, `import` for fixed lists.

A15 — **Ansible Galaxy**: Repository for roles/collections. Install via `ansible-galaxy install -r requirements.yml`.

A16 — **Connection plugins**: Manage SSH/connections. **ssh**: Remote Linux/Unix. **local**: Control node only. **network_cli**: Network devices (Cisco/Juniper).

A17 — **delegate_to**: Run task on specific host. **run_once**: Run once across all hosts. **Example:** Update DNS on `localhost` once after gathering IPs from all web servers.

A18 — Strategies:
1.  **Forks**: Increase parallelism (memory cost).
2.  **Pipelining**: Reduces SSH overhead (requires SSH config).
3.  **Fact Caching**: Store facts in DB (disk/DB cost).
4.  **Mitogen**: Async execution (complexity, external dependency).

A19 — **Jinja2**: `{{ }}` (output), `{% %}` (logic), `{# #}` (comment).
```jinja
upstream backend_servers {
{% for server in backend_list %}
    server {{ server.ip }} weight={{ server.weight }}; # Health check: {{ server.health }}
{% endfor %}
}
```

A20 — **Rolling Deploy**:
1.  Set `serial: 1` (or batch).
2.  **Pre:** Deregister from LB.
3.  **Deploy:** Update app.
4.  **Check:** `uri` module for health check.
5.  **Post:** Register to LB.
6.  **Rollback:** Use `block/rescue` to revert if health check fails.
