JS1 — Easy: `var` is function-scoped and hoisted with initialization to `undefined`. `let` and `const` are block-scoped and hoisted but not initialized (Temporal Dead Zone). `const` cannot be reassigned. Hoisting is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — Easy: The event loop manages asynchronous execution. The call stack executes synchronous code. When the stack is empty, the loop checks the microtask queue (Promises, `queueMicrotask`) first, then the macrotask queue (setTimeout, I/O). Microtasks drain completely before the next macrotask.

JS3 — Medium: A closure is a function bundled with its lexical environment. Example: A factory function returning an inner function accessing outer variables. Classic bug: `for (var i=0; i<3; i++) setTimeout(() => console.log(i), 0)` logs 3, 3, 3. Fix: Use `let` (block scope) or an IIFE to capture the current value of `i`.

JS4 — Medium: `Promise` represents eventual completion. `async/await` syntactic sugar for chaining promises. Errors are caught via `.catch()` or `try/catch`. `Promise.all` rejects immediately on first failure. `Promise.allSettled` waits for all to settle, returning status (`fulfilled`/`rejected`) for each.

JS5 — Medium: `interface` defines structure for objects/classes and supports merging. `type` defines aliases for primitives, unions, intersections, or tuples. Use `interface` for public API contracts/classes; `type` for complex unions or primitives. Discriminated unions use a literal property (tag) to narrow types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — Medium: Prototypal inheritance links objects via `[[Prototype]]`. `Object.create(proto)` creates an object with `proto` as its prototype. ES6 `class` is syntactic sugar over prototypes; `extends` sets the prototype chain, `constructor` initializes instance properties.

JS7 — Hard: `this` refers to the execution context. Binding rules (priority): 1. `new` (explicit), 2. `call`/`apply`/`bind` (explicit), 3. Object method call (implicit), 4. Default (global/window/undefined). `bind()` returns a new function with `this` permanently bound. It cannot be re-bound.

JS8 — Hard: `WeakRef` holds a weak reference to an object (GC can collect it). `FinalizationRegistry` notifies when a weakly held object is collected. Use for caching or cleanup. `WeakMap` keys must be objects because primitives are copied, not referenced, preventing garbage collection tracking.

JS9 — Hard: V8 optimizes via hidden classes (shape of objects) and inline caching (caching property access locations). Deoptimization occurs when assumptions break (e.g., adding properties dynamically, changing hidden class). Patterns causing deopt: `delete` properties, dynamic property access, `with` statement, `eval`.

JS10 — Medium: Tree-shaking removes unused exports. ES Modules (static `import`/`export`) allow static analysis for tree-shaking. CommonJS (`require`) is dynamic, so bundlers cannot statically determine dependencies, making tree-shaking ineffective without transpilation.

JS11 — Easy: `undefined` is the default value of uninitialized variables. `null` is an assignment value representing "no value". `==` converts types (`null == undefined` is true). `===` checks type and value (`null === undefined` is false). `??` returns right operand if left is `null` or `undefined`. `?.` safely accesses nested properties.

JS12 — Easy: `map` transforms elements. `filter` selects elements. `reduce` accumulates values. `reduce` with initial value starts accumulator at that value; without, it starts at index 0. `flatMap` maps then flattens one level; useful for transforming arrays of arrays into a single array.

JS13 — Easy: Template literals use backticks for multi-line strings and `${}` interpolation. Tagged templates pass the string array and interpolation values to a function. Example: `html` tag sanitizing inputs to prevent XSS.

JS14 — Easy: Destructuring extracts values from objects/arrays. Object: `{a, b}`. Array: `[a, b]`. Default values: `{a = 1}`. Renaming: `{a: b}`. Rest operator (`...`) collects remaining elements into an array or object properties.

JS15 — Easy: `for...in` iterates over enumerable property keys (including inherited). `for...of` iterates over iterable values (array elements). Avoid `for...in` on arrays because it includes non-index properties and order is not guaranteed.

JS16 — Medium: Generics (`<T>`) allow reusable types. Constraints (`T extends Foo`) limit types. Conditional types (`T extends U ? X : Y`) select types based on condition. `infer` extracts types within conditional types. Example: `type UnwrapPromise<T> = T extends Promise<infer U> ? U : T`.

JS17 — Medium: CommonJS uses `require`/`module.exports` (dynamic, synchronous). ES Modules use `import`/`export` (static, async). Node.js supports both; `type: "module"` in `package.json` forces ESM. ESM supports top-level await; CJS does not.

JS18 — Hard: TypeScript uses structural typing (shape matters, not name). Excess property checks prevent extra properties in object literals assigned to types. `extends` checks assignability; `satisfies` checks validity without widening the type. Variance: `in` (contravariant) for function arguments, `out` (covariant) for return types.

JS19 — Hard: `Proxy` intercepts operations (get, set, apply). `Reflect` provides default implementations. Example: A proxy validating property assignment or logging access. `Reflect` is used inside proxies to call default behavior.

JS20 — Hard: `SharedArrayBuffer` allows shared memory between workers. `Atomics` provides atomic operations. Workers communicate via `postMessage` or shared memory. Ordering guarantees are provided by `Atomics.wait` (blocks until notified) and `Atomics.notify` (wakes waiting threads). Requires COOP/COEP headers in browsers.

B1 — Easy: `$@` preserves arguments as separate words (respects quotes). `$*` joins arguments into a single string. `"$@"` expands to separate quoted arguments. `"$*"` expands to a single string with spaces.

B2 — Easy: `set -e` exits on error. `set -u` errors on undefined variables. `set -o pipefail` returns error if any command in a pipeline fails. Together they make scripts fail fast and safely.

B3 — Medium: `$(command)` is preferred over backticks (nested easier). `$((expression))` performs arithmetic. `expr` is an external command for arithmetic (slower, less feature-rich).

B4 — Medium: Process substitution `<(command)` creates a file descriptor pointing to command output. Used to pass command output as a file argument. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — Medium: Use `for f in *; do ...` or `find ... -print0 | while read -d '' f`. `for f in $(ls)` breaks on spaces/newlines and parses output of `ls` (unsafe).

B6 — Medium: Here document `<<EOF` reads until EOF line. Here string `<<<` passes a string. `<<-EOF` allows leading tabs to be stripped from the delimiter line.

B7 — Hard: `cmd > file 2>&1`: stdout to file, stderr to stdout (file). `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. `cmd &> file`: both to file (shorthand). First two differ because redirection order matters; `2>&1` before `> file` sends stderr to terminal.

B8 — Hard: Use `mkdir /tmp/lock` (atomic). Trap `EXIT`, `INT`, `TERM` to `rmdir /tmp/lock`. Check if dir exists before creating. Example:
```bash
trap 'rmdir /tmp/lock' EXIT INT TERM
mkdir /tmp/lock || exit 1
# ... logic ...
```

B9 — Easy: Single quotes `'...'` disable expansion. Double quotes `"..."` allow expansion. No quotes allow globbing and word splitting. Example: `echo $HOME` (no quotes) expands. `echo '$HOME'` prints literal. `echo $HOME` (no quotes) splits on spaces.

B10 — Easy: `[ -f file ]` (file), `[ -d dir ]` (directory), `[ -r file ]` (readable). `[[ ]]` is a keyword (bash builtin), supports `&&`, `||`, regex (`=~`). `[ ]` is a command, POSIX compliant.

B11 — Easy: `$?` exit status of last command. `$$` PID of current shell. `$!` PID of last background job. `$#` number of positional parameters.

B12 — Easy: `foo() { ... }` or `function foo { ... }`. Return values via `return` (exit code) or `echo` (stdout). Functions inherit environment variables.

B13 — Easy: `PATH` is colon-separated directories. Bash searches PATH for external commands. Builtins are internal (e.g., `cd`). `type` or `which` checks command type.

B14 — Medium: `trap 'cmd' EXIT` runs on exit. `trap 'cmd' INT` on Ctrl+C. `trap 'cmd' ERR` on error. Execution order: Handlers run in order of definition. Signals queue; multiple signals may coalesce.

B15 — Medium: `xargs` builds command lines from stdin. `command substitution` runs command immediately. `xargs -0` handles null-delimited input (from `find -print0`). `xargs -P` runs jobs in parallel.

B16 — Medium: Arrays: `arr=(1 2)`. Associative: `declare -A map`. Append: `arr+=(3)`. Length: `${#arr[@]}`. Syntax differs for access (`arr[0]` vs `map[key]`).

B17 — Hard: Subshell `(...)` runs in new process (variables lost). Group `{ ...; }` runs in current shell. Piping into `while` creates subshell, so variable changes inside loop are lost after loop.

B18 — Hard: SIGINT (Ctrl+C), SIGTERM (kill), SIGHUP (hangup), SIGKILL (force kill, no handler). Graceful shutdown: Trap SIGTERM, cleanup, then exit. Background jobs need `wait` or `disown`.

B19 — Hard: Split log `split -l 1000`. `xargs -P 4 -I {} sh -c 'process {}'`. Merge results. Trade-off: Parallelism vs I/O contention. Error propagation: Check exit codes of parallel jobs, aggregate failures.

B20 — Hard: `exec` replaces shell process or redirects FDs. `exec 3>file` opens FD 3. `exec 3>&-` closes it. Coprocess (`coproc`) creates bidirectional pipe. Use for persistent communication with external tools.

PS1 — Easy: `Write-Output` sends objects to pipeline. `Write-Host` writes to console (bypasses pipeline). Distinction matters because `Write-Host` output cannot be captured or piped.

PS2 — Easy: Pipeline passes objects between cmdlets. `ForEach-Object` is a cmdlet (pipeline-aware, processes each item). `foreach` is a statement (iterates collection in memory). Use `ForEach-Object` for streaming; `foreach` for in-memory collections.

PS3 — Medium: `[PSCustomObject]` creates a strongly typed object with properties. `[hashtable]` is a dictionary. Use CustomObject for structured data/pipeline. Convert: `[PSCustomObject]@{Key='Value'}` or `.GetEnumerator()` for hashtable.

PS4 — Medium: Terminating errors stop execution (throw). Non-terminating continue. `-ErrorAction Stop` converts non-terminating to terminating. `$ErrorActionPreference` sets default. `try/catch` handles terminating errors.

PS5 — Medium: Splatting `@params` passes hashtable elements as arguments. `Command -Param1 "val"` passes hashtable as single param. Splatting expands arguments.

PS6 — Medium: `Invoke-Command -ComputerName` runs one-off. `Enter-PSSession` interactive. Uses WS-Management (WinRM) or SSH (v7+).

PS7 — Hard: Returns array if multiple items, scalar if one. If 0 items, returns `$null`. `$x.GetType().Name` is `String` (if one item) or `Object[]`. Correct pattern: `$result = @(); $result += ...; return $result`.

PS8 — Medium: `[scriptblock]` is code object. Functions are named scriptblocks. Pass via `-ArgumentList`. Double-hop: CredSSP allows auth delegation. `Invoke-Command -ArgumentList` serializes variables.

PS9 — Easy: `@()` forces array. `[array]` casts. Indexing `$null` returns `$null`. Indexing empty array returns `$null`.

PS10 — Easy: `-eq` (equal), `-like` (glob), `-match` (regex). `-ceq` (case-sensitive). `-contains` checks if collection contains value. `-in` checks if value is in collection (newer, cleaner).

PS11 — Easy: Module: Reusable code. `.psm1` script, `.psd1` manifest, binary (DLL). `Import-Module` loads it. Manifest defines metadata.

PS12 — Easy: `"Hello $var"` interpolates. `'...'` literal. `@"...`@` multi-line. Expressions: `$var`, `$var + 1`.

PS13 — Easy: `Get-Member` lists members. Properties (data), Methods (functions), NoteProperties (added dynamically). `Select-Object` chooses properties. `Where-Object` filters.

PS14 — Medium: Classes (v5) support inheritance, constructors. Limitations: No interfaces, limited polymorphism. Use classes for complex state; functions for simple logic.

PS15 — Medium: `[CmdletBinding()]` enables common parameters (`-Verbose`, `-WhatIf`). `SupportsShouldProcess` enables `-Confirm`. `Begin` (init), `Process` (pipeline), `End` (cleanup).

PS16 — Medium: Providers expose data as file system. `Get-ChildItem` works on Registry (`HKLM:\`). `cd HKLM:\` navigates registry like folders.

PS17 — Hard: Runspaces manage threads. `[runspacepool]` manages pool. `ForEach-Object -Parallel` (v7+) uses pool. Manual runspace: `[PowerShell]::Create()`. Thread safety: Avoid shared mutable state.

PS18 — Medium: PSScriptAnalyzer enforces standards. Rules have severity (Error, Warning, Info). Suppress via `#pragma`. Integrate via CI (e.g., GitHub Actions).

PS19 — Hard: `$using:` passes local variable to remote block. `-ArgumentList` serializes. Limits: CLIXML depth (1000+). Complex objects may fail serialization.

PS20 — Hard: DSC configures state. Resources define actions. LCM applies config. PowerShell 7+ uses `dsc` CLI. Modern alternative: Ansible, Terraform, or native `Invoke-DscResource`.
