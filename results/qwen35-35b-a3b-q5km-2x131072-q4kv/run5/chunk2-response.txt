K1 — A **Deployment** manages stateless workloads where pods are interchangeable and can be replaced freely. A **StatefulSet** manages stateful workloads, providing stable, unique network identities (pod names), ordered deployment/termination, and persistent storage guarantees per pod.

K2 — A **ClusterIP** Service exposes a stable internal IP. **kube-proxy** (using iptables or IPVS) on each node intercepts traffic to the Service IP and redirects it to the backend Pod IPs based on the Service's Endpoints list.

K3 — **PersistentVolume (PV)** is a cluster-wide storage resource. **PersistentVolumeClaim (PVC)** is a user request for storage. **StorageClass** defines the provisioning mechanism (dynamic). A PVC binds to a PV (static) or triggers a PV creation via a StorageClass (dynamic).

K4 — 1. Check container logs (`kubectl logs`). 2. Check pod events (`kubectl describe pod`) for OOMKilled or probe failures. 3. Verify resource limits vs. actual usage. 4. Check application configuration or dependencies (e.g., database connectivity).

K5 — CoreDNS provides DNS resolution for internal Kubernetes services (Service discovery). If CoreDNS pods are down, internal service discovery fails (e.g., `my-service.default.svc`), though external DNS resolution may still work depending on the cluster's `resolv.conf`.

K6 — The scheduler uses **Filtering** (remove ineligible nodes) and **Scoring** (rank eligible nodes). Factors: 1. Resource requests (CPU/Mem) vs. node capacity. 2. Node Affinity/Taints. 3. Pod Affinity/Anti-Affinity. 4. Topology spread constraints.

K7 — Yes, it will be scheduled (3.5GB + 256Mi < 4GB). If it uses 600Mi at runtime (exceeding 512Mi limit), the container will be **OOMKilled** and restart.

K8 — 1. Request hits External LB. 2. LB forwards to Ingress Controller (NGINX). 3. NGINX matches Host/Path rules to a Kubernetes Service. 4. Service resolves to Pod IPs via Endpoints. 5. **kube-proxy** (iptables/IPVS) routes traffic to the Pod.

K9 — A **NetworkPolicy** restricts traffic flow between pods. Default behavior (no policy): All traffic is allowed. When applied: Traffic is denied by default for selected pods unless explicitly allowed by the policy rules.

K10 — **RollingUpdate** replaces pods gradually (zero downtime). **Recreate** kills all existing pods before starting new ones. Choose **Recreate** for stateful applications where concurrent instances are not supported or safe.

K11 — A **Namespace** is a virtual cluster within a physical cluster for resource isolation. Default namespaces: `default`, `kube-system`, `kube-public`, `kube-node-lease`.

K12 — `kubectl describe` shows detailed status. 1. **Events** (warnings/errors). 2. **Conditions** (Ready status details). 3. **Mounts** (Volumes/ConfigMaps). 4. **Resource Limits/Requests**. `kubectl get` only shows Phase, Age, Ready, Restarts, IP.

K13 — A **ConfigMap** stores non-confidential configuration data. Pods consume it via: 1. **Environment Variables**. 2. **Volume Mounts** (files).

K14 — **ClusterIP**: Internal cluster access only. **NodePort**: Exposes service on a static port on every node's IP. **LoadBalancer**: Provisions an external cloud load balancer to access the service.

K15 — A **DaemonSet** ensures a copy of a pod runs on all (or specific) nodes. Use for system-level agents (logging, monitoring, CNI) rather than application workloads managed by Deployments.

K16 — **Taints** repel pods (node property); **Tolerations** allow pods to schedule on tainted nodes. **Node Affinity** attracts pods to nodes. Use Taints for dedicated nodes (e.g., GPU); use Affinity for co-location (e.g., app + cache).

K17 — An **Operator** is a custom controller that manages complex stateful applications using CRDs. It differs from standard controllers by encapsulating operational knowledge (lifecycle, backup, scaling) to automate management tasks.

K18 — etcd uses **Strong Consistency** (linearizable). If quorum is lost, the cluster becomes read-only/unavailable. Recovery: Restore the remaining node from a snapshot, rejoin lost nodes, or add new nodes and restore etcd data to re-establish quorum.

K19 — **Role** (namespace-scoped permissions), **ClusterRole** (cluster-wide), **RoleBinding** (links Role to subject in namespace), **ClusterRoleBinding** (links ClusterRole to subject cluster-wide).
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata: namespace: my-ns
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list", "watch"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata: namespace: my-ns
subjects:
- kind: User
  name: developer
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io
```

K20 — `ndots` defines how many dots trigger an FQDN search. Default 5 causes queries for `host` + all search suffixes. Fix: Set `dnsConfig` in pod spec with `ndots: 1` to reduce queries for external domains.

D1 — **Rebase** rewrites history to create a linear commit chain. **Merge** preserves history with a merge commit. Avoid rebasing on shared/public branches to prevent history conflicts for collaborators.

D2 — **TCP** is connection-oriented and reliable (HTTP, SSH). **UDP** is connectionless and unreliable (DNS, VoIP).

D3 — A **race condition** occurs when output depends on the sequence/timing of uncontrollable events. Example: Two threads incrementing a shared integer `count++` without a lock, resulting in lost updates.

D4 — **Symmetric** uses one key for encryption/decryption. **Asymmetric** uses a public/private key pair. TLS uses Asymmetric for the handshake (key exchange) and Symmetric for the data session (performance).

D5 — A **layer** is a filesystem diff in a Docker image. Layer caching speeds builds: place frequently changing instructions (e.g., `COPY .`) after stable ones (e.g., `COPY package.json`) to reuse cached layers.

D6 — 1. Check application logs for errors. 2. Verify upstream service health. 3. Check resource limits (CPU throttling/OOM). 4. Review load balancer health check logs.

D7 — **CAP**: Consistency, Availability, Partition Tolerance. **CP** (ZooKeeper, HBase): Prioritizes data consistency over availability during partitions. **AP** (Cassandra, DynamoDB): Prioritizes availability, accepting eventual consistency.

D8 — **Horizontal Scaling** adds more nodes; **Vertical Scaling** adds more resources to a node. Patterns enabling horizontal scaling: Stateless design, Load Balancing, Sharding.

D9 — An **environment variable** is dynamic configuration passed to processes. Persist across sessions on Linux by exporting in `~/.bashrc` or `~/.profile`.

D10 — **JSON** is strict key-value syntax (machine-friendly). **YAML** is human-readable with indentation. JSON advantage: Strict parsing. YAML advantage: Concise configuration files.

D11 — **REST** is an architectural style for APIs. **GET** (retrieve), **POST** (create), **PUT** (replace), **DELETE** (remove).

D12 — **Version pinning** specifies exact dependency versions (e.g., `1.0.1` vs `^1.0.0`). Important for reproducible builds to prevent unexpected breakage from upstream updates.

D13 — **Compiled** (C, Go) translates code to binary before execution. **Interpreted** (Python, JS) executes code line-by-line at runtime.

D14 — **Reverse Proxy** (Nginx, HAProxy) sits in front of servers to handle client requests. **Forward Proxy** sits in front of clients. Use Nginx for web serving/caching; HAProxy for high-performance load balancing.

D15 — A **deadlock** is a permanent blocking of processes waiting for each other. Coffman Conditions: 1. Mutual Exclusion, 2. Hold and Wait, 3. No Preemption, 4. Circular Wait.

D16 — **Eventual Consistency** ensures updates propagate over time. Cassandra uses **Tunable Consistency** (e.g., ONE, QUORUM, ALL) to balance latency and consistency per read/write request.

D17 — **mTLS** authenticates both client and server via certificates. Handshake: Client Hello, Server Hello, Cert exchange, Verify, Key exchange. Used in **Service Meshes** (e.g., Istio) for secure service-to-service communication.

D18 — A **Hash Table** maps keys to indices. **Collision Resolution**: Chaining (linked lists) or Open Addressing (probing). Time Complexity: Avg O(1), Worst O(n) (all collisions).

D19 — **Circuit Breaker** prevents cascade failures. **Closed** (normal), **Open** (fail fast), **Half-Open** (test recovery). Relates to retries: Do not retry if circuit is Open; use exponential backoff when transitioning to Half-Open.

D20 — **Optimistic** (check version on write) assumes low conflict. **Pessimistic** (lock on read) assumes high conflict. Example: Optimistic uses `version` column; Pessimistic uses `SELECT FOR UPDATE`.
