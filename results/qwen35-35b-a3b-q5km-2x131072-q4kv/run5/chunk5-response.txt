DN1 — **Class vs Struct:** `class` is a reference type stored on the heap, supports inheritance, and is mutable by default. `struct` is a value type stored on the stack (or inline), does not support inheritance, and is immutable by convention. **Use a struct** for small, immutable data types (e.g., `Point`, `Color`) where value semantics are required to avoid reference copying overhead.

DN2 — **IEnumerable<T> vs IQueryable<T>:** `IEnumerable<T>` executes queries in-memory (client-side), while `IQueryable<T>` builds expression trees that are translated to server-side queries (e.g., SQL). **Matters for DB:** Using `IQueryable` pushes filtering/sorting to the database, reducing data transfer and improving performance. `IEnumerable` loads all data first, which can cause N+1 issues or memory bloat.

DN3 — **Async/Await:** Compiles to a state machine. `Task` is a reference type object representing an async operation. `ValueTask` is a struct that avoids allocation if the result is already available. **Use ValueTask** for async methods returning value types in hot paths to reduce GC pressure, but avoid it for general use due to complexity.

DN4 — **Dependency Injection (DI):** Pattern for decoupling dependencies. **Lifetimes:** `Transient` (new instance every request), `Scoped` (new per request scope), `Singleton` (one instance for app lifetime). **Bug:** Injecting a `Scoped` service into a `Singleton` causes the Singleton to capture the Scoped service from the first request, leading to stale data or `ObjectDisposedException` when the scope ends.

DN5 — **String vs ReadOnlySpan<char>:** `string` is immutable and stored on the GC heap. `ReadOnlySpan<char>` is a stack-based view over memory with no allocation. **Use Span<T>** for high-performance string processing (parsing, slicing) to avoid creating intermediate string objects and reduce GC pressure.

DN6 — **Garbage Collector (GC):** Manages heap memory. **Generations:** Gen 0 (short-lived), Gen 1 (medium), Gen 2 (long-lived). **LOH:** Large Object Heap (>85KB) for objects too big for standard heap. **IDisposable:** Required for unmanaged resources (files, DB connections) because the GC only manages managed memory; `Dispose` ensures timely cleanup.

DN7 — **ConfigureAwait(false):** Prevents capturing the `SynchronizationContext`. **Use** in library/server code to avoid deadlocks and improve performance. **Do not use** in UI code (WPF/WinForms) where you need to marshal back to the UI thread. Default behavior captures context, which can cause context-switching overhead.

DN8 — **.NET 8/9 Features:** **Primary Constructors** (C#12) allow passing args to base/ctor directly. **Collection Expressions** (C#12) simplify array/list creation (`[1, 2, 3]`). **FrozenDictionary** (C#12) is an immutable, high-performance hash map. **Example:** `public class Service(int id) : IService { }`.

DN9 — **DbContext Exception:** **Cause:** `DbContext` is not thread-safe. High concurrency + `Singleton` lifetime = shared instance causing race conditions/disposes. **Fix:** Change `DbContext` lifetime to `Scoped` (per HTTP request). EF Core expects one context per logical unit of work.

DN10 — **Record vs Class:** `record` provides value-based equality and immutability by default. `class` provides reference equality. **Record struct** combines value type semantics with record equality. **Choose record** for DTOs and immutable models; **choose class** for mutable entities or when inheritance is required.

DN11 — **ref, out, in:** `ref` passes by reference (must be initialized before call, can modify). `out` passes by reference (must be assigned inside method). `in` passes by reference (read-only, no copy). **Use `ref`** for modifying large structs, `out` for multiple return values, `in` for large read-only structs.

DN12 — **Nullable Reference Types (NRT):** `?` annotation warns about potential null dereferences on reference types at compile time. **Nullable value types** (`int?`) are actual types (`Nullable<int>`) allowing `null` values. NRT helps prevent `NullReferenceException` without runtime overhead.

DN13 — **LINQ:** Language Integrated Query. **Method vs Query:** Method (`Where`) vs Query (`from...where`); compiled to same IL. **Deferred Execution:** Query logic isn't run until enumeration (e.g., `ToList()`). Prevents unnecessary computation if results aren't used.

DN14 — **Generic Constraints:** `where T : class` (reference), `where T : struct` (value), `where T : new()` (parameterless ctor), `where T : IComparable<T>` (interface). **Combine:** `where T : class, IComparable<T>` to enforce multiple requirements on the generic type parameter.

DN15 — **Abstract Class vs Interface:** `abstract` allows code implementation + contract. `interface` defines contract only. **Default Interface Methods (C#8+):** Allow adding behavior to interfaces without breaking existing implementations, blurring the line with abstract classes.

DN16 — **Middleware Pipeline:** Request processing chain. `app.Use()` adds middleware, `app.Map()` branches for specific paths, `app.Run()` is terminal. **Ordering:** Auth/Logging before MVC. **Exception Handling:** Middleware wrapping the pipeline catches exceptions from downstream components.

DN17 — **Channel<T> vs BlockingCollection<T>:** `Channel<T>` is async-first (TPL). `BlockingCollection<T>` is sync-first. **Bounded Channels:** Set max capacity to apply backpressure, preventing producers from overwhelming consumers and causing OOM.

DN18 — **Source Generators:** Compile-time code generation. **Vs Reflection:** Faster, no runtime overhead. **Vs IL Weaving:** Happens during compilation, not post-build. **Incremental Generators:** Preferred for performance; they only regenerate code when source changes, unlike `ISourceGenerator` which runs every build.

DN19 — **TPL Dataflow:** Pipeline processing library. `ActionBlock` (process), `TransformBlock` (map), `BufferBlock` (queue). **Vs Channels:** Dataflow offers more flow control (backpressure, completion signaling, cancellation), while Channels are simpler async queues.

DN20 — **Minimal APIs:** Delegate-based endpoints without controllers. **Trade-offs:** Less boilerplate, but harder to scale complex auth/middleware. **Route Groups:** Organize endpoints by feature. **Endpoint Filters:** Apply logic (e.g., validation) to specific endpoints. OpenAPI support via Swashbuckle.

P1 — **List vs Tuple:** `List` is mutable, ordered collection. `Tuple` is immutable, ordered collection. **Set vs Frozenset:** `Set` is mutable, unique elements. `Frozenset` is immutable, hashable (can be used as dict key).

P2 — **GIL:** Global Interpreter Lock allows only one thread to execute Python bytecode at a time. **Affects:** CPU-bound workloads. **Workarounds:** Multiprocessing (separate processes), C-extensions (release GIL), or GIL-free interpreters (PyPy, Jython).

P3 — **Deepcopy vs Copy:** `copy` creates a shallow copy (new container, same objects). `deepcopy` creates a new container and recursively copies objects. **Example:** `list = [[1]]`. `copy` results in nested list sharing the inner list; modifying inner list affects both.

P4 — **Decorators:** Functions that wrap other functions. **Time Logger:**
```python
import time
def timeit(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 — **Generators:** Functions that yield values lazily. **Vs List:** No memory allocation for all items. **Yield from:** Delegates to another generator, simplifying nested iteration logic.

P6 — **Asyncio:** Single-threaded event loop. `asyncio.run()` starts loop. `asyncio.gather()` waits for multiple tasks. `asyncio.create_task()` schedules task. **time.sleep()** blocks the loop; use `asyncio.sleep()` to yield control.

P7 — **MRO:** Method Resolution Order uses C3 linearization. **Diamond Inheritance:** `D(B, C)` resolves to `D -> B -> C -> A`. `D().who()` prints "C" because `C` overrides `A` and appears later in the MRO than `B`.

P8 — **Metaclass:** Class of a class. `__init_subclass__` is a subclass hook (easier, no metaclass needed). **Use Metaclass** for enforcing structure (e.g., ORM field registration). **Use `__init_subclass__`** for configuration hooks on subclass creation.

P9 — **Multiprocessing vs Threading:** Multiprocessing bypasses GIL (CPU-bound), uses pickling (serialization overhead). Threading shares memory (I/O-bound), blocked by GIL. **Concurrent.Futures:** High-level API for both. Use Multiprocessing for CPU, Threading for I/O.

P10 — **__slots__:** Restricts attributes to a fixed set. **Prevents:** Dynamic attribute creation (`__dict__` removed). **Memory:** Significant savings for many instances (no per-instance dict).

P11 — **Virtual Environment:** Isolated Python environment. **Why:** Dependency management. **venv:** Standard library tool. **virtualenv:** Third-party, older. **conda:** Package manager + env, handles non-Python deps.

P12 — **Comprehensions:** Concise loop syntax (`[x for x in list]`). **Vs Map/Filter:** Comprehensions are more Pythonic/readable for simple logic. Prefer comprehensions for list/dict/set creation.

P13 — **== vs is:** `==` compares values. `is` compares identity (memory address). **Interning:** Small integers (-5 to 256) are cached. `a is b` returns True for small ints due to interning, not value equality.

P14 — **args/kwargs:** `*args` (positional tuple), `**kwargs` (keyword dict). **Markers:** `/` forces positional-only, `*` forces keyword-only. Example: `def f(a, /, b, *, c):`.

P15 — **Context Managers:** Manage resources (`with` statement). **__enter__/__exit__:** Class-based. **@contextmanager:** Decorator-based (yield). **Example:** Acquiring a lock or DB connection.

P16 — **Type Hinting:** Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. **mypy/pyright:** Analyze code without running to catch type errors before runtime.

P17 — **Dunder Methods:** Special methods. `__repr__` (dev), `__str__` (user). `__eq__` vs `__hash__`: If `__eq__` is defined, `__hash__` becomes `None` (unhashable) unless explicitly defined.

P18 — **Descriptors:** Objects implementing `__get__`, `__set__`, `__delete__`. **property/classmethod:** Built-in descriptors. Used to control attribute access, validation, and binding.

P19 — **Import System:** `sys.path` determines search paths. **Relative:** `from . import mod`. **Absolute:** `from pkg import mod`. **Namespace Packages:** (PEP 420) allow splitting packages across directories without `__init__.py`. **Finder/Loader:** Locate and load modules.

P20 — **Dataclasses:** Auto-generates `__init__`, `__repr__`, etc. **Vs attrs/Pydantic:** attrs (more features), Pydantic (validation). **Pydantic:** Choose for API models requiring validation and serialization. **__post_init__:** Runs after init. **Frozen:** Immutable instances.
