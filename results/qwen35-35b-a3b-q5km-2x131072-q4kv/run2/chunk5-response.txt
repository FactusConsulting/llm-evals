### .NET / C# (20 questions)

**DN1**
**Class** is a reference type stored on the heap; it supports inheritance and polymorphism. **Struct** is a value type stored on the stack (or inline on heap); it does not support inheritance (except from `System.ValueType`). Use a **struct** for small, immutable data objects (e.g., `Point`, `Vector`) where value semantics are required to avoid reference sharing overhead.

**DN2**
`IEnumerable<T>` represents an in-memory collection with deferred execution (client-side). `IQueryable<T>` represents a queryable data source (e.g., database) using expression trees (server-side). This distinction matters for DB queries because `IQueryable` translates LINQ to SQL (executed on the server), whereas `IEnumerable` loads data into memory first, then filters, causing performance issues.

**DN3**
`async/await` uses a state machine to suspend execution without blocking threads. A **Task** is a reference type representing an async operation. A **ValueTask** is a struct that avoids heap allocation if the operation completes synchronously. Use `ValueTask` in hot paths where the async operation often completes synchronously to reduce GC pressure.

**DN4**
Dependency Injection (DI) decouples object creation from usage. **Transient**: New instance every time requested. **Scoped**: New instance per request (e.g., HTTP request). **Singleton**: One instance for the app lifetime. **Bug**: Injecting a Scoped service into a Singleton captures the Scoped instance at Singleton creation time, leading to stale data or `ObjectDisposedException` when the scope ends.

**DN5**
`string` is immutable and managed by the GC. `ReadOnlySpan<char>` is a stack-only view into memory (no GC allocation). Use `Span<T>` for high-performance string processing (parsing, slicing) to avoid creating intermediate string objects and reduce memory pressure.

**DN6**
The GC manages memory by reclaiming unreachable objects. **Gen 0**: Short-lived objects. **Gen 1**: Transient. **Gen 2**: Long-lived. **Large Object Heap (LOH)**: Objects > 85KB, collected less frequently. `IDisposable` is needed for unmanaged resources (files, DB connections) because the GC only manages managed memory; `Dispose` ensures timely cleanup.

**DN7**
`ConfigureAwait(false)` prevents capturing the current `SynchronizationContext`. Use it in library code to avoid deadlocks and improve performance. Do **not** use it in UI code (WPF/WinForms) where UI updates must occur on the UI thread. Default behavior captures the context, which can cause context-switching overhead or deadlocks in console apps.

**DN8**
**Primary Constructors**: Define constructor args in class declaration. **Collection Expressions**: `[1, 2, 3]` syntax. **FrozenDictionary**: Immutable, high-performance dictionary.
Example:
```csharp
public class Service(int port) : IService { }
// DI Registration
services.AddScoped<Service>(); // Port injected via DI or default
```

**DN9**
**Cause**: Likely using a `DbContext` as a Singleton or capturing a Scoped `DbContext` in a background task. `DbContext` is not thread-safe. **Fix**: Register `DbContext` as **Scoped** in DI. Ensure it is disposed at the end of the request scope.

**DN10**
**Record**: Immutable by default, value-based equality (`==`), positional properties. **Class**: Mutable, reference equality. **Record Struct**: Value type with record semantics. Use **record** for DTOs/Models (value semantics). Use **class** for mutable entities or when inheritance is needed. Use **record struct** for high-performance value objects.

**DN11**
`ref`: Pass by reference, allows read/write. `out`: Pass by reference, must be assigned inside method. `in`: Pass by reference, read-only (avoids copy of large structs). Use `ref` for modifying args, `out` for multiple return values, `in` for large read-only structs.

**DN12**
**Nullable Reference Types (NRT)**: Compiler warnings for potential nulls on reference types (`string?`). `#nullable enable` activates this. **Nullable Value Types**: `int?` allows `null` for value types. NRTs are compile-time checks; value type nullables are runtime values.

**DN13**
**LINQ**: Language Integrated Query. **Method Syntax**: `list.Where(...)`. **Query Syntax**: `from x in list where ...`. Both compile to the same IL. **Deferred Execution**: Query logic runs only when enumerated (e.g., `ToList()`), not when defined.

**DN14**
**Generic Constraints**: `where T : class` (reference), `where T : struct` (value), `where T : new()` (parameterless ctor), `where T : IComparable<T>` (interface). Combine constraints with commas: `where T : class, IDisposable`. Use to restrict types to ensure specific capabilities.

**DN15**
**Abstract Class**: Can contain implementation and state. **Interface**: Contract only (pre-C# 8). **Default Interface Methods** (C# 8+): Allow adding methods to interfaces without breaking implementations, blurring the line between interface and abstract class.

**DN16**
**Middleware Pipeline**: HTTP request passes through components. `app.Use()`: Adds middleware (can call next). `app.Map()`: Branches pipeline for specific paths. `app.Run()`: Terminal middleware (no next). **Order**: Critical (e.g., Auth before Routing). **Exception Handling**: Middleware can catch exceptions and return responses.

**DN17**
`Channel<T>`: Async producer-consumer queue. `BlockingCollection<T>`: Sync wrapper over `ConcurrentQueue`. **Bounded Channel**: Limits capacity. If full, producers wait (backpressure). Prevents memory exhaustion under high load.

**DN18**
**Source Generators**: Compile-time code generation (IL). Faster than reflection, no runtime weaving. **Incremental Generator**: Caches results, regenerates only on input change (faster builds). Preferred over `ISourceGenerator` (full re-run) for performance in large solutions.

**DN19**
**TPL Dataflow**: Pipeline blocks. `ActionBlock` (process, no output), `TransformBlock` (process + output), `BufferBlock` (queue). **Vs Channels**: Channels are simpler (one producer/consumer). Dataflow is better for complex pipelines with fan-out/fan-in and backpressure handling.

**DN20**
**Minimal APIs**: Delegate-based endpoints, no controllers. **Trade-offs**: Less boilerplate, harder to scale complex auth/middleware, OpenAPI support via Swashbuckle. **Route Groups**: Organize endpoints (`app.MapGroup`). **Endpoint Filters**: Apply logic (e.g., validation) to specific endpoints.

### Python (20 questions)

**P1**
**List**: Mutable, ordered collection. **Tuple**: Immutable, ordered collection. **Set**: Mutable, unordered, unique elements. **Frozenset**: Immutable set. Use tuples for fixed data, sets for uniqueness.

**P2**
**GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects **CPU-bound** workloads. **Workarounds**: Multiprocessing (separate processes), C-extensions (release GIL), or PyPy/Jython.

**P3**
**Deepcopy**: Recursive copy of object and all nested objects. **Copy**: Shallow copy (new object, references to nested objects). **Bug**: `list1 = [1, [2, 3]]`, `list2 = copy(list1)`. Modifying `list2[1]` also modifies `list1[1]`.

**P4**
**Decorators**: Functions that wrap other functions.
```python
import time
def log_time(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

**P5**
**Generators**: Functions using `yield` to produce items lazily. **Vs List**: Generators save memory (no full list in RAM). `yield from`: Delegates to another generator.

**P6**
**Event Loop**: Manages async tasks. `asyncio.run()`: Starts loop. `asyncio.gather()`: Runs tasks concurrently, waits for all. `asyncio.create_task()`: Schedules task immediately. `time.sleep()` blocks the loop; use `await asyncio.sleep()`.

**P7**
**MRO**: Method Resolution Order. Python 3 uses **C3 Linearization**. **Diamond Inheritance**: Resolved by order. In `D(B, C)`, MRO is `D -> B -> C -> A`. `D().who()` calls `C.who()` ("C") because `C` appears before `A` in MRO.

**P8**
**Metaclass**: Class of a class (controls class creation). `__init_subclass__`: Hook for subclass initialization (easier, no metaclass needed). **Use Case**: Metaclass for ORM registration; `__init_subclass__` for config validation on subclass creation.

**P9**
**Multiprocessing**: CPU-bound, separate memory (no GIL). **Threading**: I/O-bound, shared memory (GIL). **Concurrent.Futures**: High-level API for both. **Serialization**: Multiprocessing requires pickling args; complex objects may fail.

**P10**
`__slots__`: Prevents creation of `__dict__` for instances. **Use**: Save memory, prevent dynamic attributes. **Impact**: Reduces memory per instance significantly but prevents adding new attributes at runtime.

**P11**
**Virtual Env**: Isolated Python environment. **Why**: Dependency management. **venv**: Standard library tool. **virtualenv**: Third-party, older. **conda**: Package manager + env, handles non-Python deps.

**P12**
**Comprehensions**: Concise loops (`[x for x in list]`). **Map/Filter**: Functional style. **Prefer**: Comprehensions for readability and speed in Python. Use `map/filter` for lazy evaluation or complex logic.

**P13**
`==`: Value equality. `is`: Identity equality. **Interning**: Small integers (-5 to 256) are cached. `a is b` returns True for small ints because they point to the same object.

**P14**
`*args`: Tuple of positional args. `**kwargs`: Dict of keyword args. `/`: Positional-only args before. `*`: Keyword-only args after. Example: `def f(a, /, b, *, c):`.

**P15**
**Context Managers**: `with` statement for resource management. `__enter__`/`__exit__`: Manual implementation. `@contextmanager`: Decorator for generator-based. **Example**: Acquiring a lock or database connection.

**P16**
**Type Hinting**: Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. **mypy/pyright**: Analyze code without running to catch type errors. `Protocol` enables structural typing (duck typing).

**P17**
**Dunder Methods**: Special methods. `__repr__` (dev), `__str__` (user). `__eq__` vs `__hash__`: If `__eq__` is defined, `__hash__` becomes `None` (unhashable) unless explicitly defined. `__getattr__` (missing attr), `__getattribute__` (all attr).

**P18**
**Descriptors**: Objects implementing `__get__`, `__set__`, `__delete__`. **property**: Uses descriptor for getter/setter. **classmethod/staticmethod**: Implemented via descriptors to bind/unbind methods.

**P19**
**Import System**: **Finder** locates module, **Loader** loads it. **Relative**: From package (`from . import x`). **Absolute**: From root (`import x`). `sys.path`: Search paths. `__init__.py`: Marks package. **Namespace Packages**: No `__init__.py`, split across paths.

**P20**
**Dataclasses**: Auto-generates `__init__`, `__repr__`, etc. **attrs**: More features (validators). **Pydantic**: Validation + serialization. `__post_init__`: Logic after init. **Frozen**: Immutable. **Choose Pydantic**: When validation (types, constraints) is required.
