G1 — **Difference:** An array is a fixed-size value type `[T]N`. A slice is a dynamic view over an array with metadata (pointer, length, capacity) `[]T`. **Append:** When appending to a slice at capacity, Go allocates a new, larger backing array (usually doubling capacity), copies elements, and updates the slice header.

G2 — **Difference:** `var x int` declares a variable with the zero value (`0`) and explicit type. `x := 0` declares and initializes, inferring the type. **Cannot use `:=`:** When redeclaring a variable in the same scope (e.g., `x := 1` after `x := 0`), or when the type must be explicit (e.g., `var x = nil` is invalid, `var x *int` is needed).

G3 — **Goroutine:** A lightweight thread managed by the Go runtime, not the OS. **Scheduler (GMP):** Maps Goroutines (G) to OS Threads (M) via Processors (P). The scheduler handles work-stealing and preemption to maximize CPU utilization without OS context switch overhead.

G4 — **Channels:** Typed conduits for communication. **Buffered:** Has capacity; send blocks only if full. **Unbuffered:** Blocks until both sender and receiver are ready. **Full Buffer:** Sending to a full buffered channel blocks the sender until a receiver reads from it.

G5 — **Usage:** Propagates cancellation, deadlines, and request-scoped values. **Example:**
```go
ctx, cancel := context.WithTimeout(context.Background(), 1*time.Second)
defer cancel()
resp, err := http.GetContext(ctx, url)
```

G6 — **Model:** Go uses structural typing (duck typing); satisfaction is implicit. Java/C# use nominal typing (explicit declaration). **Empty Interface:** `interface{}` (or `any`) accepts any type. **Danger:** Requires type assertions which can panic if incorrect; loses compile-time type safety.

G7 — **Unsafe:** The loop `for !ready {}` creates a data race. The compiler may cache `ready` in a register, and writes to `data` are not visible to the main goroutine without memory barriers. **Fix:** Use `sync/atomic` for `ready` or `sync.Mutex` to protect `data`.

G8 — **Mutex vs RWMutex:** `Mutex` is for exclusive write access. `RWMutex` allows multiple readers or one writer. **sync.Map:** Optimized for read-heavy workloads with disjoint keys. **Trade-off:** `sync.Map` is slower for writes/updates compared to a `map` + `Mutex` due to internal locking complexity.

G9 — **Modules:** Dependency management system. **go.mod:** Declares module path and dependencies. **go.sum:** Cryptographic checksums of dependencies. **tidy:** Removes unused imports/dependencies. **MVS:** Selects the lowest version of a dependency that satisfies all constraints in the dependency tree.

G10 — **Defer:** Delays function execution until the surrounding function returns. **Order:** LIFO (Last In, First Out). **Gotcha:** In a loop, the closure captures the loop variable by reference. Use a new variable inside the loop or pass the value as an argument to the deferred function.

G11 — **Pointer:** Memory address of a value. **Zero Value:** `nil`. **Struct Literal:** `&MyStruct{Field: value}`.

G12 — **init():** Runs automatically before `main()` starts. **Multiple:** Yes, multiple `init()` functions can exist. **Order:** Executed in declaration order within the package file, then across files in alphabetical order.

G13 — **make:** Allocates and initializes internal structures (map, slice, channel). **new:** Allocates zeroed memory for any type and returns a pointer. **Usage:** Use `make` for slices/maps/channels; `new` for pointers to structs.

G14 — **Handling:** Errors are values. **Interface:** `type error interface { Error() string }`. **Idiom:** `if err != nil { return err }`. **Is/As:** `errors.Is` checks logical equality (wrapping). `errors.As` performs type assertion on wrapped errors. `==` only checks exact pointer/value equality.

G15 — **Types:** Map, Slice, Channel, Struct. **Zero Values:** Map/Channel/Slice are `nil`. Struct fields are their respective zero values (e.g., `0`, `""`, `false`).

G16 — **Race:** Concurrent access to shared data without synchronization. **Flag:** `-race` adds instrumentation to detect races at runtime. **Fix:** Wrap shared access with `sync.Mutex` or use channels for communication.

G17 — **Select:** Multiplexes channel operations. **Switch:** Matches values. **Multiple Ready:** Select chooses one randomly. **Timeout:** Add a case with `time.After(duration)` to `select`.

G18 — **Escape:** Determines if a variable lives on the heap or stack. **Decision:** Based on lifetime (if it escapes the function scope). **Inspect:** `go build -gcflags="-m"`. **Performance:** Heap allocations trigger GC; stack is faster but limited by scope.

G19 — **Reflection:** Runtime introspection. **Type vs Value:** `reflect.Type` is metadata; `reflect.Value` is the actual data container. **Cost:** Slow, no compile-time checks, prevents inlining. **Justified:** Frameworks (serialization, ORM) where code generation is too complex.

G20 — **Embed:** Compiles files into the binary. **Usage:** `//go:embed file.txt` or `//go:embed dir/*`. **FS:** Returns `io/fs.FS` (e.g., `embed.FS`). **Limitations:** Static at compile time; cannot embed files generated at runtime.

R1 — **Ownership:** Resource management system. **Rules:** 1. Each value has exactly one owner. 2. When owner goes out of scope, value is dropped. 3. Can borrow via references (`&T`) but cannot mutate if immutable reference exists.

R2 — **String:** Owned, heap-allocated, mutable. **&str:** Borrowed slice of bytes (immutable). **Signatures:** Use `&str` for function arguments (no allocation). Use `String` for return values requiring ownership or mutation.

R3 — **Borrowing:** Accessing data without taking ownership. **Rules:** 1. Can have many `&T` (immutable) OR one `&mut T` (mutable). 2. References must always be valid. 3. Cannot alias mutable and immutable references simultaneously.

R4 — **Result:** `Ok(T)` or `Err(E)`. **Option:** `Some(T)` or `None`. **? Operator:** Propagates errors up the call stack. **Example:**
```rust
fn read_file() -> Result<String, io::Error> {
    fs::read_to_string("file.txt")
}
```

R5 — **Box:** Heap allocation, single owner. **Rc:** Reference counted, single thread. **Arc:** Atomic reference counted, thread-safe. **Usage:** `Box` for recursive types/heap. `Rc` for shared ownership in single thread. `Arc` for shared ownership across threads.

R6 — **Traits:** Define shared behavior. **Comparison:** Similar to Go interfaces (implicit satisfaction). **Dyn vs Generic:** `dyn Trait` uses dynamic dispatch (vtable, runtime cost). `T: Trait` uses static dispatch (monomorphization, zero cost).

R7 — **Lifetimes:** Ensure references don't outlive data. **Failure:** Compiler doesn't know which input (`x` or `y`) the return value refers to. **Fix:** `fn longest<'a>(x: &'a str, y: &'a str) -> &'a str { ... }`.

R8 — **Send:** Type can be transferred to another thread. **Sync:** Type can be referenced from multiple threads. **Rc:** Not `Send` because ref count is not atomic. **Arc:** Uses atomic ref counting, making it `Send` and `Sync`.

R9 — **Async:** State machines compiled from `async fn`. **Future:** Pollable task. **Executor:** Runs the future (e.g., Tokio). **Difference:** Go uses stackful goroutines (OS threads). Rust uses stackless futures (single thread, cooperative multitasking).

R10 — **Unsafe:** Bypasses safety checks. **Actions:** 1. Dereference raw pointers. 2. Call unsafe functions. 3. Access union fields. **Justified:** FFI, low-level optimization, implementing safe abstractions over unsafe primitives.

R11 — **Matching:** `match` exhaustively handles all patterns. **if let:** Matches a single pattern. **Exhaustiveness:** `match` requires all cases; compiler errors if a case is missing (e.g., `Option` must handle `None`).

R12 — **Vec:** Dynamic array, heap-allocated. **Array:** Fixed size, stack-allocated (usually). **Capacity:** When `Vec` exceeds capacity, it allocates a new larger buffer and copies elements (amortized O(1) push).

R13 — **Enums:** Tagged unions (data + variant). **Difference:** C/Java enums are constants; Rust enums hold data. **Option:** `Some(T)` or `None`. **No Null:** Forces explicit handling of absence, preventing null pointer exceptions.

R14 — **Impl:** Defines methods. **&self:** Borrow immutable. **&mut self:** Borrow mutable. **self:** Consumes the value. **Associated:** Functions not taking `self` (e.g., `String::new()`), act like static methods.

R15 — **Derive:** Macro generating boilerplate. **Traits:** `Debug`, `Clone`, `PartialEq`. **Limitations:** Cannot derive if fields contain types that don't support the trait (e.g., `Fn` trait in `Clone`).

R16 — **Cow:** Clone on Write. **Usage:** Borrows data if possible, clones only on mutation. **Benefit:** Avoids allocation if the data is never mutated (e.g., reading a config file that is only read).

R17 — **Modules:** `mod` defines module, `use` imports, `pub` exposes. **Filesystem:** `mod.rs` or `mod.rs` in directory. **2018:** Introduced `crate::` paths, removed `extern crate`, improved path resolution.

R18 — **Pin:** Prevents moving memory. **Need:** Self-referential structs (e.g., async) cannot move without invalidating internal pointers. **Unpin:** Marker trait indicating type can be moved safely.

R19 — **Macros:** Code generation. **Declarative:** `macro_rules!` (pattern matching). **Procedural:** Rust code generating Rust code (derive, attribute). **Hygiene:** Macros expand into the caller's scope without polluting variable names.

R20 — **Zero-Cost:** Abstractions compile to efficient machine code. **Iterators:** Loops unrolled, no heap alloc. **Generics:** Monomorphization (copy-paste code per type). **Traits:** Static dispatch (inline function calls) vs dynamic dispatch.
