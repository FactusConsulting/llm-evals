N1 — Easy: What is the difference between a Layer 2 switch and a Layer 3 switch?
A Layer 2 switch operates at the Data Link layer (OSI Layer 2), forwarding frames based on MAC addresses within a single broadcast domain. A Layer 3 switch operates at the Network layer (OSI Layer 3), forwarding packets based on IP addresses and performing routing between VLANs or subnets, often with hardware-accelerated routing performance.

N2 — Easy: A host has IP 10.0.5.37/20. What is the network address and broadcast address?
The /20 mask (255.255.240.0) creates blocks of 16 in the third octet. The IP 10.0.5.37 falls within the 10.0.0.0 block.
Network Address: 10.0.0.0
Broadcast Address: 10.0.15.255

N3 — Medium: Explain the TCP three-way handshake. What happens if the final ACK is lost?
The handshake consists of: 1) Client sends SYN, 2) Server responds with SYN-ACK, 3) Client sends ACK. If the final ACK is lost, the server does not transition to the ESTABLISHED state. The server will retransmit the SYN-ACK until the timeout expires, leaving the connection in SYN-RECEIVED state on the server side.

N4 — Medium: What is the difference between SNAT and DNAT? Give a practical example of each.
SNAT (Source NAT) modifies the source IP address of outgoing packets. Example: Masquerading internal private IPs to a public IP for internet access. DNAT (Destination NAT) modifies the destination IP address of incoming packets. Example: Port forwarding external traffic on port 80 to an internal web server at 192.168.1.10.

N5 — Medium: You run `curl -v https://example.com` and get `SSL: certificate subject name does not match target host name`. List 3 possible causes.
1. The certificate's Common Name (CN) or Subject Alternative Name (SAN) does not include the hostname being accessed.
2. The hostname in the `curl` command does not match the domain the certificate was issued for (e.g., accessing IP instead of domain).
3. The client is connecting to a different server than expected (e.g., DNS resolution points to a different IP with a different cert).

N6 — Medium: Explain the difference between a VLAN and a VxLAN. When would you choose VxLAN over VLAN?
VLAN (802.1Q) is a Layer 2 segmentation limited to a single physical broadcast domain (max 4096 IDs). VxLAN is an overlay protocol that encapsulates Layer 2 frames in UDP packets over Layer 3, allowing Layer 2 extension across Layer 3 boundaries. Choose VxLAN for multi-datacenter environments, large-scale cloud deployments requiring >4096 segments, or when physical topology limits VLAN extension.

N7 — Hard: A Linux host has two NICs: eth0 (10.0.0.5/24, default gw 10.0.0.1) and eth1 (192.168.1.5/24). Traffic arriving on eth1 from 192.168.1.0/24 is being replied via eth0. Why, and how do you fix it with policy-based routing?
This is asymmetric routing caused by the default route pointing to eth0. The kernel replies via the default gateway regardless of the incoming interface. Fix: Use Policy-Based Routing (PBR). Create a custom routing table for eth1 traffic: `ip rule add from 192.168.1.0/24 table eth1`, `ip route add default via 192.168.1.1 dev eth1 table eth1`.

N8 — Hard: Explain how BGP path selection works. List at least 5 attributes in the correct order of preference.
BGP selects the best path based on a sequence of attributes. Common order (Cisco default):
1. Weight (Local preference)
2. Local Preference
3. Locally originated routes
4. AS Path Length (Shortest)
5. Origin Type (IGP < EGP < Incomplete)
6. MED (Multi-Exit Discriminator)
7. eBGP over iBGP
8. IGP Cost to Next Hop

N9 — Hard: What is the MTU implication of running VxLAN over an IPsec tunnel? Show the math for the maximum inner payload if the physical MTU is 1500.
VxLAN adds ~50 bytes overhead (20 Outer IP + 8 UDP + 8 VxLAN + 14 Inner Ethernet). IPsec ESP adds ~50 bytes overhead (Header + Padding + Auth). Total overhead ≈ 100 bytes.
Max Inner IP Packet = 1500 - 100 = 1400 bytes.
Max Inner Payload (TCP) = 1400 - 20 (Inner IP) - 20 (Inner TCP) = 1360 bytes.

N10 — Medium: What is the purpose of ARP, and what problem does Gratuitous ARP solve?
ARP (Address Resolution Protocol) maps IP addresses to MAC addresses on a local network. Gratuitous ARP is an unsolicited ARP reply used to update ARP caches on other hosts (e.g., after a failover) and to detect IP address conflicts.

N11 — Easy: What is the difference between a public IP address and a private IP address? List the three private IPv4 ranges defined in RFC 1918.
Public IPs are routable on the internet; private IPs are reserved for internal networks and not routable globally. RFC 1918 ranges:
1. 10.0.0.0/8
2. 172.16.0.0/12
3. 192.168.0.0/16

N12 — Easy: What is DNS, and what is the difference between an A record, a CNAME record, and an AAAA record?
DNS (Domain Name System) resolves domain names to IP addresses.
A Record: Maps a hostname to an IPv4 address.
AAAA Record: Maps a hostname to an IPv6 address.
CNAME Record: Maps a hostname to another hostname (alias).

N13 — Easy: What is DHCP, and what four-step process (DORA) does a client use to obtain an IP address from a DHCP server?
DHCP (Dynamic Host Configuration Protocol) automatically assigns IP configurations to clients. DORA:
1. Discover (Client broadcasts request)
2. Offer (Server offers IP)
3. Request (Client requests the offered IP)
4. Acknowledge (Server confirms lease)

N14 — Easy: What is the difference between a hub, a switch, and a router? At which OSI layer does each primarily operate?
Hub: Layer 1 (Physical), broadcasts all traffic to all ports.
Switch: Layer 2 (Data Link), forwards frames based on MAC addresses.
Router: Layer 3 (Network), forwards packets based on IP addresses between networks.

N15 — Easy: Explain the difference between half-duplex and full-duplex communication. Which mode do modern Ethernet links use?
Half-duplex allows transmission in only one direction at a time (shared medium, collisions). Full-duplex allows simultaneous transmission and reception (separate pairs). Modern Ethernet links operate in full-duplex mode.

N16 — Medium: What is the purpose of the TTL field in an IP packet? How does traceroute exploit it to map the network path?
TTL (Time To Live) prevents packets from looping indefinitely by decrementing at each hop; if it reaches 0, the packet is dropped. Traceroute sends packets with TTL=1, 2, 3... causing routers to send ICMP "Time Exceeded" messages back, revealing the hop IP.

N17 — Medium: Explain what a NAT gateway does. What is the difference between static NAT, dynamic NAT, and PAT (port address translation)?
A NAT gateway translates private IP addresses to public ones.
Static NAT: 1:1 mapping (fixed private to fixed public).
Dynamic NAT: Private IPs mapped to a pool of public IPs.
PAT (NAT Overload): Many private IPs mapped to one public IP using unique source ports.

N18 — Hard: Explain how ECMP (Equal-Cost Multi-Path) routing works. What hashing strategies are commonly used, and what problems can arise with asymmetric flows when stateful firewalls are involved?
ECMP allows load balancing across multiple paths with the same metric. Hashing strategies typically use source/destination IP and port tuples. Asymmetric flows occur if return traffic takes a different path; stateful firewalls may drop packets because the return flow state was established on the forward path, which is not seen on the return path.

N19 — Hard: A TCP connection between two hosts shows high latency but packet captures show no drops. The bandwidth-delay product is 10MB but the TCP window size maxes out at 64KB. Explain the problem and how TCP window scaling (RFC 1323) solves it.
The problem is the TCP window size limit (16-bit field = 65,535 bytes). With a BDP of 10MB, the sender must wait for ACKs before sending more data, severely limiting throughput. TCP Window Scaling (RFC 1323) introduces a scale factor option in the TCP header, allowing the window size to be shifted left (multiplied), supporting windows up to 1GB.

N20 — Hard: Describe how 802.1Q trunk ports work. A server with a single NIC needs to communicate on VLANs 10, 20, and 30. Explain how to configure this on Linux using sub-interfaces and what the frame format looks like on the wire.
802.1Q tags Ethernet frames with a VLAN ID. On Linux, create sub-interfaces: `ip link add link eth0 name eth0.10 type vlan id 10`, etc. On the wire, the frame format is: Ethernet Header (Src/Dst MAC) + 802.1Q Tag (TPID 0x8100 + TCI with VLAN ID) + EtherType + Payload.

L1 — Easy: What is the difference between a hard link and a symbolic link?
A hard link is a direct reference to the inode of a file; both links share the same data and inode number. A symbolic (soft) link is a separate file containing a path to the target file; if the target is deleted, the symlink breaks.

L2 — Easy: How do you find all files larger than 100MB under /var/log?
`find /var/log -type f -size +100M`

L3 — Medium: A process is in state D (uninterruptible sleep). What does this mean, and can you kill it with kill -9?
State D means the process is waiting for I/O (usually disk) and cannot be interrupted. You cannot kill it with `kill -9` (SIGKILL) because the kernel ignores signals until the I/O operation completes.

L4 — Medium: Explain the difference between cgroups v1 and cgroups v2. Which does systemd prefer and why?
Cgroups v1 uses separate hierarchies for different controllers (cpu, memory, etc.). Cgroups v2 uses a unified hierarchy where all controllers share the same tree. Systemd prefers cgroups v2 because it is more efficient, supports new features (e.g., CPU weight), and simplifies management.

L5 — Medium: What is the difference between nftables and iptables? How does the kernel handle both simultaneously?
iptables is the legacy netfilter framework using multiple tables (filter, nat, mangle). nftables is the modern replacement using a single table with sets and maps. In newer kernels, iptables commands are translated to nftables rules internally, allowing both to coexist via compatibility layers.

L6 — Medium: You run `df -h` and see a filesystem at 100%, but `du -sh /*` totals far less. What is the most likely cause?
A file was deleted but is still held open by a running process. The space is not released until the process closes the file handle. Check with `lsof | grep deleted`.

L7 — Hard: Explain the Linux boot process from UEFI firmware to a running systemd target. Cover: firmware, bootloader, kernel, initramfs, root, systemd.
1. UEFI firmware initializes hardware and loads the bootloader.
2. Bootloader (e.g., GRUB) loads the kernel and initramfs into memory.
3. Kernel initializes hardware and mounts the initramfs.
4. Initramfs runs scripts to mount the real root filesystem.
5. Kernel switches to the real root and starts PID 1 (systemd).
6. systemd initializes services and transitions to the target (e.g., multi-user.target).

L8 — Hard: A server has 64GB RAM but only 2GB swap. free -h shows 60GB used, 1GB available, 15GB of that is buff/cache. The OOM killer fires. Explain why available can be low even with high buff/cache, and what kernel tunables affect this behavior.
`available` estimates memory available for new apps without swapping. If low, it means anonymous memory usage is high and cache cannot be reclaimed fast enough to satisfy allocation requests. Tunables: `vm.swappiness` (controls swap usage tendency), `vm.vfs_cache_pressure` (controls inode/dentry cache reclaim), `vm.overcommit_memory` (controls allocation policy).

L9 — Medium: What does `ip netns` do, and how is it used by container runtimes?
`ip netns` creates isolated network namespaces, providing separate network stacks (interfaces, routing tables, iptables). Container runtimes (Docker, Kubernetes) use network namespaces to isolate network traffic for each container.

L10 — Medium: Explain the difference between SIGTERM, SIGKILL, and SIGQUIT. Which can be caught by the process?
SIGTERM (15): Graceful termination request (catchable). SIGKILL (9): Forceful termination (uncatchable, immediate). SIGQUIT (3): Quit and dump core (catchable). SIGTERM and SIGQUIT can be caught; SIGKILL cannot.

L11 — Easy: What is the difference between stdout, stderr, and stdin? How do you redirect stderr to a file while still displaying stdout on the terminal?
stdin (0): Input stream. stdout (1): Standard output. stderr (2): Standard error. Redirect stderr: `command 2> error.log` (stdout remains on terminal).

L12 — Easy: What does the chmod 755 command do? Explain what each digit represents.
Sets permissions to Owner: rwx (7), Group: r-x (5), Others: r-x (5). 7 = read+write+execute, 5 = read+execute.

L13 — Easy: What is the difference between the root user and a user with sudo privileges?
Root (UID 0) has unrestricted access to all system resources. A user with sudo privileges can execute specific commands as root after authentication, following policy defined in `/etc/sudoers`.

L14 — Easy: What is a package manager? Name the default package managers for Debian/Ubuntu and RHEL/Fedora.
A package manager handles software installation, updates, and dependency resolution. Debian/Ubuntu: `apt`. RHEL/Fedora: `dnf` (or `yum` on older versions).

L15 — Easy: What is the /etc/fstab file used for? What happens if an entry is misconfigured?
`/etc/fstab` defines how disk drives and partitions are mounted at boot. If misconfigured, the system may fail to boot or enter emergency mode, as critical filesystems cannot be mounted.

L16 — Medium: Explain what LVM (Logical Volume Manager) is. What are physical volumes, volume groups, and logical volumes, and how do you extend a logical volume on a running system?
LVM abstracts physical storage into logical volumes. PV (Physical Volume): Raw disk/partition. VG (Volume Group): Pool of PVs. LV (Logical Volume): Formatted filesystem. Extend: `lvextend -L +1G /dev/vg/lv` followed by `resize2fs /dev/vg/lv`.

L17 — Hard: Explain how the Linux kernel's I/O scheduler works. Compare mq-deadline, bfq, and none schedulers. When would you choose each, and how do you change the scheduler for a block device at runtime?
Schedulers optimize disk I/O order. mq-deadline: Optimizes latency (good for SSDs). bfq: Optimizes fairness (good for shared systems). none: No scheduling (best for NVMe). Change: `echo scheduler > /sys/block/sdX/queue/scheduler`.

L18 — Hard: A system is experiencing intermittent high load averages but CPU usage is low. Explain what load average actually measures on Linux, why I/O wait can inflate it, and how you would diagnose whether the bottleneck is CPU, disk, or network using standard tools.
Load average measures runnable processes + uninterruptible sleep (I/O wait). High I/O wait inflates load without CPU usage. Diagnose: `vmstat` (check `wa`), `iostat` (check `%util`), `pidstat` (check I/O wait per process).

L19 — Hard: Explain how SELinux enforces mandatory access control. What is the difference between enforcing, permissive, and disabled modes? How do you troubleshoot a daemon that fails to bind a non-standard port due to SELinux policy?
SELinux enforces policies defined by labels (contexts). Enforcing: Blocks violations. Permissive: Logs violations but allows. Disabled: No enforcement. Troubleshoot: Check `/var/log/audit/audit.log`, use `audit2allow` to generate policy, or `semanage port -a -t http_port_t -p tcp 8080`.

L20 — Hard: Describe how the Linux kernel's virtual memory subsystem handles page faults. What is the difference between a minor and a major page fault? How does transparent huge pages (THP) affect application performance, and when would you disable it?
Page faults occur when accessing unmapped memory. Minor: Page in RAM (copy-on-write). Major: Page on disk (read from swap/file). THP uses 2MB pages to reduce TLB misses and improve performance. Disable THP for latency-sensitive apps (e.g., HFT, DB) to avoid allocation latency and fragmentation.
