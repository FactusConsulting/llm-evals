JS1: **var** is function-scoped and hoisted (declaration + initialization to `undefined`). **let** and **const** are block-scoped and hoisted only (declaration exists, but initialization is in the Temporal Dead Zone until the line is executed). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2: The **event loop** manages asynchronous execution. The **call stack** executes synchronous code. When the stack is empty, the loop checks the **microtask queue** (Promises, `queueMicrotask`) first, then the **macrotask queue** (`setTimeout`, `setInterval`, I/O). Microtasks drain completely before the next macrotask.

JS3: A **closure** is a function that retains access to its outer scope variables even after the outer function returns. **Bug:** In a `for` loop with `var`, all closures share the same `i` variable reference, logging the final value. **Fix:** Use `let` (block scope) or an IIFE to capture the current value.

JS4: **Promise** represents an async result. **async/await** is syntactic sugar for Promise chaining. Error handling uses `try/catch` with await or `.catch()` on Promises. **Promise.all** fails fast on any rejection. **Promise.allSettled** waits for all to settle (fulfilled or rejected) and returns status objects.

JS5: **Interface** defines a shape for objects (can be merged, implemented). **Type** defines a data shape (can be primitives, unions, intersections). Use **Interface** for public API contracts/classes. Use **Type** for unions, primitives, or complex compositions. **Discriminated unions** are unions with a literal type property used to narrow types (e.g., `{type: 'a', data: string}`).

JS6: **Prototypal inheritance** links objects via a prototype chain (`__proto__`). **Object.create(proto)** creates a new object with the specified prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` to set up the prototype chain and `constructor` for initialization.

JS7: **this** binding rules (priority): 1. **new** (new instance), 2. **Explicit** (call/apply/bind), 3. **Implicit** (object method), 4. **Default** (global/window or undefined in strict mode). **bind()** returns a new function with bound `this`. It **cannot** be re-bound (subsequent binds are ignored).

JS8: **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** notifies when a weakly referenced object is collected. Use for caches or cleanup. **WeakMap** keys must be objects because primitives are copied by value; objects are referenced, allowing GC to collect keys if no strong references remain.

JS9: V8 optimizes via **hidden classes** (object shapes) and **inline caching** (caching property lookup offsets). **Deoptimization** occurs when object shapes change dynamically or types change unexpectedly. Patterns causing this: adding/removing properties dynamically, polymorphic property access, or `eval`.

JS10: **Tree-shaking** eliminates unused exports. **ES Modules** use static `import/export`, allowing static analysis to remove dead code. **CommonJS** uses dynamic `require`, which cannot be analyzed statically, making tree-shaking ineffective without bundler workarounds.

JS11: **null** is an intentional absence of value. **undefined** is a variable declared but not assigned. **==** coerces types (null == undefined is true). **===** checks strict equality. **??** (nullish coalescing) returns right operand if left is null/undefined. **?.** (optional chaining) safely accesses nested properties.

JS12: **map()** transforms elements. **filter()** selects elements. **reduce()** accumulates values. **Initial value** in reduce prevents using the first array element as the accumulator. **flatMap()** maps then flattens one level; useful for transforming arrays of arrays into a single array.

JS13: **Template literals** use backticks for multi-line strings and interpolation (`${}`). **Tagged templates** call a function with the string parts and interpolation values. Example: `html` tag function to escape HTML strings before rendering to prevent XSS.

JS14: **Destructuring** extracts values from objects/arrays. **Object:** `{a, b}`. **Array:** `[a, b]`. **Defaults:** `{a = 1}`. **Renaming:** `{a: b}`. **Rest operator (...)** collects remaining elements into an array (e.g., `[...rest]` or `{...rest}`).

JS15: **for...in** iterates over enumerable property keys (including inherited). **for...of** iterates over iterable values. Avoid **for...in** on arrays because it iterates over all enumerable properties (including custom ones) and is slower; use **for...of** or standard `for` loops.

JS16: **Generics** allow reusable components with type parameters. **Constraints (`extends`)** limit types (e.g., `T extends string`). **Conditional types** (`T extends U ? X : Y`) select types based on conditions. **infer** extracts types within conditional types (e.g., extracting return type of a function).

JS17: **CommonJS** (`require`) is synchronous and dynamic. **ES Modules** (`import`) are asynchronous and static. **Node.js** supports both; ESM requires `.mjs` or `"type": "module"` in `package.json`. `"type": "module"` forces all `.js` files in the package to be treated as ES Modules.

JS18: **Structural typing** means types are compatible if they have the same shape. **Excess property checks** prevent extra properties on object literals assigned to types. **extends** (assignment) vs **satisfies** (type narrowing without widening). **Variance** (`in`/`out`) in TS 4.7+ explicitly marks generic parameters as contravariant or covariant (e.g., `ReadonlyArray`).

JS19: **Proxy** intercepts operations (get, set, etc.). **Reflect** provides default implementations for Proxy traps. **Example:** Validation Proxy intercepts `set` to validate data before assignment. **Reactive binding:** Proxy tracks changes to trigger UI updates.

JS20: **SharedArrayBuffer** allows memory sharing between workers. **Atomics** provides atomic operations. Workers communicate via `Atomics.wait` (block) and `Atomics.notify` (wake). **Ordering guarantees:** Memory barriers ensure visibility of writes across threads. Requires COOP/COEP headers in browsers.

B1: **$@** expands to individual arguments. **$*** expands to a single string (IFS joined). **"$@"** preserves arguments as separate quoted words (safe for loops). **"$*"** joins them into one quoted string.

B2: **set -e** exits on error. **set -u** errors on undefined variables. **set -o pipefail** returns error if any command in a pipeline fails. Together they make scripts fail fast and safely.

B3: **$(command)** is preferred over **`command`** (backticks) for nesting and readability. **$((expression))** is arithmetic expansion (shell built-in). **expr** is an external command; slower and less feature-rich.

B4: **Process substitution** `<(command)` creates a file descriptor pointing to the command's output. **Example:** `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5: Use **`for f in *`** or **`find . -print0 | while IFS= read -r -d '' f`**. **`for f in $(ls)`** is dangerous because `ls` output is parsed by whitespace, breaking on filenames with spaces or newlines.

B6: **Here document (`<<EOF`)** reads until EOF line. **Here string (`<<<`)** passes a string as stdin. **`<<-EOF`** strips leading tabs from the here document content.

B7: **`cmd > file 2>&1`**: stdout to file, then stderr to stdout (file). **`cmd 2>&1 > file`**: stderr to stdout (terminal), then stdout to file. **`&>`**: Bash shorthand for `> file 2>&1` (both to file). Order matters because redirections are processed left-to-right.

B8: Use **`mkdir`** for locking (atomic on POSIX). Function: `mkdir /tmp/lock` inside `trap 'rmdir /tmp/lock' EXIT`. **Why mkdir:** Creating a directory is atomic; `touch` is not. Handles race conditions and SIGTERM via trap cleanup.

B9: **Single quotes** (`'...'`) literal (no expansion). **Double quotes** (`"..."`) expand variables/commands. **No quotes** expand variables, commands, and globbing. Example: `echo '$var'` prints `$var`; `echo "$var"` prints value.

B10: **`[ -f file ]`** (file exists), **`[ -d dir ]`** (directory), **`[ -r file ]`** (readable). **`[[ ]]`** is a bash builtin (supports regex, `&&`/`||`), `[ ]` is a command (POSIX test).

B11: **`$?`** (exit code of last cmd), **`$$`** (current PID), **`$!`** (PID of last background job), **`$#`** (number of arguments). Use cases: Error checking, logging, job control, argument counting.

B12: **`func() {}`** or **`function func {}`**. Return values via **`return`** (exit code) or **`echo`** (stdout). Functions share the global scope unless declared `local`.

B13: **PATH** is a colon-separated list of directories. Bash searches PATH for external commands. **Builtin** commands are part of the shell (faster). **External** are binaries. Use **`type`** or **`which`** to check.

B14: **`trap`** sets handlers for signals. **EXIT** runs on script exit. **ERR** on error. **INT/TERM** on signals. Execution order: Handlers run in order of definition; EXIT handlers run last.

B15: **xargs** builds command lines from stdin. **`find -print0 | xargs -0`** handles filenames with spaces/newlines. **`xargs -P`** runs jobs in parallel (concurrency limit).

B16: **Arrays:** `arr=(a b)`. **Associative:** `declare -A assoc`. Append: `arr+=(c)`. Length: `${#arr[@]}`. Associative keys are strings; values can be anything.

B17: **Subshell (`(...)`)** isolates variables. **Group (`{...}`)** shares variables. **Piping into `while`** runs the loop in a subshell, so variable changes are lost after the loop. Use process substitution or redirection to avoid this.

B18: **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (unblockable). Handle graceful shutdown by trapping SIGTERM, cleaning up resources, then exiting. Background processes need `wait` or `disown`.

B19: **Script:** `split`, `xargs -P -n1 -I{} process.sh {}`, `cat results/*`. **Trade-offs:** Faster I/O but higher memory/CPU usage. **Error propagation:** Check exit codes of parallel jobs; aggregate errors in a final step.

B20: **`exec`** replaces the current shell process. **FD manipulation:** `exec 3>file` opens FD 3. **Coprocess:** `coproc` creates a bidirectional pipe. Use `exec` for persistent FDs or replacing the shell (e.g., `exec nginx`).

PS1: **Write-Output** sends objects to the pipeline. **Write-Host** writes directly to the console (bypasses pipeline). Distinction matters because Write-Host breaks piping and automation.

PS2: **Pipeline** passes objects. **ForEach-Object** is a cmdlet processing pipeline items one-by-one. **foreach** is a statement iterating over a collection in memory. Use **ForEach-Object** for streaming; **foreach** for in-memory collections.

PS3: **[PSCustomObject]** creates a typed object with properties. **[hashtable]** is a key/value collection. Convert: `[PSCustomObject]@{Key = 'Value'}`. Use **PSCustomObject** for structured data; **Hashtable** for dynamic config.

PS4: **Terminating errors** stop execution (throw). **Non-terminating** continue (warning). **-ErrorAction** overrides preference. **try/catch** catches terminating errors. `$ErrorActionPreference` sets default behavior.

PS5: **Splatting** (`@params`) passes a hashtable as separate arguments. **Hashtable parameter** passes the whole hashtable as one argument. Splatting is cleaner for dynamic arguments.

PS6: **Remoting** uses WinRM (HTTP) or SSH. **Invoke-Command** runs one-off commands remotely. **Enter-PSSession** starts an interactive session.

PS7: **Type inference:** `$result += "one"` on `@()` creates `String[]`. `$x.GetType().Name` is `String[]`. If returning 0 items, `$x` is empty array. **Correct pattern:** Explicitly initialize as array (`$result = @()`) and return it to ensure consistent type.

PS8: **[scriptblock]** is a code container. **Function** is a named scriptblock. Pass via `-ArgumentList`. **Double-hop:** Kerberos delegation issue when accessing a third server. Solve with **CredSSP** or **Invoke-Command -ArgumentList** to pass credentials securely.

PS9: **@()** creates an array. **[array]** casts to array. **Indexing $null** returns $null. **Indexing empty array** returns $null. Force single result: `$x = @($singleItem)`.

PS10: **-eq** (equal), **-like** (wildcard), **-match** (regex), **-contains** (left), **-in** (right). Case-sensitive variants: **-ceq**, **-clike**, **-cmatch**. **-contains** checks if collection contains value. **-in** checks if value is in collection.

PS11: **Module** is a reusable script/library. **.psm1** (script), **.psd1** (manifest), **Binary** (DLL). **Import-Module** loads it. Manifest defines metadata (version, commands).

PS12: **Interpolation** (`"Hello $name"`). **Single quotes** (`'...'`) no interpolation. **Here-string** (`@"..."@`) multi-line. Include expressions: `$var` or `${var}`.

PS13: **Get-Member** lists object members. **Properties** (data), **Methods** (functions), **NoteProperties** (added dynamically). **Select-Object** filters properties. **Where-Object** filters objects.

PS14: **Classes** (v5) define types. **Limitations:** No interfaces, limited inheritance (single), no static members in older versions. Use **Classes** for complex state; **Functions** for simple logic.

PS15: **CmdletBinding** enables advanced features (parameters, pipeline). **SupportsShouldProcess** for `-WhatIf`/`-Confirm`. **Begin/Process/End** blocks handle setup, streaming, and cleanup.

PS16: **Providers** expose data as file systems. **Get-ChildItem** works on Registry (`HKLM:\`), Certificates, etc. Access registry like files: `Get-ChildItem HKLM:\Software`.

PS17: **Runspaces** are isolated execution environments. **RunspacePool** manages threads. **ForEach-Object -Parallel** (v7+) simplifies parallelism. Manual runspace management offers more control but is complex.

PS18: **PSScriptAnalyzer** enforces standards. **Rules** check syntax/style. **Severity** (Error, Warning, Info). **Suppress** via comments. Integrate with **CI/CD** to fail builds on violations.

PS19: **$using:** scope passes local variables to remote sessions. **Serialization limits:** CLIXML depth limits complex objects. **ArgumentList** serializes arguments. Use **$using:** for simple vars; avoid deep objects.

PS20: **DSC** manages state. **Configurations** define state. **Resources** implement actions. **LCM** applies config. **PS7+** uses DSC v3 (cross-platform). Replaced by **Ansible/Terraform** for cloud-native IaC in many scenarios.
