JS1 — **var** is function-scoped and hoisted (initialized as `undefined`). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages execution. The **call stack** runs synchronous code. When empty, it processes the **microtask queue** (Promises, `queueMicrotask`) before the **macrotask queue** (`setTimeout`, `setInterval`). This ensures async callbacks run before the next render.

JS3 — A **closure** is a function retaining access to its lexical scope. Example: Counter factory. **Bug:** `var` in a `for` loop shares one variable reference, causing all callbacks to log the final value. **Fix:** Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents async completion. **async/await** is syntactic sugar for Promise chaining. Error handling uses `try/catch` for async or `.catch()` for Promises. **Promise.all** fails on first rejection; **Promise.allSettled** waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 — **interface** defines shape (can be merged, implemented by classes); **type** defines value (unions, primitives, intersections). Use **interface** for public API contracts/classes; **type** for unions/complex shapes. **Discriminated unions** use a literal type property to narrow types (e.g., `{type: 'a', a: number}`).

JS6 — **Prototypal inheritance** links objects via `[[Prototype]]`. **Object.create(proto)** creates an object with a specific prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` and `constructor` to manage the prototype chain.

JS7 — **this** binding rules (priority): 1. `new` (constructor), 2. Explicit (`call`/`apply`/`bind`), 3. Implicit (object method), 4. Default (global/window/undefined). **bind()** returns a new function with bound context. **Cannot re-bind** a bound function.

JS8 — **WeakRef** holds weak reference to object (GC can collect). **FinalizationRegistry** triggers cleanup when object is collected. **WeakMap** keys must be objects (GC can collect keys). Used for private data or caches where memory leaks must be avoided.

JS9 — **Hidden classes** optimize property access by grouping objects with same shape. **Inline caching** caches property lookup results. **Deoptimization** occurs when shape changes (e.g., adding/deleting properties dynamically). Patterns causing deopt: dynamic property access, deleting properties, polymorphic shapes.

JS10 — **Tree-shaking** removes unused exports. **ES Modules** have static structure, enabling static analysis. **CommonJS** (`require`) is dynamic, preventing static analysis, so bundlers cannot safely remove unused code.

JS11 — **undefined** (variable declared but not assigned); **null** (intentional absence). `== null` checks both. **Nullish coalescing (`??`)** returns right side if left is `null`/`undefined`. **Optional chaining (`?.`)** safely accesses nested properties.

JS12 — **map** transforms, **filter** selects, **reduce** accumulates. **reduce** with initial value starts from index 0; without, starts from index 1. **flatMap** maps then flattens one level; useful for transforming arrays of arrays.

JS13 — **Template literals** use backticks for multi-line strings and interpolation `${}`. **Tagged templates** call a function with strings and values. Example: `sql` tag to prevent SQL injection by escaping values.

JS14 — **Destructuring** extracts values. **Object**: `{a, b}`. **Array**: `[a, b]`. **Defaults**: `{a = 1}`. **Renaming**: `{a: b}`. **Rest operator (`...`)** collects remaining items into an array.

JS15 — **for...in** iterates enumerable keys/properties (including inherited). **for...of** iterates values of iterable objects. Avoid **for...in** on arrays as it iterates indices/properties, is slower, and includes non-index properties.

JS16 — **Generics** (`<T>`) allow reusable types. **Constraints** (`T extends Base`) limit types. **Conditional types** (`T extends U ? X : Y`) select types based on condition. **infer** extracts types from conditional types (e.g., `ReturnType<T>`).

JS17 — **CommonJS** (`require`) is dynamic, synchronous. **ES Modules** (`import`) are static, asynchronous. **Node.js** defaults to CJS; `"type": "module"` in `package.json` enables ESM. Node supports both via `.mjs` or config.

JS18 — **Structural typing** checks shape, not name. **Excess property checks** apply to object literals assigned to variables/interfaces. **extends** checks assignability; **satisfies** checks shape without widening type. **Variance** (`in`/`out`) controls generic parameter usage (TS 4.7+).

JS19 — **Proxy** intercepts operations (get, set, apply). **Reflect** provides default implementations. Example: Validation Proxy intercepts set to check values. Reactive data binding uses Proxy to trigger updates on property changes.

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic ops (wait, notify). Workers communicate via `postMessage` or SharedArrayBuffer. **Ordering guarantees** ensure memory consistency across threads using memory barriers.

B1 — **$@** preserves arguments as separate words (respects quotes). **$*** joins arguments into one string. **"$@"** expands to separate quoted words (safe for loops). **"$*"** expands to one string (space-separated).

B2 — **-e**: Exit immediately on error. **-u**: Treat unset variables as error. **-o pipefail**: Return error if any command in pipe fails. Ensures strict error handling.

B3 — **$(command)** is modern, supports nesting. **`command`** is legacy. **$((expression))** is arithmetic expansion (shell builtin). **expr** is external command, slower, less feature-rich.

B4 — **Process substitution** (`<(command)`) treats command output as a file. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — **Safe iteration**: `for f in *; do ...` (glob expansion). **`for f in $(ls)`** is dangerous because it splits on whitespace/newlines, breaking filenames with spaces. Use `find` or globbing.

B6 — **Here document** (`<<EOF`) reads stdin until EOF. **Here string** (`<<<`) passes a string as stdin. **`<<-EOF`** strips leading tabs from lines.

B7 — `cmd > file 2>&1`: stdout to file, stderr to stdout (file). **Both in file.** `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. **Stderr to terminal, stdout to file.** `&> file`: Both to file.

B8 — Use `mkdir` for locking (atomic on POSIX). Function: `mkdir "$LOCK" 2>/dev/null && trap 'rmdir "$LOCK"' EXIT`. **Why mkdir:** Atomic creation prevents race conditions; `touch` is not atomic.

B9 — **'...'**: Literal (no expansion). **"..."**: Expansion (variables, commands). **No quotes**: Word splitting and globbing. Example: `echo $HOME` (no quotes) vs `echo "$HOME"` (quotes).

B10 — **File tests**: `-e` (exists), `-d` (directory), `-r` (readable). **`[`** is external command (POSIX). **`[[`** is bash builtin (supports regex, safer).

B11 — **$?**: Exit code of last command. **$$**: PID of current shell. **$!**: PID of last background job. **$#**: Number of arguments.

B12 — **Define**: `func() {}` or `function func {}`. **Return**: `return` sets exit code; `echo` sets stdout. Functions share global scope unless `local` used.

B13 — **PATH**: Colon-separated list of directories. Bash searches PATH for commands. **Builtin** (shell internal) vs **External** (binary). Check with `type` or `command -v`.

B14 — **trap**: Sets signal handlers. `trap 'cleanup' EXIT`. **Execution order**: Handlers run on exit. Multiple signals: Last handler set usually wins unless specified.

B15 — **xargs** builds command lines from stdin. **Substitution** runs command immediately. **xargs -0**: Handles null-delimited input (`find -print0`). **xargs -P**: Parallel execution (N jobs).

B16 — **Indexed**: `arr=(1 2)`. **Associative**: `declare -A arr=([key]=val)`. **Append**: `arr+=(val)`. **Length**: `${#arr[@]}`. **Iterate**: `for k in "${!arr[@]}"`.

B17 — **Subshell** `(...)`: New process, variables lost. **Group** `{ ...; }`: Same shell, variables persist. **Pipe into while**: Runs in subshell, variable changes lost. Use process substitution or temp file.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (force kill, no handler). **Graceful shutdown**: Trap SIGTERM, cleanup resources, exit. Background jobs need `wait` or job control.

B19 — **Script**: `split -l 1000 log.log part_`. `xargs -P 4 -I {} sh -c 'process {}'`. **Merge**: `cat part_* > merged.log`. **Trade-offs**: Parallel I/O vs CPU. **Errors**: Check exit codes in worker, aggregate failures.

B20 — **exec**: Replaces shell process. **FD manipulation**: `exec 3>file` opens FD 3. `exec 3>&-` closes. **Coprocess**: `coproc CMD` creates bidirectional pipe. Use for persistent connections or background tasks.

PS1 — **Write-Output**: Sends object to pipeline. **Write-Host**: Writes to console (screen). **Distinction**: Write-Host breaks pipeline (objects not passed downstream).

PS2 — **Pipeline**: Passes objects between cmdlets. **ForEach-Object**: Cmdlet, processes pipeline items one-by-one. **foreach**: Statement, iterates collection in memory. Use **ForEach-Object** for streaming, **foreach** for pre-loaded collections.

PS3 — **[PSCustomObject]**: Strongly typed object with properties. **[hashtable]**: Key/value pairs. **Convert**: `[PSCustomObject]@{Key='Value'}`. Use **CustomObject** for structured data, **Hashtable** for dynamic config.

PS4 — **Terminating**: Stops execution (throw). **Non-terminating**: Continues (warning). **-ErrorAction Stop**: Treats non-terminating as terminating. **try/catch**: Catches terminating errors.

PS5 — **Splatting** (`@params`): Passes hashtable elements as individual parameters. **Difference**: Passing hashtable as single param passes it as one argument. Splatting allows dynamic parameter sets.

PS6 — **Invoke-Command**: One-off remote execution. **Enter-PSSession**: Interactive session. **Protocol**: WinRM (HTTP/HTTPS). **SSH**: PowerShell 7+ supports SSH remoting.

PS7 — **Type System**: Returns array if initialized `@()`. Returns scalar if not. **Snippet**: Returns Array. **0 items**: Returns empty array. **Pattern**: Always initialize `$result = @()` to ensure array return.

PS8 — **[scriptblock]**: Code object. **Pass to remote**: `-ArgumentList` or `$using:`. **Double-hop**: CredSSP needed for second hop. **Solution**: Use `-Credential` or configure delegation.

PS9 — **@()**: Casts to array. **[array]**: Type constraint. **Force single**: `@($item)`. **Indexing**: `$null[0]` returns `$null`. `@()[0]` throws error.

PS10 — **-eq**: Value equality. **-like**: Wildcard. **-match**: Regex. **-contains**: Left contains right. **-in**: Left in right. **Case-sensitive**: `-ceq`, `-clike`.

PS11 — **Module**: Reusable code. **.psm1**: Script module (code). **.psd1**: Manifest (metadata). **Binary**: DLL. **Import-Module**: Loads module into session.

PS12 — **"..."**: Interpolates variables. **'...'**: Literal. **Expressions**: `${var}`. **Here-string**: `@"..."@` (multiline, preserves formatting).

PS13 — **Get-Member**: Inspect object members. **Properties**: Data. **Methods**: Functions. **NoteProperties**: Added dynamically. **Select-Object**: Choose properties. **Where-Object**: Filter objects.

PS14 — **Classes** (v5+): Define types. **Differences**: No private members, limited inheritance. **Limitations**: No interfaces (v5), constructors limited. **Use**: Stateful objects, complex data structures.

PS15 — **[CmdletBinding]**: Enables advanced features. **SupportsShouldProcess**: `-WhatIf`/`-Confirm`. **Begin/Process/End**: Stream processing blocks. **ValueFromPipeline**: Accept pipeline input.

PS16 — **Providers**: Virtual file system. **Get-ChildItem**: Works on FileSystem, Registry, etc. **Registry**: `HKLM:\Software\...`. Access like files/directories.

PS17 — **Runspaces**: Isolated execution environments. **[runspacepool]**: Reuse threads. **ForEach-Object -Parallel**: v7+ parallel processing. **Manual**: Use `[powershell]::Create()` for v5.

PS18 — **PSScriptAnalyzer**: Linting tool. **Rules**: Code quality checks. **Severity**: Warning/Error. **Suppress**: `#pragma`. **CI/CD**: Integrate via `Invoke-ScriptAnalyzer`.

PS19 — **$using:**: Passes local vars to remote blocks. **ArgumentList**: Serializes args. **Limits**: CLIXML depth limits complex objects. **Effect**: Nested objects may fail serialization.

PS20 — **DSC**: Configuration management. **Resources**: Tasks (e.g., File). **LCM**: Manages config state. **PS7**: DSC still supported but less central. **Replaced**: Ansible, Terraform, or native package managers (winget/choco) for modern IaC.
