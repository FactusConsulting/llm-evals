N1 — Easy: A Layer 2 switch forwards frames based on MAC addresses within a broadcast domain (VLAN). A Layer 3 switch forwards packets based on IP addresses and can perform routing between VLANs/subnets. Layer 3 switches also support ACLs and QoS at the IP level.

N2 — Easy:
*   **Network Address:** 10.0.0.0
*   **Broadcast Address:** 10.0.15.255
*   **Calculation:** /20 mask (255.255.240.0) affects the 3rd octet. Block size is 16 (256 - 240). The subnet starts at 0 and ends at 15.

N3 — Medium: The handshake consists of SYN, SYN-ACK, and ACK. If the final ACK is lost, the server retransmits the SYN-ACK. If the client receives the retransmission, it sends the ACK again. If the server never receives the ACK, it times out and closes the connection.

N4 — Medium:
*   **SNAT (Source NAT):** Modifies the source IP of outgoing packets (e.g., Masquerading internal IPs to a public IP).
*   **DNAT (Destination NAT):** Modifies the destination IP of incoming packets (e.g., Port forwarding external traffic to an internal server).
*   **Example:** SNAT allows a laptop to browse the web; DNAT allows a user to access a web server via a public IP.

N5 — Medium:
1.  The hostname in the `curl` URL does not match the Common Name (CN) or Subject Alternative Name (SAN) in the certificate.
2.  The certificate is a wildcard (e.g., `*.example.com`) but the target is a specific subdomain not covered (e.g., `api.example.com` vs `*.example.com` mismatch).
3.  DNS resolution points to a different server (e.g., load balancer) that presents a certificate for a different domain.

N6 — Medium:
*   **VLAN:** Logical L2 segmentation using 802.1Q tags. Limited to 4094 IDs. Requires physical L2 connectivity.
*   **VxLAN:** Encapsulates L2 frames in UDP/IP (L3). Supports 16 million IDs. Works over L3 networks.
*   **Choice:** Choose VxLAN for multi-tenant cloud environments, data center overlays, or when L2 connectivity is not available across the physical network.

N7 — Hard:
*   **Cause:** The default route points to eth0. Return traffic from eth1 uses the default route (eth0), causing asymmetric routing.
*   **Fix:** Use Policy-Based Routing (PBR).
    1.  `ip rule add from 192.168.1.0/24 table 100`
    2.  `ip route add default via 192.168.1.1 table 100`
    3.  This forces traffic originating from eth1's subnet to reply via eth1.

N8 — Hard: BGP path selection order (simplified):
1.  **Weight** (Cisco proprietary, local only)
2.  **Local Preference** (Highest wins)
3.  **Locally Originated** (Network/Aggregate)
4.  **AS Path** (Shortest wins)
5.  **Origin Code** (IGP < EGP < Incomplete)
6.  **MED** (Lowest wins)
7.  **eBGP over iBGP**
8.  **Router ID** (Lowest wins)

N9 — Hard:
*   **Overhead:** VxLAN adds ~50 bytes (20 IP + 8 UDP + 8 VxLAN + 14 Eth header). IPsec adds ~50 bytes (ESP header + padding + auth). Total overhead ≈ 100 bytes.
*   **Math:** 1500 (Physical MTU) - 100 (Overhead) = 1400 bytes.
*   **Max Inner Payload:** 1400 bytes (excluding inner Ethernet header).

N10 — Medium:
*   **ARP:** Resolves IP addresses to MAC addresses on a local network.
*   **Gratuitous ARP:** Solves ARP cache poisoning and failover. It announces an IP/MAC mapping to update switch MAC tables and notify hosts of a new owner for an IP (e.g., after a VM migration).

N11 — Easy:
*   **Public:** Routable on the internet.
*   **Private:** Reserved for internal networks, not routable on the internet.
*   **RFC 1918 Ranges:**
    1.  10.0.0.0 – 10.255.255.255 (/8)
    2.  172.16.0.0 – 172.31.255.255 (/12)
    3.  192.168.0.0 – 192.168.255.255 (/16)

N12 — Easy:
*   **DNS:** Domain Name System, translates names to IPs.
*   **A Record:** Maps hostname to IPv4 address.
*   **CNAME Record:** Maps hostname to another hostname (alias).
*   **AAAA Record:** Maps hostname to IPv6 address.

N13 — Easy:
*   **DHCP:** Dynamic Host Configuration Protocol, assigns IP addresses automatically.
*   **DORA Process:**
    1.  **Discover:** Client broadcasts to find servers.
    2.  **Offer:** Server offers an IP.
    3.  **Request:** Client requests the offered IP.
    4.  **Acknowledge:** Server confirms and leases the IP.

N14 — Easy:
*   **Hub:** L1 device, broadcasts to all ports.
*   **Switch:** L2 device, forwards based on MAC addresses.
*   **Router:** L3 device, forwards based on IP addresses.
*   **Layers:** Hub (Physical), Switch (Data Link), Router (Network).

N15 — Easy:
*   **Half-Duplex:** Shared medium, can send or receive, but not both simultaneously (collisions possible).
*   **Full-Duplex:** Separate transmit/receive paths, simultaneous send/receive (no collisions).
*   **Modern Ethernet:** Full-duplex.

N16 — Medium:
*   **TTL:** Time To Live, decremented by each router to prevent infinite loops.
*   **Traceroute:** Sends packets with TTL=1, 2, 3... Each router decrements TTL to 0 and sends back an ICMP "Time Exceeded" message, revealing its IP.

N17 — Medium:
*   **NAT Gateway:** Translates private IPs to public IPs.
*   **Static NAT:** 1:1 mapping (fixed).
*   **Dynamic NAT:** Pool of public IPs mapped to private IPs.
*   **PAT (NAPT):** Many private IPs map to one public IP using different ports.

N18 — Hard:
*   **ECMP:** Routes traffic across multiple paths with equal cost.
*   **Hashing:** Uses 5-tuple (Src IP, Dst IP, Src Port, Dst Port, Protocol) to ensure packets for one flow take the same path.
*   **Problem:** Asymmetric flows (request path A, reply path B). Stateful firewalls on path A see the SYN but the ACK arrives on path B, causing the firewall to drop the ACK (no state).

N19 — Hard:
*   **Problem:** The TCP Window Size field is 16-bit (max 64KB). With a Bandwidth-Delay Product (BDP) of 10MB, the 64KB window cannot fill the pipe, limiting throughput significantly.
*   **Solution:** TCP Window Scaling (RFC 1323) adds a scaling factor option during the handshake, allowing the window to be shifted left (multiplied by 2^scale), supporting windows > 64KB.

N20 — Hard:
*   **802.1Q:** Inserts a 4-byte tag into the Ethernet frame (after Source MAC) containing VLAN ID and priority.
*   **Linux Config:** Create sub-interfaces (e.g., `eth0.10`, `eth0.20`) and assign IPs.
*   **Frame Format:** Destination MAC | Source MAC | 0x8100 (Tag) | VLAN ID/Priority | EtherType | Payload | FCS.

L1 — Easy:
*   **Hard Link:** Points to the same inode as the original file. Deleting the original does not affect the link.
*   **Symbolic Link:** Points to the file path. Deleting the original breaks the link. Cannot cross filesystems.

L2 — Easy: `find /var/log -type f -size +100M`

L3 — Medium:
*   **Meaning:** Process is waiting for I/O (disk, network) and cannot be interrupted.
*   **Kill:** Cannot be killed with `kill -9`. Must wait for I/O to complete or reboot.

L4 — Medium:
*   **v1:** Hierarchical controllers (cpu, memory separate).
*   **v2:** Unified hierarchy (all resources in one tree).
*   **Systemd:** Prefers v2 for better resource isolation and cgroup management capabilities.

L5 — Medium:
*   **nftables:** Unified framework, faster, single table syntax.
*   **iptables:** Legacy, multiple tables (filter, nat, mangle).
*   **Kernel:** Modern kernels use `nf_tables` backend; `iptables` commands are translated to nftables rules.

L6 — Medium: A file was deleted but a process still has it open (holding the inode). The space is not freed until the process closes the file. Check with `lsof | grep deleted`.

L7 — Hard:
1.  **UEFI:** Initializes hardware, loads bootloader.
2.  **Bootloader (GRUB):** Loads kernel and initramfs.
3.  **Kernel:** Initializes hardware, mounts initramfs.
4.  **Initramfs:** Loads drivers needed to mount root filesystem.
5.  **Root:** Mounts actual root filesystem.
6.  **Systemd:** Starts services and reaches target (e.g., multi-user).

L8 — Medium:
*   **Why:** `available` accounts for reclaimable memory. If `buff/cache` is high but `available` is low, the kernel may be configured to not reclaim aggressively (`vm.swappiness` low) or cache is dirty/unreclaimable.
*   **Tunables:** `vm.swappiness` (controls swap usage), `vm.vfs_cache_pressure` (controls inode/dentry reclaim).

L9 — Medium:
*   **Function:** Creates isolated network namespaces (network stack).
*   **Usage:** Container runtimes (Docker, Kubernetes) use namespaces to isolate network interfaces, IPs, and routing tables for each container.

L10 — Medium:
*   **SIGTERM (15):** Graceful termination. Catchable.
*   **SIGKILL (9):** Force kill. Cannot be caught or ignored.
*   **SIGQUIT (3):** Quit and dump core. Catchable.

L11 — Easy:
*   **stdin:** Input (0). **stdout:** Output (1). **stderr:** Errors (2).
*   **Redirect:** `command 2> error.log` (stdout stays on terminal, stderr goes to file).

L12 — Easy:
*   **7:** Read, Write, Execute (Owner).
*   **5:** Read, Execute (Group).
*   **5:** Read, Execute (Others).

L13 — Easy:
*   **Root:** UID 0, unrestricted access to all system resources.
*   **Sudo:** Standard user granted specific elevated privileges via `/etc/sudoers` policy.

L14 — Easy:
*   **Definition:** Tool to install, update, and remove software packages.
*   **Debian/Ubuntu:** APT (Advanced Package Tool).
*   **RHEL/Fedora:** DNF (or YUM).

L15 — Easy:
*   **Purpose:** Defines filesystems to mount at boot.
*   **Misconfiguration:** Can prevent the system from booting (e.g., if root or /boot is misconfigured).

L16 — Medium:
*   **LVM:** Abstraction layer for storage.
*   **PV:** Physical Volume (disk/partition).
*   **VG:** Volume Group (pool of PVs).
*   **LV:** Logical Volume (virtual partition).
*   **Extend:** `lvextend -L +1G /dev/vg/lv` followed by `resize2fs /dev/vg/lv`.

L17 — Hard:
*   **mq-deadline:** Optimized for SSDs, ensures latency bounds.
*   **bfq:** Optimized for HDDs, ensures fairness.
*   **none:** No scheduling, best for NVMe.
*   **Change:** `echo mq-deadline > /sys/block/sda/queue/scheduler`.

L18 — Hard:
*   **Load Average:** Number of processes in Runnable (CPU) or Uninterruptible Sleep (I/O).
*   **I/O Wait:** High I/O wait inflates load average without high CPU usage.
*   **Diagnosis:** `iostat` (disk), `vmstat` (CPU/IO), `pidstat` (process level).

L19 — Hard:
*   **Enforcement:** Labels processes/files, blocks access not in policy.
*   **Modes:** Enforcing (block), Permissive (log only), Disabled (off).
*   **Troubleshoot:** Check `dmesg`/`audit.log` for AVC denials. Use `audit2allow` to generate policy or `setenforce 0` to test.

L20 — Hard:
*   **Page Fault:** CPU requests page not in RAM.
*   **Minor:** Page in RAM, just mapped. **Major:** Page on disk, must read.
*   **THP:** Uses 2MB pages instead of 4KB. Reduces TLB misses, improves throughput.
*   **Disable:** For latency-sensitive apps (e.g., databases) where large page allocation causes jitter.
