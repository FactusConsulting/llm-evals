DN1 ‚Äî **Class vs Struct:** `class` is a reference type (heap), supports inheritance, and has a default null value. `struct` is a value type (stack/inline), does not support inheritance (except from `System.ValueType`), and is copied on assignment. **Use struct** for small, immutable data types (e.g., `Point`, `Vector2`) where value semantics are required and allocation overhead must be minimized.

DN2 ‚Äî **IEnumerable<T> vs IQueryable<T>:** `IEnumerable<T>` executes queries in-memory (LINQ to Objects). `IQueryable<T>` builds expression trees that are translated to a query language (e.g., SQL) and executed on the data source. **Distinction matters** because `IQueryable` pushes filtering/aggregation to the database (efficient), whereas `IEnumerable` pulls all data to memory first (inefficient for large datasets).

DN3 ‚Äî **Async/Await:** `async/await` creates a state machine to suspend/resume execution without blocking threads. `Task` is a reference type representing an async operation. `ValueTask` is a struct that avoids heap allocation if the result is already available synchronously. **Use ValueTask** in hot paths (e.g., high-frequency API calls) where the operation often completes synchronously to reduce GC pressure.

DN4 ‚Äî **Dependency Injection (DI):** DI decouples object creation from usage. **Lifetimes:** `Transient` (new instance every request), `Scoped` (new instance per request scope), `Singleton` (single instance for app lifetime). **Common Bug:** Injecting a `Scoped` service (e.g., DbContext) into a `Singleton` causes the scoped service to be reused across requests, leading to stale data, thread-safety issues, or `ObjectDisposedException` when the scope ends.

DN5 ‚Äî **string vs ReadOnlySpan<char>:** `string` is immutable and stored on the heap. `ReadOnlySpan<char>` is a stack-based view into memory (can be on heap or stack) with no allocation. **Use Span<T>** for high-performance parsing or string manipulation to avoid creating intermediate string objects and reduce GC pressure.

DN6 ‚Äî **Garbage Collector (GC):** The GC automatically reclaims memory. **Generations:** Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **LOH:** Large Object Heap (>85KB) is not compacted frequently. **IDisposable:** Needed for unmanaged resources (files, DB connections) because the GC only manages managed memory; `Dispose` ensures deterministic cleanup.

DN7 ‚Äî **ConfigureAwait(false):** Default `await` captures the `SynchronizationContext` (e.g., UI thread). `ConfigureAwait(false)` does not. **Use** in library code to avoid context switching and deadlocks. **Do NOT use** in UI code where you need to marshal back to the UI thread to update controls.

DN8 ‚Äî **.NET 8/9 Features:** **Primary Constructors:** `public class Service(int id) { }`. **Collection Expressions:** `[1, 2, 3]`. **FrozenDictionary:** Immutable, high-performance dictionary. **Example:**
```csharp
public class Service(int id) : IService { }
// DI Registration: services.AddSingleton(new Service(1));
```

DN9 ‚Äî **DbContext Exception:** **Cause:** DbContext is registered as Singleton or shared statically, but it is not thread-safe and expects Scoped lifetime. **Fix:** Register `DbContext` as `Scoped` in DI. **Interaction:** EF Core DbContext tracks changes per request; sharing it across threads causes race conditions and disposal errors.

DN10 ‚Äî **Record vs Class:** `record` provides value-based equality and is immutable by default. `class` provides reference equality. **Record Struct:** Value type with record semantics. **Choose Record** for DTOs/Models. **Choose Class** for mutable entities or when inheritance is required.

DN11 ‚Äî **ref, out, in:** `ref` passes by reference (must be initialized). `out` passes by reference (must be assigned inside method). `in` passes by reference (read-only, avoids copy). **Use `ref`/`out`** for modifying large structs or returning multiple values. **Use `in`** for passing large read-only data efficiently.

DN12 ‚Äî **Nullable Reference Types (NRT):** `string?` indicates a reference type can be null (compile-time warning). `int?` is a nullable value type (`Nullable<int>`). **Difference:** NRT is a compile-time safety feature for references; nullable value types are actual runtime types allowing null values for value types.

DN13 ‚Äî **LINQ:** Language Integrated Query. **Method vs Query:** Same IL, different syntax preference. **Deferred Execution:** Query is not executed until enumerated (e.g., `foreach`). **Benefit:** Allows composition and optimization before execution.

DN14 ‚Äî **Generic Constraints:** `where T : class` (reference type), `where T : struct` (value type), `where T : new()` (parameterless constructor), `where T : IComparable<T>` (interface). **Combine** using `&` (e.g., `where T : class, IComparable<T>`) to enforce multiple capabilities on the generic type.

DN15 ‚Äî **Abstract Class vs Interface:** Abstract class allows partial implementation and state. Interface defines a contract only. **Default Interface Methods (C# 8+):** Allow adding methods to interfaces without breaking existing implementations, blurring the line by allowing shared logic.

DN16 ‚Äî **Middleware Pipeline:** **Use:** Adds middleware to pipeline. **Map:** Branches pipeline for specific paths. **Run:** Terminal middleware (no next). **Ordering:** Critical for request flow (e.g., Auth before Routing). **Exception Handling:** Middleware catches exceptions and can return a response or pass to next handler.

DN17 ‚Äî **Channel<T>:** Async producer-consumer pattern. **BlockingCollection<T>** is synchronous. **Bounded Channel:** Sets capacity limit to apply backpressure (blocks producer if full). **Example:** `Channel.CreateBounded<int>(100)` limits queue size to prevent memory overflow.

DN18 ‚Äî **Source Generators:** Compile-time code generation. **Vs Reflection:** Faster, no runtime cost. **Vs IL Weaving:** Happens before compilation, not post-build. **Incremental Generator:** Only regenerates code if inputs change (performance). **Preferred** over `ISourceGenerator` for scalability and incremental builds.

DN19 ‚Äî **TPL Dataflow:** Pipeline processing library. **ActionBlock:** Consumes data. **TransformBlock:** Transforms data. **BufferBlock:** Stores data. **Vs Channels:** Dataflow provides higher-level orchestration (linking, cancellation, completion), while Channels are lower-level primitives.

DN20 ‚Äî **Minimal APIs:** Lightweight API definition without controllers. **Trade-offs:** Less boilerplate, harder to scale complex auth/middleware, OpenAPI support requires extensions (e.g., Swashbuckle). **Route Groups:** Organize endpoints by feature. **Endpoint Filters:** Intercept requests/responses globally or per endpoint.

P1 ‚Äî **List vs Tuple:** `List` is mutable, ordered collection. `Tuple` is immutable, ordered collection. **Set vs FrozenSet:** `Set` is mutable, unique elements. `FrozenSet` is immutable, hashable.

P2 ‚Äî **GIL:** Global Interpreter Lock allows only one thread to execute Python bytecode at a time. **Affects:** CPU-bound workloads. **Workarounds:** Multiprocessing (separate processes), C-extensions (release GIL), or asyncio (I/O-bound concurrency).

P3 ‚Äî **Deepcopy vs Copy:** `copy` creates a shallow copy (new container, same objects). `deepcopy` creates a new container and recursively copies objects. **Example:** `list1 = [[1]]`, `list2 = copy(list1)`. Modifying `list2[0]` modifies `list1[0]`.

P4 ‚Äî **Decorators:** Functions that wrap other functions. **Time Decorator:**
```python
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 ‚Äî **Generators:** Functions that yield values lazily. **Vs List:** No memory for all items at once. **Yield From:** Delegates to another generator, simplifying nested iteration.

P6 ‚Äî **Asyncio:** Event loop manages coroutines. `asyncio.run()` starts the loop. `asyncio.gather()` waits for multiple tasks. `asyncio.create_task()` schedules a task. **time.sleep()** blocks the loop; use `asyncio.sleep()` to yield control.

P7 ‚Äî **MRO:** Method Resolution Order. **Algorithm:** C3 Linearization. **Diamond:** `D(B, C)` where `B, C` inherit `A`. MRO: `D -> B -> C -> A`. `D().who()` prints "C" because `C` appears before `A` in the linearization order.

P8 ‚Äî **Metaclass:** Class of a class. `__init_subclass__` is easier for subclass hooks. **Metaclass:** Registry pattern (auto-register classes). **__init_subclass__:** Auto-configuration on subclass creation (e.g., adding fields).

P9 ‚Äî **Multiprocessing vs Threading:** Multiprocessing bypasses GIL (CPU-bound). Threading shares memory (I/O-bound). **Concurrent.futures:** High-level API for both. **Serialization:** Multiprocessing requires pickling objects to send between processes.

P10 ‚Äî **__slots__:** Fixed attribute list. **Use:** Save memory in large object instances. **Prevents:** Dynamic attribute creation. **Impact:** Reduces `__dict__` overhead, saving significant memory.

P11 ‚Äî **Virtual Environment:** Isolated Python environment. **Why:** Dependency management. **Difference:** `venv` (stdlib), `virtualenv` (3rd party, more features), `conda` (package manager + env, binary packages).

P12 ‚Äî **Comprehensions:** Concise loops. `map/filter` are functional. **Prefer Comprehensions** for readability and Pythonic style. `map/filter` preferred for lazy evaluation in Python 3.

P13 ‚Äî **== vs is:** `==` compares values. `is` compares identity. **Interning:** Small integers (-5 to 256) are cached. `a is b` returns True for small ints because they point to the same object.

P14 ‚Äî **args/kwargs:** `*args` (tuple), `**kwargs` (dict). **Markers:** `/` (positional only), `*` (keyword only). **Use:** Flexible function signatures, e.g., `def func(a, /, b, *, c):`.

P15 ‚Äî **Context Managers:** Manage resources (`with`). **__enter__/__exit__** vs `@contextmanager`. **Example:** Locking a resource or managing a database transaction.

P16 ‚Äî **Type Hinting:** Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. **mypy/pyright:** Use hints to detect type errors before runtime. `Protocol` enables structural typing (duck typing).

P17 ‚Äî **Data Model:** `__repr__` (dev), `__str__` (user). `__eq__` vs `__hash__`: If `__eq__` is defined, `__hash__` becomes `None` (unhashable) unless explicitly defined. `__getattr__` (fallback), `__getattribute__` (always called).

P18 ‚Äî **Descriptors:** Objects controlling attribute access (`__get__`, `__set__`, `__delete__`). **property:** Data descriptor. **classmethod:** Non-data descriptor. **Under the hood:** Attributes are descriptors stored in the class dict.

P19 ‚Äî **Import System:** `sys.path` determines search locations. **Relative:** Based on current package. **Absolute:** Full path. **Namespace Packages:** No `__init__.py` required (PEP 420). **Finder:** Locates module. **Loader:** Loads module.

P20 ‚Äî **Dataclasses:** Reduces boilerplate (`__init__`, `__repr__`). **Vs attrs/Pydantic:** attrs is more flexible, Pydantic adds validation. **__post_init__:** Logic after init. **Frozen:** Immutable. **Choose Pydantic** when validation and serialization (JSON) are required.
