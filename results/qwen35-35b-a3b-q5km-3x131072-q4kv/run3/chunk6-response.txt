JS1 — **var, let, const, Hoisting**
*   **var:** Function-scoped or globally-scoped. Hoisted and initialized with `undefined`. Can be redeclared.
*   **let:** Block-scoped. Hoisted but not initialized (Temporal Dead Zone). Cannot be redeclared in same scope.
*   **const:** Block-scoped. Must be initialized. Cannot be reassigned (though object properties can change).
*   **Hoisting:** JavaScript moves declarations to the top of their scope during compilation. `var` declarations are hoisted and initialized; `let`/`const` are hoisted but remain uninitialized until execution reaches them.

JS2 — **Event Loop**
*   **Call Stack:** Executes synchronous code.
*   **Macrotask Queue:** Tasks like `setTimeout`, `setInterval`, I/O, UI rendering.
*   **Microtask Queue:** Tasks like `Promise.then`, `queueMicrotask`, `MutationObserver`.
*   **Relationship:** The event loop checks the call stack. If empty, it processes all **microtasks** first, then one **macrotask**, then renders UI, then repeats. Microtasks always run before the next macrotask.

JS3 — **Closures**
*   **Definition:** A function bundled with its lexical environment (variables from outer scope).
*   **Example:** Counter factory.
    ```javascript
    function createCounter() {
      let count = 0;
      return () => ++count;
    }
    ```
*   **For Loop Bug:** In `for (var i=0; i<3; i++) setTimeout(() => console.log(i), 100)`, all logs print `3` because `var` is function-scoped and shared.
*   **Fix:** Use `let` (block-scoped) or wrap in an IIFE: `(function(i) { setTimeout(...) })(i)`.

JS4 — **Promises & Async/Await**
*   **Promise:** Object representing eventual completion/failure. States: pending, fulfilled, rejected.
*   **async/await:** Syntactic sugar over Promises. `async` returns a Promise; `await` pauses execution until Promise resolves.
*   **Error Handling:** `try/catch` blocks around `await`. `.catch()` on Promises.
*   **Promise.all:** Waits for all to resolve; fails fast if one rejects.
*   **Promise.allSettled:** Waits for all to settle (resolve or reject); returns array of status objects `{status, value/reason}`.

JS5 — **Interface vs Type**
*   **Interface:** Defines shape of objects. Supports inheritance (`extends`), merging, and implementation (`implements`). Cannot define primitives directly.
*   **Type:** Union, intersection, primitives, tuples. More flexible.
*   **Usage:** Use `interface` for public API contracts/classes. Use `type` for unions, primitives, or complex mappings.
*   **Discriminated Unions:** Unions with a literal type field (discriminator) used in `switch`/`if` to narrow types safely.
    ```typescript
    type Circle = { kind: 'circle'; radius: number };
    type Square = { kind: 'square'; size: number };
    type Shape = Circle | Square;
    ```

JS6 — **Prototypal Inheritance**
*   **Mechanism:** Objects link to a prototype object via `[[Prototype]]` (accessible via `__proto__` or `Object.getPrototypeOf`). Property lookup traverses this chain.
*   **Object.create(proto):** Creates new object with `proto` as its prototype.
*   **ES6 Classes:** Syntactic sugar over prototypes. `class` defines a constructor and prototype methods. `extends` sets up the prototype chain.

JS7 — **this Keyword**
*   **Binding Rules (Priority):**
    1.  **New:** `new Foo()` binds `this` to new instance.
    2.  **Explicit:** `call`, `apply`, `bind` bind `this` to argument.
    3.  **Implicit:** `obj.method()` binds `this` to `obj`.
    4.  **Default:** `this` is `undefined` (strict mode) or `window` (non-strict).
*   **bind():** Returns a new function with `this` permanently bound.
*   **Re-binding:** No. Once bound, `this` cannot be changed by `call`/`apply` on the bound function.

JS8 — **WeakRef & FinalizationRegistry**
*   **WeakRef:** Holds weak reference to an object (doesn't prevent GC). Access via `.deref()`.
*   **FinalizationRegistry:** Callback invoked when target object is garbage collected.
*   **Use Case:** Caching, cleanup of native resources, avoiding memory leaks in large caches.
*   **WeakMap:** Keys must be objects (weakly held). Values are strong. Keys are not enumerable.
*   **GC Relation:** WeakMap keys don't prevent the key object from being collected, allowing automatic cleanup when keys are no longer referenced elsewhere.

JS9 — **V8 Optimization**
*   **Hidden Classes:** Objects with same shape (properties) share a hidden class for fast property access.
*   **Inline Caching (IC):** Caches property lookup results based on hidden class to avoid lookup overhead.
*   **Deoptimization:** Occurs when assumptions (e.g., object shape) are violated (e.g., adding property to one instance but not others).
*   **Patterns causing Deopt:** Dynamic property addition, changing object shape, polymorphic calls, `arguments` object usage.

JS10 — **Tree-Shaking**
*   **Definition:** Eliminating unused code during build.
*   **ES Modules:** Static `import`/`export` allows bundlers (Webpack, Rollup) to analyze dependency graph statically and remove dead code.
*   **CommonJS:** Dynamic `require()` prevents static analysis. Bundlers must assume all code is used, making tree-shaking ineffective.
*   **Requirement:** ES modules must be side-effect free (no global side effects in top-level scope).

JS11 — **null, undefined, Operators**
*   **null:** Intentional absence of value (assignment).
*   **undefined:** Variable declared but not assigned, or missing property.
*   **== vs ===:** `==` performs type coercion (`null == undefined` is true). `===` checks value and type (`null === undefined` is false).
*   **Nullish Coalescing (??):** Returns right operand if left is `null` or `undefined` (ignores `0` or `false`).
*   **Optional Chaining (?.):** Safely accesses nested properties (`obj?.a?.b`). Returns `undefined` if path is broken.

JS12 — **Array Methods**
*   **map:** Transforms each element, returns new array.
*   **filter:** Returns elements passing test.
*   **reduce:** Accumulates value. Initial value prevents first element from being accumulator.
*   **flatMap:** Maps then flattens one level. Useful for transforming arrays of arrays into single arrays.

JS13 — **Template Literals**
*   **Definition:** Strings using backticks (`` ` ``). Support interpolation `${expr}` and multiline.
*   **Tagged Templates:** Function called with first arg as array of strings, rest as interpolated values.
*   **Example:**
    ```javascript
    function escape(strs, ...vals) {
      return strs.reduce((acc, str, i) => acc + str + (vals[i] || ''), '');
    }
    const sql = escape`SELECT * FROM users WHERE id = ${id}`;
    ```

JS14 — **Destructuring**
*   **Object:** `const { a, b } = obj`.
*   **Array:** `const [a, b] = arr`.
*   **Defaults:** `const { a = 10 } = obj`.
*   **Renaming:** `const { a: b } = obj` (renames `a` to `b`).
*   **Rest:** `const { a, ...rest } = obj` collects remaining properties.

JS15 — **for...in vs for...of**
*   **for...in:** Iterates over enumerable **property keys** (including inherited). Used for objects.
*   **for...of:** Iterates over **values** of iterable objects (arrays, strings, Maps).
*   **Avoid for...in on arrays:** Iterates over keys (indices), not values. Can include inherited properties. Slower and less predictable.

JS16 — **TypeScript Generics**
*   **Generics:** `function identity<T>(arg: T): T`. Preserves type info.
*   **Constraints:** `function log<T extends string>(arg: T)`.
*   **Conditional Types:** `type IsString<T> = T extends string ? true : false`.
*   **infer:** Extracts type within conditional. `type UnwrapPromise<T> = T extends Promise<infer U> ? U : T`.

JS17 — **Module Systems**
*   **CommonJS:** `require`/`module.exports`. Synchronous, dynamic, Node.js default (historically).
*   **ES Modules:** `import`/`export`. Static, asynchronous, browser standard.
*   **Node.js:** Supports both. ESM requires `.mjs` or `"type": "module"` in `package.json`.
*   **type: module:** Forces Node to treat `.js` files as ES Modules.

JS18 — **Structural Typing**
*   **System:** Types compatible if they have same shape (duck typing).
*   **Excess Property Checks:** Prevents assigning object literals to interfaces/types with extra properties (e.g., `const x: {a: number} = {a: 1, b: 2}` fails). Applies only to literals, not variables.
*   **extends vs satisfies:** `extends` enforces type compatibility. `satisfies` checks validity without widening type (preserves specific literal types).
*   **Variance:** `in` (contravariant) for function arguments, `out` (covariant) for return types. TS 4.7+ allows explicit variance annotations.

JS19 — **Proxy & Reflect**
*   **Proxy:** Wraps object to intercept operations (get, set, has, apply, etc.).
*   **Reflect:** Provides methods to perform operations (e.g., `Reflect.get`) matching Proxy traps.
*   **Intercept:**
    ```javascript
    const p = new Proxy(obj, {
      get(target, prop) { return target[prop] * 2; }
    });
    ```
*   **Use Case:** Validation, logging, reactive frameworks (Vue 3 uses Proxy).

JS20 — **SharedArrayBuffer & Atomics**
*   **SharedArrayBuffer:** Shared memory between threads (Web Workers).
*   **Communication:** Workers read/write same memory address.
*   **Ordering:** Memory barriers ensure visibility.
*   **Atomics.wait(value, expected):** Pauses thread until value changes or timeout.
*   **Atomics.notify(value, count):** Wakes up waiting threads.
*   **Security:** Requires specific COOP/COEP headers in browsers.

B1 — **$@ vs $***
*   **$*:** All arguments as single string (space-separated).
*   **$@:** All arguments as separate strings.
*   **"$*":** Preserves grouping if quoted (treated as one string).
*   **"$@":** Preserves individual argument boundaries (safest for loops).

B2 — **set -euo pipefail**
*   **-e:** Exit immediately if command exits non-zero.
*   **-u:** Treat unset variables as error.
*   **-o pipefail:** Pipeline returns exit code of first non-zero command (not just last).

B3 — **Command Substitution**
*   **$(command) vs `command`:** Both capture stdout. `$(...)` is preferred (nestable, readable).
*   **$((expression)) vs expr:** `$((...))` is arithmetic expansion (native, faster). `expr` is external command (slower, legacy).

B4 — **Process Substitution**
*   **Definition:** `<(command)` creates a FIFO/pipe file descriptor.
*   **Use:** Allows commands expecting filenames to read from command output.
*   **Example:** `diff <(sort a.txt) <(sort b.txt)`.

B5 — **Safe Iteration**
*   **Method:** `for f in *; do ...` or `find . -print0 | while IFS= read -r -d '' f; do ...`.
*   **Danger of `for f in $(ls)`:** Word splitting breaks filenames with spaces/newlines. `ls` parsing is fragile.

B6 — **Here Documents**
*   **<<EOF:** Reads until line containing `EOF`.
*   **<<<:** Here string (single line).
*   **<<-EOF:** Indented here document (strips leading tabs).

B7 — **Redirections**
*   **`cmd > file 2>&1`:** Redirect stdout to file, then stderr to stdout (which is now file). **Result:** Both in file.
*   **`cmd 2>&1 > file`:** Redirect stderr to stdout (terminal), then stdout to file. **Result:** Only stdout in file, stderr on terminal.
*   **`cmd &> file`:** Bash shorthand for both stdout and stderr to file.

B8 — **Lock File Function**
```bash
lock_file() {
  local lock_dir="/tmp/locks"
  mkdir -p "$lock_dir"
  local lock_file="$lock_dir/$$.lock"
  if ! mkdir "$lock_file" 2>/dev/null; then
    echo "Lock held" >&2; exit 1
  fi
  trap 'rmdir "$lock_file"' EXIT INT TERM
  # Critical section
}
```
*   **Why mkdir:** Atomic on POSIX filesystems. Race condition impossible.
*   **Cleanup:** `trap` ensures removal on exit/signals.

B9 — **Quotes**
*   **Single ('...'):** Literal. No expansion.
*   **Double ("..."):** Expansion of `$`, `` ` ``, `\`.
*   **No quotes:** Word splitting and globbing occur.
*   **Example:** `echo $HOME` (no quotes) expands. `echo '$HOME'` prints literal string.

B10 — **File Tests**
*   **Exist:** `-e file`.
*   **Directory:** `-d file`.
*   **Readable:** `-r file`.
*   **[ vs [[:** `[[` is a keyword (bash builtin), supports regex (`=~`), safer for strings. `[` is a command (`test`), POSIX compliant.

B11 — **Special Variables**
*   **$?:** Exit status of last command.
*   **$$:** PID of current shell.
*   **$!:** PID of last background process.
*   **$#:** Number of arguments passed to script/function.

B12 — **Functions**
*   **Def:** `foo() { ... }` or `function foo { ... }`.
*   **Return:** `return` sets exit code. Use `echo` for output.
*   **Scope:** Variables are global by default unless `local` used.

B13 — **PATH & Commands**
*   **PATH:** Colon-separated list of directories searched for executables.
*   **Resolution:** Searches PATH in order.
*   **Builtin vs External:** Builtins are part of shell (faster). External are binaries.
*   **Check:** `type -a cmd` or `which cmd`.

B14 — **Trap**
*   **Definition:** Catches signals. `trap 'cmd' EXIT`.
*   **Signals:** EXIT (on exit), ERR (on error), INT (Ctrl+C), TERM (kill).
*   **Order:** EXIT trap runs last. ERR runs before command exits. Multiple traps run in order defined.

B15 — **xargs**
*   **vs Substitution:** `xargs` handles arguments safely (quotes, newlines). Substitution expands immediately.
*   **-0 / -print0:** Null-delimited output for filenames with spaces/newlines.
*   **-P:** Parallel execution (run N jobs simultaneously).

B16 — **Arrays**
*   **Indexed:** `arr=(a b c)`. Access `${arr[0]}`. Length `${#arr[@]}`.
*   **Associative:** `declare -A map`. `map[key]=value`.
*   **Iterate:** `for i in "${arr[@]}"; do ...`.
*   **Diff:** Indexed uses numbers; Associative uses strings.

B17 — **Subshell vs Group**
*   **Subshell (...):** New process. Variables lost after exit.
*   **Group { ...; }:** Same shell. Variables persist.
*   **Pipe While:** `cmd | while read; do ...` runs in subshell. Variable changes lost. Use process substitution `while read ...; do ...; done < <(cmd)`.

B18 — **Signals & Job Control**
*   **SIGINT:** Interrupt (Ctrl+C).
*   **SIGTERM:** Terminate (kill).
*   **SIGHUP:** Hangup (terminal close).
*   **SIGKILL:** Force kill (cannot be caught).
*   **Graceful Shutdown:** Trap SIGTERM, cleanup, then exit. Background jobs need `wait` or `disown`.

B19 — **Parallel Log Processing**
*   **Script:** `split -l 1000 log.log part_`, `find . -name part_* -print0 | xargs -0 -P 4 -I {} bash process.sh {}`, `cat part_* > merged.log`.
*   **Trade-offs:** Speed vs complexity/error handling.
*   **Errors:** Check exit codes of parallel jobs. Aggregate logs.

B20 — **exec & File Descriptors**
*   **exec:** Replaces current shell process with command (no new PID).
*   **FD Manipulation:** `exec 3>file` opens FD 3. `exec 3>&-` closes it.
*   **Coprocess:** `coproc CMD`. Creates pipe for stdin/stdout. Used for persistent communication with external process.

PS1 — **Write-Output vs Write-Host**
*   **Write-Output:** Sends object to pipeline.
*   **Write-Host:** Writes directly to console (bypasses pipeline).
*   **Matter:** Use `Write-Output` (or default) for scripting/piping. `Write-Host` breaks pipelines and is harder to test.

PS2 — **Pipeline & Foreach**
*   **Pipeline:** Objects flow from command to command.
*   **ForEach-Object:** Script block processing each object in pipeline (streaming).
*   **foreach:** Statement iterating over a collection in memory (slower, blocks pipeline).
*   **Use:** `ForEach-Object` for streaming data. `foreach` for logic over known collections.

PS3 — **PSCustomObject vs Hashtable**
*   **PSCustomObject:** Strongly typed object with properties. Preserves order.
*   **Hashtable:** Key-value pairs. Unordered (historically).
*   **Conversion:** `[PSCustomObject]@{Key='Value'}` or `@{Key='Value'}.GetEnumerator()`.
*   **Use:** `PSCustomObject` for structured data output. Hashtable for dynamic config.

PS4 — **Error Handling**
*   **Terminating:** Stops execution (e.g., `throw`, `exit`). Caught by `try/catch`.
*   **Non-Terminating:** Continues execution (e.g., `Get-Item` on missing file).
*   **-ErrorAction:** Overrides `$ErrorActionPreference`.
*   **Interaction:** `try { ... } catch { ... }` handles terminating errors.

PS5 — **Splatting**
*   **Definition:** Passing hashtable as parameters to cmdlet. `@params`.
*   **Example:** `Get-ChildItem @params`.
*   **Vs Single Param:** Splatting passes keys as named parameters. Passing hashtable as single param treats it as one argument.

PS6 — **Remoting**
*   **Invoke-Command:** One-off command execution on remote.
*   **Enter-PSSession:** Interactive session.
*   **Protocol:** WS-Management (WinRM). PowerShell 7+ supports SSH.
*   **Relation:** WinRM is default for PS5. SSH is default for PS7+.

PS7 — **Type System & Pipeline**
*   **Issue:** `Get-Items` returns "one" (string). `$x` becomes string, not array.
*   **0 Items:** Returns `$null`.
*   **Pattern:** Always return array: `$result = @(); $result += ...; return $result`.
*   **Result:** `$x.GetType().Name` is `String` (not `Object[]`).

PS8 — **Scriptblock vs Function**
*   **Scriptblock:** Code block object (`{ ... }`). Passed as argument.
*   **Function:** Named scriptblock with scope.
*   **Remote:** Pass scriptblock via `-ScriptBlock`.
*   **Double-Hop:** CredSSP allows delegation. `Invoke-Command -ArgumentList` serializes variables to avoid double-hop auth issues.

PS9 — **@() vs [array]**
*   **@():** Array subexpression. Forces collection.
*   **[array]:** Type cast.
*   **Force Single:** `@($single)`.
*   **Indexing:** `$null[0]` throws error. `@()[0]` returns `$null`.

PS10 — **Comparison Operators**
*   **-eq:** Equal (case-insensitive).
*   **-ceq:** Case-sensitive.
*   **-like:** Wildcard match.
*   **-match:** Regex match.
*   **-contains:** Left operand contains right.
*   **-in:** Right operand in left operand (array).

PS11 — **Modules**
*   **Script (.psm1):** Code file.
*   **Manifest (.psd1):** Metadata (version, exports).
*   **Binary:** Compiled DLL.
*   **Import-Module:** Loads module into session.

PS12 — **String Interpolation**
*   **"...":** Expands variables.
*   **'...':** Literal.
*   **Expressions:** `"${var}"` or `"$var"`.
*   **Here-String:** `@"..."@` for multiline.

PS13 — **Get-Member**
*   **Purpose:** Inspect object properties/methods.
*   **Properties:** Data.
*   **Methods:** Actions.
*   **NoteProperties:** Added dynamically (e.g., via `Add-Member`).
*   **Select-Object:** Filters properties. `Where-Object`: Filters objects.

PS14 — **PowerShell Classes**
*   **v5+:** `class MyClass { ... }`.
*   **Vs C#:** Simpler, no interfaces, limited inheritance (single).
*   **Limitations:** No private fields (use `[Property()]`), no constructors with args easily.
*   **Use:** Encapsulation, strong typing over functions.

PS15 — **CmdletBinding**
*   **Enables:** Advanced features (parameters, pipeline, help).
*   **SupportsShouldProcess:** `-WhatIf`/`-Confirm`.
*   **Begin/Process/End:** `Begin` (init), `Process` (pipeline items), `End` (cleanup).

PS16 — **Providers**
*   **System:** Abstraction layer (FileSystem, Registry, Env).
*   **Get-ChildItem:** Works on all providers (e.g., `Get-ChildItem HKLM:\`).
*   **Registry:** Accessed like folders (`HKLM:\Software`).

PS17 — **Runspaces**
*   **Runspace:** Execution environment.
*   **RunspacePool:** Reuse runspaces for performance.
*   **ForEach-Object -Parallel:** PS7+ built-in parallelism (uses runspace pool).
*   **Thread Safety:** Variables not shared. Use `RunspaceState` for state.

PS18 — **PSScriptAnalyzer**
*   **Purpose:** Static code analysis.
*   **Rules:** Naming, style, security.
*   **Severity:** Error, Warning, Information.
*   **CI/CD:** Integrate via `Invoke-ScriptAnalyzer`. Suppress via `#pragma`.

PS19 — **$using: vs -ArgumentList**
*   **$using:** Injects local variable into remote block (serialization).
*   **-ArgumentList:** Passes arguments to function in remote session.
*   **Limits:** CLIXML serialization depth limits complex objects.
*   **Effect:** Complex objects may lose type info or be truncated.

PS20 — **DSC**
*   **Config:** Desired state definition.
*   **Resources:** Atomic actions (File, Service).
*   **LCM:** Manages state enforcement.
*   **WinPS vs PS7:** DSC in PS7 uses `Microsoft.PowerShell.DSC` module.
*   **Replacement:** Ansible, Terraform, Chef for modern IaC. DSC remains for Windows Server management.
