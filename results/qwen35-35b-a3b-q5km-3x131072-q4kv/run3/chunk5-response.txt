DN1 — **Class vs Struct:** `class` is a reference type (heap), supports inheritance, and has a default null value. `struct` is a value type (stack/inline), does not support inheritance (except from `System.ValueType`), and is copied on assignment. **Use struct** for small, immutable data types (e.g., `Point`, `Vector`) where value semantics are required and performance (avoiding heap allocation) is critical.

DN2 — **IEnumerable vs IQueryable:** `IEnumerable<T>` executes queries in-memory (client-side) using LINQ to Objects. `IQueryable<T>` executes queries on the server (e.g., SQL) using expression trees. **Distinction matters** because `IQueryable` pushes filtering/aggregation to the database (efficient), whereas `IEnumerable` pulls all data to memory first then filters (inefficient for large datasets).

DN3 — **Async/Await:** `async/await` enables non-blocking I/O by yielding control to the caller while awaiting an operation. `Task` is a reference type representing an async operation. `ValueTask` is a struct that avoids heap allocation if the operation completes synchronously. **Use ValueTask** for high-frequency methods that often complete synchronously (e.g., cache lookups) to reduce GC pressure.

DN4 — **Dependency Injection (DI):** DI decouples object creation from usage. **Lifetimes:** `Transient` (new instance every request), `Scoped` (new instance per request), `Singleton` (one instance for app lifetime). **Bug:** Injecting a `Scoped` service into a `Singleton` causes the Singleton to hold a reference to the Scoped instance from the first request, leading to stale data or `ObjectDisposedException` when the scope ends.

DN5 — **String vs ReadOnlySpan<char>:** `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a stack-only view into memory (string, array, buffer) allowing zero-copy manipulation. **Use Span<T>** for high-performance parsing or string manipulation where avoiding allocations and copying is critical (e.g., JSON parsing, binary protocols).

DN6 — **Garbage Collector (GC):** The GC automatically reclaims memory. **Generations:** Gen 0 (short-lived), Gen 1 (buffer), Gen 2 (long-lived). **LOH:** Large Object Heap (objects > 85KB) is not compacted frequently. **IDisposable:** Needed for unmanaged resources (file handles, DB connections) because the GC only manages managed memory; `Dispose` ensures timely release of OS resources.

DN7 — **ConfigureAwait:** `ConfigureAwait(false)` prevents capturing the current `SynchronizationContext`. **Use it** in library code to avoid deadlocks and improve performance. **Do not use it** in UI applications (WPF/WinForms) where UI updates must occur on the UI thread. **ASP.NET Core** does not have a SynchronizationContext by default, so the impact is minimal there, but it remains best practice for libraries.

DN8 — **.NET 8/9 Features:** **Primary Constructors** (C# 12) allow passing args to base/constructor directly in class declaration. **Collection Expressions** (C# 12) allow `[1, 2, 3]` syntax. **FrozenDictionary** is an immutable, high-performance dictionary. **Example:**
```csharp
public class Service(int id) : IService { public int Id => id; }
// DI: services.AddSingleton<Service>(new Service(1));
```

DN9 — **DbContext Exception:** **Cause:** `DbContext` is `Scoped` but injected into a `Singleton` service or shared globally. Under high concurrency, the Singleton holds a reference to a Scoped context that gets disposed or reused incorrectly. **Fix:** Ensure `DbContext` is injected into `Scoped` services (e.g., Controllers) or use `IDbContextFactory` for background tasks. EF Core contexts are not thread-safe.

DN10 — **Record vs Class:** `record` is a reference type with value-based equality (by default) and immutability features. `record struct` is a value type with value equality. **Choose record** for DTOs, immutable data models, or when value equality is desired. **Choose class** for mutable state, complex inheritance hierarchies, or when reference equality is required.

DN11 — **Ref/Out/In:** `ref` passes by reference (can read/write). `out` passes by reference (must be assigned before return). `in` passes by reference (read-only). **Use `ref`** for modifying large structs. **Use `out`** for returning multiple values. **Use `in`** for large structs where you don't need to modify them (avoids copy).

DN12 — **Nullable Reference Types (NRT):** NRT (`string?`) adds compile-time warnings for potential null dereferences on reference types. **Nullable value types** (`int?`) are actual runtime types (`Nullable<T>`). NRT is a compile-time safety feature; `int?` is a runtime wrapper for value types.

DN13 — **LINQ:** Language Integrated Query. **Method vs Query:** Syntactic sugar; both compile to the same IL. **Deferred Execution:** Queries are not executed until enumerated (e.g., `foreach`, `ToList()`). This allows composing queries before execution.

DN14 — **Generic Constraints:** `where T : class` (reference type), `where T : struct` (value type), `where T : new()` (parameterless constructor), `where T : IComparable<T>` (implements interface). **Combine** constraints (e.g., `where T : class, IDisposable`) to enforce multiple capabilities on a generic type.

DN15 — **Abstract Class vs Interface:** `abstract class` can contain implementation and state. `interface` defines a contract (C# 8+ allows default methods). **Default interface methods** allow adding functionality to interfaces without breaking existing implementations, blurring the line but keeping interfaces as contracts.

DN16 — **Middleware Pipeline:** Middleware components handle requests/responses. **`app.Use`** adds middleware to the pipeline. **`app.Map`** branches the pipeline for specific paths. **`app.Run`** adds a terminal handler. **Order matters:** Auth/Logging before Controllers. **Exception Handling:** Middleware wrapping the pipeline catches exceptions; `UseExceptionHandler` configures global error handling.

DN17 — **Channel<T>:** Async producer-consumer primitive. **vs BlockingCollection:** `Channel` is async-first; `BlockingCollection` is sync. **Bounded Channel:** Set `maxItems` to enforce backpressure. If the buffer is full, producers wait, preventing memory exhaustion under high load.

DN18 — **Source Generators:** Generate code at compile-time. **vs Reflection:** Reflection is runtime (slow, no compile-time safety). **vs IL Weaving:** Weaving modifies IL post-build (harder to debug). **Incremental Generator:** Only regenerates code if inputs change (faster build times). Preferred over `ISourceGenerator` for performance and incremental compilation support.

DN19 — **TPL Dataflow:** High-level async pipeline library. **ActionBlock** (consume), **TransformBlock** (transform), **BufferBlock** (store). **vs Channels:** Channels are lower-level primitives. Dataflow provides built-in backpressure, cancellation, and flow control, making it easier to build complex pipelines than raw Channels.

DN20 — **Minimal APIs:** Lightweight entry points without controllers. **vs Controllers:** Less boilerplate, faster startup. **Trade-offs:** Harder to organize complex auth/middleware, less automatic OpenAPI generation (requires Swashbuckle/NSwag). **Route Groups:** Organize endpoints by feature. **Endpoint Filters:** Intercept requests/responses (e.g., validation, logging) without middleware overhead.

P1 — **List vs Tuple:** `list` is mutable, ordered collection. `tuple` is immutable, ordered collection. **Set vs Frozenset:** `set` is mutable, unordered, unique elements. `frozenset` is immutable. Use `frozenset` as dictionary keys.

P2 — **GIL:** Global Interpreter Lock allows only one thread to execute Python bytecode at a time. **Affects:** CPU-bound threading. **Workarounds:** Use `multiprocessing` (separate processes), C-extensions (release GIL), or async I/O for I/O-bound tasks.

P3 — **Deepcopy vs Copy:** `copy` creates a shallow copy (new container, same object refs). `deepcopy` creates a new container and recursively copies objects. **Example:** `list_of_lists = [[1], [2]]`. Shallow copy `[[1], [2]]` shares inner lists; modifying inner list in copy affects original.

P4 — **Decorators:** Functions that wrap other functions. **Time Logger:**
```python
import time
def timeit(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 — **Generators:** Functions that yield values lazily. **vs List:** Generators don't store all values in memory. **`yield from`** delegates to another generator, simplifying nested iteration logic.

P6 — **Asyncio:** Event loop manages coroutines. **`run`** starts the loop. **`gather`** runs tasks concurrently. **`create_task`** schedules a coroutine. **`time.sleep`** blocks the loop; use `asyncio.sleep` to yield control to the event loop.

P7 — **MRO:** Method Resolution Order determines which method is called in inheritance. **Algorithm:** C3 Linearization. **Diamond:** `D(B, C)` where `B, C` inherit `A`. MRO: `D -> B -> C -> A`. `D().who()` calls `C.who()` because `C` appears before `A` in the linearization order.

P8 — **Metaclass:** Class of a class. **`__init_subclass__`** is easier for subclass hooks (e.g., plugin registration). **Metaclass** is for controlling class creation (e.g., ORM model registration). Use `__init_subclass__` for simple hooks, Metaclass for complex class construction logic.

P9 — **Multiprocessing vs Threading:** `multiprocessing` bypasses GIL (CPU-bound), uses separate memory (pickling required). `threading` shares memory (GIL-bound), good for I/O. **`concurrent.futures`** provides a high-level API for both. Pickling is required for passing args between processes.

P10 — **`__slots__`:** Restricts instance attributes to a fixed set. **Prevents:** Dynamic attribute creation (`__dict__` not created). **Memory:** Reduces memory footprint significantly for many instances.

P11 — **Virtual Environment:** Isolated Python environment. **Why:** Dependency management. **`venv`** (stdlib), **`virtualenv`** (3rd party, more features), **`conda`** (package manager + env, common in data science).

P12 — **Comprehensions:** Concise syntax for creating lists/dicts/sets. **vs map/filter:** Comprehensions are more Pythonic and readable. **Prefer** comprehensions for simple transformations; use `map/filter` for functional pipelines or when reusing functions.

P13 — **`==` vs `is`:** `==` checks value equality. `is` checks identity (memory address). **Interning:** Small integers (-5 to 256) are cached. `a is b` returns True for small ints because they point to the same cached object.

P14 — **`*args`/`**kwargs`:** `*args` collects positional args into a tuple. `**kwargs` collects keyword args into a dict. **`/`** marks positional-only args. **`*`** marks keyword-only args (must be passed by name).

P15 — **Context Managers:** Manage resources (e.g., locks, files). **`__enter__`/`__exit__`** define protocol. **`@contextmanager`** decorator simplifies creation. **Example:** Acquiring a lock before a block and releasing it in `__exit__`.

P16 — **Type Hinting:** Static type annotations. **`Union`** (A | B), **`Optional`** (A | None), **`Literal`** (specific value), **`TypeVar`** (generic type), **`Protocol`** (structural typing). **mypy/pyright** use these to catch type errors before runtime.

P17 — **Data Model:** Dunder methods define behavior. **`__repr__`** (dev/debug), **`__str__`** (user). **`__eq__`** vs **`__hash__`**: If `__eq__` is defined, `__hash__` becomes `None` (object becomes unhashable) unless explicitly defined.

P18 — **Descriptors:** Objects controlling attribute access (`__get__`, `__set__`, `__delete__`). **`property`** is a data descriptor. **`classmethod`/`staticmethod`** are non-data descriptors. They allow customizing attribute access logic (e.g., validation, lazy loading).

P19 — **Import System:** **`sys.path`** lists directories. **`PYTHONPATH`** overrides/adds to path. **`__init__.py`** marks package (optional in PEP 420 namespace packages). **Finder** locates modules, **Loader** loads them. **Relative imports** use `.` based on current package.

P20 — **Dataclasses:** Reduces boilerplate for `__init__`, `__repr__`, etc. **vs attrs/Pydantic:** `attrs` is more flexible, `Pydantic` adds validation. **`__post_init__`** runs after init. **Frozen** makes instance immutable. **Choose Pydantic** for API models requiring validation and serialization (e.g., FastAPI).
