**G1** — A **slice** is a dynamic view into an underlying array (header contains pointer, length, capacity), while an **array** is a fixed-size value type. When appending to a slice that exceeds its capacity, Go allocates a new, larger underlying array, copies the data, and updates the slice header.

**G2** — `var x int` declares a variable with the zero value (`0`) and requires explicit type. `x := 0` is a short variable declaration that infers the type. You cannot use `:=` if the variable already exists in the current scope (redeclaration error) or if you need to declare a variable without initializing it immediately.

**G3** — A **goroutine** is a lightweight thread managed by the Go runtime, not the OS. It uses less memory and context-switching overhead than OS threads. The **scheduler** (GMP model) maps Goroutines (G) to OS Threads (M) via Processors (P), allowing efficient multiplexing of many goroutines onto fewer OS threads.

**G4** — **Channels** are typed conduits for communication. **Unbuffered** channels block both sender and receiver until the other side is ready (synchronization). **Buffered** channels allow sending without blocking until the buffer is full. Sending to a **full buffered channel** blocks until a receiver reads from it.

**G5** — The `context` package manages deadlines, cancellation, and request-scoped values.
```go
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()
req, _ := http.NewRequestWithContext(ctx, "GET", url, nil)
resp, err := client.Do(req) // Cancels if timeout exceeded
```

**G6** — Go interfaces are satisfied **implicitly** (duck typing) rather than declared explicitly. Unlike Java/C#, you don't need to `implements` keyword. The **empty interface** (`interface{}` or `any`) accepts any type. It is powerful for generic containers but dangerous because it requires type assertions, losing compile-time type safety.

**G7** — This is unsafe due to a **data race**: one goroutine writes `data` and `ready`, while the main goroutine reads `data` without synchronization. The compiler may reorder instructions or cache values.
**Fix:** Use `sync.Mutex` to protect `data` and `ready`, or use `atomic.Bool` for `ready` and `atomic.Value` for `data`.
```go
var mu sync.Mutex
var data string
var ready atomic.Bool
go func() {
    mu.Lock()
    data = "hello"
    mu.Unlock()
    ready.Store(true)
}()
for !ready.Load() {}
mu.Lock()
fmt.Println(data)
mu.Unlock()
```

**G8** — `sync.Mutex` allows exclusive access. `sync.RWMutex` allows multiple readers or one writer. `sync.Map` is optimized for cases where keys are mostly read, rarely written, or keys are distinct per goroutine. Trade-off: `sync.Map` has higher overhead for writes and complex locking compared to a standard `map` + `Mutex` for dense access patterns.

**G9** — **Go Modules** manage dependencies. `go.mod` lists direct dependencies and versions. `go.sum` contains cryptographic checksums for reproducibility. `go mod tidy` removes unused imports and adds missing ones. **MVS** (Minimum Version Selection) selects the lowest version of a dependency that satisfies all constraints in the dependency tree.

**G10** — `defer` schedules a function call to run immediately before the surrounding function returns. Multiple defers execute in **LIFO** (Last In, First Out) order. A common gotcha in loops: `defer` captures the loop variable by reference, so all deferred calls might see the final loop value. Use a new variable inside the loop to capture the current value.

**G11** — A **pointer** holds a memory address. Its zero value is `nil`. To create a pointer to a struct literal: `&MyStruct{Field: value}`.

**G12** — `init()` is a function run automatically before `main()` executes. It runs during package initialization. A package can have multiple `init()` functions; they execute in the order they appear in the source file.

**G13** — `new(T)` allocates zeroed memory for type `T` and returns a pointer `*T`. `make(T)` initializes built-in reference types (`map`, `slice`, `channel`) and returns the initialized value (not a pointer). `new` works on all types; `make` only on built-ins.

**G14** — Errors are values implementing the `error` interface (`Error() string`). Idiomatic pattern: `if err != nil { return err }`. `errors.Is(err, target)` checks semantic equality (wrapping). `errors.As(err, &target)` performs type assertion on wrapped errors. `==` only checks exact type/value, not wrapped errors.

**G15** — **Composite types:**
*   `map`: Zero value is `nil`.
*   `slice`: Zero value is `nil`.
*   `channel`: Zero value is `nil`.
*   `struct`: Zero value has all fields set to their respective zero values.

**G16** — A **race condition** occurs when two goroutines access shared data concurrently, with at least one write, without synchronization. The `-race` flag instruments code to detect these at runtime.
**Fix:** Use `sync.Mutex` to lock access or channels to pass ownership instead of sharing memory.

**G17** — `select` waits on multiple channel operations. Unlike `switch`, it chooses a case **non-deterministically** if multiple are ready. It can block if no case is ready. Timeout is implemented by selecting on `time.After(duration)`.
```go
select {
case <-time.After(1 * time.Second):
    fmt.Println("Timeout")
case data := <-ch:
    fmt.Println(data)
}
```

**G18** — **Escape analysis** determines if a variable lives on the stack or heap. If a variable's lifetime exceeds the function scope (e.g., returned or stored in a global), it escapes to the heap. Inspect with `go build -gcflags="-m"`. Heap allocation is slower (GC overhead) than stack allocation.

**G19** — `reflect` allows runtime inspection of types and values. `reflect.Type` is metadata; `reflect.Value` is the data. It incurs significant performance costs (no compile-time checks, dynamic dispatch). Justified for frameworks (serialization, dependency injection) but avoid in hot paths.

**G20** — `//go:embed` embeds files/directories at compile time into a `string` or `[]byte`. It works with `io/fs.FS` (e.g., `embed.FS`). Limitations: files must exist at compile time, cannot embed binary blobs larger than memory limits easily without streaming.

**R1** — **Ownership** ensures memory safety without a garbage collector. Rules: 1) Each value has exactly one owner. 2) When the owner goes out of scope, the value is dropped. 3) Values can be transferred (moved) but not copied by default.

**R2** — `String` is an owned, mutable, heap-allocated string. `&str` is an immutable, borrowed slice of bytes (string slice). Use `&str` for function arguments to avoid copying; use `String` when you need to own or mutate the data.

**R3** — **Borrowing** allows temporary access without ownership. Rules: You can have either one mutable reference (`&mut T`) OR any number of immutable references (`&T`), but not both simultaneously. References must be valid within their scope.

**R4** — `Result<T, E>` represents success (`Ok`) or failure (`Err`). `Option<T>` represents `Some(T)` or `None`. The `?` operator propagates errors.
```rust
use std::fs;
fn read_file() -> Result<String, std::io::Error> {
    fs::read_to_string("file.txt")
}
```

**R5** — `Box<T>` owns data on the heap (single owner). `Rc<T>` (Reference Counted) allows multiple owners in a single thread. `Arc<T>` (Atomic Reference Counted) allows multiple owners across threads. Use `Arc` when sharing data across threads.

**R6** — **Traits** define shared behavior. Unlike Go interfaces, traits can have default implementations. `dyn Trait` enables dynamic dispatch (vtable, runtime cost). `T: Trait` enables static dispatch (monomorphization, zero cost).

**R7** — The function fails because the compiler cannot determine which input's lifetime the return value should borrow.
**Fix:** Add explicit lifetime annotations.
```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
```

**R8** — `Send` means ownership can be transferred across threads. `Sync` means shared references can be sent across threads. `Rc<T>` is not `Send` because its reference count is not atomic. `Arc<T>` uses atomic reference counting, making it `Send` and `Sync`.

**R9** — Rust `async/await` compiles to a **state machine** (`Future`). The executor (e.g., Tokio) polls the future until completion. Unlike Go's goroutines (stack-based coroutines), Rust futures are zero-cost abstractions that do not allocate a stack per task.

**R10** — `unsafe` bypasses Rust's safety checks. You can: 1) Dereference raw pointers. 2) Call unsafe functions/extern blocks (FFI). 3) Access/modify mutable statics. Justified for FFI, low-level optimizations, or implementing safe abstractions over unsafe primitives.

**R11** — **Pattern matching** (`match`) exhaustively checks all possible values of an enum. `if let` matches a single pattern. The compiler enforces **exhaustiveness** in `match`, ensuring no cases are missed, preventing runtime errors.

**R12** — `Vec<T>` is a growable heap-allocated array. `[T; N]` is a fixed-size stack-allocated array. When `Vec` capacity is exceeded, it allocates a new, larger buffer and copies elements (usually doubling capacity).

**R13** — Rust **enums** (ADTs) can hold data in variants. Unlike C/Java enums, they are powerful types. `Option<T>` is `Some(T)` or `None`. Rust has no `null` to prevent null pointer exceptions; `Option` forces handling of absence.

**R14** — `impl` defines methods. `&self` borrows the instance. `&mut self` borrows mutably. `self` consumes the instance. **Associated functions** (e.g., `String::new()`) do not take `self` and act like constructors.

**R15** — **Derive macros** automatically implement traits. Common: `Debug`, `Clone`, `PartialEq`. Cannot derive if the trait requires functionality not available for the type (e.g., `Clone` on a struct containing `Rc` without `Clone` on inner type, or `FnOnce`).

**R16** — `Cow<'a, T>` (Clone on Write) holds either a borrowed reference or an owned value. It avoids allocation if the data is only read. If mutation is needed, it clones the data internally. Useful for functions that return data that might be borrowed or owned.

**R17** — **Modules** organize code. `mod` declares a module, `use` imports paths, `pub` makes items public. File system maps to modules (`mod.rs` or `mod` folder). Rust 2018 improved path resolution (`crate::` vs `super::`) and macro hygiene.

**R18** — `Pin<T>` guarantees a value's memory address does not move. Needed for **self-referential structs** and `async` (Futures may hold self-references). `Unpin` is a marker trait; if a type is `Unpin`, it can be moved freely.

**R19** — **Macros** generate code. `macro_rules!` is declarative (pattern matching). Procedural macros (derive, attribute) are Rust code that runs at compile time. **Hygiene** ensures macro variables don't clash with local variables.

**R20** — **Zero-cost abstractions** mean high-level features compile to efficient machine code. **Generics** use monomorphization (copying code per type). **Iterators** are inlined. **Traits** use static dispatch by default. This matches C performance while providing safety.
