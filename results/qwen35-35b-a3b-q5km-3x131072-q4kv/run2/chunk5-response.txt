DN1 ‚Äî **Class vs Struct:** `class` is a reference type (heap), supports inheritance, and has a default null. `struct` is a value type (stack/heap), does not support inheritance (except from `System.ValueType`), and cannot be null. Use `struct` for small, immutable data types (e.g., `Point`, `Vector`) to avoid heap allocation and copying overhead.

DN2 ‚Äî **IEnumerable vs IQueryable:** `IEnumerable<T>` executes queries in-memory (client-side), pulling all data before filtering. `IQueryable<T>` builds expression trees that translate to server-side queries (e.g., SQL). This matters because `IQueryable` pushes filtering/sorting to the database, reducing network traffic and improving performance.

DN3 ‚Äî **Async/Await & Task:** `async/await` enables non-blocking I/O by compiling state machines. `Task` is a reference type representing an async operation. `ValueTask` is a value type that avoids heap allocation if the operation completes synchronously. Use `ValueTask` in high-throughput scenarios where the operation often completes synchronously to reduce GC pressure.

DN4 ‚Äî **Dependency Injection (DI):** DI decouples object creation from usage. Lifetimes: **Transient** (new instance every time), **Scoped** (new instance per request), **Singleton** (one instance for app lifetime). **Bug:** Injecting a Scoped service into a Singleton captures the Scoped service's lifetime, causing it to be reused across requests (state leakage) or disposed prematurely if the Singleton outlives the Scoped scope.

DN5 ‚Äî **string vs ReadOnlySpan<char>:** `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a stack-based view over memory (no allocation). `Span<T>` allows mutable views. Use `Span<T>` for high-performance parsing or string manipulation to avoid creating intermediate string objects.

DN6 ‚Äî **Garbage Collector (GC):** GC manages memory by reclaiming unreachable objects. **Generations:** Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **LOH:** Large Object Heap (objects > 85KB) is not compacted frequently. `IDisposable` is needed for deterministic cleanup of unmanaged resources (files, handles) which GC does not track.

DN7 ‚Äî **ConfigureAwait:** `ConfigureAwait(false)` prevents capturing the current `SynchronizationContext`. Use it in library code to avoid deadlocks and context switching overhead. Do **not** use it in UI code (WPF/WinForms) where you need to update the UI thread after the await.

DN8 ‚Äî **.NET 8/9 Features:** **Primary Constructors** (C# 12) allow constructor args in class definition. **Collection Expressions** (`[1, 2]`) simplify array creation. **FrozenDictionary** is an immutable, high-performance dictionary.
*Example:*
```csharp
public class Service(int id) : IService { public int Id => id; }
// DI: services.AddSingleton<Service>(new Service(1));
```

DN9 ‚Äî **DbContext Exception:** Likely cause: `DbContext` is registered as Scoped but shared across threads or requests (e.g., injected into a Singleton). EF Core `DbContext` is not thread-safe. **Fix:** Ensure `DbContext` is injected as Scoped (per HTTP request) and never stored in a static or Singleton field.

DN10 ‚Äî **Record vs Class:** `record` provides value-based equality and is immutable by default (positional properties). `class` provides reference equality and is mutable. `record struct` combines value type semantics with record equality. Use `record` for DTOs/Models, `class` for entities with identity or mutable state.

DN11 ‚Äî **Ref/Out/In:** `ref` passes by reference (read/write). `out` passes by reference (must be assigned inside method). `in` passes by reference (read-only). Use `ref` for returning multiple values, `out` for constructors/initializers, `in` for large structs to avoid copying.

DN12 ‚Äî **Nullable Reference Types (NRT):** NRT (`string?`) is a compile-time warning system for reference types to prevent null dereferences. `int?` is a nullable value type (`Nullable<int>`) that allows `null` at runtime. NRT does not change runtime behavior, only static analysis.

DN13 ‚Äî **LINQ:** Language Integrated Query. **Method vs Query:** Syntactic sugar; both compile to the same IL. **Deferred Execution:** Queries are not executed until enumerated (e.g., `foreach`). This allows composition but can lead to unexpected execution timing.

DN14 ‚Äî **Generic Constraints:** Restrict type arguments. `where T : class` (reference), `where T : struct` (value), `where T : new()` (paramless ctor), `where T : IComparable<T>` (comparable). Combine: `where T : class, IComparable<T>` to ensure T is a reference type that can be compared.

DN15 ‚Äî **Abstract vs Interface:** `abstract` allows partial implementation and state. `interface` defines a contract only. **Default Interface Methods (C# 8+)** allow adding methods to interfaces without breaking existing implementations, blurring the line by allowing stateless logic in interfaces.

DN16 ‚Äî **Middleware Pipeline:** Components processing HTTP requests. `app.Use()` adds middleware, `app.Map()` branches the pipeline, `app.Run()` is terminal. **Ordering:** Critical (e.g., Auth before Routing). **Exception Handling:** Middleware can catch exceptions and return a response, preventing the pipeline from crashing.

DN17 ‚Äî **Channel<T>:** Async producer-consumer primitive. `BlockingCollection<T>` is synchronous. **Bounded Channel:** Set capacity limit to apply backpressure (producer waits if buffer full). Example: `Channel.CreateBounded<int>(100)`.

DN18 ‚Äî **Source Generators:** Compile-time code generation. **Vs Reflection:** Faster (no runtime cost). **Vs IL Weaving:** Happens before compilation, no post-build step. **IncrementalGenerator:** Regenerates only when source changes (performance), preferred over `ISourceGenerator` for large projects.

DN19 ‚Äî **TPL Dataflow:** Pipeline processing library. `ActionBlock` (process, no output), `TransformBlock` (process, output), `BufferBlock` (queue). **Vs Channels:** Dataflow offers higher-level orchestration (linking, cancellation, completion), while Channels are lower-level primitives.

DN20 ‚Äî **Minimal APIs:** Lightweight entry points without controllers. **Trade-offs:** Less boilerplate, but harder to organize complex logic/test. **Middleware:** Same as MVC. **OpenAPI:** Supported via Swashbuckle. **Route Groups:** Organize endpoints (`MapGroup`). **Filters:** Intercept requests/responses (e.g., validation).

P1 ‚Äî **List vs Tuple:** `list` is mutable, ordered collection. `tuple` is immutable, ordered collection. `set` is mutable, unordered, unique elements. `frozenset` is immutable set. Use `tuple` for fixed data, `set` for uniqueness.

P2 ‚Äî **GIL:** Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects **CPU-bound** workloads. **Workarounds:** Use `multiprocessing` (separate processes), C-extensions (release GIL), or `asyncio` (I/O-bound).

P3 ‚Äî **Deepcopy vs Copy:** `copy` creates a shallow copy (new container, same object refs). `deepcopy` creates a new container and recursively copies objects. **Bug:** Modifying a nested object in a shallow copy affects the original.

P4 ‚Äî **Decorators:** Functions that wrap other functions.
```python
import time
def timeit(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 ‚Äî **Generators:** Functions that yield values lazily. **Vs List:** No memory for all items, execution pauses at `yield`. `yield from` delegates to another generator.

P6 ‚Äî **Asyncio:** Event loop manages tasks. `asyncio.run()` starts loop. `asyncio.gather()` runs tasks concurrently. `asyncio.create_task()` schedules task. `time.sleep()` blocks the loop; use `await asyncio.sleep()` instead.

P7 ‚Äî **MRO:** Method Resolution Order. Python 3 uses **C3 Linearization**. In `D(B, C)`, MRO is `D -> B -> C -> A`. `D().who()` calls `C.who()` because `C` appears before `A` in the linearization and overrides `A`.

P8 ‚Äî **Metaclass:** Class of a class. `__init_subclass__` is easier for subclass hooks (e.g., registry). **Metaclass** is for controlling class creation (e.g., ORM models). Use `__init_subclass__` for configuration, Metaclass for structural control.

P9 ‚Äî **Multiprocessing vs Threading:** `multiprocessing` bypasses GIL (CPU-bound), separate memory. `threading` shares memory (GIL), good for I/O. `concurrent.futures` is high-level API. **Serialization:** Multiprocessing requires pickling arguments, which fails for unserializable objects (e.g., open files).

P10 ‚Äî **__slots__:** Prevents creation of `__dict__`. Saves memory (no dict overhead) and prevents dynamic attribute assignment. Use for classes with many instances and fixed attributes.

P11 ‚Äî **Virtual Environment:** Isolated Python environment. **Why:** Dependency management. **Differences:** `venv` (stdlib), `virtualenv` (3rd party, more features), `conda` (package manager + env, binary packages).

P12 ‚Äî **Comprehensions:** Concise loops. `map/filter` return iterators. Prefer comprehensions for readability. Use `map/filter` for functional style or when function is already defined.

P13 ‚Äî **== vs is:** `==` checks value equality. `is` checks identity (memory address). **Interning:** Small integers (-5 to 256) are cached. `a is b` returns True for small ints due to interning, not value equality.

P14 ‚Äî **Args/Kwargs:** `*args` (tuple), `**kwargs` (dict). `/` marks positional-only args. `*` marks keyword-only args. Use `/` for API compatibility, `*` to enforce keyword usage.

P15 ‚Äî **Context Managers:** Manage resources (`with`). `__enter__`/`__exit__` define protocol. `@contextmanager` uses generator. **Example:** `Lock` acquisition, temporary directory creation.

P16 ‚Äî **Type Hinting:** Static analysis. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. `mypy`/`pyright` check types before runtime to catch errors early.

P17 ‚Äî **Data Model:** `__repr__` (dev), `__str__` (user). `__eq__` vs `__hash__`: If `__eq__` is defined, `__hash__` becomes `None` (unhashable) unless explicitly defined. `__getattr__` (missing attr), `__getattribute__` (all attrs).

P18 ‚Äî **Descriptors:** Manage attribute access (`__get__`, `__set__`, `__delete__`). `property`, `classmethod`, `staticmethod` are descriptors. Used for validation, computed properties, and method binding.

P19 ‚Äî **Import System:** `sys.path` lists directories. **Relative:** `from . import`. **Absolute:** `from pkg import`. `__init__.py` marks package. **Namespace:** No `__init__.py`, allows splitting packages. **Finder:** Locates module. **Loader:** Loads module.

P20 ‚Äî **Dataclasses:** Reduce boilerplate (`__init__`, `__repr__`). **Vs attrs:** attrs more flexible. **Vs Pydantic:** Pydantic adds validation/serialization. **__post_init__:** Logic after init. **Frozen:** Immutable. Choose Pydantic for validation-heavy data models.
