JS1 — **var** is function-scoped and hoisted (initialized as `undefined`). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages execution. The **call stack** runs synchronous code. When empty, the loop processes the **microtask queue** (Promises) first, then the **macrotask queue** (setTimeout, I/O). This ensures async callbacks run before the next render.

JS3 — A **closure** is a function retaining access to its lexical scope. Example: `function outer() { let x=1; return () => x; }`. **For loop bug**: `var` is function-scoped, so all callbacks share the same `i`. **Fix**: Use `let` (block-scoped) or an IIFE to capture the value.

JS4 — **Promise** represents async completion. **async/await** is syntactic sugar for Promises. Error handling: `try/catch` for await, `.catch()` for Promises. **Promise.all** fails on first rejection; **Promise.allSettled** waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 — **interface** defines shape (can extend, merge). **type** defines value (unions, primitives, intersections). Use **interface** for public API contracts, **type** for unions/complex logic. **Discriminated unions** are unions with a literal type field to narrow types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — **Prototypal inheritance** links objects via `__proto__`. **Object.create(proto)** creates an object with `proto` as its prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` and `constructor` to manage the prototype chain.

JS7 — **this** depends on call site. **4 Rules**: 1. Default (window/undefined), 2. Implicit (obj.method), 3. New (new Obj), 4. Explicit (call/apply/bind). **bind()** returns a new function with fixed `this`. It **cannot** be re-bound.

JS8 — **WeakRef** holds weak reference to object (GC safe). **FinalizationRegistry** runs cleanup when object is GC'd. **WeakMap** keys must be objects (GC can track them). This allows keys to be garbage collected if no other references exist, preventing memory leaks.

JS9 — **Hidden classes** optimize property access by grouping objects with same shape. **Inline caching** caches property lookup results. **Deoptimization** occurs when shape changes (e.g., adding property dynamically). Patterns causing deopt: `eval`, `with`, dynamic property addition.

JS10 — **Tree-shaking** removes unused exports. **ES Modules** use static `import/export`, enabling static analysis. **CommonJS** uses dynamic `require`, making static analysis impossible, so bundlers cannot safely remove unused code.

JS11 — **null** is intentional absence; **undefined** is uninitialized. `== null` checks both. **Nullish coalescing (??)** returns right side if left is null/undefined. **Optional chaining (?.)** safely accesses nested properties without throwing.

JS12 — **map** transforms, **filter** selects, **reduce** accumulates. **reduce** with initial value prevents first element from being init. **flatMap** maps then flattens one level, useful for transforming arrays of arrays into a single array.

JS13 — **Template literals** use backticks for multi-line strings and interpolation `${}`. **Tagged templates** call a function with strings and values. Example: `sql`SELECT * FROM ${table}`; function sql(strings, ...values) { return strings[0] + values[0]; }`.

JS14 — **Destructuring** extracts values. **Object**: `{a}`. **Array**: `[a]`. **Defaults**: `{a=1}`. **Rename**: `{a: b}`. **Rest**: `...rest` collects remaining items. Works in function arguments and declarations.

JS15 — **for...in** iterates enumerable keys/properties (including inherited). **for...of** iterates values of iterable objects. Avoid **for...in** on arrays because it iterates properties (including non-index), is slower, and order isn't guaranteed.

JS16 — **Generics** use type params (e.g., `T`). **Constraints**: `T extends Foo`. **Conditional types**: `T extends U ? X : Y`. **infer**: extracts type from type (e.g., `infer R` in `Promise<R>`). Example: `type Unwrap<T> = T extends Promise<infer R> ? R : T`.

JS17 — **CommonJS** (`require`) is dynamic, Node default historically. **ES Modules** (`import`) are static, browser native. Node uses **type: "module"** in `package.json` to force ESM. ESM supports top-level await; CommonJS does not.

JS18 — **Structural typing** matches shapes, not names. **Excess property checks** apply to object literals (not variables). **extends** checks assignment compatibility. **satisfies** narrows type without widening. **Variance** (4.7+): `in` (contravariant), `out` (covariant) for generic parameters.

JS19 — **Proxy** intercepts operations (get, set, etc.). **Reflect** provides default implementations. Example: `new Proxy(obj, { set: (t, k, v) => { if (v < 0) throw Error; return true; } })`. Used for validation, logging, or reactive frameworks.

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic ops. **Ordering**: Memory barriers ensure visibility. **Atomics.wait** blocks until **Atomics.notify** wakes threads. Used for lock-free data structures.

B1 — **$@** expands to individual quoted arguments. **$*** expands to all args as one string. **"$@"** preserves individual arguments (safe for loops). **"$*"** joins them with first char of IFS (usually space).

B2 — **set -e**: Exit on error. **set -u**: Error on undefined vars. **set -o pipefail**: Exit if any command in pipe fails. Together they make scripts fail fast and safely.

B3 — **$(command)** is modern command substitution. **`command`** is legacy. **$((expression))** is arithmetic expansion (shell built-in). **expr** is an external command, slower and less feature-rich.

B4 — **Process substitution** `<(command)` creates a file descriptor for command output. Used when a command expects a file but you have a stream. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — Use **`for f in *`** or **`find . -print0 | while IFS= read -r -d '' f`**. **`for f in $(ls)`** is dangerous because word splitting breaks filenames with spaces or newlines.

B6 — **Here document** `<<EOF` reads stdin until EOF. **Here string** `<<<` passes a string as stdin. **`<<-EOF`** strips leading tabs from the here document content.

B7 — `cmd > file 2>&1`: stdout to file, then stderr to stdout (file). Both in file. `cmd 2>&1 > file`: stderr to stdout (terminal), then stdout to file. Only stdout in file. `cmd &> file`: Bash shorthand, both to file.

B8 — Use `mkdir` for locking (atomic). Script:
```bash
LOCKDIR="/tmp/lock"
trap 'rm -rf "$LOCKDIR"' EXIT TERM
mkdir "$LOCKDIR" || exit 1
# ... work ...
```
`mkdir` is atomic on POSIX. `trap` ensures cleanup on kill.

B9 — **Single quotes**: Literal (no expansion). **Double quotes**: Expand vars/globs. **No quotes**: Expand vars/globs/split. Example: `echo $HOME` (no quotes) vs `echo "$HOME"` vs `echo '$HOME'`.

B10 — **`[ -f file ]`** (exists), **`[ -d dir ]`**, **`[ -r file ]`** (readable). **`[[ ]]`** is Bash-specific, supports `&&`, `||`, regex, no word splitting. `[ ]` is POSIX.

B11 — **$?**: Exit code of last cmd. **$$**: Current shell PID. **$!**: PID of last background job. **$#**: Number of arguments passed.

B12 — **`func() {}`** or **`function func {}`**. Return values via `echo` (stdout) or `return` (exit code). `return` only works for exit codes; use stdout for data.

B13 — **PATH** is colon-separated dirs. Bash searches PATH for external commands. **Builtin** is internal (e.g., `cd`). **External** is binary. Use **`type`** or **`which`** to check.

B14 — **trap** sets handlers. `trap 'cmd' EXIT` runs on exit. `trap 'cmd' INT` on Ctrl+C. Execution order: Handlers run when signal received or shell exits. Multiple signals queue or interrupt.

B15 — **xargs** builds command lines from stdin. **command substitution** runs once. **xargs -0** handles null-delimited input (from `find -print0`). **xargs -P** runs jobs in parallel.

B16 — **Arrays**: `arr=()`, append `arr+=()`, length `${#arr[@]}`. **Associative**: `declare -A`, keys are strings. Syntax differs for iteration (`for k in "${!arr[@]}"`).

B17 — **Subshell** `(...)` runs in new process (vars lost). **Group** `{ ...; }` runs in current shell (vars kept). **Pipe** `| while` runs loop in subshell, so variable changes are lost after loop.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (cannot catch). Graceful shutdown: `trap` SIGTERM to cleanup, wait for background jobs, then exit.

B19 — Split log, `xargs -P` process chunks, merge results. **Trade-off**: Parallelism vs overhead/error handling. **Error propagation**: Check exit codes in parallel jobs, aggregate failures, exit non-zero if any fail.

B20 — **exec** replaces shell process. **FD manipulation**: `exec 3>file` opens FD 3. **Coprocess** `coproc` creates bidirectional pipe. Use exec for persistent FDs, coprocess for interactive child processes.

PS1 — **Write-Output** sends objects to pipeline. **Write-Host** writes to screen only (breaks pipeline). Use Write-Output for automation; Write-Host for user interaction.

PS2 — **Pipeline** passes objects. **ForEach-Object** (cmdlet) processes pipeline items one-by-one. **foreach** (statement) iterates a collection in memory. Use cmdlet for streaming, statement for pre-loaded collections.

PS3 — **[PSCustomObject]** is ordered, typed object. **[hashtable]** is unordered, dynamic. Convert: `[PSCustomObject]@{Key='Value'}`. Use PSCustomObject for structured data, hashtable for config.

PS4 — **Terminating error** stops execution (throw). **Non-terminating** continues (warning). **-ErrorAction Stop** converts non-terminating to terminating. **try/catch** handles terminating errors.

PS5 — **Splatting** `@params` passes hashtable elements as arguments. **Hashtable param** passes the whole hashtable as one argument. Splatting is cleaner for cmdlets with many params.

PS6 — **Invoke-Command** runs one-off remote. **Enter-PSSession** interactive. Uses **WinRM** (HTTP/HTTPS) or **SSH** (v7+). SSH is preferred for cross-platform.

PS7 — Returns array even for single item. `GetType().Name` is `String[]`. If 0 items, still array. **Pattern**: Initialize `$result = @()` and ensure return is always array to avoid type confusion.

PS8 — **[scriptblock]** is code container. Pass via `-ArgumentList`. **Double-hop** = auth pass-through issue. Solve with **CredSSP** or **Invoke-Command -ArgumentList** to pass creds securely.

PS9 — **@()** forces array. **[array]** casts. **$null** index = error. **Empty array** index = error. Always check count before indexing.

PS10 — **-eq** (value), **-like** (glob), **-match** (regex). **-ceq** (case-sensitive). **-contains** (left side contains right). **-in** (right side in left).

PS11 — **Module** = collection of commands. **.psm1** (code), **.psd1** (manifest), **Binary** (DLL). **Import-Module** loads module into session.

PS12 — **"..."** interpolates. **'...'** literal. **@"..."@** here-string (multiline). Expressions: `"Value: $var"`.

PS13 — **Get-Member** (`gm`) lists properties/methods. **Properties** (data), **Methods** (func), **NoteProperties** (dynamic). **Select-Object** filters props. **Where-Object** filters objects.

PS14 — **Classes** (v5+) are C#-like. Limitations: No interfaces (v5), limited inheritance. Use for complex state management, not simple scripts.

PS15 — **[CmdletBinding()]** enables advanced features. **SupportsShouldProcess** enables `-WhatIf`. **Begin/Process/End** blocks handle streaming data.

PS16 — **Providers** virtualize data (Registry, File). **Get-ChildItem** works on all. Registry: `Get-ChildItem HKLM:\Software`.

PS17 — **[runspace]** = isolated execution. **[runspacepool]** = reuse. **ForEach-Object -Parallel** (v7+) simplifies parallelism. Manual runspace for complex control.

PS18 — **PSScriptAnalyzer** = linting. **Rules** enforce standards. **Severity** (Error, Warning). **Suppress** via comments. Integrate via CI (e.g., GitHub Actions).

PS19 — **$using:** passes local var to remote. **-ArgumentList** serializes args. **CLIXML** limits depth. Complex objects may fail serialization; use JSON or custom types.

PS20 — **DSC** = config management. **Resources** (actions), **Config** (desired state), **LCM** (enforcer). **PS7** = DSC Core. Replaced by **Ansible**, **Terraform**, or **Chocolatey** in modern IaC.
