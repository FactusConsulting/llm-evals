T1 — Easy: `tofu plan` calculates the execution plan and displays changes without applying them. `tofu apply` executes the changes to reach the desired state.

T2 — Easy: State is a JSON file mapping OpenTofu resources to real-world infrastructure. Remote state is important for team collaboration, state locking, and security (storing state off local machines).

T3 — Medium: `count` uses integer indices; `for_each` uses map/set keys. `for_each` is preferred when resources are added/removed dynamically to avoid index shifting issues that cause unintended resource recreation.

T4 — Medium: This is "drift". `tofu plan` detects the difference between state and reality. `tofu apply` will attempt to reconcile the configuration to match the desired state (or fail if the resource is immutable).

T5 — Medium:
```hcl
resource "null_resource" "vm" {
  for_each = toset(["web1", "web2", "web3"])
  triggers = {
    name = each.key
  }
}
```

T6 — Hard: `terraform_remote_state` reads raw state files directly (less secure, exposes internal structure). `output` values use module outputs (preferred, encapsulated, type-safe). Recommended alternative: Module outputs via `output` blocks.

T7 — Hard: Provisioners run asynchronously; the output is evaluated before the file write completes. Fix: Add `depends_on = [null_resource.kubeconfig]` to the dependent resource to ensure the provisioner finishes before the output is consumed.

T8 — Medium: `import` CLI is manual and hard to track. `import block` (1.5+) is declarative in config, allowing import logic to be version-controlled and integrated into CI/CD pipelines.

T9 — Easy: A variable is an input parameter passed into a module. An output is a value exposed by a module to the caller.

T10 — Easy: A provider is a plugin that interacts with cloud APIs. Without `tofu init`, providers aren't downloaded, causing `plan` to fail with "Provider not installed" errors.

T11 — Easy: The lock file records exact provider versions. It should be committed to ensure reproducible builds across different environments and developers.

T12 — Easy: `tofu destroy` removes all managed resources. Target a single resource with `tofu destroy -target=resource.address`.

T13 — Easy: A data source fetches existing data without creating resources. Example: Using `data "aws_ami"` to get the latest Ubuntu image ID instead of hardcoding it.

T14 — Medium: Locals are computed values within a module scope. Variables are inputs from outside. Use locals for internal logic (e.g., combining strings) to keep variables clean.

T15 — Medium: Lifecycle controls resource behavior. `create_before_destroy` (zero downtime), `prevent_destroy` (safety against accidental deletion), `ignore_changes` (ignore external drift).

T16 — Medium: A `moved` block allows refactoring resource paths (e.g., moving a resource to a new module) without destroying and recreating it, preserving state mapping.

T17 — Hard: The dependency graph determines execution order. Implicit dependencies come from references; explicit come from `depends_on`. `depends_on` serializes execution, reducing parallelism and slowing down plans.

T18 — Hard: 1. `-refresh=false` (faster, stale state), 2. Split modules (smaller graphs), 3. Parallelism (Cloud/Enterprise), 4. State filtering (exclude unchanged resources). Tradeoffs: Stale state, complexity, cost.

T19 — Hard: Locking prevents concurrent writes. If a lock is stale (crash), use `tofu force-unlock`. Risk: Concurrent writes can corrupt state if the lock is broken incorrectly.

T20 — Hard: 1. Backup local state. 2. Add `backend "s3"` config. 3. Run `tofu init -backend-config`. 4. `tofu state pull` (old) -> `tofu state push` (new). 5. Verify with `tofu state list`.

A1 — Easy: Playbook = Execution script. Role = Directory structure for logic. Task = Single action within a playbook.

A2 — Easy: `ansible_facts` contains gathered system info. Access OS family via `{{ ansible_os_family }}`.

A3 — Medium: `command` (no shell), `shell` (shell features like pipes), `raw` (no setup/transfer, runs immediately). Use `raw` for bootstrapping.

A4 — Medium: Idempotency = Safe to run multiple times. `apt` (idempotent), `shell: rm -rf /` (not).

A5 — Medium:
```yaml
- name: Install htop
  package:
    name: htop
    state: present
  when: ansible_os_family == "Debian"
- name: Install epel + htop
  package:
    name:
      - epel-release
      - htop
    state: present
  when: ansible_os_family == "RedHat"
```

A6 — Hard: Priority: Extra_vars > Host_vars > Group_vars > Role defaults. `set_fact` is runtime, usually overrides defaults but lower than extra_vars.

A7 — Hard: Use `serial: 3` in the play. Group nodes. Use `wait_for` or `meta: flush_handlers` to ensure the first batch completes before the next starts.

A8 — Medium: `ansible-vault encrypt_string 'value' --name 'var_name'`. Encrypts specific string, not the whole file.

A9 — Easy: Inventory = List of hosts. Static = File-based. Dynamic = Script/API-based (e.g., AWS EC2).

A10 — Easy: `ansible webservers -m shell -a "df -h"`.

A11 — Easy: Handler = Task triggered by `notify`. Runs only if notified, typically after all tasks complete.

A12 — Easy: `ansible.cfg` configures Ansible. Settings: `inventory`, `remote_user`, `forks`.

A13 — Easy: `register` saves task output to a variable. Use in `when` conditions or `debug` tasks.

A14 — Medium: `import_tasks` (static, compile time), `include_tasks` (dynamic, runtime). Use `import` for fixed lists, `include` for conditional loading.

A15 — Medium: Galaxy = Role/Collection repository. Use `ansible-galaxy install -r requirements.yml` to install dependencies.

A16 — Medium: Connection = Transport. `ssh` (remote Linux), `local` (control node), `network_cli` (network devices).

A17 — Hard: `delegate_to: localhost`, `run_once`. Example: Gather remote facts, then update a local config file on the control node.

A18 — Hard: 1. Increase `forks`, 2. Enable `pipelining`, 3. Use `fact_caching`, 4. Use `mitogen`. Tradeoffs: Resource usage, complexity.

A19 — Hard: `{{ }}` (output), `{% %}` (logic), `{# #}` (comment).
```jinja
upstream backend {
{% for server in backends %}
  server {{ server.ip }} weight={{ server.weight }}; # Health: {{ server.health }}
{% endfor %}
}
```

A20 — Hard: `serial: 1`. Pre-task: Deregister from LB. Post-task: Register. Health check: `uri` module. Rollback: `block/rescue` to revert changes on failure.
