DN1 — Class is a reference type stored on the heap, supports inheritance, and can be null. Struct is a value type stored on the stack (or inline), does not support inheritance (except from System.ValueType), and cannot be null. Use a struct for small, immutable data representing a single value (e.g., Point, Color) to avoid allocation overhead.

DN2 — IEnumerable<T> executes queries in memory (client-side), while IQueryable<T> builds expression trees for server-side execution (e.g., SQL). This distinction matters for database queries because IQueryable allows filtering and projection to be pushed to the database, reducing data transfer and improving performance.

DN3 — async/await uses a state machine to suspend/resume execution without blocking threads. Task is a reference type object; ValueTask is a struct (zero allocation). Use ValueTask for async methods that often complete synchronously or return value types to avoid allocation overhead in hot paths.

DN4 — Dependency Injection (DI) manages object lifecycles. Transient (new instance every request), Scoped (new per request scope), Singleton (one instance app-wide). Common bug: Injecting a Scoped service into a Singleton captures the Singleton's lifetime, causing stale data or ObjectDisposedException when the Scoped scope ends.

DN5 — string is immutable and heap-allocated. ReadOnlySpan<char> is a stack-allocated view (no allocation). Span<T> is used for high-performance buffer manipulation to avoid copying data and reduce GC pressure, especially when processing large byte arrays or strings.

DN6 — The GC manages heap memory automatically. Generations: Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). Large Object Heap (LOH) stores objects >85KB. IDisposable is needed for unmanaged resources (files, handles) because the GC only manages managed memory and cannot track external resource lifecycles.

DN7 — ConfigureAwait(false) prevents capturing the current SynchronizationContext. Use it in library code to avoid deadlocks and improve performance. Do NOT use it in UI code (WPF/WinForms) where the context is required for UI thread updates.

DN8 — Primary constructors (C# 12) allow defining constructor parameters in the class signature. Collection expressions create arrays/lists concisely. FrozenDictionary is immutable.
Example:
```csharp
public class MyService(int x) : IService { public int X => x; }
// Registration
services.AddSingleton<MyService>(new MyService(10));
```

DN9 — Cause: DbContext is not thread-safe. High concurrency + wrong lifetime (Singleton) causes race conditions/disposal. Fix: Register DbContext as Scoped (`AddScoped<DbContext>()`). EF Core scopes DB connections to the DI scope (usually HTTP request).

DN10 — Record provides value equality (props compared) and immutability by default. Class provides reference equality. Record struct combines value type semantics with record features. Use records for DTOs/Models, classes for mutable entities.

DN11 — ref (pass reference, modify original), out (pass reference, must assign before return), in (pass reference, read-only). Use ref for large structs to avoid copy, out for multiple return values, in for large read-only data.

DN12 — Nullable Reference Types (NRT) warn about nulls on reference types at compile-time (`string?`). Nullable Value Types (`int?`) allow value types to be null at runtime. NRT is static analysis; Value Types are runtime behavior.

DN13 — LINQ is Language Integrated Query. Method syntax (`list.Where`) vs Query syntax (`from x in list`). Deferred execution means query logic isn't run until enumeration (e.g., `foreach`).

DN14 — Generic constraints restrict type arguments (`where T : class`). `new()` requires parameterless constructor. Combine with `&` (e.g., `where T : class, IComparable<T>`). Use to enforce capabilities on generics.

DN15 — Abstract class allows code + contract. Interface defines contract only. Default interface methods (C# 8) allow adding behavior to interfaces without breaking existing implementations.

DN16 — Middleware pipeline processes requests. `app.Use()` adds middleware, `app.Map()` branches for specific paths, `app.Run()` is terminal. Ordering matters (Auth before MVC). Exception handling middleware catches errors from downstream.

DN17 — Channel<T> is an async producer-consumer queue. BlockingCollection<T> is blocking (older). Bounded channels enforce backpressure (max size), preventing memory exhaustion when producer is faster than consumer.

DN18 — Source Generators generate code at compile-time (faster than reflection). Incremental generators (C# 11) only regenerate when inputs change, improving build times vs full ISourceGenerator regeneration.

DN19 — TPL Dataflow (ActionBlock, TransformBlock, BufferBlock) creates pipelines with flow control. Channels are simpler for basic queues. Dataflow offers more complex backpressure and completion signaling.

DN20 — Minimal APIs use delegates for endpoints (no controllers). Trade-offs: Less boilerplate, harder to scale complex auth/middleware, OpenAPI support via extensions. Route groups organize endpoints. Endpoint filters apply logic to specific endpoints.

P1 — List (mutable, ordered), Tuple (immutable, ordered). Set (mutable, unique), Frozenset (immutable). Use tuple for fixed data, set for uniqueness.

P2 — GIL allows only one thread to execute Python bytecode. Affects CPU-bound workloads. Workarounds: Multiprocessing (separate processes), C-extensions (release GIL), Asyncio (I/O bound).

P3 — Deepcopy recursively copies objects. Copy (shallow) copies references. Example: `list1 = [[1]]; list2 = copy(list1); list2[0].append(2)` affects list1.

P4 — Decorator wraps function.
```python
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.perf_counter() - start:.2f}s")
        return result
    return wrapper
```

P5 — Generators yield values lazily. `yield` pauses function state. `yield from` delegates to another generator. Differs from list by not storing all items in memory.

P6 — Event loop schedules coroutines. `asyncio.run()` starts loop. `gather()` runs tasks concurrently. `create_task()` schedules task. `time.sleep()` blocks loop; use `asyncio.sleep()` for non-blocking wait.

P7 — MRO uses C3 linearization. D(B, C) -> D, B, C, A. D inherits who from C (overrides A). Prints "C". Diamond inheritance resolved by order of bases.

P8 — Metaclass controls class creation (e.g., ORM). `__init_subclass__` is simpler hook for subclass initialization. Use metaclass for complex class structure control, `__init_subclass__` for config validation.

P9 — Multiprocessing (CPU-bound, bypass GIL, heavy), Threading (I/O-bound, GIL, light), concurrent.futures (high-level API). Multiprocessing requires pickling (serialization) for args/return values.

P10 — `__slots__` defines fixed attributes. Prevents `__dict__` creation. Saves memory, prevents dynamic attribute assignment. Impact: Reduces per-instance memory overhead.

P11 — Virtual env isolates dependencies. venv (stdlib), virtualenv (3rd party), conda (env + packages). Use to avoid dependency conflicts between projects.

P12 — Comprehensions are concise loops. map/filter are functional. Prefer comprehensions for readability and Pythonic style.

P13 — `==` compares values, `is` compares identity. Object interning caches small integers (-5 to 256), so `a is b` is True for small ints.

P14 — `*args` (tuple), `**kwargs` (dict). `/` forces positional-only, `*` forces keyword-only. Use for API clarity and safety.

P15 — Context managers manage resources (`with`). `__enter__/__exit__` class-based, `@contextmanager` generator-based. Example: `threading.Lock` or DB connection.

P16 — Type hints enable static analysis. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. mypy/pyright check types before runtime to catch errors.

P17 — `__repr__` (dev/debug), `__str__` (user). `__eq__` without `__hash__` makes object unhashable. `__getattr__` (missing attr), `__getattribute__` (all attr).

P18 — Descriptors manage attribute access (`__get__`, `__set__`, `__delete__`). `property`, `classmethod`, `staticmethod` are descriptors. Used for computed properties and method binding.

P19 — Import system uses `sys.path`. Relative (`from . import`) vs Absolute (`from pkg import`). `__init__.py` marks package. Namespace packages (PEP 420) omit `__init__.py`. Finder locates, Loader executes.

P20 — Dataclasses auto-generate dunder methods. attrs (more features), Pydantic (validation). `__post_init__` runs after init. Frozen dataclasses are immutable. Choose Pydantic for validation-heavy data.
