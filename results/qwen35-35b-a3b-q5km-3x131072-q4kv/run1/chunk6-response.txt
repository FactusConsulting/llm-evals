JS1 — **var** is function-scoped and hoisted (declaration moved to top, value undefined). **let/const** are block-scoped and hoisted but in a Temporal Dead Zone (TDZ) until declaration. **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **Event Loop** manages execution order. **Call Stack** runs synchronous code. **Microtask Queue** (Promises, queueMicrotask) runs after the stack clears but before rendering. **Macrotask Queue** (setTimeout, I/O) runs after microtasks and rendering. Loop: Stack empty -> Microtasks -> Render -> Macrotasks.

JS3 — A **closure** is a function retaining access to its lexical scope even when executed outside that scope. **Bug:** In a loop, `var` shares one variable reference, causing all callbacks to log the final value. **Fix:** Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents an async result (pending/fulfilled/rejected). **async/await** is syntactic sugar for promises. **Error handling:** `try/catch` for async/await, `.catch()` for promises. **Promise.all** fails fast on first rejection. **Promise.allSettled** waits for all to settle (success/fail) and returns status objects.

JS5 — **Interface** defines shape for classes/objects, supports merging, and is erased at runtime. **Type** defines value shapes, supports primitives/unions, and is erased. Use **Interface** for public API contracts/classes. Use **Type** for unions/intersections/primitives. **Discriminated Unions** are unions with a literal type field used to narrow types (e.g., `{type: 'a', data: A} | {type: 'b', data: B}`).

JS6 — **Prototypal inheritance** links objects via a prototype chain (`__proto__`). **Object.create(proto)** creates a new object with `proto` as its prototype. **ES6 Classes** are syntactic sugar over prototypes, using `extends` to set up the prototype chain and `constructor` for initialization.

JS7 — **this** binding rules (priority): 1. **New** (constructor), 2. **Explicit** (call/apply/bind), 3. **Implicit** (obj.method), 4. **Default** (window/undefined). **bind()** returns a new function with bound `this`. It **cannot** be re-bound (subsequent binds are ignored).

JS8 — **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** notifies when a weakly held object is collected. Use for caches or cleanup. **WeakMap** keys must be objects because primitives are immutable and GC cannot track them; objects allow GC to detect when the key is no longer referenced.

JS9 — V8 optimizes via **Hidden Classes** (object shape), **Inline Caching** (caching property lookup), and **Deoptimization** (reverting to slower code if shape changes). **Deoptimization patterns:** Adding/deleting properties dynamically, changing property types, or using `arguments` object.

JS10 — **Tree-shaking** removes unused code. **ES Modules** use static `import/export`, allowing static analysis to remove dead code. **CommonJS** uses dynamic `require()`, which cannot be analyzed statically, preventing effective tree-shaking.

JS11 — **null** is intentional absence of value. **undefined** is unassigned. **==** coerces types (null == undefined is true). **===** checks strict equality (null === undefined is false). **??** (nullish coalescing) returns right side if left is null/undefined. **?.** (optional chaining) safely accesses nested properties.

JS12 — **map()** transforms elements. **filter()** selects elements. **reduce()** accumulates values. **Initial value** in reduce sets the accumulator start (otherwise first element is used). **flatMap()** maps then flattens one level; useful for transforming arrays of arrays into a single array.

JS13 — **Template literals** use backticks for multi-line strings and interpolation `${}`. **Tagged templates** call a function with string parts and values. Example: `sql`tag`SELECT * FROM users WHERE id = ${id}`; the function receives `['SELECT * FROM users WHERE id = ', '']` and `id`.

JS14 — **Destructuring** extracts values from objects/arrays. **Object:** `{a, b}`. **Array:** `[a, b]`. **Defaults:** `{a = 1}`. **Renaming:** `{a: b}`. **Rest operator** (`...`) collects remaining elements/properties into an array/object.

JS15 — **for...in** iterates over enumerable property keys (including inherited). **for...of** iterates over iterable values. Avoid **for...in** on arrays because it includes inherited properties and is slower; use **for...of** or standard `for` loops.

JS16 — **Generics** allow type parameters (e.g., `T`). **Constraints** (`T extends U`) restrict types. **Conditional types** (`T extends U ? X : Y`) select types based on condition. **infer** extracts types from conditional types. Example: `type First<T> = T extends (infer U)[] ? U : never`.

JS17 — **CommonJS** (`require`) is dynamic, Node default historically. **ES Modules** (`import`) are static, support top-level await. Node handles both based on file extension or `"type": "module"` in `package.json`, which forces all `.js` files to be treated as ESM.

JS18 — **Structural typing** checks compatibility by shape, not name. **Excess property checks** apply to object literals to prevent typos. **extends** checks assignability. **satisfies** checks type without widening the type. **Variance** (in/out) defines how subtypes relate (e.g., `Array<out T>` is covariant).

JS19 — **Proxy** intercepts operations (get, set, etc.). **Reflect** provides default implementations for Proxy traps. **Example:** A Proxy can validate property assignments before setting them, or log access for debugging.

JS20 — **SharedArrayBuffer** allows multiple threads to share memory. **Atomics** provide atomic operations (wait, notify, add). **Ordering guarantees** ensure memory visibility across threads. **Atomics.wait** blocks until notified; **Atomics.notify** wakes waiting threads.

B1 — **$@** expands to individual quoted arguments. **$*** expands to a single string of all arguments. **"$@"** preserves each argument as a separate word. **"$*"** joins all arguments into one string separated by the first character of IFS.

B2 — **set -e** exits on error. **set -u** errors on undefined variables. **set -o pipefail** returns error if any command in a pipeline fails. Together they enforce strict error handling.

B3 — **$(command)** and **`command`** are both command substitution (modern prefers `$()`). **$((expression))** is arithmetic expansion (shell built-in). **expr** is an external command for arithmetic, slower and less feature-rich.

B4 — **Process substitution** (`<(command)`) creates a file descriptor for command output. **Example:** `diff <(sort a.txt) <(sort b.txt)` compares sorted files without creating temp files.

B5 — Use `for f in *` or `while IFS= read -r -d '' f; do ... done < <(find . -print0)`. **`for f in $(ls)`** is dangerous because it splits on whitespace, breaking filenames with spaces or newlines.

B6 — **Here document** (`<<EOF`) feeds stdin from a file. **Here string** (`<<<`) feeds a string to stdin. **`<<-EOF`** strips leading tabs from the delimiter line for indentation.

B7 — `cmd > file 2>&1`: stdout to file, stderr to stdout (file). `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. `cmd &> file`: both to file (bash shorthand). First two differ because redirection order matters.

B8 — Use `mkdir` for locking (atomic).
```bash
lock() {
  mkdir lockdir 2>/dev/null || return 1
  trap 'rmdir lockdir' EXIT
  # ... work ...
}
```
`mkdir` is atomic; if it fails, lock exists. `trap` ensures cleanup on exit/SIGTERM.

B9 — **Single quotes** (`'...'`) literal (no expansion). **Double quotes** (`"..."`) expand variables/glob. **No quotes** expand variables/glob/split on whitespace. Example: `echo $HOME` (no quotes) vs `echo "$HOME"` (quoted) vs `echo '$HOME'` (literal).

B10 — **File exists:** `-e`. **Directory:** `-d`. **Readable:** `-r`. **`[[ ]]`** supports regex (`=~`), logical operators (`&&`, `||`), and is safer than `[ ]` (test).

B11 — **$?** (exit code of last cmd). **$$** (current PID). **$!** (PID of last background job). **$#** (number of arguments).

B12 — **Define:** `function name() {}` or `name() {}`. **Return:** Via `exit code` (0 success) or `echo` (stdout). Functions share global scope unless `local` is used.

B13 — **PATH** is colon-separated list of dirs. Bash searches PATH for external commands. **Builtin** (shell internal) vs **External** (binary). **Check:** `type command` or `which command`.

B14 — **trap** sets handlers for signals. **EXIT** (script end), **ERR** (command fails), **INT** (Ctrl+C), **TERM** (kill). Execution order: Signal arrives -> Handler runs -> Script continues or exits.

B15 — **xargs** builds command lines from stdin. **command substitution** runs command immediately. **xargs -0** handles null-delimited input (from `find -print0`). **xargs -P** runs jobs in parallel.

B16 — **Arrays:** `declare -a arr`; `arr+=(val)`. **Associative:** `declare -A arr`; `arr[key]=val`. **Iterate:** `for i in "${arr[@]}"`. **Length:** `${#arr[@]}`. Associative keys are strings; indexed are integers.

B17 — **Subshell** `(...)` forks a new process (vars lost). **Group** `{ ...; }` runs in current shell (vars persist). **Pipe** into `while` loop runs loop in subshell, so variable changes are lost after loop.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill, graceful), **SIGHUP** (hangup), **SIGKILL** (force, uncatchable). **Graceful shutdown:** Trap SIGTERM to kill child processes, then exit.

B19 —
```bash
split -l 1000 log.txt part_
xargs -P 4 -I {} sh -c 'process {}'
cat part_* > merged.txt
```
**Trade-offs:** Parallelism vs I/O contention. **Error propagation:** Check exit codes of `xargs` jobs; use `set -e` or explicit checks.

B20 — **exec** replaces current shell process. **FD manipulation:** `exec 3>file` opens FD 3. `exec 3>&-` closes it. **Coprocess** (`coproc`) creates bidirectional pipe to a command. Use for persistent background processes needing I/O.

PS1 — **Write-Output** sends objects to the pipeline. **Write-Host** writes directly to the console (bypasses pipeline). Distinction matters because pipeline output can be captured/chained; Write-Host cannot.

PS2 — **Pipeline** passes objects. **ForEach-Object** (cmdlet) processes items one-by-one in pipeline. **foreach** (statement) iterates a collection in memory (faster, no pipeline). Use **ForEach-Object** for streaming; **foreach** for local collections.

PS3 — **[PSCustomObject]** creates a typed object with properties. **[hashtable]** is key/value pairs. Convert: `[PSCustomObject]@{Key='Value'}`. Use **PSCustomObject** for structured data; **Hashtable** for dynamic config.

PS4 — **Terminating error** stops execution (throw). **Non-terminating** continues (warning). **-ErrorAction Stop** converts non-terminating to terminating. **try/catch** catches terminating errors.

PS5 — **Splatting** (`@params`) passes a hashtable as separate parameters. **Difference:** Passing hashtable as single param treats it as one argument; splatting expands keys to parameter names.

PS6 — **Invoke-Command -ComputerName** runs one-off commands remotely. **Enter-PSSession** starts interactive session. **Protocol:** WinRM (default) or SSH (v7+).

PS7 — `$x.GetType().Name` is `String[]`. If function returns 0 items, `$x` is an empty array `@()`. **Correct pattern:** Initialize array `$result = @()`, append, return `$result`. Avoid implicit array creation.

PS8 — **[scriptblock]** is a container for code. Pass via `-ArgumentList` or `-ScriptBlock`. **Double-hop:** CredSSP needed to delegate credentials to a second remote server. **Solve:** Use `-ArgumentList` to pass data, or configure CredSSP/SSH.

PS9 — **@()** creates an array. **[array]** casts to array. **Force single result:** `@($result)`. **Indexing:** `$null[0]` throws error. `@()[0]` returns `$null`.

PS10 — **-eq** (value), **-like** (glob), **-match** (regex). **-ceq** (case-sensitive). **-contains** (object in array). **-in** (value in array). `-contains` checks if array contains object; `-in` checks if value is in array.

PS11 — **Module** is a reusable script/library. **.psm1** (code), **.psd1** (manifest), **Binary** (DLL). **Import-Module** loads module into session.

PS12 — **"..."** interpolates variables. **'...'** literal. **@"..."@** (here-string) for multi-line strings. Expressions: `"Value: ${var}"`.

PS13 — **Get-Member** lists object members. **Properties** (data), **Methods** (func), **NoteProperties** (dynamic). **Select-Object** selects properties. **Where-Object** filters objects.

PS14 — **Classes** (v5+) define types. **Differences:** No private members, limited inheritance (no interfaces). **Use:** Complex state management vs functions for simple logic.

PS15 — **CmdletBinding** enables advanced features (parameters, ShouldProcess). **SupportsShouldProcess** adds -WhatIf/-Confirm. **Begin/Process/End** blocks handle setup, streaming, and cleanup.

PS16 — **Providers** expose data as file system. **Get-ChildItem** works on Registry (`HKLM:\`), Cert (`Cert:\`), Env (`Env:`). Access registry like files: `Get-ChildItem HKLM:\Software`.

PS17 — **[runspace]** is a thread. **[runspacepool]** manages threads. **ForEach-Object -Parallel** (v7+) uses pool. Manual runspace management offers more control but is complex.

PS18 — **PSScriptAnalyzer** enforces standards. **Rules** (e.g., naming), **Severity** (Error, Warning). **Suppress:** `#pragma warning disable`. **CI/CD:** Run in pipeline to block bad code.

PS19 — **$using:** passes local variable to remote session. **-ArgumentList** serializes data. **CLIXML** limits depth/size. **Complex objects** may fail serialization; use simple types or custom serializers.

PS20 — **DSC** manages system state. **Configurations** define state. **Resources** implement changes. **LCM** applies config. **WinPS vs PS7:** DSC core is in WinPS; PS7 uses `Microsoft.Dsc` module. **Replaced:** Ansible, Terraform, or Pulumi for cross-platform IaC.
