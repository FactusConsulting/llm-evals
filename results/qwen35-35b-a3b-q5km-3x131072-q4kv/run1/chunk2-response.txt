## Kubernetes

**K1** — A **Deployment** manages stateless applications, allowing for rolling updates and scaling without maintaining pod identity. A **StatefulSet** manages stateful applications, providing stable, unique network identifiers (hostnames), ordered deployment/termination, and persistent storage retention per pod.

**K2** — A `ClusterIP` Service creates a virtual IP (ClusterIP) within the cluster. `kube-proxy` on each node configures `iptables` or `IPVS` rules to intercept traffic sent to this IP and route it to the healthy pod IPs listed in the Service's Endpoints object.

**K3** — **PersistentVolume (PV)** is a cluster-level storage resource. **PersistentVolumeClaim (PVC)** is a user request for storage. **StorageClass** defines the provisioning mechanism (dynamic provisioning). A PVC binds to a PV (either statically provisioned or dynamically created via StorageClass).

**K4** — Troubleshooting CrashLoopBackOff:
1.  **Logs:** `kubectl logs <pod>` to see application errors.
2.  **Events:** `kubectl describe pod <pod>` to check scheduling or image pull issues.
3.  **Resources:** Check if the container is OOMKilled due to memory limits.
4.  **Probes:** Verify liveness/readiness probe configurations (e.g., failing immediately).

**K5** — CoreDNS provides DNS resolution for services within the cluster. If CoreDNS pods are down, internal service discovery (e.g., `my-service.default.svc`) fails, though external DNS resolution may still work depending on the pod's `/etc/resolv.conf` configuration.

**K6** — The scheduler filters nodes based on feasibility (taints, nodeSelector, resource availability) and scores them based on preference. **Factors:** 1. Resource requests (CPU/Memory), 2. Node Affinity/Taints, 3. Pod Affinity/Anti-affinity.

**K7** — **Yes**, it will be scheduled. The node has 4GB total, 3.5GB requested, leaving 0.5GB (512Mi) free. The pod requests 256Mi, which fits. **At runtime**, if it uses 600Mi, it exceeds the 512Mi limit and will be **OOMKilled** by the kernel.

**K8** — 1. Request hits External Load Balancer. 2. LB forwards to Ingress Controller (NGINX pod). 3. NGINX matches Ingress rules and forwards to Service ClusterIP. 4. `kube-proxy` (iptables/IPVS) routes Service IP to Pod IPs (Endpoints). 5. Traffic reaches the Application Pod.

**K9** — A **NetworkPolicy** restricts traffic flow to/from pods. **Default:** If no policy exists, all traffic is allowed. **When applied:** Traffic is denied by default for the selected pods unless explicitly allowed by the policy rules (ingress/egress).

**K10** — **RollingUpdate** replaces pods gradually (zero downtime). **Recreate** kills all existing pods before starting new ones. Choose **Recreate** for stateful applications requiring exclusive access to resources or during database migrations where concurrent versions are incompatible.

**K11** — A **Namespace** provides logical isolation for resources. **Default namespaces:** `default`, `kube-system`, `kube-public`, `kube-node-lease`.

**K12** — `kubectl describe` provides detailed diagnostic information. **3 pieces of info:** 1. Events (warnings/errors), 2. Container State (waiting/running/terminated details), 3. Mount paths and environment variables.

**K13** — A **ConfigMap** stores non-confidential configuration data. **Consumption methods:** 1. As environment variables in the pod spec. 2. As files mounted into a volume within the pod.

**K14** — **ClusterIP:** Internal cluster access only. **NodePort:** Exposes service on a static port on each node's IP. **LoadBalancer:** Provisions an external cloud load balancer to expose the service.

**K15** — A **DaemonSet** ensures a copy of a pod runs on all (or specific) nodes. Use one for **logging agents** (Fluentd), **monitoring** (Prometheus Node Exporter), or **network plugins** (CNI).

**K16** — **Taints** repel pods from a node; **Tolerations** allow pods to schedule on tainted nodes. **Node Affinity** attracts pods to nodes. **Use Case:** Taint a GPU node (no entry) and tolerate it on ML pods. Use Affinity to co-locate web and cache pods on the same zone.

**K17** — An **Operator** is a custom controller that manages complex stateful applications using CRDs. It differs from a standard controller by automating lifecycle management (backup, restore, scaling) specific to an application. It solves operational complexity for stateful apps like databases.

**K18** — etcd uses **strong consistency (linearizability)**. If quorum is lost, the cluster becomes read-only/unwritable. **Recovery:** Restore from a snapshot on a new node, then rejoin the remaining node (if any) or bring up new nodes to form a new cluster using the snapshot.

**K19** — **Role:** Namespaced permissions. **ClusterRole:** Cluster-wide permissions. **RoleBinding:** Assigns Role to a subject in a namespace. **ClusterRoleBinding:** Assigns ClusterRole cluster-wide.
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: pod-reader
  namespace: dev
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list", "watch"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: read-pods
  namespace: dev
subjects:
- kind: User
  name: developer
  apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io
```

**K20** — `ndots` defines how many dots must appear in a hostname before a search is attempted. `ndots: 5` with a short name triggers queries for every search domain suffix, causing latency. **Fix:** Set `dnsConfig` in pod spec: `options: [{name: "ndots", value: "1"}]`.

## Development

**D1** — **Rebase** rewrites history to create a linear commit chain. **Merge** preserves history and creates a merge commit. **Avoid rebasing** on shared/public branches where others have already pulled the commits.

**D2** — **TCP:** Connection-oriented, reliable (ACKs). **UDP:** Connectionless, unreliable (best effort). **TCP:** HTTP, SSH. **UDP:** DNS, VoIP.

**D3** — A **race condition** occurs when the output depends on the sequence/timing of uncontrollable events. **Example:** Two threads increment a shared counter `x++` without a lock, resulting in lost updates.

**D4** — **Symmetric:** Same key for encryption/decryption. **Asymmetric:** Public key encrypts, private key decrypts. **TLS:** Asymmetric keys exchange a session key; Symmetric key encrypts the actual data transfer.

**D5** — A **layer** is a read-only filesystem diff in a Docker image. **Caching:** Place instructions that change rarely (e.g., `COPY package.json`, `RUN npm install`) before code changes to maximize build cache hits.

**D6** — Troubleshooting HTTP 503:
1.  Check upstream service health (is the backend down?).
2.  Check database connectivity.
3.  Check resource limits (OOM/Kill).
4.  Check rate limiting or circuit breaker status.

**D7** — **CAP:** Consistency, Availability, Partition Tolerance. **CP:** Zookeeper, HBase (sacrifices availability during partition). **AP:** Cassandra, DynamoDB (sacrifices consistency during partition).

**D8** — **Horizontal:** Add more nodes. **Vertical:** Add more resources (CPU/RAM) to existing node. **Patterns:** Stateless services, sharding, load balancing, microservices.

**D9** — An **environment variable** is a dynamic value affecting process behavior. **Persist:** Add `export VAR=value` to `~/.bashrc` or `~/.profile`.

**D10** — **JSON:** Strict syntax, key-value pairs. **YAML:** Human-readable, indentation-based. **JSON:** Machine parsing is strict. **YAML:** Supports comments and easier editing.

**D11** — **REST:** Architectural style using HTTP methods. **GET:** Retrieve data. **POST:** Create resource. **PUT:** Replace resource. **DELETE:** Remove resource.

**D12** — **Version pinning** specifies exact dependency versions (e.g., `1.0.1` vs `^1.0.0`). **Importance:** Ensures reproducible builds and prevents breaking changes from upstream updates.

**D13** — **Compiled:** Source to binary before execution. **Interpreted:** Source executed line-by-line. **Compiled:** C, Go. **Interpreted:** Python, JavaScript.

**D14** — **Reverse Proxy:** Client -> Proxy -> Server (hides server). **Forward Proxy:** Client -> Proxy -> Internet (hides client). **Servers:** Nginx (web serving), HAProxy (load balancing).

**D15** — A **deadlock** is a permanent block where processes wait for each other. **Conditions:** 1. Mutual Exclusion, 2. Hold and Wait, 3. No Preemption, 4. Circular Wait.

**D16** — **Eventual Consistency:** Updates propagate asynchronously. **Cassandra:** Uses Hinted Handoff and Read Repair. **Levels:** ONE, QUORUM, LOCAL_QUORUM, ALL.

**D17** — **mTLS:** Mutual TLS where both client and server present certificates. **Handshake:** Server requests client cert; Client sends cert; Server verifies. **Use:** Service Mesh (Istio) for zero-trust communication.

**D18** — **Hash Table:** Array of buckets. **Collision:** Chaining (linked lists) or Open Addressing (probing). **Complexity:** Avg O(1), Worst O(n) (all collisions).

**D19** — **Circuit Breaker:** Prevents cascade failures. **States:** Closed (normal), Open (fail fast), Half-open (test recovery). **Relation:** Retries are disabled when the circuit is Open.

**D20** — **Optimistic:** Checks conflicts at commit (versioning). **Pessimistic:** Locks data at read (`SELECT FOR UPDATE`). **Choose Optimistic:** Low contention. **Choose Pessimistic:** High contention/conflict risk.
