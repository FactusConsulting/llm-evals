JS1 — **var** is function-scoped and hoisted (initialized as `undefined`). **let** and **const** are block-scoped. **let** is mutable; **const** is immutable reference (though object properties can change). **Hoisting** means declarations are moved to the top of their scope during compilation. `var` is hoisted and initialized; `let`/`const` are hoisted but remain in the Temporal Dead Zone (TDZ) until declaration.

JS2 — The **event loop** manages execution of code, collection, and processing of events. **Call Stack**: executes synchronous code. **Microtask Queue**: high priority (Promises, `queueMicrotask`); processed after the current stack clears, before the next macrotask. **Macrotask Queue**: lower priority (setTimeout, I/O); processed one per event loop iteration. The loop runs: Stack -> Microtasks -> Render -> Macrotask.

JS3 — A **closure** is a function bundled with its lexical environment. Example: `function outer() { let x=1; return () => x; }`. **For loop bug**: `var` creates function scope, so all closures share the same `i`. By the time callbacks run, `i` is `length`. **Fix**: Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents eventual completion/failure. **async/await** syntactic sugar for promises. **Error handling**: `try/catch` with await, or `.catch()` on promises. **Promise.all**: waits for all to resolve; rejects immediately on first failure. **Promise.allSettled**: waits for all to settle (resolve or reject), returns array of status objects `{status, value/reason}`.

JS5 — **interface** defines structure only (no implementation), supports declaration merging. **type** can define primitives, unions, intersections, tuples. Use **interface** for public API contracts/classes; **type** for unions/complex shapes. **Discriminated unions**: unions of objects with a shared literal property (tag) used to narrow types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — **Prototypal inheritance**: objects link to a prototype object via `[[Prototype]]`. **Object.create(proto)** creates a new object with `proto` as its prototype. **ES6 classes** are syntactic sugar over prototypes; `class` defines a constructor and methods on the prototype chain.

JS7 — **this** binding depends on call site. **4 Rules**: 1. `new` (binds to new instance), 2. Explicit (`call`, `apply`, `bind`), 3. Implicit (`obj.method()`), 4. Default (`window`/`undefined` in strict mode). **bind()** returns a new function with bound `this`. **Cannot re-bind** a bound function.

JS8 — **WeakRef** holds weak reference to object (GC can collect it). **FinalizationRegistry** notifies when object is collected. Use for caches or cleanup. **WeakMap** keys must be objects because primitives are copied by value; objects are references. GC can collect keys if no strong references exist, preventing memory leaks.

JS9 — **Hidden classes**: V8 optimizes property access by grouping objects with same shape. **Inline caching**: caches property lookup results. **Deoptimization**: occurs when hidden class changes (e.g., adding property dynamically). **Patterns causing deopt**: dynamic property addition, `delete`, `with` statement, changing prototype chain.

JS10 — **Tree-shaking** removes unused exports. Works with **ES Modules** (static `import`/`export` allows static analysis). **CommonJS** (`require`) is dynamic, so bundlers cannot statically analyze dependencies easily, making tree-shaking ineffective.

JS11 — **undefined**: variable declared but not assigned. **null**: intentional absence of value. `null == undefined` is `true`; `null === undefined` is `false`. **Nullish coalescing (`??`)**: returns right operand if left is `null` or `undefined` (unlike `||` which checks falsy). **Optional chaining (`?.`)**: safely accesses nested properties without throwing if intermediate is null/undefined.

JS12 — **map()**: transforms array. **filter()**: selects elements. **reduce()**: accumulates value. **Initial value**: second arg to reduce; if omitted, first element is accumulator. **flatMap()**: maps then flattens one level. Useful for transforming nested arrays into a single array without nested loops.

JS13 — **Template literals**: strings with `${}` interpolation and multi-line support. **Tagged templates**: function called with template string parts and values. Example: `html` tag to escape values preventing XSS.
```javascript
function html(strings, ...values) {
  return strings.reduce((acc, str, i) => acc + str + (values[i] || ''), '');
}
```

JS14 — **Destructuring**: unpacks values. **Object**: `{a, b}`. **Array**: `[a, b]`. **Defaults**: `{a = 1}`. **Renaming**: `{a: b}`. **Rest operator**: `...rest` collects remaining items. In destructuring, `...rest` must be last.

JS15 — **for...in**: iterates over enumerable **keys** (properties) of an object. **for...of**: iterates over **values** of iterable objects (arrays, strings). Avoid **for...in** on arrays because it iterates over inherited properties and order is not guaranteed.

JS16 — **Generics**: `function identity<T>(arg: T): T`. **Constraints**: `function log<T extends {length: number}>(arg: T)`. **Conditional types**: `type IsString<T> = T extends string ? true : false`. **infer**: extracts type from conditional. Example: `type UnwrapPromise<T> = T extends Promise<infer U> ? U : T`.

JS17 — **CommonJS**: `require`/`module.exports` (dynamic, synchronous). **ES Modules**: `import`/`export` (static, async). **Node.js**: supports both; ESM requires `.mjs` or `"type": "module"` in `package.json`. `"type": "module"` makes `.js` files treated as ESM.

JS18 — **Structural typing**: types compatible if they have same shape. **Excess property checks**: prevent extra properties in object literals assigned to types (only applies to literals, not variables). **extends**: for inheritance/constraints. **satisfies**: checks shape without widening type. **Variance**: `in` (contravariant), `out` (covariant) in generics (TS 4.7+).

JS19 — **Proxy**: intercepts operations (get, set, has, etc.). **Reflect**: provides default implementations. Example: Validation Proxy.
```javascript
const validator = new Proxy(obj, {
  set(target, prop, value) {
    if (prop === 'age' && value < 0) throw new Error('Invalid age');
    return Reflect.set(target, prop, value);
  }
});
```

JS20 — **SharedArrayBuffer**: shared memory between threads (Workers). **Atomics**: atomic operations on typed arrays. **Ordering**: memory barriers ensure visibility. **Atomics.wait**: blocks until notified. **Atomics.notify**: wakes waiting threads. Requires specific browser headers (COOP/COEP) for security.

B1 — **$@**: expands to separate arguments (`"$1" "$2"`). **$***: expands to single string (`"$1 $2"`). **"$@"**: preserves argument boundaries (safe for loops). **"$*"**: joins args with first char of IFS (dangerous with spaces).

B2 — **set -e**: exit on error. **set -u**: error on undefined vars. **set -o pipefail**: exit if any command in pipe fails (default is last command).

B3 — **$(command)**: command substitution (modern, nested). **`command`**: legacy backticks. **$((expression))**: arithmetic expansion. **expr**: external command for arithmetic (slower, legacy).

B4 — **Process substitution**: `<(command)` creates a file descriptor (e.g., `/dev/fd/63`). **diff**: `diff <(sort a) <(sort b)` compares sorted outputs without creating temp files.

B5 — **Safe iteration**: `while IFS= read -r -d '' file; do ...; done < <(find . -print0)`. **for f in $(ls)** is dangerous because it splits on whitespace, breaking filenames with spaces/newlines.

B6 — **Here document**: `<<EOF` reads until EOF line. **Here string**: `<<<` passes string as stdin. **`<<-EOF`**: strips leading tabs from lines (for indentation).

B7 — **`cmd > file 2>&1`**: stdout to file, stderr to stdout (file). **`cmd 2>&1 > file`**: stderr to stdout (terminal), stdout to file. **`&> file`**: both to file (bash specific). First two differ because redirections are processed left-to-right.

B8 — **Lock function**:
```bash
lock() {
  mkdir "$1.lock" 2>/dev/null && trap 'rmdir "$1.lock"' EXIT INT TERM
}
```
**mkdir** is atomic (mkdir fails if exists), unlike `touch` which can race. **trap** ensures cleanup on kill.

B9 — **Single quotes**: literal (no expansion). **Double quotes**: expansion allowed (`$`, `` ` ``). **No quotes**: word splitting/globbing. Example: `echo $HOME` (double) vs `echo '$HOME'` (single).

B10 — **Test**: `[ -f file ]` (file), `[ -d dir ]` (directory), `[ -r file ]` (readable). **`[`** is a command (external `test`). **`[[`** is a keyword (bash builtin), supports `&&`, `||`, regex.

B11 — **$?**: exit status of last command. **$$**: PID of current shell. **$!**: PID of last background job. **$#**: number of arguments passed to script.

B12 — **Function**: `foo() { ... }` or `function foo { ... }`. **Return**: `return` sets exit code; use `echo` for output.

B13 — **PATH**: colon-separated dirs for command lookup. **Builtin**: internal to shell (faster). **External**: separate binary. **Check**: `type -a cmd` or `which cmd`.

B14 — **trap**: `trap 'cmd' EXIT`. **Order**: EXIT handlers run last. **Signals**: INT (Ctrl+C), TERM (kill), HUP (hangup). Multiple signals: queued or handled sequentially depending on shell.

B15 — **xargs**: builds command lines from stdin. **command substitution**: `$(...)` runs immediately. **xargs -0**: handles null-delimited input (`find -print0`). **xargs -P**: parallel execution (N jobs).

B16 — **Arrays**: `arr=(a b)`. **Associative**: `declare -A arr; arr[key]=val`. **Iterate**: `for i in "${arr[@]}"; do ...`. **Length**: `${#arr[@]}`.

B17 — **Subshell**: `(...)` runs in new process (vars lost). **Group**: `{ ...; }` runs in current shell (vars persist). **Pipe to while**: `cmd | while read; do ...` runs in subshell, so vars inside loop don't persist outside.

B18 — **Signals**: SIGINT (interrupt), SIGTERM (terminate), SIGHUP (hangup), SIGKILL (force kill, uncatchable). **Graceful shutdown**: `trap` SIGTERM to cleanup background jobs (`wait`) before exit.

B19 — **Parallel script**: `split -l 1000 log.log part_`, `find . -name 'part_*' -print0 | xargs -0 -P 4 -I {} bash process.sh {}`. **Trade-offs**: I/O vs CPU. **Error propagation**: check exit codes of all jobs, aggregate failures.

B20 — **exec**: replaces shell process (no new PID). **File descriptors**: `exec 3>file` opens FD 3. **Coprocess**: `coproc CMD` creates bidirectional pipe. Use for persistent connections or long-running processes.

PS1 — **Write-Output**: sends object to pipeline (default). **Write-Host**: writes directly to console (bypasses pipeline). **Matter**: Pipeline objects can be processed by next cmd; Host output is just text.

PS2 — **Pipeline**: passes objects between cmdlets. **ForEach-Object**: cmdlet (pipeline-aware, processes each item). **foreach**: statement (iterates collection in memory, faster for large arrays, no pipeline).

PS3 — **[PSCustomObject]**: ordered, typed object. **[hashtable]**: unordered, key/value pairs. **Convert**: `[PSCustomObject]@{a=1}` or `@{a=1} | ConvertTo-Json | ConvertFrom-Json`. Use CustomObject for structured data; Hashtable for dynamic config.

PS4 — **Terminating**: stops execution (exceptions). **Non-terminating**: continues (errors). **-ErrorAction**: overrides `$ErrorActionPreference`. **try/catch**: catches terminating errors.

PS5 — **Splatting**: `@params` passes hashtable as arguments. **Difference**: Passing hashtable as single param treats it as one object; splatting expands keys to parameters.

PS6 — **Invoke-Command**: runs command on remote machine (one-off). **Enter-PSSession**: interactive session. **Protocol**: WS-Management (WinRM). **SSH**: PowerShell 7+ supports SSH remoting.

PS7 — **Type Inference**: `Get-Items` returns string, not array. `$x.GetType().Name` is `String`. **0 items**: returns `$null`. **Pattern**: Always return array: `$result = @(); $result += ...; return $result`.

PS8 — **[scriptblock]**: code container. **Pass to remote**: `-ArgumentList` or `$using:`. **Double-hop**: CredSSP allows auth delegation. **Solve**: Use `-Credential` with `-Authentication Kerberos` or configure CredSSP.

PS9 — **@()**: array subexpression (forces array). **[array]**: type cast. **Indexing**: `$null[0]` throws; `@()[0]` returns `$null`.

PS10 — **-eq**: equality. **-like**: wildcard. **-match**: regex. **-contains**: checks if collection contains value. **-in**: checks if value is in collection. **Case-sensitive**: `-ceq`, `-clike`.

PS11 — **Module**: reusable code package. **.psm1**: script module. **.psd1**: manifest (metadata). **Import-Module**: loads module into session.

PS12 — **Interpolation**: `"Hello $name"`. **'...'**: literal. **Here-string**: `@"..."@` for multi-line. **Expression**: `$var` inside double quotes.

PS13 — **Get-Member**: lists properties/methods. **Properties**: data. **Methods**: functions. **NoteProperties**: added dynamically. **Select-Object**: selects properties. **Where-Object**: filters objects.

PS14 — **Classes**: v5+. **C#**: PS classes are wrappers. **Limitations**: No interfaces, limited inheritance. **Use**: Encapsulation, state management. **Function**: Simpler logic.

PS15 — **[CmdletBinding]**: enables advanced features. **SupportsShouldProcess**: `-WhatIf`/`-Confirm`. **Begin/Process/End**: Init, Stream, Finalize. **ValueFromPipeline**: binds pipeline input.

PS16 — **Providers**: abstract data stores. **Get-ChildItem**: works on File, Registry, Env. **Registry**: `Get-ChildItem HKLM:\Software`.

PS17 — **Runspaces**: isolated execution environments. **RunspacePool**: manages threads. **ForEach-Object -Parallel**: v7+ parallelism. **Manual**: `[RunspaceFactory]::CreateRunspacePool()`.

PS18 — **PSScriptAnalyzer**: linting tool. **Rules**: static analysis checks. **Severity**: Error, Warning, Info. **Suppress**: `#pragma`. **CI/CD**: `Invoke-ScriptAnalyzer` in pipeline.

PS19 — **$using:**: captures local var in remote session. **-ArgumentList**: passes data. **Serialization**: CLIXML limits depth/complexity. **Complex objects**: Convert to JSON or use `Invoke-Command -Session`.

PS20 — **DSC**: config management. **Resources**: atomic units. **LCM**: manages state. **WinPS vs PS7**: PS7 uses native modules. **Replaced**: Ansible, Terraform, or native PS7 DSC modules.
