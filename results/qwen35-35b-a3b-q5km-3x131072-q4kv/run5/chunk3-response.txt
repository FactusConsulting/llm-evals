T1 — **Plan** calculates the execution plan (what will change) without making changes. **Apply** executes the plan to create, update, or destroy resources. Plan is for review; Apply is for action.

T2 — **State** is a JSON file mapping real-world resources to your configuration. **Remote state** stores this file in a shared location (e.g., S3), enabling team collaboration, state locking, and preventing local state loss.

T3 — **count** uses integer indices; **for_each** uses a map or set of strings. **for_each** is preferred because it is more stable during refactoring (removing an item doesn't shift indices and cause resource recreation).

T4 — This is **state drift**. OpenTofu detects it during `tofu plan` by comparing the state file to the live infrastructure. `tofu apply` will then reconcile the configuration to match the desired state.

T5 —
```hcl
resource "aws_instance" "web" {
  for_each = toset(["web1", "web2", "web3"])
  ami      = "ami-12345"
  tags     = { Name = each.key }
}
```

T6 — `terraform_remote_state` reads state files from a backend. **Recommended Alternative:** Pass outputs directly via module arguments if modules are in the same run, or use a CI/CD pipeline to pass values. **Why:** `terraform_remote_state` creates tight coupling between state files and requires backend access permissions; direct outputs are more explicit and decoupled.

T7 — The `null_resource` provisioner runs during apply, but the dependent resource may try to read the output before the file is written. **Fix:** Add `depends_on = [null_resource.kubeconfig]` to the dependent resource to ensure the file exists before it is consumed.

T8 — `tofu import` adds existing resources to state without creating config. **Import Block** (1.5+) allows defining imports in config, enabling `tofu plan` to show the import as a change. **Limitation:** CLI import doesn't show diff; Import Block does.

T9 — A **variable** is an input parameter passed into the module. An **output** is a value exposed by the module to be used by the caller.

T10 — A **provider** defines the API for a cloud service (e.g., AWS). Without `tofu init`, providers are not downloaded, and `tofu plan` fails with "Provider not installed" errors.

T11 — The `.terraform.lock.hcl` file records exact provider versions and checksums. It should be committed to ensure reproducible builds and prevent unexpected provider updates.

T12 — `tofu destroy` removes all managed resources. Target a single resource with `-target=resource.type.name`.

T13 — A **data source** reads data from an external system without managing its lifecycle. **Example:** `data "aws_ami"` to find the latest Ubuntu image ID instead of managing the AMI itself.

T14 — **Variables** are inputs from outside the module. **Locals** are computed values within the module configuration. Use **locals** for intermediate calculations or constants that don't need external input.

T15 — **Lifecycle** controls resource behavior. `create_before_destroy` creates new resource before deleting old (zero downtime). `prevent_destroy` stops accidental deletion. `ignore_changes` ignores drift in specific attributes.

T16 — The **moved block** allows renaming or moving resources in configuration without changing their state ID. It helps refactor module structure without destroying and recreating resources.

T17 — The **dependency graph** determines execution order. **Implicit** dependencies come from resource references. **Explicit** dependencies use `depends_on`. `depends_on` reduces parallelism because it forces sequential execution, slowing down plans.

T18 — 1. **`-refresh=false`**: Skips state refresh (faster, but may miss drift). 2. **Exclude resources**: Use `-exclude` for non-critical resources (faster, less visibility). 3. **Remote State Caching**: Cache state files (faster reads, requires setup). 4. **Parallelism**: Increase `parallelism` setting (faster apply, higher resource contention).

T19 — State locking prevents concurrent writes. If a lock isn't released, subsequent applies fail. **Break lock:** `tofu force-unlock <LOCK_ID>`. **Risk:** Data corruption if another apply is actually running.

T20 — 1. Configure new backend in `backend` block. 2. Run `tofu init -migrate-state`. 3. OpenTofu copies local state to S3. 4. Verify with `tofu state list` and `tofu state show`.

A1 — **Playbook**: YAML file defining automation steps. **Role**: Directory structure for reusable logic. **Task**: Single unit of work (e.g., install package).

A2 — `ansible_facts` contains gathered system info. Access OS family via `{{ ansible_os_family }}`.

A3 — **command**: Executes without shell (no pipes). **shell**: Executes with shell (supports pipes). **raw**: Executes without setup (no Python module), useful for initial bootstrap.

A4 — **Idempotency** means running a task multiple times yields the same result. **Idempotent:** `apt: state=present`. **Not Idempotent:** `shell: echo "hello" > file` (without `content` module).

A5 —
```yaml
- name: Install htop
  package:
    name: htop
    state: present
  when: ansible_os_family == "Debian"
- name: Install htop on RedHat
  package:
    name:
      - epel-release
      - htop
    state: present
  when: ansible_os_family == "RedHat"
```

A6 — **Precedence (High to Low):** Extra_vars > Play vars > Role vars > Host vars > Group vars > Role defaults.

A7 — Use **serial** (e.g., `serial: 3`) to batch nodes. Group nodes in inventory. Use `wait_for` to ensure the first 3 nodes are ready before the next batch starts.

A8 — **ansible-vault** encrypts sensitive data. To encrypt a single variable, use the `!vault` block syntax within the YAML file, or encrypt the entire vars file.

A9 — **Inventory** defines hosts. **Static:** YAML/INI file. **Dynamic:** Script/API that generates inventory dynamically (e.g., AWS EC2).

A10 — `ansible webservers -m shell -a "df -h"`

A11 — A **handler** is a task that runs only when notified. It differs from a regular task by running asynchronously after the play completes. Triggered via `notify: handler_name`.

A12 — **ansible.cfg** configures Ansible behavior. Common settings: `host_key_checking`, `forks`, `remote_user`.

A13 — **register** stores task output in a variable. Use it in subsequent tasks via `{{ registered_var.stdout }}`.

A14 — **include_tasks** is evaluated at runtime (dynamic). **import_tasks** is evaluated at compile time (static). Use `include` for conditionals; `import` for performance.

A15 — **Ansible Galaxy** is a repository for roles/collections. Use `ansible-galaxy install -r requirements.yml` to install them.

A16 — **Connection plugins** manage communication. **ssh:** Remote Linux/Unix. **local:** Control node. **network_cli:** Network devices (Cisco/Juniper).

A17 — **delegate_to** runs a task on a specific host. **run_once** ensures it runs once. **Example:** Gather facts from remote host, then run a script on localhost using those facts.

A18 — 1. **Forks:** Increase parallel connections (faster, more memory). 2. **Pipelining:** Reduces SSH overhead (faster, requires SSH config). 3. **Fact Caching:** Avoids gathering facts every run (faster, stale data risk). 4. **Mitogen:** Optimized transport (faster, external dependency).

A19 — `{{ }}` evaluates expressions. `{% %}` executes logic (loops/ifs). `{# #}` comments.
```jinja
upstream backend {
{% for server in backend_servers %}
    server {{ server }}; # Health check: {{ server_health_check }}
{% endfor %}
}
```

A20 — 1. Set **serial** (e.g., 5). 2. **Pre-task:** Deregister from LB. 3. **Deploy:** Update app. 4. **Health Check:** `uri` module. 5. **Post-task:** Re-register. 6. **Rollback:** Use `block/rescue` to revert changes if health check fails.
