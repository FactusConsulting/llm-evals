JS1 ‚Äî **var** is function-scoped and hoisted (initialized as `undefined`). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 ‚Äî The **event loop** manages execution. The **call stack** runs synchronous code. **Microtask queue** (promises) runs after the call stack clears but before the next macrotask. **Macrotask queue** (setTimeout, I/O) runs after the microtask queue.

JS3 ‚Äî A **closure** is a function retaining access to its lexical scope. Example: Counter factory. **Bug:** `var` in loops shares one variable reference. **Fix:** Use `let` (block scope) or an IIFE to capture the current value.

JS4 ‚Äî **Promise** represents async completion. **async/await** is syntactic sugar. Error handling: `try/catch` for await, `.catch()` for promises. **Promise.all** fails fast on first rejection. **Promise.allSettled** waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 ‚Äî **Interface** defines shape (can be merged/extended). **Type** defines value (unions, primitives, intersections). Use **Interface** for public API contracts, **Type** for unions/complex shapes. **Discriminated unions** are unions with a literal type tag to narrow types safely.

JS6 ‚Äî **Prototypal inheritance** links objects via `[[Prototype]]`. **Object.create(proto)** creates an object with a specific prototype. **ES6 classes** are syntactic sugar over prototype chains, using `extends` and `constructor`.

JS7 ‚Äî **this** is determined by call site. **Rules:** 1. Explicit (bind/call/apply), 2. Implicit (obj.method), 3. New (new), 4. Default (window/undefined). **bind()** returns a new function with bound `this`. It **cannot** be re-bound.

JS8 ‚Äî **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** runs cleanup when weak refs are collected. **WeakMap** keys must be objects because primitives are immutable and GC tracks object references.

JS9 ‚Äî V8 uses **Hidden Classes** (shapes) for fast property access and **Inline Caching** (caches lookup results). **Deoptimization** occurs when assumptions break (e.g., adding properties dynamically, polymorphic calls). Patterns: Dynamic property access, `arguments` object, `eval`.

JS10 ‚Äî **Tree-shaking** removes unused code. **ES Modules** allow static analysis (static imports), enabling tree-shaking. **CommonJS** (`require`) is dynamic, preventing static analysis, so tree-shaking is ineffective.

JS11 ‚Äî **null** is intentional absence; **undefined** is unassigned. **==** coerces types; **===** checks strict equality. **??** (nullish coalescing) returns right side if left is `null`/`undefined`. **?.** (optional chaining) short-circuits on `null`/`undefined`.

JS12 ‚Äî **map()** transforms, **filter()** selects, **reduce()** accumulates. **reduce()** with initial value prevents type errors on empty arrays. **flatMap()** maps then flattens one level; useful for transforming arrays of arrays.

JS13 ‚Äî **Template literals** use backticks for multi-line strings and interpolation `${}`. **Tagged templates** call a function with the string parts and values. Example: `html` tag to escape HTML or build SQL queries safely.

JS14 ‚Äî **Destructuring** extracts values. **Object:** `{a, b}`. **Array:** `[a, b]`. **Defaults:** `{a = 1}`. **Renaming:** `{a: b}`. **Rest:** `...rest` collects remaining items.

JS15 ‚Äî **for...in** iterates over enumerable keys/properties (including inherited). **for...of** iterates over values of iterable objects. Avoid **for...in** on arrays because it includes non-index properties and is slower.

JS16 ‚Äî **Generics** use type parameters (`<T>`). **Constraints:** `T extends U`. **Conditional types:** `T extends U ? X : Y`. **infer:** Extracts type from a type (e.g., `infer ReturnType`). Example: Generic function returning input type.

JS17 ‚Äî **CommonJS** (`require`) is dynamic/synchronous. **ES Modules** (`import`) are static/async. Node.js supports both. **"type": "module"** in `package.json` treats `.js` files as ES Modules by default.

JS18 ‚Äî **Structural typing** matches shapes, not names. **Excess property checks** apply to object literals during assignment. **extends** is for inheritance/constraints. **satisfies** checks type without widening the inferred type. **Variance** (`in`, `out`) controls generic type parameter usage (TS 4.7+).

JS19 ‚Äî **Proxy** intercepts operations (get, set, etc.). **Reflect** provides default implementations. Example: Proxy for validation or reactive data binding (Vue/React use similar patterns).

JS20 ‚Äî **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic ops. **Ordering:** Memory barriers ensure visibility. **Atomics.wait** blocks until notified. **Atomics.notify** wakes waiting threads.

B1 ‚Äî **$@** expands to separate quoted arguments. **$*** expands to a single string. **"$@"** preserves individual arguments (safe for loops). **"$*"** joins them into one string (dangerous with spaces).

B2 ‚Äî **-e**: Exit immediately on error. **-u**: Treat unset variables as errors. **-o pipefail**: Pipeline fails if any command fails.

B3 ‚Äî **$(command)** is modern, supports nesting. **`command`** is legacy. **$((expression))** is built-in arithmetic. **expr** is an external command (slower, less feature-rich).

B4 ‚Äî **Process substitution** `<(cmd)` creates a file descriptor. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 ‚Äî Use globbing `for f in *` or `find`. **`for f in $(ls)`** is dangerous because it splits on whitespace/newlines, breaking filenames with spaces.

B6 ‚Äî **Here document** `<<EOF` reads until EOF. **Here string** `<<<` passes a string. **`<<-EOF`** strips leading tabs from the script body.

B7 ‚Äî `cmd > file 2>&1`: stdout to file, then stderr to stdout (file). **Both captured.** `cmd 2>&1 > file`: stderr to stdout (terminal), then stdout to file. **Only stdout captured.** `&> file`: Both to file.

B8 ‚Äî Use `mkdir` for locking (atomic). Function: `mkdir /tmp/lock` then `trap 'rmdir /tmp/lock' EXIT`. Handle SIGTERM by trapping and cleaning up before exit.

B9 ‚Äî **Single quotes**: Literal (no expansion). **Double quotes**: Expand variables/commands. **No quotes**: Word splitting/globbing. Example: `echo $HOME` (double) vs `echo '$HOME'` (single).

B10 ‚Äî **`[ -e file ]`** exists, **`-d`** directory, **`-r`** readable. **`[[ ]]`** is a bash builtin (supports regex, `&&`), `[ ]` is a command (POSIX).

B11 ‚Äî **$?**: Exit code. **$$**: Current PID. **$!**: Last background PID. **$#**: Number of arguments.

B12 ‚Äî **`function foo {}`** or **`foo() {}`**. Return values via `exit` code or `echo` (captured via `$()`).

B13 ‚Äî **PATH**: Colon-separated dirs. Bash searches PATH. **Builtin**: Shell internal. **External**: Binary. Check with **`type`** or **`which`**.

B14 ‚Äî **trap** sets handlers. **EXIT**: On exit. **ERR**: On error. **INT/TERM**: On signals. Order: Registered handlers run in order of registration.

B15 ‚Äî **xargs** builds command lines. **`find -print0`** + **`xargs -0`** handles spaces/newlines. **`-P`** runs jobs in parallel.

B16 ‚Äî **Arrays**: `arr=(1 2)`. **Associative**: `declare -A map`. Append: `arr+=(val)`. Length: `${#arr[@]}`.

B17 ‚Äî **Subshell** `(...)` forks a new process (vars lost). **Group** `{...}` runs in current shell. **Pipe to while** runs in subshell, so vars don't persist outside.

B18 ‚Äî **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (unblockable). Handle graceful shutdown by trapping SIGTERM, waiting for bg jobs, then exiting.

B19 ‚Äî Script: `split`, `xargs -P`, process function, `cat` results. **Trade-offs:** Parallel I/O vs CPU. **Error:** Check exit codes of parallel jobs; aggregate errors.

B20 ‚Äî **exec** replaces shell process. **FD manipulation**: `exec 3>file` opens FD 3. **Coprocess**: `coproc` creates bidirectional pipe. Use for persistent background communication.

PS1 ‚Äî **Write-Output** sends objects to pipeline. **Write-Host** writes to console (bypasses pipeline). Distinction matters for piping data to other commands.

PS2 ‚Äî **Pipeline** passes objects. **ForEach-Object** (cmdlet) processes pipeline items. **foreach** (statement) iterates a collection. Use **ForEach-Object** for streaming, **foreach** for pre-loaded collections.

PS3 ‚Äî **[PSCustomObject]** is a typed object. **[hashtable]** is key/value pairs. Convert: `[PSCustomObject]@{Key="Val"}`. Use CustomObject for structured data, Hashtable for dynamic config.

PS4 ‚Äî **Terminating** (throw) stops execution. **Non-terminating** (catch) continues. **-ErrorAction** overrides preference. **try/catch** handles terminating errors.

PS5 ‚Äî **Splatting** `@params` passes hashtable as arguments. Differs from passing hashtable as a single parameter. Example: `Get-Process @params`.

PS6 ‚Äî **Remoting** uses WinRM (HTTP) or SSH. **Invoke-Command** runs one-off commands. **Enter-PSSession** starts interactive session.

PS7 ‚Äî Returns array even if 1 item. **Type:** `String[]`. If 0 items, returns empty array `@()` (Object[]). **Pattern:** Always initialize `$result = @()` and return it.

PS8 ‚Äî **[scriptblock]** is code object. Pass via `-ArgumentList`. **Double-hop:** Auth fails when accessing remote resource from remote session. **Solve:** CredSSP or `-ArgumentList` with credentials.

PS9 ‚Äî **@()** creates array. **[array]** casts. **Index $null** throws error. **Index empty array** returns empty. Force single result: `@($result)`.

PS10 ‚Äî **-eq** (equal), **-like** (wildcard), **-match** (regex). **-c** variants are case-sensitive. **-contains** checks member. **-in** checks if value is in array.

PS11 ‚Äî **Module** (.psm1, .psd1, dll). **Import-Module** loads it. **Script module**: Code. **Manifest**: Metadata. **Binary**: Compiled DLL.

PS12 ‚Äî **Interpolation** `"$var"`. **Here-string** `@"..."@` for multi-line. **Expressions** inside `$()`.

PS13 ‚Äî **Get-Member** introspects objects. **Properties** (data), **Methods** (actions), **NoteProperties** (added dynamically). **Select-Object** (subset), **Where-Object** (filter).

PS14 ‚Äî **Classes** (v5) support state. **Limitations:** No interfaces, limited inheritance. Use **Class** for stateful logic, **Function** for stateless.

PS15 ‚Äî **CmdletBinding** enables advanced features. **SupportsShouldProcess** (Confirm). **Begin/Process/End** (streaming). **ValueFromPipeline** binds pipeline input.

PS16 ‚Äî **Providers** expose data as file system. **Get-ChildItem** works on Registry, Certs, Env. Access registry: `Get-ChildItem HKLM:\Software`.

PS17 ‚Äî **Runspace** (thread), **RunspacePool** (pool). **ForEach-Object -Parallel** (v7+) uses pool. Manual runspace for complex control.

PS18 ‚Äî **PSScriptAnalyzer** enforces standards. **Rules** (severity). **Suppress** via `.psd1` or comments. Integrate via CI/CD (exit code).

PS19 ‚Äî **$using:** passes local var to remote. **Serialization:** CLIXML limits depth. Complex objects may fail serialization.

PS20 ‚Äî **DSC** (Desired State Config). **Resources** (config units), **LCM** (manager). **PS 7+**: Cross-platform DSC. **Replaced by:** Ansible/Terraform for broader IaC, but DSC remains for Windows config.
