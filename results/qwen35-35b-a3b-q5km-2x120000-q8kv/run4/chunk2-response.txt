K1 — Easy: A **Deployment** manages stateless applications, creating and replacing Pods without guaranteed order or identity. A **StatefulSet** manages stateful applications, providing stable, unique network identities (hostnames) and ordered, graceful deployment and scaling.

K2 — Easy: A **ClusterIP** Service exposes a virtual IP within the cluster. **kube-proxy** (using iptables or IPVS) maintains network rules on each node to route traffic from the Service IP to the backend Pod IPs based on the Service’s selector.

K3 — Medium: A **PersistentVolume (PV)** is a cluster-level storage resource. A **PersistentVolumeClaim (PVC)** is a user request for storage. A **StorageClass** defines the provisioning class (dynamic provisioning). A PVC binds to a PV (static) or triggers a StorageClass to create a PV (dynamic).

K4 — Medium: 1. Check container logs (`kubectl logs`). 2. Check pod events (`kubectl describe pod`). 3. Verify resource limits vs. usage (OOMKilled). 4. Check liveness/readiness probe configuration and health endpoint status.

K5 — Medium: CoreDNS provides DNS resolution for Services and pods within the cluster. If CoreDNS pods are down, internal service discovery (e.g., `service-name.namespace`) fails, though external DNS resolution may still work depending on the cluster’s `resolv.conf` configuration.

K6 — Hard: The scheduler uses **Filtering** (remove nodes that don't meet requirements) and **Scoring** (rank remaining nodes). Factors include: 1. Resource requests (CPU/Memory), 2. Node Affinity/Taints, 3. Topology Spread Constraints (e.g., spread across zones).

K7 — Hard: Yes, it will be scheduled (256Mi request < 512Mi available). If it uses 600Mi at runtime, it exceeds the 512Mi limit and will be **OOMKilled** by the kernel, triggering a restart.

K8 — Hard: 1. Request hits External LB. 2. LB forwards to Ingress Controller (NGINX). 3. NGINX matches Ingress rules (host/path). 4. NGINX forwards to Kubernetes Service IP. 5. **kube-proxy** (iptables/IPVS) routes to Pod IP. 6. Pod receives request.

K9 — Medium: A **NetworkPolicy** restricts traffic flow between pods. Default behavior (no policy): All traffic is allowed. When applied: Traffic is denied by default unless explicitly allowed by the policy's ingress/egress rules.

K10 — Medium: **RollingUpdate** replaces pods gradually (zero downtime). **Recreate** kills all existing pods before starting new ones. Choose **Recreate** for stateful apps requiring exclusive access or when atomic updates are safer than gradual rollout.

K11 — Easy: A **Namespace** is a virtual cluster partition for resource isolation. Default namespaces: `default`, `kube-system`, `kube-public`, `kube-node-lease`.

K12 — Easy: `kubectl describe` shows detailed status. 1. **Events** (warnings/errors). 2. **Conditions** (Ready status). 3. **Mounts/Env vars** (detailed config). `kubectl get` only shows basic status/age.

K13 — Easy: A **ConfigMap** stores non-sensitive configuration data. Pods consume it via: 1. **Environment Variables** in the pod spec. 2. **Volume Mounts** (files inside the container).

K14 — Easy: **ClusterIP** (internal cluster access), **NodePort** (exposes port on every node's IP), **LoadBalancer** (provisions external cloud load balancer).

K15 — Easy: A **DaemonSet** ensures a copy of a pod runs on all (or specific) nodes. Use for logging agents, monitoring daemons, or storage daemons that must run on every node.

K16 — Medium: **Taints** repel pods from a node; **Tolerations** allow pods to schedule on tainted nodes. **Node Affinity** attracts pods to nodes. Use case: Taint for dedicated GPU nodes; Affinity for co-locating related services.

K17 — Medium: An **Operator** is a custom controller that manages complex stateful applications using CRDs. It differs from standard controllers by automating operational tasks (backup, scaling, upgrades). It solves the complexity of managing stateful apps in Kubernetes.

K18 — Hard: etcd uses **linearizability** (strong consistency). If quorum is lost, the cluster becomes read-only/unavailable. Recovery: Restore from the latest snapshot on a new etcd cluster or the remaining node, then rejoin nodes to form a new quorum.

K19 — Hard: **Role/ClusterRole** define permissions. **RoleBinding/ClusterRoleBinding** assign roles to users/groups.
```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata: namespace: my-ns
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata: namespace: my-ns
subjects:
- kind: User
  name: developer
roleRef:
  kind: Role
  name: pod-reader
```

K20 — Hard: `ndots` (default 5) causes the resolver to try appending up to 5 suffixes to short names. If a name is short, it queries multiple domains, causing timeouts. Fix: Set `dnsConfig: { options: [{ name: "ndots", value: "1" }] }` in the pod spec.

D1 — Easy: **Rebase** rewrites history to create a linear commit chain. **Merge** preserves history with a merge commit. Avoid rebasing on shared branches where others have pulled the commits.

D2 — Easy: **TCP** is connection-oriented and reliable (HTTP, SSH). **UDP** is connectionless and faster (DNS, VoIP).

D3 — Medium: A **race condition** occurs when output depends on the sequence/timing of uncontrollable events. Example: Two threads incrementing a shared counter without a lock, resulting in lost updates.

D4 — Medium: **Symmetric** uses one key for encryption/decryption. **Asymmetric** uses a public/private key pair. TLS uses Asymmetric for the handshake (key exchange) and Symmetric for data transfer (performance).

D5 — Medium: A **layer** is a read-only filesystem diff in a Docker image. Layer caching improves build speed; place frequently changing instructions (e.g., `COPY .`) after stable dependencies to maximize cache hits.

D6 — Medium: 1. Check upstream service health. 2. Review application logs for errors. 3. Check resource limits (CPU/Memory throttling). 4. Verify rate limiting or circuit breaker status.

D7 — Hard: **CAP** states a distributed system can only guarantee 2 of 3: Consistency, Availability, Partition Tolerance. **CP** (Zookeeper, HBase) prioritizes consistency. **AP** (Cassandra, DynamoDB) prioritizes availability during partitions.

D8 — Medium: **Horizontal** adds more nodes; **Vertical** adds more resources to a node. Patterns enabling horizontal scaling: Stateless design, Load Balancing, Sharding.

D9 — Easy: An **environment variable** is a dynamic value affecting process behavior. Persist across sessions on Linux by exporting in `~/.bashrc` or `~/.profile`.

D10 — Easy: **JSON** is strict key-value (machine-friendly). **YAML** is human-readable (indentation-based). Advantage JSON: Strict parsing. Advantage YAML: Readable config files.

D11 — Easy: **REST** is an architectural style using HTTP. **GET** (retrieve), **POST** (create), **PUT** (replace), **DELETE** (remove).

D12 — Easy: **Version pinning** locks dependencies to specific versions. Important for reproducible builds, security (avoiding unexpected vulnerabilities), and stability.

D13 — Easy: **Compiled** (source to binary, e.g., C, Go). **Interpreted** (source to execution, e.g., Python, JavaScript).

D14 — Medium: A **reverse proxy** sits in front of servers (client -> proxy -> server). A **forward proxy** sits in front of clients (client -> proxy -> internet). NGINX (web server), HAProxy (load balancer).

D15 — Medium: A **deadlock** is a permanent blocking of processes. Coffman conditions: 1. Mutual Exclusion, 2. Hold and Wait, 3. No Preemption, 4. Circular Wait.

D16 — Medium: **Eventual consistency** guarantees updates propagate eventually. Cassandra uses **Read Repair** and **Hinted Handoff**. Tunable levels: ONE, QUORUM, ALL.

D17 — Hard: **mTLS** authenticates both client and server via certificates. Handshake: Client Hello, Server Hello, Server Cert, Client Cert, Verify, Finished. Commonly used in **Service Meshes** (e.g., Istio) for secure service-to-service communication.

D18 — Hard: A **hash table** maps keys to indices. Collision resolution: **Chaining** (linked lists) or **Open Addressing** (probing). Average time: O(1). Worst case: O(n) (all keys collide).

D19 — Hard: A **circuit breaker** prevents cascade failures. States: **Closed** (normal), **Open** (fail fast), **Half-open** (test recovery). It relates to retries by disabling retries when the circuit is open to prevent overload.

D20 — Hard: **Optimistic** (check version on commit) assumes low conflict. **Pessimistic** (lock on read) assumes high conflict. Example: Optimistic uses `version` column; Pessimistic uses `SELECT FOR UPDATE`.
