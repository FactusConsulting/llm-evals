DN1 ‚Äî **Class vs Struct:** `class` is a reference type (heap-allocated, nullable, supports inheritance). `struct` is a value type (stack-allocated, copy-on-assignment, no inheritance). Use `struct` for small, immutable data (e.g., `Point`, `Vector`) where value semantics are required and performance (avoiding allocation) matters.

DN2 ‚Äî **IEnumerable<T> vs IQueryable<T>:** `IEnumerable<T>` executes queries in-memory (client-side). `IQueryable<T>` builds expression trees translated to server-side queries (e.g., SQL). This distinction matters for DB queries because `IQueryable` allows filtering/paging on the server, reducing network traffic and improving performance.

DN3 ‚Äî **Async/Await:** `async/await` enables non-blocking I/O using a state machine. `Task` is a reference type representing an async operation. `ValueTask` is a value type (struct) representing an operation that might be completed synchronously. Use `ValueTask` in hot paths to reduce heap allocations when the operation often completes synchronously.

DN4 ‚Äî **Dependency Injection (DI):** DI decouples object creation from usage. **Lifetimes:** `Transient` (new instance every request), `Scoped` (new instance per request), `Singleton` (one instance for app lifetime). **Bug:** Injecting a `Scoped` service into a `Singleton` causes the Singleton to hold a reference to the Scoped service from the first request, leading to stale data or `ObjectDisposedException` when the scope ends.

DN5 ‚Äî **string vs ReadOnlySpan<char>:** `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a mutable view over memory (stack or heap) allowing zero-copy parsing. Use `Span<T>` for high-performance string manipulation (e.g., parsing, tokenizing) to avoid creating intermediate string allocations.

DN6 ‚Äî **Garbage Collector (GC):** GC manages memory via mark-and-sweep. **Generations:** Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **LOH:** Large Object Heap (objects > 85KB) is collected less frequently and not compacted by default. **IDisposable:** Needed for unmanaged resources (files, handles) because GC only manages managed memory; `Dispose` ensures timely cleanup.

DN7 ‚Äî **ConfigureAwait(false):** Default `await` captures the current `SynchronizationContext` (e.g., UI thread) to resume on it. `ConfigureAwait(false)` skips this. **Use:** In library code or background services to avoid deadlocks and improve throughput. **Don't use:** In UI/ASP.NET Core request context if you need to update UI or access request-specific data after the await.

DN8 ‚Äî **.NET 8/9 Features:** **Primary Constructors:** Define constructor parameters directly in class declaration (reduces boilerplate). **Collection Expressions:** `[1, 2, 3]` syntax for arrays/lists. **FrozenDictionary:** Immutable, high-performance hash map. **Example:**
```csharp
public class ServiceConfig(string Name, int Port) { }
// DI Registration: services.AddSingleton(new ServiceConfig("API", 8080));
```

DN9 ‚Äî **DbContext Concurrency:** **Cause:** `DbContext` is `Scoped`. If injected into a `Singleton` or reused across threads, it causes race conditions and `ObjectDisposedException`. **Fix:** Ensure `DbContext` is registered as `Scoped` and injected into `Scoped` services/controllers. Never share `DbContext` instances across requests.

DN10 ‚Äî **Record vs Class:** `record` provides value-based equality and immutability by default. `record struct` is a value type with value equality. `class` uses reference equality. **Choose:** `record` for DTOs/immutable data. `record struct` for small, immutable value objects. `class` for mutable, reference-based entities.

DN11 ‚Äî **ref, out, in:** `ref` passes by reference (read/write, must be initialized). `out` passes by reference (write only, must be assigned inside method). `in` passes by reference (read-only, prevents copy of large structs). Use `ref` for swapping, `out` for multiple return values, `in` for large read-only structs.

DN12 ‚Äî **Nullable Reference Types (NRT):** `?` annotation warns about potential null dereferences for reference types at compile-time. Differs from `int?` (nullable value type) which allows `null` at runtime. NRT requires `#nullable enable` and helps prevent `NullReferenceException`.

DN13 ‚Äî **LINQ:** Language Integrated Query. **Method vs Query:** Method (`Where`) is more flexible; Query (`from`) is more readable for complex joins. **Deferred Execution:** Query is not executed until enumerated (e.g., `ToList()`). This allows composition but can hide errors until execution.

DN14 ‚Äî **Generic Constraints:** `where T : class` (reference type), `where T : struct` (value type), `where T : new()` (parameterless constructor), `where T : IComparable<T>` (implements interface). **Combine:** `where T : class, IDisposable` to ensure T is a reference type that can be disposed.

DN15 ‚Äî **Abstract Class vs Interface:** Abstract class can have implementation and state; Interface defines contract only. **C# 8+ Default Interface Methods:** Interfaces can now have method implementations, blurring the line. Use abstract class for shared code/state, interface for polymorphic contracts.

DN16 ‚Äî **Middleware Pipeline:** Middleware components form a pipeline. `app.Use()` adds middleware that calls `next()`. `app.Map()` branches the pipeline for specific paths. `app.Run()` is terminal (no `next`). **Ordering:** Critical (e.g., Auth before Routing). **Exception Handling:** Middleware can catch exceptions and return custom responses.

DN17 ‚Äî **Channel<T>:** Async queue for producer-consumer. `BlockingCollection<T>` is synchronous. **Bounded Channel:** Set `boundedCapacity` to limit queue size, applying backpressure to producers when full. Example: Limiting database writes during high load.

DN18 ‚Äî **Source Generators:** Compile-time code generation (IL) vs Reflection (runtime) or IL Weaving (post-build). **Incremental Generator:** Runs only when source files change, improving build performance. Preferred over `ISourceGenerator` for scalability and incremental compilation support.

DN19 ‚Äî **TPL Dataflow:** Async pipeline library. `ActionBlock` (process, no output), `TransformBlock` (process, output), `BufferBlock` (store). **Vs Channels:** Dataflow offers built-in backpressure, completion signaling, and complex graph topologies. Channels are lower-level primitives for simple queues.

DN20 ‚Äî **Minimal APIs:** Lightweight entry points without controllers. **Trade-offs:** Less boilerplate, but less structure for large apps. **Middleware:** Same as standard. **OpenAPI:** Supported via Swashbuckle. **Route Groups:** Organize endpoints (`MapGroup`). **Filters:** `EndpointFilter` for cross-cutting concerns (e.g., validation).

P1 ‚Äî **List vs Tuple:** `list` is mutable, ordered collection. `tuple` is immutable, ordered collection. `set` is mutable, unordered unique items. `frozenset` is immutable set. Use `tuple` for fixed data structures, `set` for membership testing.

P2 ‚Äî **GIL:** Global Interpreter Lock allows only one thread to execute Python bytecode at a time. **Affects:** CPU-bound multi-threading. **Workarounds:** Use `multiprocessing` (separate processes), C-extensions (release GIL), or async I/O.

P3 ‚Äî **Deepcopy vs Copy:** `copy` creates a new container but references same objects (shallow). `deepcopy` recursively copies objects. **Example:** `list1 = [[1]]`, `list2 = copy(list1)`. Modifying `list2[0][0]` changes `list1[0][0]`.

P4 ‚Äî **Decorators:** Functions that wrap other functions. **Time Decorator:**
```python
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 ‚Äî **Generators:** Functions that yield values lazily. **Vs List:** Generators save memory (no full list in RAM). **Yield from:** Delegates to another generator (Python 3.3+).

P6 ‚Äî **Asyncio:** Single-threaded event loop. `asyncio.run()` starts the loop. `asyncio.gather()` runs coroutines concurrently. `asyncio.create_task()` schedules a coroutine. **time.sleep()** blocks the loop; use `await asyncio.sleep()` instead.

P7 ‚Äî **MRO:** Method Resolution Order. Python 3 uses **C3 Linearization**. **Diamond Inheritance:** Resolves ambiguity by prioritizing left-to-right order. **Output:** `C`. MRO: D -> B -> C -> A. `C` overrides `who`, so `D().who()` calls `C.who()`.

P8 ‚Äî **Metaclass:** Class of a class. Controls class creation. **__init_subclass__:** Easier hook for subclass initialization (Python 3.6+). **Metaclass:** Use for enforcing class structure (e.g., ORM models). **Use Case:** `__init_subclass__` for plugin registration; Metaclass for framework-level validation.

P9 ‚Äî **Multiprocessing vs Threading:** `multiprocessing` bypasses GIL (CPU-bound). `threading` shares memory (GIL-bound). `concurrent.futures` provides high-level API for both. **Serialization:** `multiprocessing` requires pickling arguments, which fails for unserializable objects (e.g., open file handles).

P10 ‚Äî **__slots__:** Restricts instance attributes to a fixed set. **Use:** Save memory (no `__dict__`) and prevent dynamic attribute assignment. **Impact:** Reduces memory footprint significantly for large numbers of instances.

P11 ‚Äî **Virtual Environment:** Isolated Python environment. **Why:** Dependency management. **Differences:** `venv` (stdlib, Python 3.3+), `virtualenv` (3rd party, supports older Python), `conda` (package manager + env, supports non-Python deps).

P12 ‚Äî **Comprehensions:** Concise syntax for creating collections. **Vs Map/Filter:** Comprehensions are more Pythonic and readable for simple logic. **Prefer:** Comprehensions for lists/sets/dicts; `map/filter` for functional pipelines or lazy evaluation.

P13 ‚Äî **== vs is:** `==` compares values. `is` compares identity (memory address). **Interning:** Small integers (-5 to 256) are cached. `a is b` returns `True` for small ints because they point to the same object.

P14 ‚Äî **args/kwargs:** `*args` collects positional args into a tuple. `**kwargs` collects keyword args into a dict. **Markers:** `/` forces positional-only (before), `*` forces keyword-only (after).

P15 ‚Äî **Context Managers:** Manage resources (`with` statement). **__enter__/__exit__:** Manual implementation. **@contextmanager:** Decorator for generator-based managers. **Example:** Database connection manager that commits on success, rolls back on error.

P16 ‚Äî **Type Hinting:** Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. **Mypy/Pyright:** Use hints to detect type errors before runtime. `Protocol` enables structural typing (duck typing).

P17 ‚Äî **Data Model:** Dunder methods define behavior. `__repr__` (dev/debug), `__str__` (user). `__eq__` defines equality. **Hash:** If `__eq__` is defined, `__hash__` becomes `None` (unhashable) unless explicitly defined.

P18 ‚Äî **Descriptors:** Objects controlling attribute access (`__get__`, `__set__`, `__delete__`). **property/staticmethod:** Implemented via descriptors. **Use:** Custom validation, lazy loading, or managing class-level state.

P19 ‚Äî **Import System:** `sys.path` lists directories. **Finders:** Locate modules. **Loaders:** Load code. **Relative vs Absolute:** Relative (`from . import x`) depends on package location. **Namespace Packages:** Allow splitting a package across directories without `__init__.py`.

P20 ‚Äî **Dataclasses:** Auto-generates `__init__`, `__repr__`, etc. **Vs attrs/Pydantic:** `attrs` is more flexible; `Pydantic` adds validation. **__post_init__:** Logic after init. **Frozen:** Immutable instances. **Choose Pydantic:** When input validation (e.g., API payloads) is required.
