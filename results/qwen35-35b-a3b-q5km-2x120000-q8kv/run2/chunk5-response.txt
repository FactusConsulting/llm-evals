DN1 — **Class vs Struct:** `class` is a reference type (heap, nullable, inheritance supported). `struct` is a value type (stack/heap, copy-on-assign, no inheritance). Use `struct` for small, immutable data (e.g., coordinates, IDs) to avoid heap allocation and ensure value semantics.

DN2 — **IEnumerable<T> vs IQueryable<T>:** `IEnumerable<T>` executes queries in-memory (client-side). `IQueryable<T>` builds expression trees for server-side execution (e.g., SQL). This matters for DB queries because `IQueryable` translates to SQL (efficient filtering), while `IEnumerable` pulls all data to memory first (inefficient).

DN3 — **Async/Await:** `async/await` enables non-blocking I/O using a state machine. `Task` is a reference type (object allocation). `ValueTask` is a struct (zero allocation). Use `ValueTask` when the operation is often already completed (e.g., cached data) to reduce GC pressure.

DN4 — **Dependency Injection (DI):** DI decouples object creation from usage. Lifetimes: `Transient` (new instance every time), `Scoped` (new per request), `Singleton` (one instance for app lifetime). **Bug:** Injecting a `Scoped` service into a `Singleton` captures the first request's scope, leading to stale data or disposal errors when the request ends.

DN5 — **string vs ReadOnlySpan<char>:** `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a stack-based view (no allocation). Use `Span<T>` for high-performance string manipulation (parsing, slicing) to avoid creating intermediate string objects.

DN6 — **Garbage Collector (GC):** GC manages memory by reclaiming unreachable objects. Generations: Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). Large Object Heap (LOH) stores objects >85KB. `IDisposable` is needed for unmanaged resources (files, DB connections) because GC only manages managed memory; `Dispose` ensures timely cleanup.

DN7 — **ConfigureAwait(false):** Default behavior captures `SynchronizationContext` (e.g., UI thread). `ConfigureAwait(false)` avoids capturing it, allowing continuation on any thread. Use in library code to prevent deadlocks. Do NOT use in UI code where context is needed to update UI controls.

DN8 — **.NET 8/9 Features:**
*   **Primary Constructors:** `public class Service(int id) : IService { }`.
*   **Collection Expressions:** `int[] arr = [1, 2, 3];`.
*   **FrozenDictionary:** Immutable, high-performance dictionary.
*   **DI Example:** `public class MyService(int id) : IMyService { }` (Constructor injected directly).

DN9 — **DbContext ObjectDisposedException:** Likely cause: `DbContext` registered as `Transient` or captured in a `Singleton`/background task. Fix: Register `DbContext` as `Scoped` in DI (`AddDbContext`). EF Core `DbContext` is not thread-safe; it must be scoped to the request lifecycle to ensure proper disposal and isolation.

DN10 — **Record vs Class:** `class` uses reference equality. `record` uses value equality (props compared). `record struct` is a value-type record. Choose `record` for DTOs/Models where equality matters. Choose `class` for mutable entities or when inheritance is complex.

DN11 — **ref, out, in:** `ref` passes by reference (must be initialized before call). `out` passes by reference (must be assigned inside method). `in` passes by reference (read-only). Use `ref` for modification, `out` for multiple returns, `in` for large read-only structs.

DN12 — **Nullable Reference Types (NRT):** `?` on reference types (e.g., `string?`) warns about nullability at compile time. Differs from `int?` (nullable value type) which is a struct wrapper. NRT helps prevent `NullReferenceException` without runtime overhead.

DN13 — **LINQ:** Language Integrated Query. Method syntax (`list.Where(...)`) vs Query syntax (`from x in list`). Both compile to same IL. **Deferred Execution:** Query is not executed until enumerated (e.g., `ToList()`), allowing composition.

DN14 — **Generic Constraints:** `where T : class` (reference), `where T : struct` (value), `where T : new()` (parameterless ctor), `where T : IComparable<T>` (interface). Combine with `&` (e.g., `where T : class, IComparable<T>`). Use to enforce capabilities on generic types.

DN15 — **Abstract Class vs Interface:** Abstract class provides partial implementation + contract. Interface provides contract only. C# 8+ **default interface methods** allow interfaces to have implementation logic, blurring the line (interfaces can now share code).

DN16 — **Middleware Pipeline:** `app.Use()` adds middleware (can call next). `app.Map()` branches pipeline. `app.Run()` terminal middleware. Ordering matters (Auth before endpoints). Exception handling middleware (`UseExceptionHandler`) wraps the pipeline to catch unhandled errors globally.

DN17 — **Channel<T> vs BlockingCollection<T>:** `Channel<T>` is async-first, supports backpressure (bounded/unbounded). `BlockingCollection<T>` is blocking/thread-safe. Use bounded channels to limit memory usage and apply backpressure to producers when consumers are slow.

DN18 — **Source Generators:** Compile-time code generation. Faster/safer than reflection, less invasive than IL weaving. **Incremental Generators** (preferred) cache results and only regenerate changed parts, improving build performance over `ISourceGenerator` (full regeneration).

DN19 — **TPL Dataflow:** Pipeline library. `ActionBlock` (process), `TransformBlock` (map), `BufferBlock` (queue). Compared to `Channels`, Dataflow offers more complex flow control (fan-out, fan-in, cancellation) but higher overhead.

DN20 — **Minimal APIs:** Delegate-based endpoints (no controllers). Trade-offs: Less boilerplate, harder to scale complex auth/middleware, OpenAPI support requires Swashbuckle. **Route Groups** organize endpoints (`MapGroup`). **Endpoint Filters** apply logic (e.g., validation) to specific endpoints.

P1 — **List vs Tuple:** `list` is mutable, ordered. `tuple` is immutable, ordered. `set` is mutable, unique, unordered. `frozenset` is immutable. Use tuples for fixed data, sets for uniqueness.

P2 — **GIL:** Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects CPU-bound workloads. Workarounds: Multiprocessing (separate processes), C-extensions (release GIL), or async I/O.

P3 — **Deepcopy vs Copy:** `copy` creates shallow copy (nested objects shared). `deepcopy` creates recursive copy. **Example:** `list1 = [[1]]`, `list2 = copy(list1)`. Modifying `list2[0][0]` changes `list1[0][0]` because inner list is shared.

P4 — **Decorators:** Functions wrapping others.
```python
import time
def log_time(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.perf_counter() - start}s")
        return result
    return wrapper
```

P5 — **Generators:** Functions with `yield` return a generator object (lazy evaluation). Unlike lists, they don't store all values in memory. `yield from` delegates to another generator.

P6 — **Asyncio:** Event loop manages async tasks. `asyncio.run()` starts loop. `asyncio.gather()` runs tasks concurrently. `asyncio.create_task()` schedules task. `time.sleep()` blocks the loop; use `await asyncio.sleep()` to yield control.

P7 — **MRO:** Method Resolution Order. Python 3 uses **C3 Linearization**. In `D(B, C)`, MRO is `D -> B -> C -> A`. `D().who()` calls `C.who()` because `C` appears after `B` in MRO and overrides `A`. Output: "C".

P8 — **Metaclass:** Class of a class. Use `__init_subclass__` for subclass hooks (simpler, no metaclass needed). Use Metaclass for enforcing class structure globally (e.g., auto-registering plugins).

P9 — **Multiprocessing vs Threading:** `multiprocessing` (processes, bypass GIL, CPU-bound, pickling overhead). `threading` (threads, GIL, I/O-bound, shared memory). `concurrent.futures` provides high-level API for both. Pickling required for multiprocessing args.

P10 — **__slots__:** Restricts instance attributes to a fixed set. Prevents `__dict__` creation. Saves memory (no dict per instance) and prevents dynamic attribute assignment.

P11 — **Virtual Environment:** Isolates dependencies. `venv` (stdlib, Python 3.3+), `virtualenv` (3rd party, older), `conda` (env + package manager, data science). Prevents dependency conflicts.

P12 — **Comprehensions:** Concise loop syntax (`[x for x in list]`). `map()`/`filter()` are functional. Prefer comprehensions for readability. Use `map()`/`filter()` for lazy evaluation in Python 3.

P13 — **== vs is:** `==` checks value equality. `is` checks identity (memory address). **Interning:** Small integers (-5 to 256) are cached. `a is b` returns True for small ints due to interning, not value equality.

P14 — **args/kwargs:** `*args` (tuple of positional), `**kwargs` (dict of keyword). `/` marks positional-only params. `*` marks keyword-only params. Used for flexible function signatures.

P15 — **Context Managers:** `with` statement for setup/teardown. `__enter__`/`__exit__` (class) or `@contextmanager` (generator). Example: DB connection manager ensuring commit/rollback.

P16 — **Type Hinting:** Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. `mypy`/`pyright` use these to catch type errors before runtime without runtime overhead.

P17 — **Data Model:** `__repr__` (dev/debug), `__str__` (user). `__eq__` defines equality. If `__eq__` defined, `__hash__` becomes `None` (unhashable) unless explicitly defined. `__getattr__` (fallback), `__getattribute__` (always called).

P18 — **Descriptors:** Objects implementing `__get__`, `__set__`, `__delete__`. `property`, `classmethod`, `staticmethod` are descriptors. Used to customize attribute access (e.g., validation, lazy loading).

P19 — **Import System:** `sys.path` determines search paths. `PYTHONPATH` overrides. `__init__.py` marks package (optional in 3.3+ for namespace). **Finder** locates module, **Loader** loads it. Relative imports (`from . import x`) vs Absolute (`from pkg import x`).

P20 — **Dataclasses:** Auto-generates `__init__`, `__repr__`, etc. `attrs` is more feature-rich. `Pydantic` adds validation. Use `Pydantic` for API models requiring validation (e.g., FastAPI). `__post_init__` runs after init. `frozen=True` makes immutable.
