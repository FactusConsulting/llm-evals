JS1 — **var** is function-scoped and hoisted (initialized as undefined). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages async execution. The **call stack** executes sync code. When async ops complete, callbacks go to the **microtask queue** (promises) or **macrotask queue** (setTimeout). The event loop processes the call stack, then all microtasks, then one macrotask, repeating.

JS3 — A **closure** is a function bundled with its lexical environment. Example: A factory function returning an inner function accessing outer vars. **Bug:** In a `for` loop using `var`, all closures share the same `i` variable. **Fix:** Use `let` (block scope) or an IIFE to capture the value.

JS4 — **Promise** represents an async result. **async/await** is syntactic sugar for promises. Error handling: `try/catch` for await, `.catch()` for promises. **Promise.all** fails fast on first rejection. **Promise.allSettled** waits for all to settle (fulfilled or rejected) and returns status objects.

JS5 — **Interface** defines shape, supports merging, and is erased at runtime. **Type** defines shape, supports primitives/unions/intersections. Use **Interface** for public API contracts, **Type** for unions or primitives. **Discriminated unions** are unions with a literal type field used to narrow types (e.g., `{type: 'a', a: 1} | {type: 'b', b: 2}`).

JS6 — **Prototypal inheritance** is a chain of objects where property lookup traverses the prototype chain. **Object.create(proto)** creates an object with a specific prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` to set the prototype chain.

JS7 — **this** refers to the execution context. **Binding rules (priority):** 1. Explicit (call/apply/bind), 2. New (constructor), 3. Implicit (obj.method), 4. Default (window/undefined). **bind()** returns a new function with bound `this`. It **cannot** be re-bound.

JS8 — **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** notifies when an object is collected. Use for caches or cleanup. **WeakMap** keys must be objects because primitives are immutable and GC cannot track them; this allows keys to be garbage collected when no longer referenced.

JS9 — V8 optimizes via **Hidden Classes** (shapes for objects) and **Inline Caching** (caching property lookup). **Deoptimization** occurs when assumptions break (e.g., adding properties dynamically, polymorphic calls). Patterns causing deopt: `delete` properties, dynamic `this`, or changing object shapes.

JS10 — **Tree-shaking** removes unused exports. **ES Modules** allow static analysis (static `import`), enabling effective tree-shaking. **CommonJS** (`require`) is dynamic, so bundlers cannot statically analyze dependencies, making tree-shaking ineffective.

JS11 — **null** is an intentional absence of value. **undefined** is a variable declared but not assigned. **==** coerces types (null == undefined is true). **===** checks strict equality. **??** (nullish coalescing) returns right side if left is null/undefined. **?.** (optional chaining) safely accesses nested props.

JS12 — **map()** transforms elements. **filter()** selects elements. **reduce()** accumulates values. **Initial value** in reduce prevents undefined on first iteration. **flatMap()** maps then flattens one level; useful for transforming arrays of arrays into a single array.

JS13 — **Template literals** use backticks for multi-line strings and interpolation (`${}`). **Tagged templates** call a function with the string parts and values. Example: `html`<div>${name}</div>` calls `html(['<div>', '</div>'], name)`.

JS14 — **Destructuring** extracts values from objects/arrays. **Object:** `{a, b}`. **Array:** `[a, b]`. **Defaults:** `{a = 1}`. **Renaming:** `{a: b}`. **Rest:** `...rest` collects remaining elements.

JS15 — **for...in** iterates over enumerable property keys (including inherited). **for...of** iterates over iterable values. Avoid **for...in** on arrays because it iterates keys, includes non-index properties, and is slower.

JS16 — **Generics** allow type parameters (e.g., `T`). **Constraints:** `T extends SomeType`. **Conditional types:** `T extends U ? X : Y`. **infer:** Extracts a type from a type (e.g., `infer R` in `Promise<R>`). Example: `type Unwrap<T> = T extends Promise<infer R> ? R : T`.

JS17 — **CommonJS** (`require`) is synchronous, dynamic, and Node.js default. **ES Modules** (`import`) are static, async, and browser standard. Node.js supports both. `"type": "module"` in `package.json` forces Node to treat `.js` files as ES Modules.

JS18 — **Structural typing** means types are compatible if they have the same shape. **Excess property checks** apply to object literals to prevent typos. **extends** is for inheritance/assignment. **satisfies** checks type without widening (preserves literal types). **Variance** in TS is implicit (covariant/contravariant) based on usage, not explicit keywords.

JS19 — **Proxy** intercepts operations (get, set, etc.). **Reflect** provides default implementations. Example: `new Proxy(obj, { set: (t, k, v) => { console.log(k); return true; } })`. Used for validation, logging, or reactive frameworks (Vue/React).

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provides atomic ops. **Ordering:** Memory barriers ensure visibility. **Atomics.wait** blocks until notified. **Atomics.notify** wakes waiting threads. Requires Cross-Origin Isolation headers in browsers.

B1 — **$@** expands to separate quoted arguments. **$*** expands to a single string. **"$@"** preserves each argument as a separate word (safe for loops). **"$*"** joins all args into one string (dangerous with spaces).

B2 — **set -e** exits on error. **set -u** errors on undefined vars. **set -o pipefail** makes pipeline fail if any command fails. Together they ensure robust scripts.

B3 — **$(command)** is modern command substitution (nested). **`command`** is legacy. **$((expression))** is arithmetic expansion (built-in). **expr** is an external command (slower, less feature-rich).

B4 — **Process substitution** `<(command)` creates a file descriptor (e.g., `/dev/fd/63`). Use when a command expects a filename but needs dynamic input. Example: `diff <(sort a) <(sort b)`.

B5 — Safe iteration: `for f in *; do ...` or `find . -print0 | while IFS= read -r -d '' f; do ...`. **`for f in $(ls)`** is dangerous because word splitting breaks filenames with spaces.

B6 — **Here document** (`<<EOF`) reads stdin from a file. **Here string** (`<<<`) reads stdin from a string. **`<<-EOF`** strips leading tabs from the input lines.

B7 — `cmd > file 2>&1`: stdout to file, then stderr to stdout (file). Both captured. `cmd 2>&1 > file`: stderr to stdout (terminal), then stdout to file. Only stdout captured. `&> file` is shorthand for `> file 2>&1`.

B8 — Use `mkdir` for locking (atomic). Function: `mkdir /tmp/lock` (trap `rmdir` on EXIT). Handle SIGTERM by trapping it to cleanup lock. `mkdir` is preferred because it fails atomically if lock exists.

B9 — **Single quotes** (`'...'`) literal (no expansion). **Double quotes** (`"..."`) expand vars/globbing. **No quotes** expand vars, globbing, and word splitting. Example: `echo $HOME` (no quotes) vs `echo "$HOME"` (safe) vs `echo '$HOME'` (literal).

B10 — **File tests:** `-e` (exists), `-d` (dir), `-r` (readable). **`[ ]`** is POSIX test. **`[[ ]]`** is Bash builtin (supports regex, `&&`/`||`).

B11 — **$?** (exit code of last cmd), **$$** (current PID), **$!** (PID of last bg job), **$#** (number of args).

B12 — **Function:** `name() { ... }` or `function name { ... }`. **Return:** Via `exit code` (0-255) or `echo` (stdout).

B13 — **PATH** is colon-separated dirs. Bash searches PATH for external commands. **Builtin** is part of shell (e.g., `cd`). **External** is binary. Check with `type` or `which`.

B14 — **trap** sets handlers. `trap 'cleanup' EXIT INT TERM`. **Execution order:** EXIT runs last. Multiple signals: handler runs once per signal unless blocked.

B15 — **xargs** builds command lines from stdin. **Substitution** runs cmd immediately. **xargs -0** + **find -print0** handles newlines/spaces. **xargs -P** runs jobs in parallel.

B16 — **Arrays:** `arr=(1 2)`. **Associative:** `declare -A map`. **Append:** `arr+=(val)`. **Length:** `${#arr[@]}`. **Iterate:** `for i in "${arr[@]}"`.

B17 — **Subshell** `(...)` forks a new process (vars lost). **Group** `{ ...; }` runs in current shell (vars persist). **Pipe** `cmd | while` runs `while` in subshell, so vars set inside are lost.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (force kill, no trap). **Graceful shutdown:** Trap SIGTERM, cleanup bg jobs, then exit.

B19 — Script: `split -l 1000 log.log part_`, `find . -name part_* -print0 | xargs -0 -P 4 -I {} bash process.sh {}`, `cat part_* > merged.log`. **Trade-offs:** Parallelism vs I/O contention. **Error:** Check exit codes of parallel jobs, aggregate failures.

B20 — **exec** replaces current shell process. **FD manipulation:** `exec 3>file` opens fd 3. `exec 3>&-` closes it. **Coprocess** (`coproc`) creates bidirectional pipe. Use for persistent background processes needing I/O.

PS1 — **Write-Output** sends objects to the pipeline. **Write-Host** writes to the console (screen). **Distinction:** Write-Output is pipeline-safe; Write-Host breaks pipelines and is harder to capture.

PS2 — **Pipeline** passes objects. **ForEach-Object** is a cmdlet (pipeline-aware, processes items one-by-one). **foreach** is a statement (loops over collection in memory). Use **ForEach-Object** for streaming; **foreach** for in-memory collections.

PS3 — **[PSCustomObject]** is a typed object with properties. **[hashtable]** is dynamic key-value. Use **PSCustomObject** for structured data. Convert: `[PSCustomObject]@{Key='Value'}`.

PS4 — **Terminating error** (throws exception, stops script). **Non-terminating** (continues). **-ErrorAction** overrides preference. **try/catch** catches terminating errors.

PS5 — **Splatting** `@params` passes a hashtable as arguments. **Difference:** Passing hashtable as single param vs expanding it into named parameters.

PS6 — **Remoting** uses WinRM (HTTP/HTTPS) or SSH (v7+). **Invoke-Command** runs one-off commands. **Enter-PSSession** starts interactive session.

PS7 — Returns **Object[]** even for single item. Returns **$null** if 0 items. **Pattern:** `$result = @(); ... return $result`. Check `$result.Count` or `$null -eq $result`.

PS8 — **[scriptblock]** is a code container. Pass via `-ArgumentList`. **Double-hop** = Kerberos delegation issue (client -> server -> target). **Solve:** CredSSP (insecure) or constrained delegation.

PS9 — **@()** creates array. **[array]** casts to array. **Index null** throws error. **Index empty array** returns empty. Force single: `@($single)`.

PS10 — **-eq** (equal), **-like** (wildcard), **-match** (regex). **-c** variants are case-sensitive. **-contains** checks if array contains object. **-in** checks if value is in array.

PS11 — **Module** is a reusable script/library. **.psm1** (script), **.psd1** (manifest), **.dll** (binary). **Import-Module** loads it into session.

PS12 — **Interpolation** `"$var"` expands vars. **'...'** is literal. **Here-string** `@"..."@` allows multi-line strings.

PS13 — **Get-Member** lists properties/methods. **Properties** (data), **Methods** (functions), **NoteProperties** (added dynamically). **Select-Object** (subset props), **Where-Object** (filter objects).

PS14 — **Classes** (v5) are C#-like. **Limitations:** No interfaces, limited inheritance. **Use:** Stateful objects, encapsulation. **Function:** Stateless logic.

PS15 — **CmdletBinding** enables advanced features. **SupportsShouldProcess** (Confirm/WhatIf). **Begin/Process/End** (streaming). **ValueFromPipeline** allows pipeline input.

PS16 — **Providers** expose data as file system. **Get-ChildItem** works on Registry (`HKLM:\`), Cert (`Cert:\`), Env (`Env:`). Access registry like files.

PS17 — **Runspace** (thread), **RunspacePool** (pool). **ForEach-Object -Parallel** (v7+) uses pool. Manual runspace for complex control. Thread safety: avoid shared mutable state.

PS18 — **PSScriptAnalyzer** (linting). **Rules** (severity). **Suppress** via `.ps1xml` or comments. **CI/CD:** Run in pipeline, fail build on errors.

PS19 — **$using:** passes local var to remote. **-ArgumentList** serializes objects. **Limits:** CLIXML depth (complex objects may fail). Use primitive types or serialize to JSON.

PS20 — **DSC** (Desired State Config). **Config** (code), **Resource** (action), **LCM** (manager). **WinPS** vs **PS7**: PS7 uses DSC Core (cross-platform). **Replaced:** Often by Ansible/Choco/Pkgmgr for simpler tasks, but DSC remains for Windows compliance.
