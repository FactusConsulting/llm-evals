N1 — A Layer 2 switch forwards frames based on MAC addresses within a broadcast domain (VLAN). A Layer 3 switch performs routing based on IP addresses, allowing traffic to move between VLANs/subnets without an external router.

N2 — Network Address: 10.0.0.0. Broadcast Address: 10.0.15.255.
*Calculation:* /20 mask (255.255.240.0) creates blocks of 16 in the third octet. 10.0.0.0 to 10.0.15.255.

N3 — 1. Client sends SYN. 2. Server replies SYN-ACK. 3. Client sends ACK. If the final ACK is lost, the server times out the connection and closes it. The client may retry the SYN, but the server will not retransmit the SYN-ACK.

N4 — SNAT (Source NAT) modifies the source IP (e.g., internal hosts accessing the internet). DNAT (Destination NAT) modifies the destination IP (e.g., port forwarding external traffic to an internal server).
*Example:* SNAT for internet access; DNAT for exposing an internal web server on port 80.

N5 — 1. The certificate Common Name (CN) or Subject Alternative Name (SAN) does not match the hostname in the URL. 2. The certificate chain is incomplete or invalid. 3. The system clock on the client is skewed, causing certificate validity errors.

N6 — VLAN is a Layer 2 broadcast domain limited by physical topology and 4096 IDs. VxLAN is an overlay network encapsulating Layer 2 frames in UDP over Layer 3, supporting 16 million segments. Choose VxLAN for multi-tenant clouds, datacenter interconnects, or when exceeding VLAN limits.

N7 — The default route likely points to eth0, so return traffic from eth1 uses the default gateway via eth0. Fix with Policy-Based Routing:
1. `ip rule add from 192.168.1.0/24 table eth1`
2. `ip route add default via 192.168.1.1 dev eth1 table eth1`

N8 — BGP selects the best path based on attributes. Common order (Cisco IOS):
1. Weight (Local to router)
2. Local Preference
3. Locally originated
4. AS Path Length (Shortest)
5. Origin Code (IGP < EGP < Incomplete)

N9 — VxLAN adds ~50 bytes (14 Eth + 20 IP + 8 UDP + 8 VxLAN). IPsec adds ~50 bytes (ESP header/trailer). Total overhead ≈ 100 bytes.
*Max Inner Payload:* 1500 - 100 = 1400 bytes.

N10 — ARP resolves IP addresses to MAC addresses on a local subnet. Gratuitous ARP announces an IP/MAC binding to update neighbor caches and detect IP address conflicts.

N11 — Public IPs are routable on the internet; Private IPs are for internal use. RFC 1918 ranges:
1. 10.0.0.0/8
2. 172.16.0.0/12
3. 192.168.0.0/16

N12 — DNS resolves domain names to IP addresses. A record maps to IPv4. AAAA record maps to IPv6. CNAME record creates an alias for another domain name.

N13 — DHCP dynamically assigns IP configuration. DORA Process:
1. **D**iscover (Client broadcasts)
2. **O**ffer (Server responds)
3. **R**equest (Client requests IP)
4. **A**cknowledge (Server confirms)

N14 — Hub (Layer 1, broadcasts to all ports), Switch (Layer 2, forwards based on MAC), Router (Layer 3, routes based on IP).

N15 — Half-duplex allows transmission in only one direction at a time. Full-duplex allows simultaneous transmission and reception. Modern Ethernet links use Full-duplex.

N16 — TTL (Time To Live) prevents packets from looping indefinitely. Traceroute sends packets with incrementing TTL (1, 2, 3...); routers decrement TTL and return ICMP "Time Exceeded" when it hits 0, revealing the hop.

N17 — NAT Gateway translates private IPs to public IPs. Static NAT is 1:1 mapping. Dynamic NAT maps private to a pool of public IPs. PAT (Port Address Translation) maps multiple private IPs to one public IP using unique ports.

N18 — ECMP uses multiple paths with equal cost to load balance. Hashing (e.g., src/dst IP/port) determines the path. Asymmetric flows occur if request and response take different paths; stateful firewalls may drop the response if it arrives on a different interface than the request.

N19 — The TCP window size (64KB) is smaller than the Bandwidth-Delay Product (10MB), limiting throughput. TCP Window Scaling (RFC 1323) allows the window to be shifted left (multiplied) via a scale factor in the SYN packet, enabling windows > 64KB.

N20 — 802.1Q trunk ports tag frames with a VLAN ID. On Linux: `ip link add link eth0 name eth0.10 type vlan id 10`. Frame format: Ethernet Header + 802.1Q Tag (4 bytes: TPID 0x8100 + TCI) + Payload.

L1 — A hard link is a direct reference to the same inode (file data). A symbolic link is a separate file containing a path to the target. Hard links cannot cross filesystems; symlinks can.

L2 — `find /var/log -type f -size +100M`

L3 — State D means the process is in uninterruptible sleep, typically waiting for I/O (disk, NFS). It cannot be killed with `kill -9` until the I/O operation completes.

L4 — Cgroups v1 uses separate hierarchies for each resource type. Cgroups v2 uses a unified hierarchy. Systemd prefers v2 for better resource control, simplicity, and support for newer kernel features.

L5 — nftables is the modern replacement for iptables, using a single table syntax. The kernel handles both via the nftables backend; iptables commands are translated to nftables rules for compatibility.

L6 — A file was deleted but a process still has it open. The space is not freed until the process closes the file. Check with `lsof | grep deleted`.

L7 — 1. UEFI firmware initializes hardware. 2. Bootloader (GRUB) loads kernel. 3. Kernel initializes hardware and mounts initramfs. 4. Initramfs mounts the real root filesystem. 5. Systemd takes over as PID 1 and starts the target.

L8 — OOM killer fires when the kernel cannot reclaim enough memory (including cache) to satisfy an allocation. `vm.swappiness` controls swap usage; `vm.vfs_cache_pressure` controls cache reclaim priority.

L9 — `ip netns` creates isolated network namespaces. Container runtimes use network namespaces to isolate network interfaces, routing tables, and firewall rules for each container.

L10 — SIGTERM (15) requests graceful termination (catchable). SIGKILL (9) forces immediate termination (uncatchable). SIGQUIT (3) requests termination with core dump (catchable).

L11 — stdin (0), stdout (1), stderr (2). To redirect stderr to a file while keeping stdout on terminal: `command 2>error.log`

L12 — `chmod 755` sets permissions to `rwxr-xr-x`. 7 (Owner: Read/Write/Execute), 5 (Group: Read/Execute), 5 (Others: Read/Execute).

L13 — Root (UID 0) has unrestricted system access. A user with sudo privileges can execute specific commands as root after authentication, adhering to the principle of least privilege.

L14 — A package manager installs, updates, and removes software. Debian/Ubuntu: `apt`/`dpkg`. RHEL/Fedora: `dnf`/`yum`.

L15 — `/etc/fstab` defines filesystem mount points at boot. A misconfiguration can prevent the system from booting, often requiring single-user mode to fix.

L16 — LVM abstracts physical storage. PV (Physical Volume) is the disk/partition. VG (Volume Group) pools PVs. LV (Logical Volume) is the virtual disk. Extend: `lvextend -L +1G /dev/vg/lv` then `resize2fs /dev/vg/lv`.

L17 — I/O schedulers order disk requests. `mq-deadline` (latency), `bfq` (fairness), `none` (NVMe). Change at runtime: `echo mq-deadline > /sys/block/sda/queue/scheduler`.

L18 — Load average measures processes running or waiting for CPU/I/O. High I/O wait inflates load without high CPU usage. Diagnose with `iostat` (disk), `vmstat` (memory/swap), and `pidstat` (process).

L19 — SELinux enforces policies via labels. Enforcing (blocks), Permissive (logs), Disabled. Troubleshoot: Check `audit.log` for denials, use `semanage port -a` to allow ports, or `audit2allow` to generate rules.

L20 — Page fault occurs when accessing unmapped memory. Minor: Page in RAM (cache). Major: Page on disk (read required). THP uses 2MB pages to reduce TLB misses but can cause latency spikes during compaction. Disable for latency-sensitive apps.
