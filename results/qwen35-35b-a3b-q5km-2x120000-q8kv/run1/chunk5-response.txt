DN1 — **Class vs Struct**: `class` is a reference type (heap, inheritance supported), `struct` is a value type (stack/inline, no inheritance). Use `struct` for small, immutable data (e.g., coordinates, IDs) to avoid heap allocation and copying overhead.

DN2 — **IEnumerable<T> vs IQueryable<T>**: `IEnumerable<T>` executes in-memory (LINQ to Objects). `IQueryable<T>` builds expression trees for server-side execution (LINQ to DB). Distinction matters because `IQueryable` translates queries to SQL; `IEnumerable` pulls all data to memory first.

DN3 — **Async/Await**: Compiles to a state machine. `Task` is a reference type (object allocation). `ValueTask` is a struct (zero allocation). Use `ValueTask` for async methods that often complete synchronously or in hot paths to reduce GC pressure.

DN4 — **Dependency Injection**: Pattern for loose coupling. Lifetimes: `Transient` (new every resolve), `Scoped` (new per request), `Singleton` (one instance). Bug: Injecting a `Scoped` service into a `Singleton` causes the Singleton to hold a stale reference to the first request's scope, leading to data leakage or disposal errors.

DN5 — **string vs ReadOnlySpan<char>**: `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a stack-only view (no allocation). Use `Span<T>` for high-performance string manipulation (parsing, slicing) to avoid creating intermediate string objects.

DN6 — **Garbage Collector**: Manages heap memory. Generations: Gen 0 (short-lived), Gen 1 (medium), Gen 2 (long-lived). Large Object Heap (LOH) stores objects >85KB. `IDisposable` is needed for unmanaged resources (files, DB connections) because GC only manages managed memory.

DN7 — **ConfigureAwait(false)**: Prevents capturing the `SynchronizationContext`. Use in library code to avoid deadlocks and improve performance. Do NOT use in UI/ASP.NET Core request context if you need to marshal back to the UI thread (though ASP.NET Core has no context by default).

DN8 — **.NET 8/9 Features**: Primary constructors (C# 12) reduce boilerplate. Collection expressions (`[1, 2]`) simplify array creation. `FrozenDictionary` is immutable and fast. Example: `public class Service(int Id) : IService { }`.

DN9 — **DbContext Exception**: Likely cause: `DbContext` registered as `Singleton` instead of `Scoped`. `DbContext` is not thread-safe and holds a connection. Fix: Register as `AddDbContext<Context>(ServiceLifetime.Scoped)`. High concurrency on a Singleton causes race conditions and disposal issues.

DN10 — **Record vs Class**: `record` provides value-based equality and immutability (init-only setters). `class` provides reference equality. `record struct` combines value type semantics with record features. Choose `record` for DTOs/immutable data, `class` for mutable entities.

DN11 — **ref, out, in**: `ref` passes by reference (modify original). `out` passes by reference (must assign before return). `in` passes by reference (read-only). Use `ref`/`out` for large structs to avoid copy; use `in` for read-only large parameters.

DN12 — **Nullable Reference Types (NRT)**: `?` annotation warns about potential null references on reference types (e.g., `string?`). Differs from `int?` (nullable value type), which is a struct wrapper (`Nullable<int>`) allowing `null` on value types.

DN13 — **LINQ**: Language Integrated Query. Method syntax (`list.Where(...)`) vs Query syntax (`from x in list`). Both compile to same IL. Deferred execution means query logic runs only when enumerated (e.g., `ToList()`).

DN14 — **Generic Constraints**: `where T : class` (reference), `struct` (value), `new()` (parameterless ctor), `IComparable<T>`. Combine with commas (e.g., `where T : class, IComparable<T>`). Use to enforce capabilities on generic types.

DN15 — **Abstract Class vs Interface**: Abstract class provides partial implementation + contract. Interface provides contract only. C# 8+ default interface methods allow adding methods without breaking implementations, blurring the line.

DN16 — **Middleware Pipeline**: `app.Use()` adds middleware (calls next). `app.Map()` branches pipeline. `app.Run()` ends pipeline. Ordering matters (Auth before Routing). Exception handling middleware catches exceptions thrown downstream.

DN17 — **Channel<T>**: Async producer-consumer primitive. `BlockingCollection<T>` is blocking/synchronous. Use bounded channels (`CreateBoundedChannel`) to apply backpressure, preventing memory spikes when producers outpace consumers.

DN18 — **Source Generators**: Compile-time code generation. Faster than reflection, no IL weaving. Incremental generators (C# 12) cache results and only regenerate on source change, preferred over `ISourceGenerator` which runs fully every build.

DN19 — **TPL Dataflow**: Pipeline library. `ActionBlock` (process), `TransformBlock` (map), `BufferBlock` (queue). Compared to `Channels`, Dataflow offers more flow control (linking, cancellation, completion) for complex pipelines.

DN20 — **Minimal APIs**: Delegate-based endpoints (no Controllers). Less boilerplate, faster startup. Trade-offs: less structure for large apps, harder to organize complex auth. Route groups (`MapGroup`) organize endpoints. Endpoint filters apply logic to specific endpoints.

P1 — **List vs Tuple**: `list` is mutable, ordered. `tuple` is immutable, ordered. `set` is mutable, unique, unordered. `frozenset` is immutable. Use tuples for fixed data, sets for uniqueness.

P2 — **GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects CPU-bound workloads. Workarounds: `multiprocessing` (separate processes), C-extensions (release GIL), or `asyncio` (I/O bound).

P3 — **Deepcopy vs Copy**: `copy` creates shallow copy (new container, same references). `deepcopy` creates new container and recursively copies objects. Shallow copy fails if nested mutable objects are modified.

P4 — **Decorators**: Functions that wrap others. Example:
```python
import time
def timeit(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

P5 — **Generators**: Functions using `yield` to produce items lazily (memory efficient). `yield from` delegates to another generator. Unlike lists, they don't store all items in memory.

P6 — **Asyncio**: Event loop manages async tasks. `asyncio.run()` starts loop. `asyncio.gather()` waits for multiple tasks. `asyncio.create_task()` schedules task. `time.sleep()` blocks the loop; use `await asyncio.sleep()`.

P7 — **MRO**: Method Resolution Order. Python 3 uses C3 linearization. For `D(B, C)`, MRO is `D -> B -> C -> A`. `D().who()` calls `C.who()` because `B` inherits `A` but `C` overrides `who`. Output: "C".

P8 — **Metaclass**: Class of a class. Use `__init_subclass__` for subclass hooks (easier, no metaclass needed). Use Metaclass for enforcing class structure at creation (e.g., registering plugins).

P9 — **Multiprocessing vs Threading**: `multiprocessing` (CPU-bound, separate memory, pickling required). `threading` (I/O-bound, shared memory, GIL). `concurrent.futures` provides high-level API for both.

P10 — **__slots__**: Restricts instance attributes to a fixed set. Prevents `__dict__` creation, saving memory. Prevents adding dynamic attributes.

P11 — **Virtual Environment**: Isolates project dependencies. `venv` (stdlib), `virtualenv` (3rd party), `conda` (env + packages + binaries). Prevents dependency conflicts.

P12 — **Comprehensions**: Concise loops (`[x for x in list]`). `map()`/`filter()` are functional. Prefer comprehensions for readability and Pythonic style.

P13 — **== vs is**: `==` checks value equality. `is` checks identity (memory address). Object interning caches small integers (-5 to 256), so `a is b` can be True for small ints.

P14 — **args/kwargs**: `*args` (tuple of positional), `**kwargs` (dict of keyword). `/` marks positional-only params. `*` marks keyword-only params.

P15 — **Context Managers**: `with` statement ensures cleanup. Implement via `__enter__`/`__exit__` or `@contextmanager`. Example: Database connection lock or temporary directory.

P16 — **Type Hinting**: Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. `mypy`/`pyright` use these for static analysis to catch errors before runtime.

P17 — **Data Model**: `__repr__` (dev), `__str__` (user). `__eq__` vs `__hash__`. Defining `__eq__` without `__hash__` makes instance unhashable (cannot be dict key). `__getattr__` (fallback), `__getattribute__` (always called).

P18 — **Descriptors**: Control attribute access (`__get__`, `__set__`, `__delete__`). `property`, `classmethod`, `staticmethod` are descriptors. Used for validation, computed properties.

P19 — **Import System**: `sys.path` determines search locations. `__init__.py` marks packages. Relative imports (`from . import x`) vs Absolute (`from pkg import x`). Finders locate modules; Loaders execute them. Namespace packages (PEP 420) omit `__init__.py`.

P20 — **Dataclasses**: Auto-generates `__init__`, `__repr__`, etc. `__post_init__` runs after init. `frozen=True` makes immutable. Pydantic adds validation and serialization. Choose Pydantic for API models requiring validation.
