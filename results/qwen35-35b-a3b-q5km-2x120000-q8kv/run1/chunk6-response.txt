JS1 — **var** is function-scoped and hoisted (initialized as undefined). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **Event Loop** manages execution. The **Call Stack** runs synchronous code. When empty, the loop checks the **Microtask Queue** (Promises, queueMicrotask) first, then the **Macrotask Queue** (setTimeout, I/O). Microtasks drain completely before the next macrotask.

JS3 — A **closure** is a function retaining access to its lexical scope even when executed outside it. **Bug:** In a `for` loop using `var`, all closures share the same `i` variable. **Fix:** Use `let` (block scope) or an IIFE to capture the current value.

JS4 — **Promise** represents async completion. **async/await** is syntactic sugar for promises. Error handling: `.catch()` for promises, `try/catch` for async. **Promise.all** fails fast on first rejection. **Promise.allSettled** waits for all to settle (fulfilled/rejected) and returns status objects.

JS5 — **Interface** defines shape (can be merged, implemented). **Type** defines data shape (unions, primitives, intersections). Use **Interface** for public API contracts, **Type** for unions or primitives. **Discriminated Unions** are unions with a literal type field used to narrow types (e.g., `{type: 'a'} | {type: 'b'}`).

JS6 — **Prototypal inheritance** links objects via a prototype chain. **Object.create(proto)** creates an object with `proto` as its internal prototype. **ES6 Classes** are syntactic sugar over prototypes, using `extends` and `constructor` to manage the chain.

JS7 — **this** binding rules (priority): 1. `new` (new instance), 2. Explicit (`call`, `apply`, `bind`), 3. Implicit (object method), 4. Default (global/window or undefined in strict mode). **bind()** returns a new function with bound `this`. It **cannot** be re-bound.

JS8 — **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** notifies when a weakly held object is collected. **WeakMap** keys must be objects because primitives are immutable and cannot be garbage collected based on reference strength.

JS9 — **V8 Optimization:** **Hidden Classes** define object shape for fast property access. **Inline Caching** caches property lookup results. **Deoptimization** occurs when shape changes (e.g., adding properties dynamically). Patterns causing deopt: dynamic property access, `arguments` object, `eval`.

JS10 — **Tree-shaking** removes unused code. **ES Modules** use static `import/export`, allowing static analysis. **CommonJS** uses dynamic `require`, making static analysis impossible, so bundlers often cannot tree-shake effectively.

JS11 — **null** is intentional absence; **undefined** is unassigned. `==` treats them equal; `===` distinguishes them. **Nullish Coalescing (`??`)** returns right operand if left is `null`/`undefined`. **Optional Chaining (`?.`)** safely accesses nested properties.

JS12 — **map()** transforms, **filter()** selects, **reduce()** accumulates. **Initial value** in reduce prevents using the first array element as the accumulator. **flatMap()** maps then flattens one level, useful for transforming arrays of arrays.

JS13 — **Template literals** use backticks for multi-line strings and interpolation `${}`. **Tagged templates** pass strings and values to a function. Example: `sql`SELECT * FROM ${table}`; function sql(strings, ...values) { return strings[0] + values[0]; }`.

JS14 — **Destructuring** extracts values. **Object:** `{a, b}`. **Array:** `[a, b]`. **Defaults:** `{a = 1}`. **Rename:** `{a: b}`. **Rest:** `{...rest}` collects remaining properties.

JS15 — **for...in** iterates enumerable **keys/properties** (including inherited). **for...of** iterates **values** of iterable objects. Avoid **for...in** on arrays because it iterates properties (including custom ones) and is slower.

JS16 — **Generics** use type parameters (`<T>`). **Constraints:** `T extends Foo`. **Conditional Types:** `T extends U ? X : Y`. **Infer:** `type Unpack<T> = T extends Promise<infer U> ? U : T`.

JS17 — **CommonJS:** Dynamic `require/module.exports` (Node). **ES Modules:** Static `import/export` (Browser/Node). **`"type": "module"`** in `package.json` forces Node to treat `.js` files as ES Modules.

JS18 — **Structural typing:** Compatibility based on shape. **Excess property checks** apply to object literals assigned to types. **extends** checks assignability; **satisfies** checks type without widening. **Variance:** `in` (contravariant), `out` (covariant) control type parameter usage.

JS19 — **Proxy** intercepts operations (get, set, apply). **Reflect** provides default implementations. Example: `new Proxy(obj, { set: (t, k, v) => { if (v < 0) throw Error; t[k] = v; } })`.

JS20 — **SharedArrayBuffer** allows shared memory between workers. **Atomics** provide atomic ops (wait, notify). **Ordering:** Memory barriers ensure visibility. **Atomics.wait** blocks until notified; **notify** wakes waiting threads.

B1 — **$@** expands to separate quoted arguments. **$*** expands to a single string. **"$@"** preserves individual arguments (safe for loops). **"$*"** joins them into one string.

B2 — **`set -e`**: Exit on error. **`-u`**: Exit on undefined vars. **`-o pipefail`**: Exit if any command in a pipeline fails.

B3 — **$(command)** is preferred over **`command`** (backticks). **$((expression))** is arithmetic expansion (shell built-in). **expr** is an external command (slower).

B4 — **Process substitution** `<(command)` creates a file descriptor. Example: `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — Use **`while IFS= read -r -d '' file; do ... done < <(find . -print0)`**. `for f in $(ls)` breaks on spaces/newlines and parses output instead of filenames.

B6 — **Here document** (`<<EOF`) feeds stdin. **Here string** (`<<<`) feeds a single string. **`<<-EOF`** strips leading tabs from the script lines.

B7 — `cmd > file 2>&1`: stdout to file, stderr to stdout (file). `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. `&>`: shorthand for `> file 2>&1`. First two differ due to redirection order.

B8 — Use **`mkdir`** for locking (atomic). Function: `mkdir /tmp/lock` inside `trap 'rmdir /tmp/lock' EXIT`. Handles race conditions via mkdir failure.

B9 — **Single quotes**: Literal (no expansion). **Double quotes**: Expand variables/glob. **No quotes**: Expand, glob, word split. Example: `echo $HOME` (no quotes) vs `'echo $HOME'` (literal).

B10 — **`[ -f file ]`** (exists), **`[ -d dir ]`**, **`[ -r file ]`**. **`[[ ]]`** supports regex and is safer (no word splitting). `[ ]` is POSIX.

B11 — **$?**: Exit code. **$$**: Script PID. **$!**: Last background PID. **$#**: Number of arguments.

B12 — **`func() {}`** or **`function func {}`**. Return values via **`return`** (exit code) or **`echo`** (stdout).

B13 — **PATH**: Colon-separated dirs. Bash searches PATH. **Builtin**: Shell internal (e.g., `cd`). **External**: Binary (e.g., `/bin/ls`). Check with **`type`** or **`which`**.

B14 — **`trap`** sets handlers. **EXIT**: On script exit. **ERR**: On error. **INT/TERM**: On signals. Execution order: Handlers run in order defined; multiple signals may be queued.

B15 — **xargs** builds command lines. **`find -print0`** + **`xargs -0`** handles spaces/newlines. **`-P`** runs jobs in parallel.

B16 — **Arrays**: `arr=()`, `arr+=()`, `${#arr[@]}`. **Associative**: `declare -A`, `arr[key]=val`. Syntax differs for iteration (`for k in "${!arr[@]}"`).

B17 — **Subshell** `(...)` forks a new process (vars lost). **Group** `{ ...; }` runs in current shell. **Pipe** `while` runs in subshell, so variable changes are lost after the loop.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (uncatchable). Handle graceful shutdown by trapping TERM, waiting for background jobs, then exiting.

B19 — Script: `split`, `xargs -P N -I {} process {}`, `cat results`. **Trade-offs**: Parallelism vs I/O contention. **Error**: Check exit codes of parallel jobs; aggregate failures.

B20 — **exec** replaces shell process. **FD manipulation**: `exec 3>file` opens FD 3. **Coprocess** (`coproc`) creates bidirectional pipe. Use for persistent communication channels.

PS1 — **Write-Output** sends objects to the pipeline. **Write-Host** writes to the console (bypasses pipeline). Pipeline breaks if Write-Host is used because no object is passed downstream.

PS2 — **Pipeline** passes objects. **ForEach-Object** (cmdlet) processes items one-by-one in pipeline. **foreach** (statement) iterates a collection in memory. Use **ForEach-Object** for streaming; **foreach** for logic control.

PS3 — **[PSCustomObject]** is a typed object with properties. **[hashtable]** is key/value pairs. Convert: `[PSCustomObject]@{Key='Value'}`. Use **PSCustomObject** for structured data; **hashtable** for dynamic config.

PS4 — **Terminating error** stops execution (throw). **Non-terminating** continues (catch). **-ErrorAction Stop** converts non-terminating to terminating. **try/catch** handles terminating errors.

PS5 — **Splatting** `@params` passes a hashtable as separate arguments. **Passing hashtable** passes it as a single parameter. Splatting is cleaner for cmdlets with many parameters.

PS6 — **Remoting** uses WinRM (HTTP) or SSH. **Invoke-Command** runs one-off commands remotely. **Enter-PSSession** starts an interactive session.

PS7 — Returns scalar if 1 item, array if >1. If 0 items, returns `$null`. **Pattern:** `$result = @(); $result += ...` ensures array type.

PS8 — **[scriptblock]** is code object. Pass via `-ArgumentList`. **Double-hop**: CredSSP allows delegation. **Invoke-Command -ArgumentList** serializes variables.

PS9 — **@()** creates array. **[array]** casts. **Indexing $null** throws error. **Indexing empty array** returns empty.

PS10 — **-eq** (value), **-like** (glob), **-match** (regex). **-ceq** (case-sensitive). **-contains** (object in collection), **-in** (value in collection).

PS11 — **Module** (.psm1/.psd1). **Script module**: Code file. **Manifest**: Metadata. **Binary**: Compiled DLL. **Import-Module** loads it into session.

PS12 — **"..."** interpolates. **'...'** literal. **Here-string** `@""@` allows multi-line. Expressions: `"${var}"`.

PS13 — **Get-Member** lists properties/methods. **Properties** (data), **Methods** (functions), **NoteProperties** (added dynamically). **Select-Object** (subset), **Where-Object** (filter).

PS14 — **Classes** (v5) support inheritance/constructors. **Limitations**: No interfaces, limited reflection. Use **classes** for stateful logic; **functions** for simple tasks.

PS15 — **[CmdletBinding()]** enables advanced features. **SupportsShouldProcess** (Confirm). **Begin/Process/End** blocks (setup, stream, cleanup).

PS16 — **Providers** expose data as file system. **Get-ChildItem HKLM:** lists registry keys. Access registry like files via `Registry:` drive.

PS17 — **Runspaces** manage threads. **RunspacePool** manages pool. **ForEach-Object -Parallel** (v7+) simplifies parallelism. Manual runspace for complex control.

PS18 — **PSScriptAnalyzer** enforces standards. **Rules** (severity). **Suppress** via comments. **CI/CD**: Integrate via `Invoke-ScriptAnalyzer`.

PS19 — **$using:** passes local vars to remote. **ArgumentList** serializes. **Limits**: CLIXML depth, complex objects may fail. Use **$using:** for simple vars.

PS20 — **DSC** configures state. **Resources** (tasks). **LCM** applies config. **WinPS**: XML based. **PS7**: PowerShell based. **Replaced by**: Ansible, Terraform, or native PS modules.
