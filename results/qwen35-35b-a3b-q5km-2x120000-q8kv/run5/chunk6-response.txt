JS1 — **var** is function-scoped and hoisted (initialized as `undefined`). **let** and **const** are block-scoped and hoisted but not initialized (Temporal Dead Zone). **Hoisting** is the behavior where declarations are moved to the top of their scope during compilation.

JS2 — The **event loop** manages asynchronous execution. The **call stack** executes synchronous code. When the stack is empty, the loop checks the **microtask queue** (promises, `queueMicrotask`) first, then the **macrotask queue** (`setTimeout`, `setInterval`).

JS3 — A **closure** is a function retaining access to its lexical scope even when executed outside that scope. **Bug:** In a `for` loop using `var`, all closures share the same `i` variable. **Fix:** Use `let` (block scope) or an IIFE to capture the value.

JS4 — **Promise** represents an async result. **async/await** is syntactic sugar for promises. Error handling: `try/catch` for `await`, `.catch()` for promises. **Promise.all** fails fast on first rejection; **Promise.allSettled** waits for all to settle (fulfilled or rejected).

JS5 — **Interface** defines a shape (can be implemented, merged). **Type** defines a value shape (unions, intersections, primitives). Use **interface** for public API contracts, **type** for complex unions or primitives. **Discriminated unions** are unions with a literal type field used to narrow types.

JS6 — **Prototypal inheritance** links objects via a prototype chain. **Object.create(proto)** creates an object with `proto` as its prototype. **ES6 classes** are syntactic sugar over prototypes, using `extends` and `constructor` to manage the chain.

JS7 — **this** binding rules (priority): 1. **new** (new instance), 2. **Explicit** (call/apply/bind), 3. **Implicit** (object.method), 4. **Default** (global/window or undefined in strict mode). **bind()** returns a new function with bound `this`. It cannot be re-bound.

JS8 — **WeakRef** holds a weak reference to an object (GC can collect it). **FinalizationRegistry** notifies when a weakly held object is collected. **WeakMap** keys must be objects because primitives are not GC-tracked. This allows keys to be garbage collected when no longer referenced.

JS9 — V8 uses **hidden classes** (shapes) to optimize property access. **Inline caching** caches the property offset. **Deoptimization** occurs when hidden classes change (e.g., adding properties dynamically). Patterns causing deopt: dynamic property addition, polymorphic calls.

JS10 — **Tree-shaking** removes unused code. **ES Modules** (static `import/export`) allow static analysis for tree-shaking. **CommonJS** (`require`) is dynamic, preventing static analysis, so bundlers often cannot tree-shake effectively.

JS11 — **null** is an intentional absence of value; **undefined** is a variable declared but not assigned. **==** coerces types (`null == undefined` is true); **===** checks type and value. **??** (nullish coalescing) returns right operand if left is `null`/`undefined`. **?.** (optional chaining) safely accesses nested properties.

JS12 — **map()** transforms elements. **filter()** selects elements. **reduce()** accumulates values. **Initial value** in reduce prevents using the first array element as the accumulator. **flatMap()** maps then flattens one level; useful for transforming arrays of arrays.

JS13 — **Template literals** use backticks for multi-line strings and interpolation (`${var}`). **Tagged templates** call a function with the string parts and values. Example: `html` function escaping values to prevent XSS.

JS14 — **Destructuring** extracts values from objects/arrays. **Object:** `{ a, b } = obj`. **Array:** `[a, b] = arr`. **Defaults:** `{ a = 1 }`. **Renaming:** `{ a: b }`. **Rest:** `{ ...rest }` gathers remaining properties/values.

JS15 — **for...in** iterates over enumerable property keys (including inherited). **for...of** iterates over iterable values. Avoid **for...in** on arrays because it includes non-index properties and is slower.

JS16 — **Generics** allow reusable components with type parameters. **Constraints:** `T extends SomeType`. **Conditional types:** `T extends U ? X : Y`. **infer:** Extracts a type from a conditional type (e.g., `infer R` in `Promise<R>`).

JS17 — **CommonJS** (`require`) is synchronous and dynamic. **ES Modules** (`import`) are asynchronous and static. Node.js supports both; `"type": "module"` in `package.json` forces `.js` files to be treated as ESM.

JS18 — **Structural typing** means types are compatible if they have the same shape. **Excess property checks** apply to object literals assigned to types. **extends** checks inheritance; **satisfies** checks shape without widening the type. **Variance:** Covariant (output), Contravariant (input).

JS19 — **Proxy** intercepts operations (get, set, apply). **Reflect** provides default implementations. Example: A Proxy can validate property assignments before setting them.

JS20 — **SharedArrayBuffer** allows memory sharing between workers. **Atomics** provide atomic operations (load, store, compareExchange). **Ordering guarantees** ensure visibility across threads. **Atomics.wait** blocks until **Atomics.notify** wakes it.

B1 — **$@** expands to separate quoted arguments. **$*** expands to a single string. **"$@"** preserves individual arguments. **"$*"** joins them into one string.

B2 — **set -e** exits on error. **set -u** errors on undefined variables. **set -o pipefail** returns the exit code of the first failed command in a pipeline.

B3 — **$(command)** is modern command substitution. **`command`** is legacy. **$((expression))** is arithmetic expansion (built-in). **expr** is an external command for arithmetic.

B4 — **Process substitution** (`<(cmd)`) creates a file descriptor for command output. Used when a command expects a filename but needs dynamic input. Example: `diff <(sort a) <(sort b)`.

B5 — Use `for f in *.txt` (globbing). **`for f in $(ls)`** is dangerous because it splits on whitespace/newlines, breaking filenames with spaces.

B6 — **Here document** (`<<EOF`) feeds stdin. **Here string** (`<<<`) feeds a string. **`<<-EOF`** strips leading tabs from the input lines.

B7 — `cmd > file 2>&1`: stdout to file, stderr to stdout (file). `cmd 2>&1 > file`: stderr to stdout (terminal), stdout to file. `&> file`: both to file. Order matters because redirections are processed left-to-right.

B8 — Use `mkdir` for locking (atomic). Function: `mkdir /tmp/lock` inside `trap 'rmdir /tmp/lock' EXIT`. `mkdir` fails if exists, preventing race conditions. `touch` is not atomic.

B9 — **'...'** literal (no expansion). **"..."** expands variables/globbing. **No quotes** expands variables, globbing, and word splitting. Example: `echo $HOME` vs `echo '$HOME'`.

B10 — **`[ -e file ]`** exists, **`[ -d dir ]`** directory, **`[ -r file ]`** readable. **`[[ ]]`** is a bash builtin (supports regex, logical operators). **`[ ]`** is POSIX `test` command.

B11 — **$?** exit code. **$$** script PID. **$!** last background PID. **$#** number of arguments.

B12 — **`function name {}`** or **`name() {}`**. Return values via `exit code` (0-255) or `echo` (captured via `$()`).

B13 — **PATH** is colon-separated directories. Bash searches PATH. **Builtin** is internal (e.g., `cd`), **External** is binary. **`type cmd`** shows which is used.

B14 — **trap** sets handlers. **EXIT** runs on exit. **ERR** on error. **INT/TERM** on signals. Execution order: Signal handlers run immediately; EXIT handlers run last.

B15 — **xargs** builds command lines from stdin. **`find -print0`** + **`xargs -0`** handles newlines/spaces. **`-P`** runs jobs in parallel.

B16 — **Arrays:** `arr=(1 2)`. **Associative:** `declare -A assoc`. Append: `arr+=(val)`. Length: `${#arr[@]}`. Associative keys are strings.

B17 — **Subshell** `(...)` forks a new process (vars lost). **Group** `{ ...; }` runs in current shell. **Pipe to while** creates a subshell, so variable changes inside the loop are lost.

B18 — **SIGINT** (Ctrl+C), **SIGTERM** (kill), **SIGHUP** (hangup), **SIGKILL** (force kill, uncatchable). Handle graceful shutdown by trapping SIGTERM, killing child jobs, then exiting.

B19 — Script: `split -l 1000 log log_part`, `xargs -P 4 -I {} bash process.sh {}`, `cat log_part_* > merged.log`. Trade-offs: Parallelism vs I/O contention. Error propagation: Check exit codes of parallel jobs, aggregate failures.

B20 — **exec** replaces the shell process. **FD manipulation:** `exec 3>file` opens FD 3. **Coprocess** (`coproc`) creates a bidirectional pipe. Use for long-running interactive tools.

PS1 — **Write-Output** sends objects to the pipeline. **Write-Host** writes directly to the console (bypasses pipeline). Distinction matters for piping data to other commands.

PS2 — **Pipeline** passes objects. **ForEach-Object** is a cmdlet (pipeline-aware). **foreach** is a statement (loop). Use **ForEach-Object** for pipeline streaming; **foreach** for pre-loaded collections.

PS3 — **[PSCustomObject]** creates a typed object with properties. **[hashtable]** is key/value pairs. Convert: `[PSCustomObject]@{Key="Value"}`. Use CustomObject for structured data, Hashtable for dynamic config.

PS4 — **Terminating** errors stop execution (throw). **Non-terminating** continue (warning). **-ErrorAction Stop** converts non-terminating to terminating. **try/catch** catches terminating errors.

PS5 — **Splatting** (`@params`) passes a hashtable as separate arguments. Passing a hashtable as a single parameter passes it as one object. Splatting is cleaner for cmdlets with many params.

PS6 — **Invoke-Command** runs one-off commands remotely. **Enter-PSSession** starts an interactive session. Uses **WinRM** (HTTP/HTTPS) or **SSH** (v7+).

PS7 — Returns an array even if one item. If 0 items, returns `$null`. Correct pattern: `$result = @(); $result += ...; return $result`. Check `if ($result.Count -eq 0)`.

PS8 — **[scriptblock]** is a code container. Pass via `-ArgumentList`. **Double-hop** fails when authenticating to a third machine. Solve with **CredSSP** (insecure) or **Invoke-Command -ArgumentList** with pre-authenticated tokens.

PS9 — **@()** creates an array. **[array]** casts to array. **$null** index throws error. **Empty array** index returns empty. Force single result: `@($result)[0]`.

PS10 — **-eq** (equal), **-like** (wildcard), **-match** (regex). **-ceq** (case-sensitive). **-contains** checks if collection contains item. **-in** checks if item is in collection.

PS11 — **Module** is a reusable script/library. **.psm1** (code), **.psd1** (manifest), **Binary** (DLL). **Import-Module** loads it into the session.

PS12 — **Interpolation** works in double quotes (`"$var"`). Single quotes are literal. **Here-string** (`@"..."@`) allows multi-line strings.

PS13 — **Get-Member** lists object members. **Properties** (data), **Methods** (functions), **NoteProperties** (added dynamically). **Select-Object** selects columns. **Where-Object** filters rows.

PS14 — **Classes** (v5+) define types. Limitations: No interfaces in v5, limited inheritance. Use **classes** for complex state, **functions** for simple logic.

PS15 — **[CmdletBinding()]** enables advanced features. **SupportsShouldProcess** enables `-WhatIf/-Confirm`. **Begin/Process/End** blocks handle setup, streaming, and cleanup.

PS16 — **Providers** expose data as file systems. **Get-ChildItem** works on Registry (`Registry::HKEY_LOCAL_MACHINE`). Access registry keys like folders.

PS17 — **Runspaces** are isolated execution environments. **RunspacePool** manages threads. **ForEach-Object -Parallel** (v7+) simplifies parallelism. Manual runspace for complex control.

PS18 — **PSScriptAnalyzer** enforces standards. **Rules** check code quality. **Severity** (Error, Warning, Info). Suppress via comments. Integrate via CI (e.g., GitHub Actions).

PS19 — **$using:** passes local vars to remote/parallel. **$args** passes positional. **Serialization limits** (CLIXML) affect complex objects (e.g., circular refs). Use JSON for complex data.

PS20 — **DSC** manages state. **Configurations** define state. **Resources** implement logic. **LCM** applies config. **PS 7+** uses DSC v3 (cross-platform). Replaced by **Ansible/Terraform** in some IaC contexts.
