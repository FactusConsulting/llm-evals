DN1 — **Class vs Struct**: `class` is a reference type (heap-allocated, nullable), `struct` is a value type (stack-allocated, non-nullable by default). Use `struct` for small, immutable data (e.g., coordinates, IDs) to avoid heap allocation and copying overhead. Use `class` for complex, mutable objects or inheritance hierarchies.

DN2 — **IEnumerable<T> vs IQueryable<T>**: `IEnumerable<T>` executes queries in-memory (client-side). `IQueryable<T>` builds expression trees translated to server-side queries (e.g., SQL). This distinction matters for databases because `IQueryable` pushes filtering/aggregation to the DB, reducing data transfer and improving performance.

DN3 — **Async/Await & Task/ValueTask**: `async/await` uses a state machine to suspend/resume execution without blocking threads. `Task` is a reference type (heap allocation). `ValueTask` is a struct (stack allocation). Use `ValueTask` for high-frequency, hot-path methods that often complete synchronously to avoid allocation overhead.

DN4 — **Dependency Injection (DI)**: DI decouples object creation from usage. Lifetimes: **Transient** (new instance every request), **Scoped** (new instance per request/session), **Singleton** (single instance for app lifetime). **Bug**: Injecting a Scoped service into a Singleton captures the Singleton's lifetime, causing the Scoped service to persist across requests (stale data/memory leak).

DN5 — **string vs ReadOnlySpan<char>**: `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a stack-only, zero-copy view over memory. Use `Span<T>` for high-performance string processing (parsing, slicing) to avoid creating intermediate string allocations.

DN6 — **Garbage Collector (GC)**: GC manages memory by reclaiming unreachable objects. **Generations**: Gen 0 (short-lived), Gen 1 (transient), Gen 2 (long-lived). **LOH**: Large Object Heap (>85KB) for arrays/strings, collected less frequently. **IDisposable**: Required for unmanaged resources (files, DB connections) because GC only manages managed memory; `Dispose` ensures deterministic cleanup.

DN7 — **ConfigureAwait(false)**: Default behavior captures the `SynchronizationContext` (e.g., UI thread) to resume on the same context. `ConfigureAwait(false)` skips this. Use `false` in library code to prevent deadlocks and improve throughput. Do NOT use `false` in UI code where context is needed for UI updates.

DN8 — **.NET 8/9 Features**: **Primary Constructors** (C# 12) allow constructor parameters in class definition. **Collection Expressions** simplify array/list creation. **FrozenDictionary** is an immutable, high-performance hash map.
*Example:*
```csharp
public class User(string id, string name) { } // Primary Constructor
// DI usage: services.AddScoped<User>();
```

DN9 — **DbContext Exception**: `ObjectDisposedException` under high load usually indicates a `DbContext` is registered as **Singleton** but used across multiple requests. EF Core `DbContext` is not thread-safe. **Fix**: Register `DbContext` as **Scoped** (`AddDbContext`) so a new instance is created per HTTP request.

DN10 — **Record vs Class**: `class` is mutable with reference equality. `record` is immutable (by default) with value equality (compares properties). `record struct` combines value type semantics with record equality. Choose `record` for DTOs/Models, `class` for mutable entities, `record struct` for small value objects.

DN11 — **ref, out, in**: `ref` passes by reference (read/write, must be initialized). `out` passes by reference (write-only, must be assigned in method). `in` passes by reference (read-only, avoids copy of large structs). Use `ref` for swapping, `out` for multiple return values, `in` for large read-only structs.

DN12 — **Nullable Reference Types (NRT)**: NRT (`string?`) is compile-time safety for reference types to warn about null dereferences. `int?` (nullable value type) is a runtime nullable wrapper for value types. NRT relies on `#nullable enable` and does not affect runtime behavior.

DN13 — **LINQ**: Language Integrated Query. **Method syntax** (`list.Where(...)`) is more flexible; **Query syntax** (`from x in list`) is SQL-like. **Deferred Execution**: Queries execute only when enumerated (e.g., `ToList()`), allowing composition before execution.

DN14 — **Generic Constraints**: `where T : class` (reference type), `where T : struct` (value type), `where T : new()` (parameterless constructor), `where T : IComparable<T>` (implements interface). Combine with `&` (e.g., `where T : class, IComparable<T>`). Use to enforce capabilities on generic types.

DN15 — **Abstract Class vs Interface**: Abstract class can have implementation and state; Interface defines a contract only. **C# 8+ Default Interface Methods** allow interfaces to have method implementations, blurring the line and allowing interface evolution without breaking implementations.

DN16 — **Middleware Pipeline**: Requests pass through a chain of middleware. `app.Use()` adds middleware (calls `next()`). `app.Map()` branches the pipeline. `app.Run()` is terminal (no `next()`). **Order matters**: Auth before MVC. Exception handling middleware (`UseExceptionHandler`) wraps the pipeline to catch unhandled errors.

DN17 — **Channel<T>**: Async producer-consumer queue. `BlockingCollection<T>` is synchronous. **Bounded Channels** (`CreateBounded`) limit capacity, applying backpressure to producers when full. Example: Limiting DB writes during high load.

DN18 — **Source Generators**: Compile-time code generation (faster than reflection, no IL weaving). **Incremental Generators** (C# 12) process files incrementally, improving build performance over `ISourceGenerator`. Preferred for reducing runtime reflection overhead.

DN19 — **TPL Dataflow**: Async pipeline library. `ActionBlock` (processes items), `TransformBlock` (processes and returns), `BufferBlock` (stores items). **Vs Channels**: Dataflow offers built-in backpressure, buffering, and complex flow control; Channels are lower-level primitives for simple queues.

DN20 — **Minimal APIs**: Lightweight, single-file endpoints without controllers. **Trade-offs**: Less boilerplate, faster startup, but less structure for large apps. **Route Groups** organize endpoints. **Endpoint Filters** apply logic (e.g., auth) to specific endpoints. OpenAPI support is native in .NET 8+.

P1 — **List vs Tuple**: `list` is mutable, ordered collection. `tuple` is immutable, ordered collection. `set` is mutable, unordered unique items. `frozenset` is immutable unique items. Use tuples for fixed data structures; sets for membership testing.

P2 — **GIL**: Global Interpreter Lock allows only one thread to execute Python bytecode at a time. Affects **CPU-bound** workloads. **Workarounds**: Use `multiprocessing` (separate processes), C-extensions (release GIL), or async I/O for I/O-bound tasks.

P3 — **Deepcopy vs Copy**: `copy` creates a new container but references original objects (shallow). `deepcopy` recursively copies objects. **Example**: `list1 = [[1]]`, `list2 = copy(list1)`. Modifying `list2[0][0]` changes `list1[0][0]`.

P4 — **Decorators**: Functions wrapping other functions.
```python
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.perf_counter() - start:.2f}s")
        return result
    return wrapper
```

P5 — **Generators**: Functions using `yield` to produce a sequence lazily (memory efficient). Unlike `return` (ends function), `yield` pauses state. `yield from` delegates to another generator.

P6 — **Asyncio**: Single-threaded event loop. `asyncio.run()` starts the loop. `asyncio.gather()` runs coroutines concurrently. `asyncio.create_task()` schedules a coroutine. `time.sleep()` blocks the loop; use `await asyncio.sleep()` instead.

P7 — **MRO**: Method Resolution Order. Python 3 uses **C3 Linearization**. In `D(B, C)`, MRO is `D -> B -> C -> A`. `D().who()` calls `C.who()` because `C` appears before `A` in the linearization. Prints "C".

P8 — **Metaclass**: Class of a class. Use `__init_subclass__` for simple subclass hooks (easier, no metaclass needed). Use Metaclass for complex class creation control (e.g., ORM field registration).

P9 — **Multiprocessing vs Threading**: `multiprocessing` bypasses GIL (CPU-bound), uses separate memory (requires pickling). `threading` shares memory (GIL-bound), good for I/O. `concurrent.futures` provides high-level API for both. Pickling is required for passing objects between processes.

P10 — **__slots__**: Prevents creation of `__dict__` for instances. Saves memory and prevents dynamic attribute assignment. Use when creating many instances of a class with fixed attributes.

P11 — **Virtual Environment**: Isolated Python environment. `venv` (stdlib), `virtualenv` (3rd party, more features), `conda` (binary packages, data science). Prevents dependency conflicts.

P12 — **Comprehensions**: Concise syntax for creating lists/dicts/sets. `map()`/`filter()` are functional. Prefer comprehensions for readability and Pythonic style.

P13 — **== vs is**: `==` checks value equality. `is` checks identity (memory address). **Interning**: CPython caches small integers (-5 to 256), so `a is b` can be True for small ints even if created separately.

P14 — **args/kwargs**: `*args` collects positional args into a tuple. `**kwargs` collects keyword args into a dict. `/` marks positional-only args. `*` marks keyword-only args.

P15 — **Context Managers**: Manage resources (e.g., DB connections). Implement `__enter__`/`__exit__` or use `@contextmanager`. Example: `with db.connect() as conn:` ensures connection closes on exit.

P16 — **Type Hinting**: Static type annotations. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. `mypy`/`pyright` use these to catch type errors before runtime.

P17 — **Data Model**: `__repr__` (dev/debug), `__str__` (user). `__eq__` defines equality. Defining `__eq__` without `__hash__` makes the object unhashable (cannot be dict key/set member). `__getattr__` (fallback), `__getattribute__` (always called).

P18 — **Descriptors**: Objects controlling attribute access (`__get__`, `__set__`, `__delete__`). `property`, `classmethod`, `staticmethod` are descriptors. Used for validation, computed properties, and method binding.

P19 — **Import System**: `sys.path` lists directories. `PYTHONPATH` adds to it. `__init__.py` marks a directory as a package. **Namespace Packages** (PEP 420) allow splitting packages without `__init__.py`. **Finder** locates modules, **Loader** loads them.

P20 — **Dataclasses**: Reduce boilerplate (`__init__`, `__repr__`). `attrs` is more feature-rich. `Pydantic` adds validation. Use `Pydantic` when input validation is needed. `__post_init__` runs after init. `frozen=True` makes instances immutable.
