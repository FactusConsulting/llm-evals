JS1 — **var, let, const, Hoisting**
*   **var:** Function-scoped. Hoisted and initialized with `undefined`. Can be redeclared.
*   **let:** Block-scoped. Hoisted but not initialized (Temporal Dead Zone). Cannot be redeclared in same scope.
*   **const:** Block-scoped. Must be initialized. Cannot be reassigned (though object properties can change).
*   **Hoisting:** Declarations are moved to the top of their scope during compilation. `var` is initialized; `let`/`const` are not.

JS2 — **Event Loop**
*   **Call Stack:** Executes synchronous code.
*   **Microtask Queue:** High priority (Promises, `queueMicrotask`). Processed after the current stack clears, before rendering.
*   **Macrotask Queue:** Lower priority (`setTimeout`, `setInterval`, I/O). Processed after microtasks and rendering.
*   **Loop:** Stack empty → Run all microtasks → Render → Run one macrotask → Repeat.

JS3 — **Closures**
*   **Definition:** A function bundled with its lexical environment (variables from outer scope).
*   **Example:** Counter function retaining `count` variable.
*   **Bug:** `var i` in a `for` loop shares one reference. All callbacks log the final value (e.g., 10).
*   **Fix:** Use `let i` (block scope) or an IIFE to capture the current value.

JS4 — **Promises & Async/Await**
*   **Promise:** Object representing eventual completion/failure. States: pending, fulfilled, rejected.
*   **async/await:** Syntactic sugar over Promises. `async` returns a Promise; `await` pauses execution until resolved.
*   **Error Handling:** `try/catch` blocks work with `await`. `.catch()` handles Promise rejections.
*   **Promise.all:** Waits for all to resolve; fails immediately if one rejects.
*   **Promise.allSettled:** Waits for all to settle (fulfilled or rejected), returns status for each.

JS5 — **Interface vs Type**
*   **Interface:** For defining object shapes. Supports declaration merging. Better for classes.
*   **Type:** More flexible (unions, primitives, tuples). Cannot merge.
*   **Discriminated Unions:** Union types with a literal property (e.g., `{type: 'a'}` | `{type: 'b'}`) allowing type narrowing based on that property.

JS6 — **Prototypal Inheritance**
*   **Mechanism:** Objects link to a prototype. Property lookup traverses the chain.
*   **Object.create(proto):** Creates new object with specified prototype.
*   **ES6 Classes:** Syntactic sugar over prototypes. `class` defines constructor and prototype methods.

JS7 — **`this` Keyword**
*   **Binding Rules (Priority):**
    1.  **New:** `new` keyword binds to new instance.
    2.  **Explicit:** `call`, `apply`, `bind`.
    3.  **Implicit:** Object method call (`obj.method()`).
    4.  **Default:** Global object (`window`) or `undefined` (strict mode).
*   **bind():** Returns a new function with bound `this`. Cannot be re-bound.

JS8 — **WeakRef & FinalizationRegistry**
*   **WeakRef:** Holds weak reference to object (GC can collect it).
*   **FinalizationRegistry:** Callback when object is garbage collected.
*   **Use Case:** Caching, cleanup of native resources.
*   **WeakMap:** Keys must be objects (GC can collect keys). Prevents memory leaks if keys are no longer referenced elsewhere.

JS9 — **V8 Optimization**
*   **Hidden Classes:** Objects with same shape share a class for fast property access.
*   **Inline Caching (IC):** Caches property lookup results for specific hidden classes.
*   **Deoptimization:** Occurs when hidden class changes (e.g., adding property dynamically).
*   **Patterns:** Avoid adding/deleting properties dynamically after object creation.

JS10 — **Tree-Shaking**
*   **Definition:** Eliminating unused code exports.
*   **ES Modules:** Static `import`/`export` allows static analysis for tree-shaking.
*   **CommonJS:** Dynamic `require` prevents static analysis, making tree-shaking ineffective without bundler heuristics.

JS11 — **null vs undefined**
*   **undefined:** Variable declared but not assigned.
*   **null:** Intentional absence of value.
*   **== vs ===:** `null == undefined` is true. `null === undefined` is false.
*   **??:** Nullish coalescing (returns right side if left is `null` or `undefined`).
*   **?.:** Optional chaining (returns `undefined` if path is nullish).

JS12 — **Array Methods**
*   **map:** Transforms elements.
*   **filter:** Selects elements.
*   **reduce:** Accumulates into single value. Initial value prevents using first element as accumulator.
*   **flatMap:** Maps then flattens one level. Useful for transforming arrays of arrays.

JS13 — **Template Literals**
*   **Definition:** Strings using backticks (`` ` ``). Support interpolation `${}` and multiline.
*   **Tagged Templates:** Function called with string array and values.
*   **Example:** `sql` tag to escape values and prevent SQL injection.

JS14 — **Destructuring**
*   **Object:** `{ name } = obj`.
*   **Array:** `[ a, b ] = arr`.
*   **Defaults:** `{ name = 'default' }`.
*   **Renaming:** `{ name: fullName }`.
*   **Rest:** `const { a, ...rest } = obj` collects remaining properties.

JS15 — **for...in vs for...of**
*   **for...in:** Iterates over enumerable **keys/properties** (including inherited).
*   **for...of:** Iterates over **values** of iterable objects.
*   **Avoid for...in on arrays:** Iterates indices, includes prototype properties, slower.

JS16 — **TypeScript Generics**
*   **Generics:** `<T>` allows reusable components with type safety.
*   **Constraints:** `T extends SomeType` limits what T can be.
*   **Conditional Types:** `T extends U ? X : Y` selects type based on condition.
*   **infer:** Extracts type within conditional type (e.g., `infer R` from `Promise<R>`).

JS17 — **Module Systems**
*   **CommonJS:** `require`/`module.exports`. Dynamic, synchronous, Node default (historically).
*   **ES Modules:** `import`/`export`. Static, asynchronous, browser standard.
*   **Node:** `.mjs` or `"type": "module"` in `package.json` enables ESM.

JS18 — **Structural Typing**
*   **System:** Types compatible if shapes match (duck typing).
*   **Excess Property Checks:** Prevents extra properties in object literals assigned to types.
*   **extends vs satisfies:** `extends` checks assignability (widens type). `satisfies` checks validity without widening.
*   **Variance:** TS infers covariance (return types) and contravariance (function params). No explicit `in`/`out` keywords.

JS19 — **Proxy & Reflect**
*   **Proxy:** Wraps object to intercept operations (get, set, apply, etc.).
*   **Reflect:** API counterpart to Proxy traps (e.g., `Reflect.get`).
*   **Example:** Validation Proxy that checks types on property assignment.

JS20 — **SharedArrayBuffer & Atomics**
*   **SharedArrayBuffer:** Shared memory between threads (Workers).
*   **Atomics:** Atomic operations (`load`, `store`, `compareExchange`) for synchronization.
*   **Communication:** Workers post messages or share memory.
*   **Ordering:** Memory barriers ensure visibility across threads. `Atomics.wait` blocks until `notify` is called.

B1 — **$@ vs $***
*   **$@:** Expands to separate quoted arguments (`"$1" "$2"`). Preserves word boundaries.
*   **$*:** Expands to single string (`"$1 $2"`).
*   **"$@":** Safest for preserving arguments.
*   **"$*":** Treats all args as one string.

B2 — **set -euo pipefail**
*   **-e:** Exit immediately if command exits non-zero.
*   **-u:** Treat unset variables as errors.
*   **-o pipefail:** Pipeline fails if any command fails (not just last).
*   *(Note: -x is often added for debugging)*.

B3 — **Command Substitution & Arithmetic**
*   **$(command):** Modern, nested, preferred.
*   **`command`:** Legacy, harder to nest.
*   **$((expression)):** Bash arithmetic expansion (fast, built-in).
*   **expr:** External command, slower, legacy.

B4 — **Process Substitution**
*   **Definition:** `<(command)` creates a file descriptor (e.g., `/dev/fd/63`) with command output.
*   **Use:** Allows commands expecting files to read from command output.
*   **Example:** `diff <(sort a) <(sort b)` compares sorted files without creating temp files.

B5 — **Safe Iteration**
*   **Safe:** `for f in *; do ...` or `while IFS= read -r -d '' f; do ...` (with `find -print0`).
*   **Danger:** `for f in $(ls)` breaks on spaces/newlines in filenames and glob expansion issues.

B6 — **Here Documents**
*   **<<EOF:** Reads stdin until EOF marker.
*   **<<<:** Here string (single line).
*   **<<-EOF:** Strips leading tabs from lines (for indentation).

B7 — **Redirections**
*   **`cmd > file 2>&1`:** Stdout to file, then Stderr to wherever Stdout is (file). **Both to file.**
*   **`cmd 2>&1 > file`:** Stderr to Stdout (terminal), then Stdout to file. **Stdout to file, Stderr to terminal.**
*   **`&> file`:** Bash specific, redirects both to file.

B8 — **Lock File Function**
*   **Logic:** Use `mkdir` for atomic locking. `trap` for cleanup on exit.
*   **Code:**
    ```bash
    lock() {
      mkdir "$1.lock" 2>/dev/null || return 1
      trap 'rmdir "$1.lock"' EXIT
    }
    ```
*   **Why mkdir:** Atomic operation (mkdir fails if exists). File creation is not atomic.

B9 — **Quotes**
*   **'...':** Literal. No expansion.
*   **"...":** Expansion allowed (`$`, `` ` ``, `\`).
*   **No quotes:** Expansion + globbing + word splitting.
*   **Example:** `echo $HOME` (no quotes) expands. `echo '$HOME'` prints literal.

B10 — **File Tests**
*   **Tests:** `-e` (exists), `-d` (directory), `-r` (readable).
*   **[ vs [[:** `[[` is Bash keyword (supports regex, safer). `[` is POSIX `test` command.

B11 — **Special Variables**
*   **$?:** Exit status of last command.
*   **$$:** PID of current shell.
*   **$!:** PID of last background job.
*   **$#:** Number of arguments passed.

B12 — **Functions**
*   **Def:** `func() {}` or `function func {}`.
*   **Return:** `return` sets exit code. Output via `echo` to stdout.
*   **Scope:** Global by default unless `local` used.

B13 — **PATH & Commands**
*   **PATH:** Colon-separated list of directories.
*   **Resolution:** Bash searches PATH in order.
*   **Builtin vs External:** Builtins are shell commands (faster). External are binaries.
*   **Check:** `type` or `which`.

B14 — **Trap**
*   **Definition:** Sets handlers for signals.
*   **Syntax:** `trap 'cmd' EXIT INT TERM`.
*   **Order:** Handlers run in order of definition. EXIT runs on shell exit.

B15 — **xargs**
*   **vs Substitution:** `xargs` batches input (memory efficient). Substitution loads all at once.
*   **Spaces/Newlines:** Use `find -print0` + `xargs -0`.
*   **-P:** Parallel execution (N jobs).

B16 — **Arrays**
*   **Indexed:** `arr=(a b c)`. `${arr[@]}`.
*   **Associative:** `declare -A arr`. `arr[key]=val`.
*   **Length:** `${#arr[@]}`.

B17 — **Subshell vs Group**
*   **(...):** Subshell (new process). Variables lost after exit.
*   **{ ...; }:** Group (same shell). Variables persist.
*   **Pipe:** `cmd | while` runs loop in subshell. Variable changes lost.

B18 — **Signals & Job Control**
*   **SIGINT:** Interrupt (Ctrl+C).
*   **SIGTERM:** Terminate (kill).
*   **SIGHUP:** Hangup (terminal close).
*   **SIGKILL:** Kill (cannot catch).
*   **Graceful:** Trap SIGTERM, cleanup, exit.

B19 — **Parallel Log Processing**
*   **Script:** `split -l 1000 log`, `xargs -P 4 -I {} bash process.sh {}`, `cat result* > final`.
*   **Trade-offs:** Speed vs complexity/error handling.
*   **Errors:** Check exit codes of parallel jobs; aggregate non-zero codes.

B20 — **exec & Coprocess**
*   **exec:** Replaces shell process. Used for FD manipulation.
*   **FD:** `exec 3>file` opens FD 3. `exec 3>&-` closes it.
*   **Coprocess:** `coproc CMD`. Creates pipe for stdin/stdout. Used for persistent background processes.

PS1 — **Write-Output vs Write-Host**
*   **Write-Output:** Sends object to pipeline.
*   **Write-Host:** Writes directly to console (screen).
*   **Matter:** Write-Host breaks pipeline (cannot be captured/redirected). Use Write-Output.

PS2 — **Pipeline & ForEach**
*   **Pipeline:** Streams objects between commands.
*   **ForEach-Object:** Cmdlet, processes items one-by-one in pipeline.
*   **foreach:** Statement, iterates a collection in memory. Faster for large collections not in pipeline.

PS3 — **PSCustomObject vs Hashtable**
*   **PSCustomObject:** Ordered, typed, supports methods/properties.
*   **Hashtable:** Unordered, key/value pairs.
*   **Convert:** `[PSCustomObject]@{Key = 'Value'}`.

PS4 — **Error Handling**
*   **Terminating:** Stops execution (e.g., `throw`).
*   **Non-terminating:** Continues execution (e.g., file not found).
*   **-ErrorAction:** Overrides `$ErrorActionPreference`. `Stop` converts non-terminating to terminating.
*   **try/catch:** Catches terminating errors.

PS5 — **Splatting**
*   **Definition:** Passing hashtable as parameters using `@`.
*   **Example:** `Get-Process @params`.
*   **Difference:** Splatting passes keys as named parameters. Passing hashtable as single arg passes it as one object.

PS6 — **Remoting**
*   **Invoke-Command:** One-off command execution.
*   **Enter-PSSession:** Interactive session.
*   **Protocol:** WinRM (HTTP/HTTPS). PowerShell 7+ supports SSH.

PS7 — **Type System & Pipeline**
*   **Issue:** Single item returned as scalar, not array.
*   **Code:** `$x.GetType().Name` is `String` (if one item).
*   **Empty:** Returns `$null`.
*   **Pattern:** Always initialize array `$result = @()` and append.

PS8 — **ScriptBlock & Remoting**
*   **ScriptBlock:** Container for code. Passed to remote sessions.
*   **Double-Hop:** Credentialed session cannot delegate creds to third server.
*   **Solution:** CredSSP (insecure) or `-ArgumentList` to pass data, not creds.

PS9 — **@() vs [array]**
*   **@():** Creates array. Forces scalar to array.
*   **[array]:** Type cast.
*   **Indexing:** `$null[0]` returns `$null`. `@()[0]` throws error.

PS10 — **Comparison Operators**
*   **-eq:** Equal. **-like:** Wildcard. **-match:** Regex.
*   **Case-Sensitive:** `-ceq`, `-clike`, `-cmatch`.
*   **-contains:** Checks if collection contains value. **-in:** Checks if value is in collection.

PS11 — **Modules**
*   **.psm1:** Script module (code).
*   **.psd1:** Manifest module (metadata).
*   **Binary:** Compiled DLL.
*   **Import-Module:** Loads module into session.

PS12 — **String Interpolation**
*   **"...":** Expands variables/expressions.
*   **'...':** Literal.
*   **Expressions:** `"Value: $var"`.
*   **Here-String:** `@"..."@` for multiline strings.

PS13 — **Get-Member**
*   **Purpose:** Inspect object properties/methods.
*   **Properties:** Data. **Methods:** Functions. **NoteProperties:** Dynamic properties added at runtime.
*   **Select-Object:** Selects properties. **Where-Object:** Filters objects.

PS14 — **PowerShell Classes**
*   **v5+:** Supports classes, constructors, inheritance.
*   **Limitations:** No interfaces, limited polymorphism compared to C#.
*   **Use:** Encapsulating stateful logic. Functions for stateless logic.

PS15 — **CmdletBinding**
*   **Enables:** Advanced function features (parameters, pipeline).
*   **SupportsShouldProcess:** Prompts for confirmation.
*   **Begin/Process/End:** Setup, Stream processing, Cleanup.

PS16 — **Providers**
*   **System:** Abstraction layer (FileSystem, Registry, Env).
*   **Get-ChildItem:** Works across providers (e.g., `HKLM:\`).
*   **Registry:** Accessed like file system paths.

PS17 — **Runspaces**
*   **[runspace]:** Single thread execution context.
*   **[runspacepool]:** Pool of threads for parallelism.
*   **-Parallel:** PowerShell 7+ built-in parallel processing.
*   **Thread Safety:** Variables not shared across runspaces.

PS18 — **PSScriptAnalyzer**
*   **Purpose:** Linting/Static analysis.
*   **Rules:** Enforce standards. **Severity:** Warning, Error, Info.
*   **CI/CD:** Integrate to block builds on errors.
*   **Suppress:** `#pragma` comments.

PS19 — **$using: Scope**
*   **$using:** Passes local variable value to remote session.
*   **-ArgumentList:** Passes arguments to scriptblock.
*   **Serialization:** Objects serialized to CLIXML. Limits depth/complexity.
*   **Effect:** Complex objects may lose type info.

PS20 — **DSC**
*   **Config:** Desired state definition.
*   **LCM:** Manages configuration application.
*   **WinPS vs PS7:** WinPS uses DSC v2. PS7 uses DSC v3 (cross-platform).
*   **Replacement:** Often replaced by Terraform/Ansible for modern IaC, but DSC remains for Windows config.
