## .NET / C# (20 questions)

**DN1 — Easy:** `class` is a reference type stored on the heap; `struct` is a value type stored on the stack (or inline). `struct` supports value semantics (copy on assignment) and cannot inherit from other classes. Use `struct` for small, immutable data types (e.g., `Point`, `Color`) to avoid heap allocation and improve cache locality.

**DN2:** `IEnumerable<T>` executes queries in-memory (client-side), while `IQueryable<T>` builds expression trees that translate to server-side queries (e.g., SQL). This distinction matters for databases because `IQueryable` allows filtering/paging on the server, reducing network traffic. Using `IEnumerable` on a DB context forces the entire dataset into memory before filtering.

**DN3:** `async/await` uses a state machine to suspend execution without blocking threads. `Task` is a reference type representing an async operation. `ValueTask` is a struct that avoids allocation if the result is already available. Use `ValueTask` in hot paths (e.g., high-frequency async methods returning value types) to reduce GC pressure.

**DN4:** Dependency Injection (DI) decouples object creation from usage. Lifetimes: **Transient** (new instance every request), **Scoped** (new instance per request), **Singleton** (single instance for app lifetime). **Bug:** Injecting a Scoped service into a Singleton captures the Singleton's lifetime, causing the Scoped service to persist across requests (stale data/memory leak).

**DN5:** `string` is immutable and heap-allocated. `ReadOnlySpan<char>` is a stack-only view into memory (no allocation). `Span<T>` allows mutable views. Use `Span<T>` for high-performance string/buffer processing to avoid allocations and enable SIMD optimizations.

**DN6:** The GC manages memory by reclaiming unreachable objects. **Gen 0** (short-lived), **Gen 1** (buffer), **Gen 2** (long-lived). **LOH** (Large Object Heap) stores objects >85KB (not compacted by default). `IDisposable` is needed for unmanaged resources (files, DB connections) because the GC only manages managed memory; `Dispose` ensures timely cleanup.

**DN7:** `ConfigureAwait(false)` prevents capturing the current `SynchronizationContext`. Use it in library code to avoid deadlocks and improve performance. Do **not** use it in UI code (WPF/WinForms) where you need to marshal back to the UI thread for updates. Default behavior captures context, which can cause context switches.

**DN8:** **Primary Constructors** (`.NET 9`): `public class Service(int id) : IService { }`. **Collection Expressions** (`.NET 8`): `[1, 2, 3]`. **FrozenDictionary** (`.NET 8`): Immutable, fast lookup. Example:
```csharp
public class UserService(int id) : IUserService {
    public int Id => id;
}
// DI Registration: services.AddScoped<UserService>();
```

**DN9:** **Cause:** `DbContext` is not thread-safe. Likely registered as `Singleton` instead of `Scoped`. **Fix:** Register `DbContext` as `Scoped` (`AddDbContext`). EF Core expects one context per request; sharing it causes race conditions and `ObjectDisposedException` when the underlying connection closes prematurely.

**DN10:** `record` provides value-based equality and immutability by default. `class` provides reference equality. `record struct` combines value semantics with record features. Choose `record` for DTOs/Models where equality matters. Choose `class` for mutable entities or when inheritance is required.

**DN11:** `ref` (pass by reference, must be initialized before call), `out` (pass by reference, must be assigned inside method), `in` (pass by reference, read-only). Use `ref`/`out` for large structs to avoid copying. Use `in` to pass large structs efficiently without allowing modification.

**DN12:** Nullable Reference Types (NRT) warn about potential null dereferences on reference types (`string?`). Nullable Value Types (`int?`) are actual `Nullable<T>` structs allowing `null` values. NRT is a compile-time feature (no runtime overhead); `int?` is a runtime type.

**DN13:** LINQ (Language Integrated Query). Method syntax (`list.Where(...)`) vs Query syntax (`from x in list`). Both compile to the same IL. **Deferred Execution:** Queries are not executed until enumerated (e.g., `ToList()`). This allows composition but can hide errors until execution.

**DN14:** Generic constraints limit type arguments. `where T : class` (reference), `where T : struct` (value), `where T : new()` (paramless constructor), `where T : IComparable<T>` (interface). Combine with commas: `where T : class, IComparable<T>`. Use to enforce capabilities on generic types.

**DN15:** `abstract class` can contain implementation and state; `interface` defines a contract (pre-C# 8). C# 8+ **default interface methods** allow interfaces to have implementations, blurring the line. Use abstract classes for shared code/state; interfaces for polymorphism/contracts.

**DN16:** Middleware pipeline processes requests. `app.Use()` adds middleware (can call next), `app.Map()` branches by path, `app.Run()` is terminal (no next). **Order matters:** Authentication before Authorization. Exception handling middleware (`UseExceptionHandler`) wraps the pipeline to catch unhandled exceptions globally.

**DN17:** `Channel<T>` is an async producer-consumer primitive. `BlockingCollection<T>` is synchronous. **Bounded Channel:** Limits capacity. If full, producers wait (backpressure). Prevents memory exhaustion in high-throughput systems. Example: `Channel.CreateBounded<int>(100)`.

**DN18:** Source Generators generate code at compile-time (faster than reflection, no IL weaving). **Incremental Generators** (Roslyn 4.0+) process files incrementally, improving build times compared to `ISourceGenerator` which regenerates all code. Preferred for performance in large solutions.

**NET19:** TPL Dataflow builds pipelines. `ActionBlock` (process, no output), `TransformBlock` (process, output), `BufferBlock` (queue). Compared to `Channels`, Dataflow offers more complex flow control (fan-out, fan-in, completion propagation) but higher overhead.

**DN20:** Minimal APIs use delegates for routing (no controllers). **Trade-offs:** Less boilerplate, harder to test complex logic, limited OpenAPI support (requires Swashbuckle). **Route Groups:** Organize endpoints (`MapGroup`). **Endpoint Filters:** Apply logic (e.g., validation) to specific endpoints without middleware overhead.

## Python (20 questions)

**P1:** `list` is mutable, ordered. `tuple` is immutable, ordered. `set` is mutable, unordered, unique elements. `frozenset` is immutable. Use `tuple` for fixed data, `set` for membership testing.

**P2:** GIL (Global Interpreter Lock) allows only one thread to execute Python bytecode at a time. Affects **CPU-bound** workloads. Workarounds: `multiprocessing` (processes bypass GIL), C-extensions (release GIL), or PyPy/Jython.

**P3:** `copy` creates a shallow copy (new container, same objects). `deepcopy` creates a new container and recursively copies objects. **Bug:** `list1 = [[1]]; list2 = copy(list1); list2[0].append(2)` modifies `list1` because inner lists are shared.

**P4:** Decorators wrap functions.
```python
import time
def log_time(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.time() - start}s")
        return result
    return wrapper
```

**P5:** Generators use `yield` to produce values lazily (memory efficient). `yield from` delegates to another generator. Unlike lists, generators cannot be re-iterated and don't store all values in memory.

**P6:** Event loop schedules coroutines. `asyncio.run()` starts the loop. `asyncio.gather()` waits for multiple tasks. `asyncio.create_task()` schedules a coroutine. `time.sleep()` blocks the loop; use `await asyncio.sleep()` to yield control.

**P7:** MRO uses **C3 Linearization**. In `D(B, C)`, MRO is `D -> B -> C -> A`. `D().who()` calls `C.who()` because `C` appears before `A` in the linearization. Diamond inheritance is resolved by order of inheritance.

**P8:** Metaclass controls class creation (e.g., ORM models). `__init_subclass__` is a simpler hook for subclass initialization. Use Metaclass for complex class construction (e.g., Django models). Use `__init_subclass__` for enforcing configuration on subclasses (e.g., registering plugins).

**P9:** `multiprocessing` (processes, bypass GIL, pickle serialization), `threading` (threads, GIL, shared memory), `concurrent.futures` (high-level API). Use Multiprocessing for CPU, Threading for I/O. Pickling can fail for complex objects or lambdas.

**P10:** `__slots__` defines fixed attributes, preventing `__dict__`. Saves memory (no dict per instance) and prevents dynamic attribute assignment. Use for large numbers of instances with fixed schema.

**P11:** Virtual environment isolates dependencies. `venv` (stdlib, Python 3.3+), `virtualenv` (older, more features), `conda` (package manager + env, binary packages). Prevents dependency conflicts between projects.

**P12:** Comprehensions (`[x for x in list]`) are concise loops. `map()`/`filter()` are functional. Prefer comprehensions for readability. `map()` returns iterator in Python 3.

**P13:** `==` checks value equality. `is` checks identity (memory address). **Interning:** Small integers (-5 to 256) are cached. `a is b` is True for small ints because they reference the same object.

**P14:** `*args` (tuple of positional), `**kwargs` (dict of keyword). `/` forces positional-only (before `/`). `*` forces keyword-only (after `*`). Example: `def f(a, /, b, *, c):`.

**P15:** Context managers manage resources (`with`). `__enter__`/`__exit__` define protocol. `@contextmanager` uses generator. Example: Acquiring a lock or creating a temp directory.

**P16:** Type hints add static type info. `Union`, `Optional`, `Literal`, `TypeVar`, `Protocol`. `mypy`/`pyright` analyze code without running it to catch type errors early.

**P17:** `__repr__` (dev/debug), `__str__` (user). `__eq`/`__hash`: If `__eq` is defined, `__hash` becomes `None` (unhashable) unless explicitly defined. `__getattr__` (missing attr), `__getattribute__` (all attr).

**P18:** Descriptors manage attribute access (`__get__`, `__set__`, `__delete__`). `property`, `classmethod`, `staticmethod` are descriptors. Use descriptors for computed properties or validation logic.

**P19:** Import system uses `sys.path`. **Finders** locate modules, **Loaders** load them. Relative imports (`from . import x`) vs Absolute (`from pkg import x`). `__init__.py` marks packages. Namespace packages allow splitting a package across directories without `__init__.py`.

**P20:** `dataclasses` auto-generates `__init__`, `__repr__`, etc. `attrs` is more feature-rich. `Pydantic` adds validation. `__post_init__` runs after init. Use `Pydantic` when input validation/serialization is required (e.g., API payloads).
